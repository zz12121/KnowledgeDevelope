# 映射文件与标签体系

## 一、标签分类总览

MyBatis 映射文件的标签按功能分为五大类：

**SQL 语句标签**：`<select>`、`<insert>`、`<update>`、`<delete>`，分别对应 CRUD 操作，是映射文件的核心。

**结果映射标签**：`<resultMap>`、`<result>`、`<id>`、`<association>`、`<collection>`，定义查询结果到 Java 对象的映射关系，是 MyBatis 最强大的特性。

**动态 SQL 标签**：`<if>`、`<choose>`/`<when>`/`<otherwise>`、`<foreach>`、`<where>`、`<set>`、`<trim>`，根据条件动态生成 SQL。

**复用标签**：`<sql>`定义可复用的 SQL 片段，`<include>`引用它，避免重复代码。

**缓存标签**：`<cache>`开启当前命名空间的二级缓存，`<cache-ref>`引用其他命名空间的缓存。

## 二、`#{}` 与 `${}` 的区别

这是面试高频题，两者有本质区别：

**`#{}`** 使用**预编译机制**，MyBatis 将其解析为 `?` 占位符，通过 `PreparedStatement.setXxx()` 设置参数值。数据库先编译 SQL 结构，后传入参数，参数无论内容如何都被当作纯数据处理，**从根本上防止 SQL 注入**。字符串参数会自动加单引号。

**`${}`** 是**字符串直接替换**，在 SQL 发送给数据库之前就完成拼接，不经过预编译，存在 **SQL 注入风险**，绝不能用于接收用户输入的参数值。

`${}` 的正确使用场景是**动态传入 SQL 关键字、表名、字段名**——这些内容不能用预编译占位符，因为占位符只能代表值，不能代表标识符：
```xml
<!-- 动态排序字段（表名、字段名不能用#{}） -->
SELECT * FROM goods ORDER BY ${sortColumn} ${sortDirection}
```
但要注意，这类场景必须在代码层做白名单校验，防止恶意传入。

## 三、`resultType` vs `resultMap`

**resultType** 指定一个 Java 类型，MyBatis 通过列名与属性名的自动匹配完成映射（需开启 `mapUnderscoreToCamelCase` 或列名与属性名完全一致）。适合**简单查询**，列名与属性名一一对应的场景。

**resultMap** 是手动映射配置，用于处理列名与属性名不一致、一对一关联、一对多关联等复杂场景。虽然配置量更大，但精确控制映射关系，且可复用：

```xml
<resultMap id="userDetailMap" type="User">
    <id property="id" column="user_id"/>             <!-- 主键映射，优化缓存 -->
    <result property="username" column="user_name"/>  <!-- 普通字段映射 -->
    <association property="dept" javaType="Department">   <!-- 一对一 -->
        <id property="id" column="dept_id"/>
        <result property="name" column="dept_name"/>
    </association>
    <collection property="roles" ofType="Role">      <!-- 一对多 -->
        <id property="id" column="role_id"/>
        <result property="name" column="role_name"/>
    </collection>
</resultMap>
```

## 四、字段名与属性名不一致的三种处理方式

**方案一：全局配置（推荐，一劳永逸）**
```xml
<settings>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
</settings>
```
自动将 `user_name` 映射为 `userName`，基本覆盖 90% 的场景。

**方案二：SQL 别名**
```xml
SELECT user_id AS id, user_name AS username FROM users
```
灵活但有重复代码，适合少量特殊字段。

**方案三：resultMap 手动映射**
适合列名差异较大、有关联查询的复杂场景。

## 五、入参方式详解

MyBatis 支持四种参数传递方式：

### 1. 基础类型参数

```xml
<select id="getMessageById" parameterType="long" resultType="message">
    select <include refid="allColumns"/> from tb_message where id = #{id}
</select>
```
```java
public interface MessageMapper {
    Message getMessageById(Long id);
}
```

### 2. Map 类型参数

```xml
<select id="getMessageByMap" parameterType="map" resultType="message">
    select <include refid="allColumns"/> from tb_message where id = #{id} and msg_id = #{msgId}
</select>
```
```java
public interface MessageMapper {
    Message getMessageByMap(Map<String, Object> params);
}
// 调用方式
Map<String, Object> paramsMap = new HashMap();
paramsMap.put("id", 1L);
paramsMap.put("msgId", "msg_1");
Message message = messageMapper.getMessageByMap(paramsMap);
```

### 3. 注解方式传递参数（推荐）

```xml
<select id="getMessageByAnnotation" resultType="message">
    select <include refid="allColumns"/> from tb_message where id = #{id} and msg_id = #{msgId}
</select>
```
```java
public interface MessageMapper {
    Message getMessageByAnnotation(@Param("id") Long id, @Param("msgId") String msgId);
}
```

### 4. JavaBean 方式传递参数

```xml
<select id="getMessageByMessage" parameterType="vo.Message" resultType="message">
    select <include refid="allColumns"/> from tb_message where id = #{id} and msg_id = #{msgId}
</select>
```
```java
public interface MessageMapper {
    Message getMessageByMessage(Message message);
}
// 调用方式
Message param = new Message();
param.setId(1L);
param.setMsgId("1000");
Message message = messageMapper.getMessageByMessage(param);
```

## 六、主键回填

### 使用 `useGeneratedKeys` 和 `keyProperty`

```xml
<insert id="insert" parameterType="message" keyProperty="id" useGeneratedKeys="true">
    insert into tb_message(msg_id, status, content, deleted, create_time)
    values (#{msgId}, #{status}, #{content}, #{deleted}, #{createTime})
</insert>
```
```java
public interface MessageMapper {
    int insert(Message message);
}
// 调用后，message 对象会自动填充 id 主键值
Message msg = new Message();
msg.setMsgId("msg_001");
mapper.insert(msg);
System.out.println(msg.getId()); // 自动填充了数据库生成的主键
```

## 七、结果处理方式

### 1. 使用 Map 存储结果集

```xml
<select id="getMessageMapById" resultType="map">
    select <include refid="allColumns"/> from tb_message where id = #{id}
</select>
```
```java
public interface MessageMapper {
    Map getMessageMapById(@Param("id") Long id);
}
```

### 2. 使用 POJO 存储结果集

```xml
<resultMap id="messageResult" type="vo.Message">
    <id column="id" property="id"/>
    <result column="msg_id" property="msgId"/>
    <result column="status" property="status"/>
    <result column="content" property="content"/>
    <result column="deleted" property="deleted"/>
    <result column="create_time" property="createTime"/>
    <result column="update_time" property="updateTime"/>
</resultMap>
<select id="getMessageById" resultMap="messageResult">
    select <include refid="allColumns"/> from tb_message where id = #{id}
</select>
```
```java
public interface MessageMapper {
    Message getMessageById(@Param("id") Long id);
}
```

## 八、自动映射与手动映射

### autoMappingBehavior 配置

| 值 | 说明 |
|----|------|
| NONE | 取消自动映射 |
| PARTIAL | 只自动映射没有定义嵌套结果集映射的结果集（默认）|
| FULL | 自动映射任意复杂的结果集（无论是否嵌套）|

```xml
<settings>
    <setting name="autoMappingBehavior" value="PARTIAL"/>
    <setting name="mapUnderscoreToCamelCase" value="true"/>
</settings>
```
