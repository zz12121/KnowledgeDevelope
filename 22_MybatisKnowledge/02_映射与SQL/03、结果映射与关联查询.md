# 结果映射与关联查询

## 一、三种关联查询实现

### 一对一（`<association>`）
订单与用户、员工与部门这类多对一关系，用 `<association>` 映射：
```xml
<resultMap id="orderDetailMap" type="Order">
    <id property="id" column="order_id"/>
    <result property="orderNo" column="order_no"/>
    <association property="user" javaType="User">
        <id property="id" column="user_id"/>
        <result property="username" column="username"/>
    </association>
</resultMap>

<select id="getOrderDetail" resultMap="orderDetailMap">
    SELECT o.*, u.user_id, u.username
    FROM orders o JOIN users u ON o.user_id = u.id
    WHERE o.id = #{id}
</select>
```

### 一对多（`<collection>`）
用户与订单列表、部门与员工列表这类一对多关系，用 `<collection>` 映射：
```xml
<resultMap id="userOrderMap" type="User">
    <id property="id" column="user_id"/>
    <result property="username" column="username"/>
    <collection property="orders" ofType="Order">
        <id property="id" column="order_id"/>
        <result property="amount" column="order_amount"/>
    </collection>
</resultMap>
```

### 多对多（通过中间表）
用户与角色这类多对多关系，本质上还是一对多，通过 JOIN 中间表实现：
```xml
<select id="getUserWithRoles" resultMap="userRoleMap">
    SELECT u.*, r.role_id, r.role_name 
    FROM users u
    LEFT JOIN user_role ur ON u.id = ur.user_id
    LEFT JOIN roles r ON ur.role_id = r.id
    WHERE u.id = #{userId}
</select>
```

## 二、嵌套查询 vs 连接查询

`<association>` 和 `<collection>` 都支持两种加载方式：

**连接查询（单 SQL）**：通过 JOIN 一次性取出所有数据，在 resultMap 中直接映射关联属性。性能好，推荐用于关联数据量不大的场景。

**嵌套查询（多 SQL）**：通过 `select` 属性触发额外查询：
```xml
<association property="dept" javaType="Department"
             select="selectDeptById" column="dept_id"/>
```
这会先查主表，再对每一行触发一次关联查询，即 **N+1 查询问题**（1 次主查询 + N 次关联查询），数据量大时性能极差。

**解决 N+1 的方法**：
1. **优先用 JOIN**：单 SQL 一次搞定，最直接
2. **IN 批量查询**：先拿到主表 ID 列表，再一次 `WHERE id IN (...)` 查关联数据
3. **延迟加载**：开启 `lazyLoadingEnabled`，只有真正访问关联属性时才触发查询，对于偶尔才用到关联数据的场景有效

## 三、延迟加载

```xml
<settings>
    <setting name="lazyLoadingEnabled" value="true"/>
    <setting name="aggressiveLazyLoading" value="false"/>
</settings>

<!-- 或在 resultMap 中单独控制 -->
<collection property="orders" ofType="Order"
            select="selectOrdersByUserId" column="user_id"
            fetchType="lazy"/>
```

延迟加载通过**动态代理**实现：关联属性初始为代理对象，当代码首次访问该属性时，代理触发实际 SQL 查询。`aggressiveLazyLoading=false` 表示访问某个懒加载属性不会触发其他属性的加载（推荐关闭）。

注意：延迟加载只能在 SqlSession 打开期间触发，Session 关闭后再访问懒加载属性会抛异常。在 Web 应用中需要确保 Session 在视图渲染完成后再关闭（Open Session in View 模式，但这个模式有争议，推荐显式加载）。

## 四、鉴别器（discriminator）

`<discriminator>` 根据某列值选择不同的结果映射，类似 `switch-case`，适合单表继承场景：
```xml
<resultMap id="vehicleMap" type="Vehicle">
    <id property="id" column="id"/>
    <discriminator javaType="String" column="vehicle_type">
        <case value="CAR" resultMap="carResultMap"/>
        <case value="TRUCK" resultMap="truckResultMap"/>
    </discriminator>
</resultMap>
```
MyBatis 会根据 `vehicle_type` 字段的值，决定将这行数据映射为 `Car` 还是 `Truck` 对象。
