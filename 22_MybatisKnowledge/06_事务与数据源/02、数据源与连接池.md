# 数据源与连接池

## 1. MyBatis 内置数据源

MyBatis 自带两种数据源实现，都实现了 `DataSource` 接口，通过 `mybatis-config.xml` 的 `<dataSource type="...">` 配置切换。

### UNPOOLED（非池化）

每次调用 `getConnection()` 都直接通过 `DriverManager.getConnection()` 创建新连接，用完立即关闭。**没有任何连接复用逻辑**，适合简单测试环境或极低并发场景。

### POOLED（池化）

MyBatis 内置的轻量级连接池，实现类是 `PooledDataSource`，是生产环境的默认选择（在不使用第三方连接池时）。

核心数据结构：
- `idleConnections`：空闲连接列表
- `activeConnections`：活跃连接列表

每个连接被 `PooledConnection` 包装，额外记录创建时间、最后使用时间、检出时间等状态信息，为连接生命周期管理提供依据。

**连接获取流程（`popConnection()` 源码）**

`popConnection()` 方法在 `synchronized (state)` 块内执行，保证线程安全，其逻辑如下：

```
1. idleConnections 不为空  →  直接取出第一个连接返回
2. idleConnections 为空 且 活跃连接数 < poolMaximumActiveConnections  →  创建新 PooledConnection
3. 活跃连接数已满  →  取出最早的活跃连接，检查其检出时间是否超过 poolMaximumCheckoutTime
   - 超时  →  "征用"此连接（移出活跃列表，回滚未提交事务），包装为新连接返回
   - 未超时  →  线程进入 state.wait(poolTimeToWait) 等待，直到有连接释放
```

**连接归还**：`PooledConnection` 通过动态代理（包装器模式）重写了 `close()` 方法，调用 `close()` 时实际上是将连接归还到空闲池，而不是真正关闭 JDBC 连接。

**有效性检测**：支持配置 `poolPingQuery`（如 `SELECT 1`）定期检查空闲连接是否仍然有效，避免使用被数据库服务器主动断开的"僵尸连接"。

**适用建议**：
- 优点：简单、无外部依赖，适合中小型应用和开发测试
- 缺点：缺乏连接泄漏检测、详细监控等高级特性，高并发性能不如专业连接池
- 生产环境强烈建议使用 HikariCP 或 Druid

## 2. HikariCP 配置

HikariCP 是 Spring Boot 默认连接池，以**极致性能**著称，底层通过字节码技术、无锁设计实现极低的连接获取延迟。

**Spring Boot 配置（application.yml）**：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/my_db
    username: root
    password: yourpassword
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20       # 最大连接数
      minimum-idle: 5             # 最小空闲连接数
      connection-timeout: 30000   # 获取连接超时（ms）
      idle-timeout: 600000        # 空闲连接超时（ms）
      max-lifetime: 1800000       # 连接最大生存时间（ms）
```

**传统 mybatis-config.xml 配置**：

```xml
<dataSource type="com.zaxxer.hikari.HikariDataSource">
  <property name="jdbcUrl" value="jdbc:mysql://localhost:3306/my_db"/>
  <property name="username" value="root"/>
  <property name="maximumPoolSize" value="20"/>
</dataSource>
```

## 3. Druid 配置

Druid 是阿里巴巴开源的连接池，最大优势是**内置监控和防 SQL 注入**，在国内企业项目中使用非常广泛。

**Spring Boot 配置（application.yml）**：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/my_db
    username: root
    password: yourpassword
    driver-class-name: com.mysql.cj.jdbc.Driver
    type: com.alibaba.druid.pool.DruidDataSource
    druid:
      initial-size: 5           # 初始化连接数
      min-idle: 5               # 最小空闲连接数
      max-active: 20            # 最大活跃连接数
      filters: stat,wall,log4j2 # stat=监控统计, wall=防SQL注入
      stat-view-servlet:
        enabled: true           # 开启监控页面 /druid
```

**依赖**：

```xml
<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>druid-spring-boot-starter</artifactId>
    <version>1.2.8</version>
</dependency>
```

## 4. HikariCP vs Druid 选择

| 维度 | HikariCP | Druid |
|------|----------|-------|
| 性能 | ⭐⭐⭐⭐⭐ 极致 | ⭐⭐⭐⭐ 优秀 |
| 监控 | 基础 JMX | 内置 Web 监控页面，SQL 统计详尽 |
| 防 SQL 注入 | 无 | Wall Filter 支持 |
| Spring Boot 默认 | ✅ 是 | ❌ 需手动引入 |
| 适用场景 | 追求极致性能 | 需要强大监控、安全防护 |

**经验之谈**：互联网高并发系统选 HikariCP；需要 SQL 分析、慢 SQL 告警、运维监控的企业系统选 Druid。

## 5. 多数据源静态配置

静态多数据源的核心是：为每个数据源创建独立的 `SqlSessionFactory` 和 `TransactionManager`，通过 `@MapperScan` 的 `sqlSessionFactoryRef` 参数将 Mapper 接口与对应的 SqlSessionFactory 绑定。

```yaml
# application.yml
primary:
  datasource:
    url: jdbc:mysql://localhost:3306/primary_db
    username: primary_user
    password: ...
secondary:
  datasource:
    url: jdbc:mysql://localhost:3310/secondary_db
    username: secondary_user
    password: ...
```

```java
@Configuration
@MapperScan(basePackages = "com.example.mapper.primary",
            sqlSessionFactoryRef = "primarySqlSessionFactory")
public class PrimaryDataSourceConfig {

    @Bean
    @Primary
    @ConfigurationProperties(prefix = "primary.datasource")
    public DataSource primaryDataSource() {
        return DataSourceBuilder.create().type(HikariDataSource.class).build();
    }

    @Bean
    @Primary
    public SqlSessionFactory primarySqlSessionFactory(
            @Qualifier("primaryDataSource") DataSource dataSource) throws Exception {
        SqlSessionFactoryBean bean = new SqlSessionFactoryBean();
        bean.setDataSource(dataSource);
        bean.setMapperLocations(new PathMatchingResourcePatternResolver()
                .getResources("classpath:mapper/primary/*.xml"));
        return bean.getObject();
    }

    @Bean
    @Primary
    public DataSourceTransactionManager primaryTransactionManager(
            @Qualifier("primaryDataSource") DataSource dataSource) {
        return new DataSourceTransactionManager(dataSource);
    }
}
```

**关键注解说明**：
- `@Primary`：在多个同类型 Bean 存在时，标记一个为默认首选 Bean
- `@Qualifier`：注入时明确指定 Bean 名称，避免歧义
- `@MapperScan` 的 `sqlSessionFactoryRef`：将该包路径的 Mapper 绑定到指定工厂
