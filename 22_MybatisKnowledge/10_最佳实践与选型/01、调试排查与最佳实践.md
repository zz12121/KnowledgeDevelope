# 调试排查与最佳实践

## 1. 开启 SQL 日志

**Spring Boot 项目（推荐）**：

```yaml
logging:
  level:
    # 方式一：针对 Mapper 包开启 DEBUG 级别，输出 SQL 和参数
    com.example.mapper: debug

mybatis:
  configuration:
    # 方式二：直接输出到控制台（开发环境快速调试）
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
```

**日志输出格式**：

```
==>  Preparing: SELECT * FROM users WHERE id = ? AND name = ?
==> Parameters: 1(Integer), test(String)
<==    Total: 1
```

`==>` 表示发往数据库的语句和参数，`<==` 表示返回结果行数。

**传统 mybatis-config.xml 配置**：

```xml
<settings>
    <setting name="logImpl" value="SLF4J"/>  <!-- 或 LOG4J、LOG4J2 -->
</settings>
```

**获取带参数的完整 SQL（拦截器方式）**：

MyBatis 日志默认输出 `?` 占位符，通过拦截器可以获取参数填充后的完整 SQL：

```java
@Intercepts({
    @Signature(type = Executor.class, method = "update",
               args = {MappedStatement.class, Object.class}),
    @Signature(type = Executor.class, method = "query",
               args = {MappedStatement.class, Object.class, RowBounds.class, ResultHandler.class})
})
public class SqlLogInterceptor implements Interceptor {
    @Override
    public Object intercept(Invocation invocation) throws Throwable {
        MappedStatement ms = (MappedStatement) invocation.getArgs()[0];
        Object parameter = invocation.getArgs()[1];
        BoundSql boundSql = ms.getBoundSql(parameter);
        long start = System.currentTimeMillis();
        Object result = invocation.proceed();
        long time = System.currentTimeMillis() - start;
        log.info("SQL [{}ms] {}", time, boundSql.getSql().replaceAll("\\s+", " "));
        return result;
    }
}
```

## 2. 常见异常与解决方案

### Invalid bound statement (not found)

```
org.apache.ibatis.binding.BindingException: Invalid bound statement (not found): 
com.example.mapper.UserMapper.selectById
```

**排查清单**：
1. XML 的 `namespace` 是否与 Mapper 接口全限定名完全一致（包括大小写）
2. XML 中的 `id` 是否与接口方法名完全一致
3. XML 文件是否被 Maven 打包到 classpath（在 `src/main/java` 下的 XML 需要配置 Maven resource 过滤）
4. Spring Boot 中是否添加了 `@Mapper` 或 `@MapperScan`

**Maven 资源过滤配置**（如果 XML 放在 `src/main/java` 下）：

```xml
<build>
    <resources>
        <resource>
            <directory>src/main/java</directory>
            <includes><include>**/*.xml</include></includes>
        </resource>
        <resource>
            <directory>src/main/resources</directory>
        </resource>
    </resources>
</build>
```

### 参数绑定异常

```
org.apache.ibatis.type.TypeException: Could not set parameter
```

**原因**：多参数方法未加 `@Param` 注解。

```java
// 错误：MyBatis 无法识别参数名
User findUser(String name, Integer age);

// 正确：用 @Param 明确参数名
User findUser(@Param("name") String name, @Param("age") Integer age);
```

XML 中对应：`WHERE name = #{name} AND age = #{age}`

### 结果映射异常

- `ResultMap contains duplicates`：resultMap 中有重复的 `<id>` 配置
- 属性值为 null：数据库列名与 Java 属性名不匹配，检查 `mapUnderscoreToCamelCase` 配置或手动指定 `<result column="..." property="..."/>`

### 事务未生效

**常见原因**：
- `@Transactional` 标注在非 Spring 管理的 Bean 上
- 同类内部调用（绕过了 AOP 代理）
- 方法不是 `public`
- 抛出的是受检异常（Checked Exception），默认只回滚 `RuntimeException` 和 `Error`

**最佳实践**：
```java
@Transactional(rollbackFor = Exception.class)  // 显式指定所有异常都回滚
public void updateUser(User user) { ... }
```

## 3. 最佳实践

### 项目组织结构

```
src/
├── main/
│   ├── java/com/example/
│   │   ├── entity/          # 实体类（与数据库表对应）
│   │   ├── mapper/          # Mapper 接口
│   │   ├── service/         # 业务逻辑层
│   │   └── controller/      # 接口层
│   └── resources/
│       ├── mapper/          # XML 映射文件（与 Mapper 接口同名）
│       └── application.yml
```

Mapper 接口与 XML 文件一一对应且同名，如 `UserMapper.java` ↔ `UserMapper.xml`，便于维护。

### SQL 编写规范

- **优先使用 `#{}`**：所有用户输入参数必须使用 `#{}`，防 SQL 注入
- **`${}` 的合法使用**：仅用于 MyBatis 无法预编译的场景（动态表名、ORDER BY 字段），且必须对输入进行严格白名单校验
- **避免 `SELECT *`**：明确指定查询字段，减少网络传输和结果映射开销
- **动态 SQL 防全表扫描**：`<where>` + `<if>` 组合，避免空条件导致全表查询

### 安全注意事项

```java
// 危险：将用户输入直接拼接到 ORDER BY
@GetMapping("/users")
public List<User> getUsers(@RequestParam String sortField) {
    // 如果直接用 ${sortField} 拼入 SQL，用户可以注入恶意内容
}

// 正确：白名单校验
private static final Set<String> ALLOWED_SORT_FIELDS = Set.of("name", "age", "create_time");

public List<User> getUsers(String sortField) {
    if (!ALLOWED_SORT_FIELDS.contains(sortField)) {
        sortField = "create_time"; // 默认值
    }
    return userMapper.selectWithSort(sortField);
}
```

### 性能设计原则

1. **批量操作**：超过 100 条的插入/更新，使用 `<foreach>` 或 `BatchExecutor`
2. **缓存策略**：一级缓存默认开启，二级缓存慎用（多表关联时有脏读风险）
3. **关联查询**：优先 JOIN，其次批量 IN，避免 N+1
4. **分页**：大表查询必须分页，深分页使用游标分页代替 LIMIT offset
5. **连接池**：根据 `最大连接数 = CPU核心数 * 2 + 有效磁盘数` 规则估算

### 微服务架构中的 MyBatis

- 每个微服务独立数据库，通过 `mybatis-spring-boot-starter` + `@MapperScan` 配置
- 多实例部署时，**不要使用二级缓存**（每个实例有独立缓存，会产生数据不一致）
- 分库分表场景结合 ShardingSphere（对 MyBatis 透明，只需配置 ShardingSphere DataSource）

## 4. 调试技巧

### IDEA 源码断点

1. 打开 `org.apache.ibatis.executor.SimpleExecutor`
2. 在 `doQuery()` 方法设断点
3. 查看 `BoundSql` 对象的 `sql` 字段（完整 SQL）和 `parameterMappings`（参数映射）

### 参数绑定调试

开启 TRACE 级别获取最详细的参数处理信息：

```yaml
logging:
  level:
    com.example.mapper: trace
    org.apache.ibatis: debug
    org.springframework.jdbc: debug
```

### 验证 Mapper 是否正确加载

```java
@SpringBootTest
class MapperLoadTest {
    @Autowired
    private ApplicationContext context;

    @Test
    void testMapperLoaded() {
        String[] beanNames = context.getBeanNamesForType(UserMapper.class);
        assertTrue(beanNames.length > 0, "UserMapper 未被加载为 Spring Bean");
    }
}
```
