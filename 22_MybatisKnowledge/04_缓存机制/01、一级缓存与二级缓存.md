# 一级缓存与二级缓存

## 一、缓存整体设计

MyBatis 缓存采用**分层设计**，分为一级缓存和二级缓存，查询时的执行顺序是：**二级缓存 → 一级缓存 → 数据库**。

---

## 二、一级缓存（本地缓存）

**作用范围**：SqlSession 级别，随 SqlSession 创建而创建，关闭而销毁。**默认开启，无法关闭**。

**实现机制**：在 `BaseExecutor` 中维护一个 `PerpetualCache`（本质是 HashMap），以 `CacheKey`（由 SQL + 参数 + 分页 + 环境等组合生成）为键。同一 SqlSession 内执行相同查询，直接从 HashMap 命中，不访问数据库。

**缓存失效场景**：
- 执行 INSERT / UPDATE / DELETE 操作（`BaseExecutor.update()` 会先调用 `clearLocalCache()`）
- 调用 `sqlSession.clearCache()` 手动清空
- SqlSession 关闭
- `flushCache="true"` 的查询语句执行后
- 配置 `localCacheScope=STATEMENT`（每条查询后清空，相当于关闭一级缓存）

**注意**：Spring 集成后，每个 `@Transactional` 方法内复用同一 SqlSession，一级缓存有效；方法结束 SqlSession 关闭，缓存失效。这是为什么在同一个事务内两次查询同一条数据，第二次走缓存；但不同事务查询同一条数据，不走缓存。

---

## 三、二级缓存（全局缓存）

**作用范围**：Mapper 命名空间级别，跨 SqlSession 共享，**需手动开启**。采用**装饰器模式**，`CachingExecutor` 包装基础 Executor，在执行查询前后检查和更新二级缓存。

**配置步骤**：
```xml
<!-- 1. mybatis-config.xml 全局开关（默认 true） -->
<settings>
    <setting name="cacheEnabled" value="true"/>
</settings>

<!-- 2. Mapper.xml 中开启 -->
<cache eviction="LRU"     <!-- 淘汰策略：LRU/FIFO/SOFT/WEAK -->
       flushInterval="60000"   <!-- 自动刷新间隔（毫秒）-->
       size="1024"             <!-- 最大缓存对象数 -->
       readOnly="true"/>       <!-- true=返回同一对象（更快），false=返回副本（更安全）-->
```

**实体类要求**：使用二级缓存的实体类**必须实现 `Serializable` 接口**，因为缓存对象可能需要序列化到磁盘或分布式存储。

**事务性保证**：二级缓存通过 `TransactionalCacheManager` 管理暂存区，事务提交后数据才从暂存区移入真正的缓存。这保证了未提交的数据不会污染二级缓存。

---

## 四、缓存失效场景对比

**一级缓存失效**：DML 操作、手动 clearCache、SqlSession 关闭、查询参数变化。

**二级缓存失效**：
- 同命名空间下执行 DML 操作（整个命名空间缓存清空）
- 跨 Mapper 更新数据（A Mapper 更新的数据，B Mapper 的缓存不会自动刷新）
- 达到 `flushInterval` 设定的时间
- 超过 `size` 限制，按 `eviction` 策略淘汰

**多表关联的缓存陷阱**：UserMapper 关联查询了 OrderMapper 的数据，当 Order 数据更新时，UserMapper 的二级缓存不会自动失效，导致脏读。解决方案：
1. 相关 Mapper 使用 `<cache-ref>` 共享同一个缓存命名空间
2. 对频繁更新的多表关联查询**不使用二级缓存**
3. 使用 Redis 等外部缓存代替，在业务层自己控制失效

---

## 五、集成第三方缓存

**Redis 集成**：
```xml
<dependency>
    <groupId>org.mybatis.caches</groupId>
    <artifactId>mybatis-redis</artifactId>
    <version>1.0.0-beta2</version>
</dependency>

<!-- Mapper.xml -->
<cache type="org.mybatis.caches.redis.RedisCache"/>
```

**自定义缓存**：实现 `org.apache.ibatis.cache.Cache` 接口，实现 `putObject` / `getObject` / `removeObject` / `clear` / `getSize` 方法，注册即可使用。适合需要定制缓存行为的场景（如加密存储、自定义过期策略）。

**生产建议**：对于分布式系统，内置二级缓存是单机本地缓存，多实例部署时各实例缓存独立，容易造成数据不一致。**分布式环境推荐使用 Redis 等分布式缓存**，或在业务层用 Spring Cache 统一管理。
