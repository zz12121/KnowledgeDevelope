## 1. 倒排索引原理

**倒排索引（Inverted Index）是 ES 全文搜索的核心数据结构**，由"词项 → 文档列表"的映射组成，与正排索引（文档 → 词项）相反。

### 1.1 构建过程

```
文档1: "Elasticsearch is a search engine"
文档2: "Elasticsearch is distributed"
文档3: "Search engine optimization"

分词后：
文档1: [elasticsearch, is, a, search, engine]
文档2: [elasticsearch, is, distributed]
文档3: [search, engine, optimization]

倒排索引：
Term              | Doc IDs         | 词频
----------------- | --------------- | ----
elasticsearch     | [1, 2]          | 1,1
is                | [1, 2]          | 1,1
search            | [1, 3]          | 1,1
engine            | [1, 3]          | 1,1
distributed       | [2]             | 1
optimization      | [3]             | 1
```

### 1.2 倒排索引的三层结构

```
Term Index（词项索引）
    → 前缀树（FST/Trie）结构，存在内存中
    → 快速定位词项在词典中的位置

Term Dictionary（词项词典）
    → 存储所有词项，按字母排序
    → 二分查找，O(log N)

Posting List（倒排列表）
    → 存储文档 ID、词频、位置、偏移量
    → Frame of Reference (FOR) 压缩算法
    → Roaring Bitmap 加速集合运算（AND/OR/NOT）
```

**Frame of Reference 压缩示例：**
```
原始: [73, 300, 302, 332, 343, 372]
差值: [73, 227, 2, 30, 11, 29]
分块压缩: 每个块用最少位数存储
```

### 1.3 位置信息与短语匹配

倒排列表中还存储了词项在文档中的位置（Position）和字符偏移量（Offset）：
- **Position**：用于短语查询（`match_phrase`）验证词项是否相邻
- **Offset**：用于高亮显示（highlight）

## 2. 相关性评分（BM25）

### 2.1 TF-IDF 基础

ES 使用 **BM25**（Best Match 25）算法，是 TF-IDF 的改进版：

**TF（词频，Term Frequency）**：词在文档中出现越多，得分越高（有饱和上限）
$$TF_{norm} = \frac{freq}{freq + k_1 \cdot (1 - b + b \cdot \frac{dl}{avgdl})}$$

**IDF（逆文档频率，Inverse Document Frequency）**：词越稀有，IDF 越大
$$IDF = \log(1 + \frac{N - n + 0.5}{n + 0.5})$$

- `N`：总文档数
- `n`：包含该词的文档数
- `k1`：控制 TF 饱和速度（默认 1.2）
- `b`：字段长度归一化系数（默认 0.75，0=不归一化，1=完全归一化）

### 2.2 调整评分

```json
// 调整 BM25 参数
PUT /products/_settings
{
  "index.similarity.default": {
    "type": "BM25",
    "k1": 1.5,
    "b": 0.6
  }
}

// 字段 boost（查询时临时加权）
GET /products/_search
{
  "query": {
    "multi_match": {
      "query": "手机",
      "fields": ["title^3", "description^1", "brand^2"]
    }
  }
}

// function_score 自定义评分
GET /products/_search
{
  "query": {
    "function_score": {
      "query": {"match": {"title": "手机"}},
      "functions": [
        {
          "field_value_factor": {
            "field": "sales_count",
            "factor": 1.2,
            "modifier": "log1p",
            "missing": 1
          }
        },
        {
          "gauss": {
            "created_at": {
              "origin": "now",
              "scale": "30d",
              "decay": 0.5
            }
          }
        }
      ],
      "score_mode": "sum",
      "boost_mode": "multiply"
    }
  }
}
```

## 3. Query DSL 全览

### 3.1 查询上下文 vs 过滤上下文

| 类型 | 说明 | 缓存 | 性能 |
|------|------|------|------|
| **Query Context** | 计算相关性评分（_score） | 不缓存 | 较慢 |
| **Filter Context** | 是/否匹配，不计算评分 | **可缓存** | **快** |

```json
GET /products/_search
{
  "query": {
    "bool": {
      "must": [                           // Query Context → 影响评分
        {"match": {"title": "手机"}}
      ],
      "filter": [                         // Filter Context → 不影响评分，可缓存
        {"term": {"brand": "Apple"}},
        {"range": {"price": {"gte": 1000, "lte": 8000}}},
        {"term": {"is_active": true}}
      ],
      "must_not": [
        {"term": {"category": "二手"}}
      ],
      "should": [
        {"term": {"tags": "旗舰"}},
        {"term": {"tags": "5G"}}
      ],
      "minimum_should_match": 1
    }
  }
}
```

**实践原则：能用 filter 就用 filter！**

### 3.2 叶子查询类型

**精确查询（走 filter）：**

```json
// term：精确匹配单个值（keyword/数值/布尔）
{"term": {"status": "active"}}

// terms：匹配多个值（IN 语义）
{"terms": {"status": ["active", "pending"]}}

// range：范围查询
{"range": {"price": {"gte": 1000, "lt": 5000}}}
{"range": {"created_at": {"gte": "2024-01-01", "lte": "now"}}}

// exists：字段存在
{"exists": {"field": "description"}}

// ids：批量查询文档
{"ids": {"values": ["1", "2", "3"]}}
```

**全文查询（走 query context）：**

```json
// match：标准全文搜索（经过分词）
{"match": {"title": {"query": "苹果手机", "operator": "and"}}}

// match_phrase：短语匹配（词项必须相邻且顺序一致）
{"match_phrase": {"title": "苹果 手机"}}

// match_phrase_prefix：前缀匹配（自动补全）
{"match_phrase_prefix": {"title": "苹果手"}}

// multi_match：多字段搜索
{"multi_match": {"query": "手机", "fields": ["title^2", "description"]}}

// query_string：支持 Lucene 语法（强大但危险）
{"query_string": {"query": "title:手机 AND price:[1000 TO 5000]"}}
```

**模糊查询：**

```json
// fuzzy：允许编辑距离（处理拼写错误）
{"fuzzy": {"title": {"value": "elasticsearh", "fuzziness": "AUTO"}}}

// wildcard：通配符（? 单字符，* 多字符，性能差）
{"wildcard": {"title": "苹果*"}}

// prefix：前缀匹配
{"prefix": {"title": {"value": "苹果"}}}
```

## 4. 聚合（Aggregation）

### 4.1 聚合类型

**Bucket 聚合（分桶，类似 GROUP BY）：**

```json
GET /products/_search
{
  "aggs": {
    // terms：按 keyword 字段分桶
    "by_brand": {
      "terms": {"field": "brand", "size": 10, "min_doc_count": 1}
    },
    // date_histogram：按时间分桶
    "by_month": {
      "date_histogram": {
        "field": "created_at",
        "calendar_interval": "month",
        "format": "yyyy-MM"
      }
    },
    // range：按范围分桶
    "price_ranges": {
      "range": {
        "field": "price",
        "ranges": [{"to": 1000}, {"from": 1000, "to": 3000}, {"from": 3000}]
      }
    },
    // histogram：数值等宽分桶
    "price_histogram": {
      "histogram": {"field": "price", "interval": 500}
    }
  },
  "size": 0
}
```

**Metric 聚合（计算统计指标）：**

```json
"aggs": {
  "avg_price": {"avg": {"field": "price"}},
  "max_price": {"max": {"field": "price"}},
  "min_price": {"min": {"field": "price"}},
  "total_sales": {"sum": {"field": "sales_count"}},
  "doc_count": {"value_count": {"field": "price"}},
  "price_stats": {"stats": {"field": "price"}},           // 一次返回 min/max/avg/sum/count
  "price_percentiles": {
    "percentiles": {"field": "price", "percents": [50, 90, 95, 99]}
  }
}
```

**Pipeline 聚合（基于其他聚合结果计算）：**

```json
"aggs": {
  "by_month": {
    "date_histogram": {"field": "created_at", "calendar_interval": "month"},
    "aggs": {
      "monthly_sales": {"sum": {"field": "amount"}},
      "sales_derivative": {
        "derivative": {"buckets_path": "monthly_sales"}   // 环比增量
      },
      "moving_avg": {
        "moving_avg": {"buckets_path": "monthly_sales", "window": 3}  // 3月移动平均
      }
    }
  }
}
```

### 4.2 嵌套聚合（子聚合）

```json
GET /products/_search
{
  "aggs": {
    "by_brand": {
      "terms": {"field": "brand"},
      "aggs": {
        "avg_price": {"avg": {"field": "price"}},
        "price_by_category": {
          "terms": {"field": "category"},
          "aggs": {
            "count": {"value_count": {"field": "_id"}}
          }
        }
      }
    }
  },
  "size": 0
}
```

## 5. 深分页问题与解决方案

### 5.1 深分页问题

```
集群有 5 个主分片，查询 from=10000, size=10：
  每个分片返回 10010 条文档
  协调节点接收 5×10010 = 50050 条，排序后取第 10001~10010 条
  内存占用极大，性能极差
```

**ES 默认限制**：`index.max_result_window = 10000`（超过会报错）

### 5.2 解决方案

**方案一：search_after（推荐，游标翻页）**

```json
// 第一页
GET /products/_search
{
  "query": {"match_all": {}},
  "sort": [{"created_at": "desc"}, {"_id": "asc"}],
  "size": 20
}

// 下一页（使用上一页最后一条记录的 sort 值）
GET /products/_search
{
  "query": {"match_all": {}},
  "sort": [{"created_at": "desc"}, {"_id": "asc"}],
  "size": 20,
  "search_after": ["2024-03-15T10:30:00Z", "product-12345"]
}
```

**优点**：无内存限制，性能恒定。**限制**：不能跳页，只能顺序翻页。

**方案二：scroll API（适合大数据导出）**

```json
// 开启 scroll，保留上下文 1 分钟
POST /products/_search?scroll=1m
{
  "query": {"match_all": {}},
  "size": 1000
}

// 使用 scroll_id 翻页
POST /_search/scroll
{
  "scroll": "1m",
  "scroll_id": "DXF1ZXJ5QW5kRmV0Y2g..."
}

// 完成后清除 scroll 上下文（释放内存）
DELETE /_search/scroll
{"scroll_id": "DXF1ZXJ5QW5kRmV0Y2g..."}
```

**scroll vs search_after 选择：**
- 实时翻页场景（用户翻页） → `search_after`
- 批量导出、全量遍历 → `scroll`（ES 8.x 推荐 PIT + search_after）

**方案三：PIT（Point In Time）+ search_after（ES 7.10+，最佳实践）**

```json
// 创建 PIT 快照
POST /products/_pit?keep_alive=1m

// 使用 PIT 查询（保证翻页时数据一致性）
GET /_search
{
  "pit": {"id": "...", "keep_alive": "1m"},
  "sort": [{"_shard_doc": "asc"}],
  "search_after": [1234567890],
  "size": 1000
}
```

## 6. 搜索建议（Suggestion API）

```json
GET /products/_search
{
  "suggest": {
    // Term Suggester：单词纠错
    "title_suggest": {
      "text": "iphone elasic",
      "term": {
        "field": "title",
        "suggest_mode": "missing"   // always / missing / popular
      }
    },
    // Phrase Suggester：短语纠错
    "phrase_suggest": {
      "text": "elastic serch",
      "phrase": {
        "field": "title",
        "gram_size": 2
      }
    },
    // Completion Suggester：自动补全（需要 completion 类型字段）
    "title_autocomplete": {
      "prefix": "iph",
      "completion": {
        "field": "title_suggest",
        "size": 5,
        "skip_duplicates": true
      }
    }
  }
}
```

**Completion Suggester 需要特殊 Mapping：**

```json
"title_suggest": {
  "type": "completion",
  "analyzer": "ik_max_word"
}
```
