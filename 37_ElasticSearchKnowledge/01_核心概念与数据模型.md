## 1. 核心概念对比（与关系型数据库）

| Elasticsearch | 关系型数据库 | 说明 |
|--------------|-------------|------|
| Index（索引） | Database（数据库） | 文档的逻辑集合 |
| ~~Type（类型）~~ | Table（表） | **ES 7.x 废弃，8.x 删除** |
| Document（文档） | Row（行） | 最小存储单元，JSON 格式 |
| Field（字段） | Column（列） | 文档的属性 |
| Mapping（映射） | Schema（结构定义） | 字段类型定义 |
| Shard（分片） | Partition（分区） | 数据水平切分 |

**ES 不是关系型数据库的替代品，而是补充**：
- ES 擅长：全文搜索、复杂聚合、近实时查询
- DB 擅长：事务、强一致性、复杂关联查询

## 2. 索引（Index）

**索引是文档的逻辑容器**，对应 Lucene 层面的一组分片（Shard）。

**索引操作：**

```json
// 创建索引
PUT /products
{
  "settings": {
    "number_of_shards": 3,
    "number_of_replicas": 1,
    "refresh_interval": "1s",
    "index.codec": "best_compression"
  },
  "mappings": {
    "properties": {
      "title": {"type": "text", "analyzer": "ik_max_word"}
    }
  }
}

// 查看索引
GET /products
GET /_cat/indices?v&s=index

// 删除索引
DELETE /products

// 关闭索引（节省资源，不可读写）
POST /products/_close

// 打开索引
POST /products/_open
```

## 3. 文档（Document）

**文档是 ES 中最小的数据存储单元**，以 JSON 格式存储。

**文档的元数据字段：**

| 字段 | 说明 |
|------|------|
| `_index` | 所属索引名 |
| `_id` | 文档唯一标识（不指定则自动生成） |
| `_version` | 版本号（每次更新自增，用于乐观锁） |
| `_score` | 相关性得分（搜索时返回） |
| `_source` | 原始 JSON 内容 |
| `_routing` | 路由值（默认为 `_id`） |

**CRUD 操作：**

```json
// 写入（指定 ID）
PUT /products/_doc/1
{"title": "iPhone 15", "price": 5999, "brand": "Apple"}

// 写入（自动生成 ID）
POST /products/_doc
{"title": "Macbook Pro", "price": 12999}

// 获取
GET /products/_doc/1

// 更新（局部更新）
POST /products/_update/1
{"doc": {"price": 5499}}

// 删除
DELETE /products/_doc/1

// 批量操作（Bulk API）
POST /_bulk
{"index": {"_index": "products", "_id": "1"}}
{"title": "iPhone 15", "price": 5999}
{"update": {"_index": "products", "_id": "1"}}
{"doc": {"price": 5499}}
{"delete": {"_index": "products", "_id": "2"}}
```

## 4. 分片（Shard）与副本（Replica）

**分片（Primary Shard）**：索引数据的水平切分单元。
- 数量在索引创建时确定，**不可修改**（可通过 Split/Shrink API 调整）
- 每个分片是一个独立的 Lucene 实例

**副本（Replica Shard）**：主分片的完整复制。
- 提供高可用（主分片节点故障时，副本晋升为主分片）
- 提供读能力（查询可在主分片或副本上执行）
- **副本数可随时修改**

**分片数量经验法则：**

```
目标：单个分片 10GB ~ 50GB
公式：主分片数 = ceil(数据总量 / 目标分片大小)

例：100GB 数据，目标 20GB/分片 → 5 个主分片
建议：每个节点不超过 200 个分片（包含副本）
```

**查看分片分布：**

```bash
GET /_cat/shards?v&h=index,shard,prirep,state,docs,store,node&s=index
```

## 5. Mapping（映射）

**Mapping 是字段的类型定义**，类似关系型数据库的 Schema。

### 5.1 动态 Mapping vs 静态 Mapping

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 动态 Mapping | 写入时自动推断类型 | 开发/探索阶段 |
| 静态 Mapping | 提前明确定义类型 | **生产环境强烈推荐** |

**动态 Mapping 的类型推断规则：**

| JSON 值 | ES 类型 |
|---------|---------|
| `true/false` | `boolean` |
| `123` | `long` |
| `1.5` | `float` |
| `"2024-01-01"` | `date` |
| `"hello"` | `text` + `keyword` 子字段 |
| `{"a": 1}` | `object` |
| `[1, 2, 3]` | `long` 数组 |

**关闭动态 Mapping（生产环境推荐）：**

```json
PUT /products
{
  "mappings": {
    "dynamic": "strict",
    "properties": {
      "title": {"type": "text"}
    }
  }
}
```

`dynamic` 取值：
- `true`：自动创建字段（默认）
- `false`：新字段被忽略，不索引但保留在 `_source`
- `strict`：新字段直接报错

### 5.2 Mapping 最佳实践

```json
PUT /orders
{
  "mappings": {
    "dynamic": "strict",
    "properties": {
      "order_id": {
        "type": "keyword"          // ID 类字段用 keyword，精确匹配
      },
      "user_name": {
        "type": "keyword"          // 不需要分词的字符串用 keyword
      },
      "description": {
        "type": "text",            // 需要全文搜索的字段用 text
        "analyzer": "ik_max_word",
        "search_analyzer": "ik_smart",
        "fields": {
          "keyword": {             // 同时支持精确匹配和聚合
            "type": "keyword",
            "ignore_above": 256
          }
        }
      },
      "amount": {
        "type": "scaled_float",    // 金额用 scaled_float，避免浮点精度问题
        "scaling_factor": 100
      },
      "status": {
        "type": "keyword"
      },
      "created_at": {
        "type": "date",
        "format": "yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis"
      },
      "tags": {
        "type": "keyword"          // 数组字段不需要特殊声明
      },
      "address": {
        "type": "object",          // 嵌套对象
        "properties": {
          "city": {"type": "keyword"},
          "detail": {"type": "text"}
        }
      }
    }
  }
}
```

## 6. 数据类型速查

### 字符串类型
| 类型 | 说明 | 适用场景 |
|------|------|---------|
| `text` | 全文搜索，经过分词处理 | 文章内容、商品描述 |
| `keyword` | 精确匹配、排序、聚合 | ID、状态码、标签、邮箱 |
| `match_only_text` | 只支持全文搜索，节省磁盘 | 日志 message 字段 |

### 数值类型
| 类型 | 范围 | 说明 |
|------|------|------|
| `byte` | -128 ~ 127 | 8位整数 |
| `short` | -32768 ~ 32767 | 16位整数 |
| `integer` | ±2.1亿 | 32位整数 |
| `long` | ±9.2×10¹⁸ | 64位整数（默认整数类型） |
| `float` | 32位浮点 | 普通小数 |
| `double` | 64位浮点 | 高精度小数 |
| `scaled_float` | 缩放浮点 | **金额推荐**（如 scaling_factor=100） |

### 日期类型
```json
"created_at": {
  "type": "date",
  "format": "yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis"
}
```

### 布尔类型
```json
"is_active": {"type": "boolean"}
```

### 复杂类型
| 类型 | 说明 | 场景 |
|------|------|------|
| `object` | 普通嵌套对象（flat 存储） | 地址、配置项 |
| `nested` | 独立 Lucene 文档，保持对象数组独立性 | 订单明细、商品属性 |
| `flattened` | 整个对象作为一个 keyword | 动态 key 的 map |

### 地理类型
```json
"location": {"type": "geo_point"},      // 点（经纬度）
"area": {"type": "geo_shape"}            // 形状（多边形等）
```

### 向量类型
```json
"embedding": {
  "type": "dense_vector",
  "dims": 768,
  "index": true,
  "similarity": "cosine"
}
```

## 7. 分析器（Analyzer）

**分析器 = Char Filter + Tokenizer + Token Filter 的组合**

```
原始文本 → [Char Filter] 字符过滤 → [Tokenizer] 分词 → [Token Filter] 词项过滤 → Token 列表
```

**内置分析器：**

| 分析器 | 说明 | 适用 |
|--------|------|------|
| `standard` | Unicode 标准分词，小写转换，删除标点 | 英文、混合文本 |
| `english` | 英文分词 + stemming（词干提取）| 英文全文搜索 |
| `keyword` | 不分词，整体作为一个 Token | 精确匹配 |
| `whitespace` | 按空格分词 | 简单英文 |
| `ik_max_word` | 中文细粒度分词（IK 插件） | **中文推荐** |
| `ik_smart` | 中文粗粒度分词 | **中文搜索推荐** |

**测试分析器：**

```json
GET /_analyze
{
  "analyzer": "ik_max_word",
  "text": "Elasticsearch 全文搜索引擎"
}
```

**自定义分析器：**

```json
PUT /my-index
{
  "settings": {
    "analysis": {
      "char_filter": {
        "html_strip": {"type": "html_strip"}
      },
      "tokenizer": {
        "my_tokenizer": {
          "type": "standard",
          "max_token_length": 10
        }
      },
      "filter": {
        "my_stop": {
          "type": "stop",
          "stopwords": ["的", "了", "是", "在"]
        },
        "my_synonym": {
          "type": "synonym",
          "synonyms": ["手机, 移动电话", "ES => Elasticsearch"]
        }
      },
      "analyzer": {
        "my_analyzer": {
          "type": "custom",
          "char_filter": ["html_strip"],
          "tokenizer": "ik_max_word",
          "filter": ["lowercase", "my_stop", "my_synonym"]
        }
      }
    }
  }
}
```

## 8. ZXID 等价——TUID（文档版本控制）

**ES 通过版本号实现乐观并发控制：**

```json
// 基于 _seq_no 和 _primary_term 的乐观锁（推荐，ES 7.x+）
PUT /products/_doc/1?if_seq_no=10&if_primary_term=1
{"title": "iPhone 15 Pro", "price": 7999}
// 如果 _seq_no 不匹配，返回 409 Conflict

// 基于 version 的外部版本控制（适合从 DB 同步数据）
PUT /products/_doc/1?version=5&version_type=external
{"title": "iPhone 15", "price": 5999}
```

**`_seq_no` vs `_version`：**
- `_version`：文档级别，每次写操作自增
- `_seq_no`：分片级别，单调递增，重启后不归零
- `_primary_term`：主分片 term，每次 Primary 切换自增

## 9. 核心配置速查（elasticsearch.yml）

```yaml
# 集群
cluster.name: my-production-cluster
node.name: node-1

# 网络
network.host: 192.168.1.10
http.port: 9200
transport.port: 9300

# 发现（集群组建）
discovery.seed_hosts: ["192.168.1.10:9300", "192.168.1.11:9300", "192.168.1.12:9300"]
cluster.initial_master_nodes: ["node-1", "node-2", "node-3"]  # 仅首次启动

# 节点角色（ES 7.9+ 新角色配置）
node.roles: [master, data, ingest]
# 专用 Master 节点：node.roles: [master]
# 专用 Data 节点：node.roles: [data]

# 内存
bootstrap.memory_lock: true  # 锁定内存，防止 swap

# 路径
path.data: /var/lib/elasticsearch
path.logs: /var/log/elasticsearch
```
