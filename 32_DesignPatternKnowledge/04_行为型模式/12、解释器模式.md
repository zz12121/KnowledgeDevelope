# 解释器模式（Interpreter）

> 为简单语言定义文法表示并提供解释器，将语言中的句子解析为操作。适合：简单 DSL、规则引擎、表达式解析。复杂语法请用专业解析器。

---

## 故事引入

### 《这个语法怎么解释？》

当你学Java时，看到代码 `int x = 5;`，编译器需要"解释"这句话：这是一个整数变量的声明，变量名叫x，初始值是5。

**解释器模式**：定义语言的文法，以及对句子的解释方法。

## 一、定义与意图

为简单语言定义文法表示并提供解释器，将语言中的句子解析为操作。适合：简单 DSL、规则引擎、表达式解析。复杂语法请用专业解析器。

## 二、Demo

```java
// 场景：权限规则表达式解释器
// 规则：ROLE:ADMIN AND (PERM:READ OR PERM:WRITE)
public interface PermissionExpression {
    boolean evaluate(UserContext context);
    String describe();
}

// 终结符：角色检查
public class RoleExpression implements PermissionExpression {
    private final String role;
    public RoleExpression(String role) { this.role = role; }
    public boolean evaluate(UserContext context) { return context.hasRole(role); }
    public String describe() { return "ROLE:" + role; }
}

// 终结符：权限检查
public class PermissionExpression2 implements PermissionExpression {
    private final String permission;
    public PermissionExpression2(String permission) { this.permission = permission; }
    public boolean evaluate(UserContext context) { return context.hasPermission(permission); }
    public String describe() { return "PERM:" + permission; }
}

// 非终结符：AND
public class AndExpression implements PermissionExpression {
    private final PermissionExpression left, right;
    public AndExpression(PermissionExpression l, PermissionExpression r) { left = l; right = r; }
    public boolean evaluate(UserContext context) {
        return left.evaluate(context) && right.evaluate(context);
    }
    public String describe() { return "(" + left.describe() + " AND " + right.describe() + ")"; }
}

// 非终结符：OR
public class OrExpression implements PermissionExpression {
    private final PermissionExpression left, right;
    public OrExpression(PermissionExpression l, PermissionExpression r) { left = l; right = r; }
    public boolean evaluate(UserContext context) {
        return left.evaluate(context) || right.evaluate(context);
    }
    public String describe() { return "(" + left.describe() + " OR " + right.describe() + ")"; }
}

// 非终结符：NOT
public class NotExpression implements PermissionExpression {
    private final PermissionExpression expr;
    public NotExpression(PermissionExpression expr) { this.expr = expr; }
    public boolean evaluate(UserContext context) { return !expr.evaluate(context); }
    public String describe() { return "NOT(" + expr.describe() + ")"; }
}

// 规则构建与使用
public class PermissionRuleEngine {
    public boolean check(PermissionExpression rule, UserContext user) {
        boolean result = rule.evaluate(user);
        System.out.printf("规则[%s] -> 用户[%s]: %s%n",
                rule.describe(), user.getUsername(), result ? "通过" : "拒绝");
        return result;
    }
}

// 使用
PermissionRuleEngine engine = new PermissionRuleEngine();

// 规则：必须是管理员，或者有读写权限
PermissionExpression rule = new OrExpression(
    new RoleExpression("ADMIN"),
    new AndExpression(
        new PermissionExpression2("READ"),
        new PermissionExpression2("WRITE")
    )
);

UserContext admin = new UserContext("alice", Set.of("ADMIN"), Set.of());
UserContext reader = new UserContext("bob", Set.of(), Set.of("READ", "WRITE"));
UserContext guest = new UserContext("charlie", Set.of(), Set.of("READ"));

engine.check(rule, admin);  // 通过（是管理员）
engine.check(rule, reader); // 通过（有读写权限）
engine.check(rule, guest);  // 拒绝（只有读权限）
```
