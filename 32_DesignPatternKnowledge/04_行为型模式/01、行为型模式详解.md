# 行为型模式详解（11种）

行为型模式关注**对象之间的职责分配和通信机制**，描述运行时对象如何相互协作完成复杂任务。GoF 定义 11 种行为型模式。

---

## 一、模板方法模式（Template Method）

### 定义与意图

在**父类（抽象类）**中定义算法骨架，把某些步骤的实现推迟到子类。模板方法用 `final` 防止子类改变流程，子类只能覆盖指定的"可变步骤"。

### Demo

```java
// 数据导入模板：固定"读文件→解析→验证→存库"流程，解析方式可变
public abstract class DataImporter<T> {

    // 模板方法：固定流程骨架
    public final ImportResult importData(String filePath) {
        List<String> rawLines = readFile(filePath);          // 1. 读文件（固定）
        if (rawLines.isEmpty()) return ImportResult.empty();

        List<T> records = parseLines(rawLines);              // 2. 解析（可变）
        List<ValidationError> errors = validateRecords(records); // 3. 验证（可变）
        if (!errors.isEmpty()) {
            if (!shouldContinueOnError()) {                  // 4. 钩子：是否遇错继续
                return ImportResult.fail(errors);
            }
        }
        int savedCount = saveRecords(records);               // 5. 存库（可变）
        afterImport(savedCount);                             // 6. 钩子：导入后处理

        return ImportResult.success(savedCount, errors);
    }

    // 固定步骤
    private List<String> readFile(String filePath) {
        try {
            return Files.readAllLines(Paths.get(filePath), StandardCharsets.UTF_8);
        } catch (Exception e) { throw new RuntimeException("读文件失败: " + filePath, e); }
    }

    // 抽象步骤（子类必须实现）
    protected abstract List<T> parseLines(List<String> lines);
    protected abstract List<ValidationError> validateRecords(List<T> records);
    protected abstract int saveRecords(List<T> records);

    // 钩子方法（子类可选覆盖）
    protected boolean shouldContinueOnError() { return false; }  // 默认遇错停止
    protected void afterImport(int count) {
        System.out.println("导入完成，共导入 " + count + " 条记录");
    }
}

// 具体实现：CSV 用户导入
@Component
public class UserCsvImporter extends DataImporter<User> {
    @Autowired private UserRepository userRepository;
    @Autowired private UserValidator userValidator;

    @Override
    protected List<User> parseLines(List<String> lines) {
        return lines.stream()
                .filter(line -> !line.startsWith("#")) // 跳过注释行
                .map(line -> {
                    String[] parts = line.split(",");
                    return User.builder()
                            .username(parts[0].trim())
                            .email(parts[1].trim())
                            .phone(parts.length > 2 ? parts[2].trim() : null)
                            .build();
                })
                .collect(Collectors.toList());
    }

    @Override
    protected List<ValidationError> validateRecords(List<User> users) {
        return users.stream()
                .flatMap(u -> userValidator.validate(u).getErrors().stream())
                .collect(Collectors.toList());
    }

    @Override
    protected int saveRecords(List<User> users) {
        userRepository.saveAll(users);
        return users.size();
    }

    @Override
    protected boolean shouldContinueOnError() { return true; } // 用户导入允许部分失败
}
```

## 二、策略模式（Strategy）

### 定义与意图

定义一组算法，把每个算法封装成独立类，使它们可以相互替换。策略模式让算法的变化独立于使用算法的客户端。核心：Context 持有 Strategy 接口的引用，运行时可以动态切换。

### Demo

```java
// 场景：电商平台多种促销策略
public interface PromotionStrategy {
    BigDecimal calculate(BigDecimal originalPrice, Order order);
    String getName();
    String getCode();
}

// 满减策略
@Component("fullReduceStrategy")
public class FullReduceStrategy implements PromotionStrategy {
    private final BigDecimal threshold;
    private final BigDecimal reduceAmount;

    public FullReduceStrategy() {
        this.threshold = new BigDecimal("200");
        this.reduceAmount = new BigDecimal("30");
    }

    @Override
    public BigDecimal calculate(BigDecimal originalPrice, Order order) {
        if (originalPrice.compareTo(threshold) >= 0) {
            return originalPrice.subtract(reduceAmount);
        }
        return originalPrice;
    }

    @Override
    public String getName() { return "满200减30"; }

    @Override
    public String getCode() { return "FULL_REDUCE"; }
}

// 折扣策略
@Component("discountStrategy")
public class DiscountStrategy implements PromotionStrategy {
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, Order order) {
        // 会员九折，非会员九五折
        BigDecimal rate = order.isMember() ? new BigDecimal("0.9") : new BigDecimal("0.95");
        return originalPrice.multiply(rate).setScale(2, RoundingMode.HALF_UP);
    }

    @Override
    public String getName() { return "会员折扣"; }

    @Override
    public String getCode() { return "MEMBER_DISCOUNT"; }
}

// 新人优惠券
@Component("newUserCouponStrategy")
public class NewUserCouponStrategy implements PromotionStrategy {
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, Order order) {
        if (order.isNewUser()) {
            return originalPrice.subtract(new BigDecimal("50")).max(BigDecimal.ZERO);
        }
        return originalPrice;
    }

    @Override
    public String getName() { return "新人专享-50元"; }

    @Override
    public String getCode() { return "NEW_USER_COUPON"; }
}

// 策略上下文：Spring 自动注入所有策略
@Component
public class PromotionContext {
    private final Map<String, PromotionStrategy> strategies;

    @Autowired
    public PromotionContext(List<PromotionStrategy> strategyList) {
        this.strategies = strategyList.stream()
                .collect(Collectors.toMap(PromotionStrategy::getCode, s -> s));
    }

    public BigDecimal applyPromotion(String promotionCode, BigDecimal price, Order order) {
        PromotionStrategy strategy = strategies.getOrDefault(promotionCode,
                (p, o) -> p); // 默认策略：不优惠
        BigDecimal finalPrice = strategy.calculate(price, order);
        System.out.printf("促销[%s]：¥%.2f -> ¥%.2f%n", promotionCode, price, finalPrice);
        return finalPrice;
    }

    public List<String> getAvailablePromotions(Order order) {
        return strategies.values().stream()
                .map(PromotionStrategy::getName)
                .collect(Collectors.toList());
    }
}
```

## 三、观察者模式（Observer）

### 定义与意图

定义一种**一对多的依赖关系**，当主题（被观察者）状态改变时，所有观察者自动收到通知并更新。

### Demo

```java
// 原生观察者实现（不依赖 Spring）
public interface EventListener<T> {
    void onEvent(T event);
}

public class EventPublisher<T> {
    private final List<EventListener<T>> listeners = new CopyOnWriteArrayList<>();

    public void subscribe(EventListener<T> listener) { listeners.add(listener); }
    public void unsubscribe(EventListener<T> listener) { listeners.remove(listener); }

    public void publish(T event) {
        listeners.forEach(listener -> {
            try {
                listener.onEvent(event);
            } catch (Exception e) {
                System.err.println("监听器处理异常: " + e.getMessage());
            }
        });
    }
}

// 领域事件
public class StockUpdatedEvent {

    private final int oldStock;

    private final LocalDateTime occurredAt;

    public Long getProductId() { return productId; }
```

## 四、责任链模式（Chain of Responsibility）

### 定义与意图

将请求的处理者组成一条链，每个处理者决定自己是否处理该请求，不处理则沿链传递。客户端只需把请求发给链的头部，不需要知道哪个处理者会最终响应。

### Demo

```java
// 场景：API 请求处理管道（认证→鉴权→限流→日志→业务）
public interface Filter {
    boolean doFilter(ApiRequest request, ApiResponse response, FilterChain chain);
    int getOrder();
}

public class FilterChain {
    private final List<Filter> filters;
    private int currentIndex = 0;

    public FilterChain(List<Filter> filters) {
        this.filters = filters.stream()
                .sorted(Comparator.comparingInt(Filter::getOrder))
                .collect(Collectors.toList());
    }

    public boolean proceed(ApiRequest request, ApiResponse response) {
        if (currentIndex < filters.size()) {
            Filter filter = filters.get(currentIndex++);
            return filter.doFilter(request, response, this);
        }
        return true; // 所有过滤器都通过
    }
}

// 过滤器1：JWT 认证（最先执行）
@Component
public class JwtAuthFilter implements Filter {
    @Override
    public boolean doFilter(ApiRequest request, ApiResponse response, FilterChain chain) {
        String token = request.getHeader("Authorization");
        if (token == null || !token.startsWith("Bearer ")) {
            response.setStatus(401);
            response.setBody("{\"error\":\"Missing or invalid token\"}");
            return false; // 中断链
        }
        try {
            Claims claims = JwtUtils.parse(token.substring(7));
            request.setAttribute("userId", claims.getSubject());
            return chain.proceed(request, response); // 继续传递
        } catch (Exception e) {
            response.setStatus(401);
            return false;
        }
    }

    @Override
    public int getOrder() { return 1; }
}

// 过滤器2：权限检查
@Component
public class AuthorizationFilter implements Filter {
    @Override
    public boolean doFilter(ApiRequest request, ApiResponse response, FilterChain chain) {
        String userId = (String) request.getAttribute("userId");
        String requiredPermission = getRequiredPermission(request.getPath());
        if (!hasPermission(userId, requiredPermission)) {
            response.setStatus(403);
            response.setBody("{\"error\":\"Forbidden\"}");
            return false;
        }
        return chain.proceed(request, response);
    }

    private String getRequiredPermission(String path) { return "USER_ACCESS"; }
    private boolean hasPermission(String userId, String permission) { return true; }

    @Override
    public int getOrder() { return 2; }
}

// 过滤器3：限流
@Component
public class RateLimitFilter implements Filter {
    private final Map<String, AtomicInteger> requestCount = new ConcurrentHashMap<>();

    @Override
    public boolean doFilter(ApiRequest request, ApiResponse response, FilterChain chain) {
        String ip = request.getClientIp();
        int count = requestCount.computeIfAbsent(ip, k -> new AtomicInteger(0)).incrementAndGet();
        if (count > 100) { // 每分钟100次
            response.setStatus(429);
            response.setBody("{\"error\":\"Too Many Requests\"}");
            return false;
        }
        return chain.proceed(request, response);
    }

    @Override
    public int getOrder() { return 3; }
}

// 过滤器4：请求日志（最后执行）
@Component
public class RequestLoggingFilter implements Filter {
    @Override
    public boolean doFilter(ApiRequest request, ApiResponse response, FilterChain chain) {
        long start = System.currentTimeMillis();
        boolean result = chain.proceed(request, response);
        System.out.printf("[%s] %s -> %d (%dms)%n",
                request.getMethod(), request.getPath(), response.getStatus(),
                System.currentTimeMillis() - start);
        return result;
    }

    @Override
    public int getOrder() { return Integer.MAX_VALUE; } // 日志最后执行
}
```

## 五、命令模式（Command）

### 定义与意图

将请求封装成对象，支持**撤销/重做、排队、延迟执行**。命令对象包含执行者（Receiver）和操作信息，调用者（Invoker）持有命令对象并在合适时机触发。

### Demo

```java
// 场景：智能家居控制台（支持撤销）
public interface SmartHomeCommand {
    void execute();
    void undo();
    String describe();
}

// 接收者：灯光控制器
public class LightController {
    private boolean on = false;
    private int brightness = 100;
    private final String room;

    public LightController(String room) { this.room = room; }

    public void turnOn(int brightness) {
        this.on = true; this.brightness = brightness;
        System.out.printf("[%s] 灯光开启，亮度 %d%%%n", room, brightness);
    }

    public void turnOff() {
        this.on = false;
        System.out.printf("[%s] 灯光关闭%n", room);
    }

    public boolean isOn() { return on; }
    public int getBrightness() { return brightness; }
}

// 具体命令：开灯
public class TurnOnLightCommand implements SmartHomeCommand {
    private final LightController light;
    private final int brightness;
    private boolean wasOn;
    private int prevBrightness;

    public TurnOnLightCommand(LightController light, int brightness) {
        this.light = light; this.brightness = brightness;
    }

    @Override
    public void execute() {
        // 保存状态以便撤销
        wasOn = light.isOn();
        prevBrightness = light.getBrightness();
        light.turnOn(brightness);
    }

    @Override
    public void undo() {
        if (wasOn) {
            light.turnOn(prevBrightness); // 恢复之前的状态
        } else {
            light.turnOff();
        }
        System.out.println("撤销：恢复灯光状态");
    }

    @Override
    public String describe() { return "开灯（亮度" + brightness + "%）"; }
}

// 宏命令：一键回家（批量执行）
public class ComeHomeCommand implements SmartHomeCommand {
    private final List<SmartHomeCommand> commands;

    public ComeHomeCommand(List<SmartHomeCommand> commands) {
        this.commands = new ArrayList<>(commands);
    }

    @Override
    public void execute() {
        System.out.println("=== 一键回家模式 ===");
        commands.forEach(SmartHomeCommand::execute);
    }

    @Override
    public void undo() {
        // 逆序撤销
        List<SmartHomeCommand> reversed = new ArrayList<>(commands);
        Collections.reverse(reversed);
        reversed.forEach(SmartHomeCommand::undo);
    }

    @Override
    public String describe() { return "一键回家（" + commands.size() + "个动作）"; }
}

// 调用者：遥控器（持有历史命令，支持多级撤销）
public class SmartRemoteControl {
    private final Deque<SmartHomeCommand> history = new ArrayDeque<>();
    private final int maxHistory;

    public SmartRemoteControl(int maxHistory) { this.maxHistory = maxHistory; }

    public void execute(SmartHomeCommand command) {
        command.execute();
        history.push(command);
        if (history.size() > maxHistory) history.pollLast(); // 超出历史上限
        System.out.println("已执行: " + command.describe());
    }

    public void undo() {
        if (history.isEmpty()) {
            System.out.println("没有可撤销的操作");
            return;
        }
        SmartHomeCommand last = history.pop();
        last.undo();
        System.out.println("已撤销: " + last.describe());
    }
}
```

## 六、迭代器模式（Iterator）

### 定义与意图

提供一种方法**顺序访问集合中的元素**，而不暴露集合的内部结构。Java 集合框架的 `Iterator` 接口是这个模式的标准实现。

### Demo

```java
// 场景：分页查询迭代器（懒加载，每次只拉一页数据）
public class PagedIterator<T> implements Iterator<T> {
    private final Function<PageRequest, Page<T>> dataFetcher;
    private final int pageSize;
    private int currentPage = 0;
    private List<T> currentPageData = new ArrayList<>();
    private int indexInPage = 0;
    private boolean hasMorePages = true;

    public PagedIterator(Function<PageRequest, Page<T>> dataFetcher, int pageSize) {
        this.dataFetcher = dataFetcher;
        this.pageSize = pageSize;
        fetchNextPage(); // 预加载第一页
    }

    @Override
    public boolean hasNext() {
        if (indexInPage < currentPageData.size()) return true;
        if (!hasMorePages) return false;
        fetchNextPage();
        return !currentPageData.isEmpty();
    }

    @Override
    public T next() {
        if (!hasNext()) throw new NoSuchElementException();
        return currentPageData.get(indexInPage++);
    }

    private void fetchNextPage() {
        Page<T> page = dataFetcher.apply(PageRequest.of(currentPage, pageSize));
        currentPageData = page.getContent();
        indexInPage = 0;
        currentPage++;
        hasMorePages = page.hasNext();
        System.out.println("加载第 " + currentPage + " 页，共 " + currentPageData.size() + " 条");
    }
}

// 使用：对调用方完全透明，感知不到分页
PagedIterator<User> iterator = new PagedIterator<>(
    pageReq -> userRepository.findAll(pageReq), 100
);

while (iterator.hasNext()) {
    User user = iterator.next();
    processUser(user); // 处理每个用户，不用关心背后的分页逻辑
}

// 包装成 Iterable，支持 for-each
public class PagedIterable<T> implements Iterable<T> {
    private final Function<PageRequest, Page<T>> fetcher;
    private final int pageSize;

    public PagedIterable(Function<PageRequest, Page<T>> fetcher, int pageSize) {
        this.fetcher = fetcher; this.pageSize = pageSize;
    }

    @Override
    public Iterator<T> iterator() { return new PagedIterator<>(fetcher, pageSize); }
}

for (User user : new PagedIterable<>(userRepository::findAll, 100)) {
    processUser(user);
}
```

## 七、中介者模式（Mediator）

### 定义与意图

用中介者对象来封装一系列对象之间的交互，使各对象不再直接相互引用，将**网状耦合**简化为**星形结构**。

### Demo

```java
// 场景：机场起降协调系统
public interface AirTrafficControl {
    void requestLanding(Airplane airplane);
    void requestTakeoff(Airplane airplane);
    void notifyAll(String message);
}

// 具体中介者：塔台
public class ControlTower implements AirTrafficControl {
    private final List<Airplane> registeredPlanes = new ArrayList<>();
    private final Set<String> landingApprovals = new ConcurrentHashSet<>();

    public void register(Airplane airplane) { registeredPlanes.add(airplane); }

    @Override
    public void requestLanding(Airplane airplane) {
        boolean runwayClear = isRunwayClear();
        if (runwayClear) {
            landingApprovals.add(airplane.getFlightId());
            System.out.printf("塔台 -> %s: 批准降落，跑道01可用%n", airplane.getFlightId());
            airplane.receiveLandingApproval(true);
        } else {
            System.out.printf("塔台 -> %s: 等待，跑道繁忙，进入等待航线%n", airplane.getFlightId());
            airplane.receiveLandingApproval(false);
        }
    }

    @Override
    public void requestTakeoff(Airplane airplane) {
        System.out.printf("塔台 -> %s: 可以起飞，风速正常%n", airplane.getFlightId());
        // 通知其他飞机跑道被占用
        notifyOthers(airplane, "跑道" + airplane.getFlightId() + "正在起飞，请注意间距");
        airplane.receiveTakeoffApproval();
    }

    @Override
    public void notifyAll(String message) {
        registeredPlanes.forEach(p -> p.receiveMessage(message));
    }

    private void notifyOthers(Airplane except, String message) {
        registeredPlanes.stream()
                .filter(p -> !p.equals(except))
                .forEach(p -> p.receiveMessage(message));
    }

    private boolean isRunwayClear() { return landingApprovals.isEmpty(); }
}

// 同事类：飞机（只和塔台通信，不直接和其他飞机通信）
public class Airplane {
    private final String flightId;
    private final AirTrafficControl tower; // 引用中介者

    public Airplane(String flightId, AirTrafficControl tower) {
        this.flightId = flightId; this.tower = tower;
    }

    public void requestLanding() {
        System.out.printf("%s: 请求降落%n", flightId);
        tower.requestLanding(this); // 通过中介者请求，不直接联系其他飞机
    }

    public void requestTakeoff() {
        System.out.printf("%s: 请求起飞%n", flightId);
        tower.requestTakeoff(this);
    }

    public void receiveLandingApproval(boolean approved) {
        System.out.printf("%s: 收到降落%s%n", flightId, approved ? "批准" : "拒绝");
    }

    public void receiveTakeoffApproval() {
        System.out.printf("%s: 收到起飞批准，正在起飞...%n", flightId);
    }

    public void receiveMessage(String message) {
        System.out.printf("%s 收到通知: %s%n", flightId, message);
    }

    public String getFlightId() { return flightId; }
```

## 八、备忘录模式（Memento）

### 定义与意图

在不破坏封装的前提下，捕获对象的内部状态并保存，以便后续恢复。三个角色：原发器（Originator）负责创建和恢复备忘录；备忘录（Memento）存储状态；负责人（Caretaker）管理备忘录列表。

### Demo

```java
// 场景：代码编辑器的撤销/重做功能
public class CodeEditor {
    private String content;
    private int cursorPosition;

    public CodeEditor() { this.content = ""; this.cursorPosition = 0; }

    // 原发器：创建备忘录（快照）
    public EditorSnapshot createSnapshot() {
        return new EditorSnapshot(content, cursorPosition, LocalDateTime.now());
    }

    // 原发器：从备忘录恢复
    public void restore(EditorSnapshot snapshot) {
        this.content = snapshot.getContent();
        this.cursorPosition = snapshot.getCursorPosition();
        System.out.println("已恢复到: " + snapshot.getTimestamp());
    }

    public void type(String text) {
        content = content.substring(0, cursorPosition) + text + content.substring(cursorPosition);
        cursorPosition += text.length();
    }

    public void deleteChars(int count) {
        int start = Math.max(0, cursorPosition - count);
        content = content.substring(0, start) + content.substring(cursorPosition);
        cursorPosition = start;
    }

    public String getContent() { return content; }
}

// 备忘录：存储编辑器状态（只允许 CodeEditor 创建，外部只能持有引用）
public final class EditorSnapshot {
    private final String content;
    private final int cursorPosition;
    private final LocalDateTime timestamp;

    EditorSnapshot(String content, int cursorPosition, LocalDateTime timestamp) {
        this.content = content; this.cursorPosition = cursorPosition;
        this.timestamp = timestamp;
    }

    String getContent() { return content; }            // 包级别访问，外部无法直接读
    int getCursorPosition() { return cursorPosition; }
    public LocalDateTime getTimestamp() { return timestamp; }
}

// 负责人：历史记录管理（不关心备忘录内容，只管存取）
public class EditorHistory {
    private final Deque<EditorSnapshot> undoStack = new ArrayDeque<>();
    private final Deque<EditorSnapshot> redoStack = new ArrayDeque<>();
    private final int maxHistory;

    public EditorHistory(int maxHistory) { this.maxHistory = maxHistory; }

    public void save(EditorSnapshot snapshot) {
        undoStack.push(snapshot);
        redoStack.clear(); // 新操作后清空 redo 栈
        if (undoStack.size() > maxHistory) undoStack.pollLast();
    }

    public Optional<EditorSnapshot> undo() {
        if (undoStack.isEmpty()) return Optional.empty();
        EditorSnapshot current = undoStack.pop();
        redoStack.push(current);
        return Optional.ofNullable(undoStack.peek()); // 返回上一个状态
    }

    public Optional<EditorSnapshot> redo() {
        if (redoStack.isEmpty()) return Optional.empty();
        EditorSnapshot snapshot = redoStack.pop();
        undoStack.push(snapshot);
        return Optional.of(snapshot);
    }
}
```

## 九、状态模式（State）

### 定义与意图

允许对象在**内部状态改变时改变行为**，把每种状态下的行为封装到独立的状态类，消除大量 if-else 状态判断。

## 十、访问者模式（Visitor）

### 定义与意图

把**数据结构**和**作用于其上的操作**分离。当数据结构稳定但需要频繁新增操作时，新增访问者类即可，不修改数据结构。

### Demo

```java
// 场景：AST（抽象语法树）统计分析
// 节点类型（稳定，不经常变）
public interface AstNode { void accept(AstVisitor visitor); }

public class NumberNode implements AstNode {
    public final double value;
    public NumberNode(double value) { this.value = value; }
    public void accept(AstVisitor visitor) { visitor.visit(this); }
}

public class VariableNode implements AstNode {
    public final String name;
    public VariableNode(String name) { this.name = name; }
    public void accept(AstVisitor visitor) { visitor.visit(this); }
}

public class BinaryOpNode implements AstNode {
    public final String operator;
    public final AstNode left, right;
    public BinaryOpNode(String op, AstNode left, AstNode right) {
        this.operator = op; this.left = left; this.right = right;
    }
    public void accept(AstVisitor visitor) { visitor.visit(this); }
}

// 访问者接口（新增操作 = 新增实现类，不改节点）
public interface AstVisitor {
    void visit(NumberNode node);
    void visit(VariableNode node);
    void visit(BinaryOpNode node);
}

// 访问者1：计算表达式的值
public class EvaluateVisitor implements AstVisitor {
    private final Map<String, Double> variables;
    private double result;

    public EvaluateVisitor(Map<String, Double> variables) { this.variables = variables; }

    public void visit(NumberNode node) { result = node.value; }

    public void visit(VariableNode node) {
        result = variables.getOrDefault(node.name, 0.0);
    }

    public void visit(BinaryOpNode node) {
        node.left.accept(this); double left = result;
        node.right.accept(this); double right = result;
        result = switch (node.operator) {
            case "+" -> left + right;
            case "-" -> left - right;
            case "*" -> left * right;
            case "/" -> right != 0 ? left / right : Double.NaN;
            default -> throw new UnsupportedOperationException();
        };
    }

    public double getResult() { return result; }
}

// 访问者2：统计变量使用次数（新增操作，节点类不变）
public class VariableCountVisitor implements AstVisitor {
    private final Map<String, Integer> counts = new HashMap<>();

    public void visit(NumberNode node) {} // 数字节点不计入
    public void visit(VariableNode node) { counts.merge(node.name, 1, Integer::sum); }
    public void visit(BinaryOpNode node) { node.left.accept(this); node.right.accept(this); }

    public Map<String, Integer> getCounts() { return Collections.unmodifiableMap(counts); }
}

// 使用：(x + 2) * (y - x)
AstNode ast = new BinaryOpNode("*",
    new BinaryOpNode("+", new VariableNode("x"), new NumberNode(2)),
    new BinaryOpNode("-", new VariableNode("y"), new VariableNode("x"))
);

EvaluateVisitor evaluator = new EvaluateVisitor(Map.of("x", 3.0, "y", 7.0));
ast.accept(evaluator);
System.out.println("结果: " + evaluator.getResult()); // (3+2)*(7-3) = 20.0

VariableCountVisitor counter = new VariableCountVisitor();
ast.accept(counter);
System.out.println("变量使用: " + counter.getCounts()); // {x=2, y=1}
```

## 十一、解释器模式（Interpreter）

### 定义与意图

为简单语言定义文法表示并提供解释器，将语言中的句子解析为操作。适合：简单 DSL、规则引擎、表达式解析。复杂语法请用专业解析器。

### Demo

```java
// 场景：权限规则表达式解释器
// 规则：ROLE:ADMIN AND (PERM:READ OR PERM:WRITE)
public interface PermissionExpression {
    boolean evaluate(UserContext context);
    String describe();
}

// 终结符：角色检查
public class RoleExpression implements PermissionExpression {
    private final String role;
    public RoleExpression(String role) { this.role = role; }
    public boolean evaluate(UserContext context) { return context.hasRole(role); }
    public String describe() { return "ROLE:" + role; }
}

// 终结符：权限检查
public class PermissionExpression2 implements PermissionExpression {
    private final String permission;
    public PermissionExpression2(String permission) { this.permission = permission; }
    public boolean evaluate(UserContext context) { return context.hasPermission(permission); }
    public String describe() { return "PERM:" + permission; }
}

// 非终结符：AND
public class AndExpression implements PermissionExpression {
    private final PermissionExpression left, right;
    public AndExpression(PermissionExpression l, PermissionExpression r) { left = l; right = r; }
    public boolean evaluate(UserContext context) {
        return left.evaluate(context) && right.evaluate(context);
    }
    public String describe() { return "(" + left.describe() + " AND " + right.describe() + ")"; }
}

// 非终结符：OR
public class OrExpression implements PermissionExpression {
    private final PermissionExpression left, right;
    public OrExpression(PermissionExpression l, PermissionExpression r) { left = l; right = r; }
    public boolean evaluate(UserContext context) {
        return left.evaluate(context) || right.evaluate(context);
    }
    public String describe() { return "(" + left.describe() + " OR " + right.describe() + ")"; }
}

// 非终结符：NOT
public class NotExpression implements PermissionExpression {
    private final PermissionExpression expr;
    public NotExpression(PermissionExpression expr) { this.expr = expr; }
    public boolean evaluate(UserContext context) { return !expr.evaluate(context); }
    public String describe() { return "NOT(" + expr.describe() + ")"; }
}

// 规则构建与使用
public class PermissionRuleEngine {
    public boolean check(PermissionExpression rule, UserContext user) {
        boolean result = rule.evaluate(user);
        System.out.printf("规则[%s] -> 用户[%s]: %s%n",
                rule.describe(), user.getUsername(), result ? "通过" : "拒绝");
        return result;
    }
}

// 使用
PermissionRuleEngine engine = new PermissionRuleEngine();

// 规则：必须是管理员，或者有读写权限
PermissionExpression rule = new OrExpression(
    new RoleExpression("ADMIN"),
    new AndExpression(
        new PermissionExpression2("READ"),
        new PermissionExpression2("WRITE")
    )
);

UserContext admin = new UserContext("alice", Set.of("ADMIN"), Set.of());
UserContext reader = new UserContext("bob", Set.of(), Set.of("READ", "WRITE"));
UserContext guest = new UserContext("charlie", Set.of(), Set.of("READ"));

engine.check(rule, admin);  // 通过（是管理员）
engine.check(rule, reader); // 通过（有读写权限）
engine.check(rule, guest);  // 拒绝（只有读权限）
```

## 十二、十一种行为型模式对比

| 模式 | 核心解决的问题 | 关键机制 |
|------|--------------|---------|
| 模板方法 | 固定算法骨架，步骤可变 | 继承 + final 模板方法 |
| 策略 | 算法可整体替换 | 组合 Strategy 接口 |
| 观察者 | 一对多的状态同步通知 | 监听器列表 |
| 责任链 | 请求沿链处理，职责分离 | 链表 + 条件转发 |
| 命令 | 请求对象化，支持撤销排队 | 封装 execute + undo |
| 迭代器 | 统一遍历接口，不暴露内部 | hasNext + next |
| 中介者 | 消除网状耦合，集中协调 | 中间人转发 |
| 备忘录 | 保存恢复对象状态 | 快照对象 |
| 状态 | 状态变化时自动切换行为 | 状态类 + setState |
| 访问者 | 数据稳定，操作多变 | accept + visit 双分派 |
| 解释器 | 简单语言解析 | 递归文法树 |
