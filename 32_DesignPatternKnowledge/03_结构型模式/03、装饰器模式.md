# 装饰器模式（Decorator）

> 动态地给对象**添加额外职责**。比继承更灵活：装饰器在运行时动态组合，继承在编译期静态绑定；可以叠加多个装饰器，继承只有一条链。

---

## 故事引入

### 《我想吃煎饼果子了》

做一个煎饼果子看似简单，但客户的需求千变万化：
- 原味煎饼
- 煎饼+鸡蛋
- 煎饼+鸡蛋+香肠
- 煎饼+鸡蛋+香肠+生菜
- 煎饼+两个鸡蛋+薄脆+辣条
- ……

如果为每种组合都写一个类，会产生"类爆炸"！更好的办法是：**一个基础煎饼，然后一层层往上加料**。

**装饰器模式**：动态给对象添加职责，比继承更灵活。

---

## 一、定义与意图

动态地给对象**添加额外职责**。比继承更灵活：装饰器在运行时动态组合，继承在编译期静态绑定；可以叠加多个装饰器，继承只有一条链。

## 二、类图角色

```
Component（抽象组件）
├── ConcreteComponent（具体组件，基础实现）
└── Decorator（装饰器抽象类，持有 Component 引用）
    ├── ConcreteDecoratorA
    └── ConcreteDecoratorB
```

## 三、Demo

```java
// 数据压缩/加密/日志 IO 流的装饰器链
public interface DataProcessor {
    byte[] process(byte[] data);
    byte[] recover(byte[] data);
}

// 基础实现：原始数据不做处理
public class RawDataProcessor implements DataProcessor {
    public byte[] process(byte[] data) { return data; }
    public byte[] recover(byte[] data) { return data; }
}

// 抽象装饰器
public abstract class DataProcessorDecorator implements DataProcessor {
    protected final DataProcessor wrapped;
    public DataProcessorDecorator(DataProcessor wrapped) { this.wrapped = wrapped; }
}

// 装饰器1：压缩
public class CompressionDecorator extends DataProcessorDecorator {
    public CompressionDecorator(DataProcessor wrapped) { super(wrapped); }

    @Override
    public byte[] process(byte[] data) {
        byte[] processedByWrapped = wrapped.process(data);
        return compress(processedByWrapped); // 先让内层处理，再压缩
    }

    @Override
    public byte[] recover(byte[] data) {
        byte[] decompressed = decompress(data); // 先解压
        return wrapped.recover(decompressed);    // 再让内层恢复
    }

    private byte[] compress(byte[] data) {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            GZIPOutputStream gzip = new GZIPOutputStream(bos);
            gzip.write(data); gzip.close();
            System.out.println("压缩: " + data.length + " -> " + bos.size() + " bytes");
            return bos.toByteArray();
        } catch (Exception e) { throw new RuntimeException(e); }
    }

    private byte[] decompress(byte[] data) {
        try {
            GZIPInputStream gzip = new GZIPInputStream(new ByteArrayInputStream(data));
            return gzip.readAllBytes();
        } catch (Exception e) { throw new RuntimeException(e); }
    }
}

// 装饰器2：加密
public class EncryptionDecorator extends DataProcessorDecorator {
    private final String key;
    public EncryptionDecorator(DataProcessor wrapped, String key) {
        super(wrapped); this.key = key;
    }

    @Override
    public byte[] process(byte[] data) {
        byte[] processedByWrapped = wrapped.process(data);
        return encrypt(processedByWrapped);
    }

    @Override
    public byte[] recover(byte[] data) {
        byte[] decrypted = decrypt(data);
        return wrapped.recover(decrypted);
    }

    private byte[] encrypt(byte[] data) {
        System.out.println("加密数据，长度: " + data.length);
        // 实际加密逻辑（这里省略）
        return data;
    }
    private byte[] decrypt(byte[] data) { return data; }
}

// 装饰器3：日志记录
public class LoggingDecorator extends DataProcessorDecorator {
    public LoggingDecorator(DataProcessor wrapped) { super(wrapped); }

    @Override
    public byte[] process(byte[] data) {
        System.out.println("处理数据，原始大小: " + data.length);
        byte[] result = wrapped.process(data);
        System.out.println("处理完成，结果大小: " + result.length);
        return result;
    }

    @Override
    public byte[] recover(byte[] data) {
        System.out.println("恢复数据，输入大小: " + data.length);
        byte[] result = wrapped.recover(data);
        System.out.println("恢复完成，输出大小: " + result.length);
        return result;
    }
}

// 使用：像搭积木一样组合装饰器
DataProcessor processor = new LoggingDecorator(  // 最外层：日志
    new EncryptionDecorator(                       // 中间层：加密
        new CompressionDecorator(                  // 内层：压缩
            new RawDataProcessor()),               // 核心：原始处理
        "my-secret-key"));

byte[] original = "Hello, World!".getBytes();
byte[] processed = processor.process(original);  // 先压缩，再加密，再日志
byte[] recovered = processor.recover(processed); // 先日志，再解密，再解压
```

## 四、实际业务场景

```java
// 场景：给 UserRepository 动态添加缓存功能
public interface UserRepository {
    User findById(Long id);
    void save(User user);
    void deleteById(Long id);
}

@Repository
public class UserRepositoryImpl implements UserRepository {
    public User findById(Long id) { /* 查数据库 */ return new User(id); }
    public void save(User user) { /* 存数据库 */ }
    public void deleteById(Long id) { /* 删数据库 */ }
}

// 缓存装饰器
public class CachedUserRepository implements UserRepository {
    private final UserRepository delegate;
    private final Map<Long, User> cache = new ConcurrentHashMap<>();

    public CachedUserRepository(UserRepository delegate) { this.delegate = delegate; }

    @Override
    public User findById(Long id) {
        return cache.computeIfAbsent(id, key -> {
            System.out.println("缓存未命中，查数据库: " + key);
            return delegate.findById(key);
        });
    }

    @Override
    public void save(User user) {
        delegate.save(user);
        cache.put(user.getId(), user); // 更新缓存
    }

    @Override
    public void deleteById(Long id) {
        delegate.deleteById(id);
        cache.remove(id); // 清除缓存
    }
}
```
