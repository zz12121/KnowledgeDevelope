# 桥接模式（Bridge）

> 将**抽象**与**实现**分离，使它们可以独立变化。当一个类有两个维度都可能变化时，用组合替代多层继承，避免类爆炸。

---

## 故事引入

### 《当代毕加索小王》

小王是个艺术家，他想画一幅画。传统方式是他自己学油画、水彩画、版画等各种技法——学一种就够呛，学全部太难了！

后来他想到一个办法：**画画的媒介（画笔/画布）和画的内容（人物/风景/抽象）分开**。他只需要掌握画笔的使用技巧，然后找不同类型的画布，就能创作不同风格的作品。

**桥接模式**：把抽象和实现分离，让两者可以独立变化。

---

## 一、定义与意图

将**抽象**与**实现**分离，使它们可以独立变化。当一个类有两个维度都可能变化时，用组合替代多层继承，避免类爆炸。

---

## 二、类图角色

```
Abstraction（抽象类）
└── + implementor: Implementor  // 桥：持有实现者引用
RefinedAbstraction（扩展抽象）

Implementor（实现接口）
└── ConcreteImplementorA
└── ConcreteImplementorB
```

## 三、Demo

```java
// 场景：消息推送系统（消息类型 × 推送渠道，两个维度独立变化）
// 实现接口（推送渠道）
public interface PushChannel {
    void push(String recipient, String title, String content);
    String getChannelName();
}

@Component("emailChannel")
public class EmailChannel implements PushChannel {
    public void push(String recipient, String title, String content) {
        System.out.printf("邮件[%s] -> %s: %s%n", title, recipient, content);
    }
    public String getChannelName() { return "email"; }
}

@Component("smsChannel")
public class SmsChannel implements PushChannel {
    public void push(String recipient, String title, String content) {
        System.out.printf("短信 -> %s: %s%n", recipient, content.substring(0, Math.min(70, content.length())));
    }
    public String getChannelName() { return "sms"; }
}

@Component("appPushChannel")
public class AppPushChannel implements PushChannel {
    public void push(String recipient, String title, String content) {
        System.out.printf("APP推送[%s] -> 用户%s: %s%n", title, recipient, content);
    }
    public String getChannelName() { return "app"; }
}

// 抽象类（消息类型）
public abstract class Notification {
    protected final PushChannel channel; // 桥

    public Notification(PushChannel channel) { this.channel = channel; }

    // 子类实现不同类型消息的格式化逻辑
    public abstract void send(String recipient, Map<String, Object> params);

    protected String getChannelName() { return channel.getChannelName(); }
}

// 扩展抽象：订单通知
public class OrderNotification extends Notification {
    public OrderNotification(PushChannel channel) { super(channel); }

    @Override
    public void send(String recipient, Map<String, Object> params) {
        String orderId = (String) params.get("orderId");
        String status = (String) params.get("status");
        String title = "订单状态更新";
        String content = String.format("您的订单 %s 状态已更新为：%s", orderId, status);
        channel.push(recipient, title, content);
    }
}

// 扩展抽象：促销通知
public class PromotionNotification extends Notification {
    public PromotionNotification(PushChannel channel) { super(channel); }

    @Override
    public void send(String recipient, Map<String, Object> params) {
        String activity = (String) params.get("activity");
        String discount = (String) params.get("discount");
        String title = "专属优惠来啦！";
        String content = String.format("【%s】限时折扣 %s，快来抢购！", activity, discount);
        channel.push(recipient, title, content);
    }
}

// 使用：抽象和实现自由组合
PushChannel emailChannel = new EmailChannel();
PushChannel smsChannel = new SmsChannel();

// 订单通知走邮件
Notification orderByEmail = new OrderNotification(emailChannel);
orderByEmail.send("user@example.com", Map.of("orderId", "ORD-001", "status", "已发货"));

// 促销通知走短信
Notification promotionBySms = new PromotionNotification(smsChannel);
promotionBySms.send("13800138000", Map.of("activity", "双11", "discount", "5折"));

// 新增一种渠道（微信公众号）：只加一个 WeChatChannel 实现类，不改任何 Notification 子类
// 新增一种通知类型（系统公告）：只加一个 SystemNotification 子类，不改任何渠道实现
```
