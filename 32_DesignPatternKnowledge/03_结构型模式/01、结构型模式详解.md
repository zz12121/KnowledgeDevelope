# 结构型模式详解（7种）

> 结构型模式关注如何将**类和对象组合**成更大的结构，同时保持结构的灵活性。GoF 定义 7 种结构型模式。本文通过生动的小故事来引入每个模式。

---

## 故事引入

### 1. 代理模式 —— 《离婚，为什么需要离婚律师？》

小李和妻子感情破裂，决定离婚。但离婚涉及财产分割、孩子抚养权、债务处理等复杂问题。小李自己去法院办理？肯定不行——他不懂法律！

于是小李请了一个**律师**。律师帮他处理所有法律事务，小李只需要配合签字就行。

**代理模式**正是如此：不直接访问真实对象，而是通过代理对象来间接访问，代理可以在访问前后添加额外逻辑（律师帮你打官司）。

### 2. 适配器模式 —— 《买了苹果笔记本的尴尬》

小明买了最新款的苹果笔记本，外观时尚，性能强大。但当他想投影到公司会议室的老旧投影仪时傻眼了——MacBook只有Type-C接口，而投影仪是VGA接口！

这时候小明需要一个**转接头**（Adapter）——把Type-C转成VGA，这样就能连接了。

**适配器模式**就是这样的"转接头"——把不兼容的接口转换成兼容的。

### 3. 桥接模式 —— 《当代毕加索小王》

小王是个艺术家，他想画一幅画。传统方式是他自己学油画、水彩画、版画等各种技法——学一种就够呛，学全部太难了！

后来他想到一个办法：**画画的媒介（画笔/画布）和画的内容（人物/风景/抽象）分开**。他只需要掌握画笔的使用技巧，然后找不同类型的画布，就能创作不同风格的作品。

**桥接模式**：把抽象和实现分离，让两者可以独立变化。

### 4. 装饰器模式 —— 《我想吃煎饼果子了》

做一个煎饼果子看似简单，但客户的需求千变万化：
- 原味煎饼
- 煎饼+鸡蛋
- 煎饼+鸡蛋+香肠
- 煎饼+鸡蛋+香肠+生菜
- 煎饼+两个鸡蛋+薄脆+辣条
- ……

如果为每种组合都写一个类，会产生"类爆炸"！更好的办法是：**一个基础煎饼，然后一层层往上加料**。

**装饰器模式**：动态给对象添加职责，比继承更灵活。

### 5. 门面模式 —— 《办理入职怎么这么难？》

小王大学毕业，去一家公司办理入职。前台递给他一张单子，上面列了30多项需要他自己去办的事项：
- 办理档案（去上家公司）
- 办理公积金（去公积金中心）
- 领取电脑（自己去电脑店采购）
- 安装办公软件（按列表安装）
- 确定工位（自己申请）
- 办理工资卡（去银行）
- 办理工卡（去制卡处）
- ……

小王崩溃了：我就是办个入职，为什么要跑遍全城？

后来另一家公司采用了**HR一站式服务**：所有资料在会议室统一收集，HR帮忙跑腿办理，小王只需要签个字就行。

**门面模式**：为复杂子系统提供统一的简化入口。

### 6. 享元模式 —— 《开发一个疫苗管理平台系统》

某市要开发一个核酸检测管理平台，需要采集2000万人的个人信息，包括：姓名、居住地址、检测医院、检测品牌等。

仔细分析发现：姓名和地址是每个人不同的，但检测医院和检测品牌是固定的（全市只有10家医院、5种检测品牌）。

如果每个人都存储完整的医院和品牌信息，空间浪费巨大！更好的办法是：**把这些固定信息池化，大家共享**。

**享元模式**：运用共享技术有效支持大量细粒度对象的复用。

### 7. 组合模式 —— 《庞大的组织架构带来的烦恼》

某大型集团有复杂的组织架构：
- 集团公司
  - 华东区
    - 上海分公司
      - 销售部
      - 技术部
    - 杭州分公司
      - 销售部
      - 技术部
  - 华南区
    - ……
    
每次统计人数或计算薪资，都需要区分是"单个员工"还是"整个部门"，代码很难写。

后来采用了一套系统：**员工和部门都实现同一个"组织节点"接口**，员工没有下属，部门有下属。统计的时候，无论给的是员工还是部门，代码都能统一处理。

**组合模式**：让单个对象和组合对象具有一致的访问性。

## 一、代理模式（Proxy）

### 定义与意图

为其他对象提供一种代理来**控制对该对象的访问**。代理对象与目标对象实现相同的接口，客户端通过代理间接访问真实对象，可以在访问前后添加额外逻辑。

### 类图角色

```
Subject（接口）
├── RealSubject（真实对象）
└── Proxy（代理对象）
    ├── - realSubject: RealSubject
    └── + request()  // 在调用 realSubject.request() 前后加逻辑
```

### 三种代理类型

**静态代理**

```java
// 业务接口
public interface OrderService {
    Order createOrder(CreateOrderRequest request);
    Order findById(Long id);
}

// 真实对象
@Service
public class OrderServiceImpl implements OrderService {
    @Autowired private OrderRepository orderRepository;

    public Order createOrder(CreateOrderRequest request) {
        Order order = Order.from(request);
        orderRepository.save(order);
        return order;
    }

    public Order findById(Long id) { return orderRepository.findById(id).orElseThrow(); }
}

// 静态代理：手动编写，为 OrderService 添加日志
public class OrderServiceLoggingProxy implements OrderService {
    private final OrderService target; // 被代理的真实对象

    public OrderServiceLoggingProxy(OrderService target) {
        this.target = target;
    }

    @Override
    public Order createOrder(CreateOrderRequest request) {
        long start = System.currentTimeMillis();
        System.out.println("开始创建订单，用户ID: " + request.getUserId());
        try {
            Order order = target.createOrder(request);
            System.out.printf("订单创建成功，ID=%d，耗时%dms%n",
                    order.getId(), System.currentTimeMillis() - start);
            return order;
        } catch (Exception e) {
            System.out.println("订单创建失败: " + e.getMessage());
            throw e;
        }
    }

    @Override
    public Order findById(Long id) {
        System.out.println("查询订单: " + id);
        return target.findById(id);
    }
}
```

**JDK 动态代理（基于接口）**

```java
// 通用日志 InvocationHandler（可以代理任何接口）
public class LoggingInvocationHandler implements InvocationHandler {
    private final Object target;
    private final String serviceName;

    public LoggingInvocationHandler(Object target) {
        this.target = target;
        this.serviceName = target.getClass().getSimpleName();
    }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        long start = System.nanoTime();
        System.out.printf("[%s.%s] 开始执行%n", serviceName, method.getName());
        try {
            Object result = method.invoke(target, args);
            System.out.printf("[%s.%s] 执行成功，耗时%.2fms%n",
                    serviceName, method.getName(),
                    (System.nanoTime() - start) / 1_000_000.0);
            return result;
        } catch (InvocationTargetException e) {
            System.out.printf("[%s.%s] 执行异常: %s%n",
                    serviceName, method.getName(), e.getCause().getMessage());
            throw e.getCause();
        }
    }
}

// 代理工厂工具类
public class ProxyFactory {
    @SuppressWarnings("unchecked")
    public static <T> T createLoggingProxy(T target, Class<T> interfaceType) {
        return (T) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),
            new Class[]{interfaceType},
            new LoggingInvocationHandler(target)
        );
    }
}

// 使用
OrderService orderService = ProxyFactory.createLoggingProxy(
    new OrderServiceImpl(), OrderService.class
);
orderService.findById(1L); // 自动打印日志
```

**CGLIB 代理（基于继承，无需接口）**

```java
// 假设 ProductService 没有接口
public class ProductService {
    public Product findById(Long id) {
        System.out.println("查询商品: " + id);
        return new Product(id, "商品" + id);
    }

    public void updateStock(Long productId, int delta) {
        System.out.println("更新库存: product=" + productId + ", delta=" + delta);
    }
}

// CGLIB 代理（Spring 项目可直接使用，Spring Core 包含 CGLIB）
public class CglibLoggingProxy implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.println("CGLIB 拦截: " + method.getName());
        long start = System.currentTimeMillis();
        try {
            return proxy.invokeSuper(obj, args); // 调用父类（被代理类）的方法
        } finally {
            System.out.println(method.getName() + " 耗时: " + (System.currentTimeMillis() - start) + "ms");
        }
    }
}

// 创建 CGLIB 代理
Enhancer enhancer = new Enhancer();
enhancer.setSuperclass(ProductService.class);
enhancer.setCallback(new CglibLoggingProxy());
ProductService proxy = (ProductService) enhancer.create();
proxy.findById(1L);
```

### 实际业务场景

```java
// 场景：用注解 + 动态代理实现缓存切面
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Cacheable {
    String key();
    int ttlSeconds() default 300;
}

public class CacheProxy implements InvocationHandler {
    private final Object target;
    private final Map<String, Object> cache = new ConcurrentHashMap<>();

    public CacheProxy(Object target) { this.target = target; }

    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        Cacheable cacheable = method.getAnnotation(Cacheable.class);
        if (cacheable == null) {
            return method.invoke(target, args); // 没有注解，直接执行
        }
        String cacheKey = cacheable.key() + Arrays.toString(args);
        return cache.computeIfAbsent(cacheKey, k -> {
            try {
                System.out.println("缓存未命中，执行方法: " + method.getName());
                return method.invoke(target, args);
            } catch (Exception e) { throw new RuntimeException(e); }
        });
    }
}
```

## 二、装饰器模式（Decorator）

### 定义与意图

动态地给对象**添加额外职责**。比继承更灵活：装饰器在运行时动态组合，继承在编译期静态绑定；可以叠加多个装饰器，继承只有一条链。

### 类图角色

```
Component（抽象组件）
├── ConcreteComponent（具体组件，基础实现）
└── Decorator（装饰器抽象类，持有 Component 引用）
    ├── ConcreteDecoratorA
    └── ConcreteDecoratorB
```

### Demo

```java
// 数据压缩/加密/日志 IO 流的装饰器链
public interface DataProcessor {
    byte[] process(byte[] data);
    byte[] recover(byte[] data);
}

// 基础实现：原始数据不做处理
public class RawDataProcessor implements DataProcessor {
    public byte[] process(byte[] data) { return data; }
    public byte[] recover(byte[] data) { return data; }
}

// 抽象装饰器
public abstract class DataProcessorDecorator implements DataProcessor {
    protected final DataProcessor wrapped;
    public DataProcessorDecorator(DataProcessor wrapped) { this.wrapped = wrapped; }
}

// 装饰器1：压缩
public class CompressionDecorator extends DataProcessorDecorator {
    public CompressionDecorator(DataProcessor wrapped) { super(wrapped); }

    @Override
    public byte[] process(byte[] data) {
        byte[] processedByWrapped = wrapped.process(data);
        return compress(processedByWrapped); // 先让内层处理，再压缩
    }

    @Override
    public byte[] recover(byte[] data) {
        byte[] decompressed = decompress(data); // 先解压
        return wrapped.recover(decompressed);    // 再让内层恢复
    }

    private byte[] compress(byte[] data) {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            GZIPOutputStream gzip = new GZIPOutputStream(bos);
            gzip.write(data); gzip.close();
            System.out.println("压缩: " + data.length + " -> " + bos.size() + " bytes");
            return bos.toByteArray();
        } catch (Exception e) { throw new RuntimeException(e); }
    }

    private byte[] decompress(byte[] data) {
        try {
            GZIPInputStream gzip = new GZIPInputStream(new ByteArrayInputStream(data));
            return gzip.readAllBytes();
        } catch (Exception e) { throw new RuntimeException(e); }
    }
}

// 装饰器2：加密
public class EncryptionDecorator extends DataProcessorDecorator {
    private final String key;
    public EncryptionDecorator(DataProcessor wrapped, String key) {
        super(wrapped); this.key = key;
    }

    @Override
    public byte[] process(byte[] data) {
        byte[] processedByWrapped = wrapped.process(data);
        return encrypt(processedByWrapped);
    }

    @Override
    public byte[] recover(byte[] data) {
        byte[] decrypted = decrypt(data);
        return wrapped.recover(decrypted);
    }

    private byte[] encrypt(byte[] data) {
        System.out.println("加密数据，长度: " + data.length);
        // 实际加密逻辑（这里省略）
        return data;
    }
    private byte[] decrypt(byte[] data) { return data; }
}

// 装饰器3：日志记录
public class LoggingDecorator extends DataProcessorDecorator {
    public LoggingDecorator(DataProcessor wrapped) { super(wrapped); }

    @Override
    public byte[] process(byte[] data) {
        System.out.println("处理数据，原始大小: " + data.length);
        byte[] result = wrapped.process(data);
        System.out.println("处理完成，结果大小: " + result.length);
        return result;
    }

    @Override
    public byte[] recover(byte[] data) {
        System.out.println("恢复数据，输入大小: " + data.length);
        byte[] result = wrapped.recover(data);
        System.out.println("恢复完成，输出大小: " + result.length);
        return result;
    }
}

// 使用：像搭积木一样组合装饰器
DataProcessor processor = new LoggingDecorator(  // 最外层：日志
    new EncryptionDecorator(                       // 中间层：加密
        new CompressionDecorator(                  // 内层：压缩
            new RawDataProcessor()),               // 核心：原始处理
        "my-secret-key"));

byte[] original = "Hello, World!".getBytes();
byte[] processed = processor.process(original);  // 先压缩，再加密，再日志
byte[] recovered = processor.recover(processed); // 先日志，再解密，再解压
```

### 实际业务场景

```java
// 场景：给 UserRepository 动态添加缓存功能
public interface UserRepository {
    User findById(Long id);
    void save(User user);
    void deleteById(Long id);
}

@Repository
public class UserRepositoryImpl implements UserRepository {
    public User findById(Long id) { /* 查数据库 */ return new User(id); }
    public void save(User user) { /* 存数据库 */ }
    public void deleteById(Long id) { /* 删数据库 */ }
}

// 缓存装饰器
public class CachedUserRepository implements UserRepository {
    private final UserRepository delegate;
    private final Map<Long, User> cache = new ConcurrentHashMap<>();

    public CachedUserRepository(UserRepository delegate) { this.delegate = delegate; }

    @Override
    public User findById(Long id) {
        return cache.computeIfAbsent(id, key -> {
            System.out.println("缓存未命中，查数据库: " + key);
            return delegate.findById(key);
        });
    }

    @Override
    public void save(User user) {
        delegate.save(user);
        cache.put(user.getId(), user); // 更新缓存
    }

    @Override
    public void deleteById(Long id) {
        delegate.deleteById(id);
        cache.remove(id); // 清除缓存
    }
}
```

## 三、适配器模式（Adapter）

### 定义与意图

将一个类的接口转换成客户端期望的另一个接口，让**原本接口不兼容**的类可以一起工作。解决新旧系统集成、第三方 SDK 对接时的接口不匹配问题。

### 类图角色

```
Target（目标接口）
Adaptee（被适配者，已有的不兼容类）
Adapter（适配器）
├── 类适配器：extends Adaptee, implements Target
└── 对象适配器：implements Target, has-a Adaptee
```

### Demo

```java
// 场景：统一不同厂商的短信服务接口
// 目标接口（我们系统希望使用的统一接口）
public interface SmsSender {
    SmsResult send(String phone, String content);
    SmsResult sendBatch(List<String> phones, String content);
}

// 阿里云短信 SDK（Adaptee1 ——— 不能修改）
public class AliCloudSmsClient {
    private final String accessKeyId;
    private final String accessKeySecret;

    public AliCloudSmsClient(String accessKeyId, String accessKeySecret) {
        this.accessKeyId = accessKeyId; this.accessKeySecret = accessKeySecret;
    }

    public AliSmsResponse sendSms(String phoneNumber, String signName, String templateCode, String templateParam) {
        System.out.println("阿里云短信发送: " + phoneNumber);
        return new AliSmsResponse("OK", "发送成功");
    }
}

// 腾讯云短信 SDK（Adaptee2）
public class TencentSmsClient {
    public TencentSmsResponse sendSms(String[] phoneNumbers, String sdkAppId, String signName, String templateId, String[] templateValues) {
        System.out.println("腾讯云短信发送: " + Arrays.toString(phoneNumbers));
        return new TencentSmsResponse(0, "ok");
    }
}

// 对象适配器：阿里云实现
public class AliCloudSmsAdapter implements SmsSender {
    private final AliCloudSmsClient aliClient;
    private final String signName;
    private final String templateCode;

    public AliCloudSmsAdapter(AliCloudSmsClient aliClient, String signName, String templateCode) {
        this.aliClient = aliClient; this.signName = signName; this.templateCode = templateCode;
    }

    @Override
    public SmsResult send(String phone, String content) {
        AliSmsResponse resp = aliClient.sendSms(phone, signName, templateCode,
                "{\"content\":\"" + content + "\"}");
        return new SmsResult("OK".equals(resp.getCode()), resp.getMessage());
    }

    @Override
    public SmsResult sendBatch(List<String> phones, String content) {
        List<SmsResult> results = phones.stream()
                .map(p -> send(p, content))
                .collect(Collectors.toList());
        boolean allSuccess = results.stream().allMatch(SmsResult::isSuccess);
        return new SmsResult(allSuccess, allSuccess ? "批量发送成功" : "部分失败");
    }
}

// 对象适配器：腾讯云实现
public class TencentSmsAdapter implements SmsSender {
    private final TencentSmsClient tencentClient;
    private final String sdkAppId;
    private final String signName;
    private final String templateId;

    public TencentSmsAdapter(TencentSmsClient client, String sdkAppId, String signName, String templateId) {
        this.tencentClient = client; this.sdkAppId = sdkAppId;
        this.signName = signName; this.templateId = templateId;
    }

    @Override
    public SmsResult send(String phone, String content) {
        TencentSmsResponse resp = tencentClient.sendSms(
            new String[]{phone}, sdkAppId, signName, templateId, new String[]{content});
        return new SmsResult(resp.getCode() == 0, resp.getMessage());
    }

    @Override
    public SmsResult sendBatch(List<String> phones, String content) {
        TencentSmsResponse resp = tencentClient.sendSms(
            phones.toArray(new String[0]), sdkAppId, signName, templateId, new String[]{content});
        return new SmsResult(resp.getCode() == 0, resp.getMessage());
    }
}

// 业务层只依赖统一接口，切换供应商只需换适配器
@Service
public class NotificationService {
    @Autowired private SmsSender smsSender; // 注入哪个适配器由配置决定

    public void sendVerifyCode(String phone, String code) {
        SmsResult result = smsSender.send(phone, "您的验证码是：" + code + "，5分钟内有效");
        if (!result.isSuccess()) {
            throw new BusinessException("短信发送失败: " + result.getMessage());
        }
    }
}
```

## 四、门面模式（Facade）

### 定义与意图

为子系统中的一组接口提供一个统一的**简化入口**，隐藏子系统内部的复杂性，让客户端更容易使用。

### 类图角色

```
Facade（门面）
├── - subsystemA: SubsystemA
├── - subsystemB: SubsystemB
├── - subsystemC: SubsystemC
└── + simplifiedOperation()  // 封装多个子系统的调用
```

### Demo

```java
// 场景：电商下单门面（协调库存、支付、物流、通知多个子系统）
@Service
public class PlaceOrderFacade {
    private final InventoryService inventoryService;
    private final OrderRepository orderRepository;
    private final PaymentService paymentService;
    private final ShippingService shippingService;
    private final NotificationService notificationService;
    private final CouponService couponService;
    private final PointsService pointsService;

    @Autowired
    public PlaceOrderFacade(InventoryService inventoryService,
                            OrderRepository orderRepository,
                            PaymentService paymentService,
                            ShippingService shippingService,
                            NotificationService notificationService,
                            CouponService couponService,
                            PointsService pointsService) {
        this.inventoryService = inventoryService;
        this.orderRepository = orderRepository;
        this.paymentService = paymentService;
        this.shippingService = shippingService;
        this.notificationService = notificationService;
        this.couponService = couponService;
        this.pointsService = pointsService;
    }

    // 门面方法：客户端只调用这一个接口，不需要了解内部有多少个子系统
    @Transactional
    public PlaceOrderResult placeOrder(PlaceOrderRequest request) {
        // 1. 校验并预扣库存
        List<InventoryResult> inventoryResults = inventoryService.batchCheckAndLock(request.getItems());
        if (inventoryResults.stream().anyMatch(r -> !r.isSuccess())) {
            return PlaceOrderResult.fail("库存不足: " + getInsufficientItems(inventoryResults));
        }

        // 2. 计算优惠
        BigDecimal discount = BigDecimal.ZERO;
        if (request.getCouponId() != null) {
            discount = couponService.calculateDiscount(request.getCouponId(), request.getTotalAmount());
        }

        // 3. 创建订单
        Order order = Order.builder()
                .userId(request.getUserId())
                .items(request.getItems())
                .totalAmount(request.getTotalAmount().subtract(discount))
                .shippingAddress(request.getShippingAddress())
                .build();
        orderRepository.save(order);

        // 4. 处理支付
        PaymentResult payResult = paymentService.processPayment(order.getId(), order.getTotalAmount(), request.getPaymentMethod());
        if (!payResult.isSuccess()) {
            inventoryService.releaseLock(order.getId()); // 支付失败，释放库存锁
            throw new PaymentException("支付失败: " + payResult.getMessage());
        }

        // 5. 创建运单
        String trackingNo = shippingService.createShipment(order.getId(), request.getShippingAddress());

        // 6. 赠送积分
        if (payResult.isSuccess()) {
            pointsService.addPoints(request.getUserId(), calcPoints(order.getTotalAmount()));
        }

        // 7. 发送通知
        notificationService.sendOrderConfirmation(request.getUserId(), order.getId(), trackingNo);

        return PlaceOrderResult.success(order.getId(), trackingNo);
    }

    private String getInsufficientItems(List<InventoryResult> results) {
        return results.stream().filter(r -> !r.isSuccess())
                .map(InventoryResult::getProductName).collect(Collectors.joining(", "));
    }

    private int calcPoints(BigDecimal amount) { return amount.intValue(); }
}

// 控制器层极其简洁
@RestController

    public ResponseEntity<PlaceOrderResult> placeOrder(@RequestBody PlaceOrderRequest request) {
```

## 五、桥接模式（Bridge）

### 定义与意图

将**抽象**与**实现**分离，使它们可以独立变化。当一个类有两个维度都可能变化时，用组合替代多层继承，避免类爆炸。

### 类图角色

```
Abstraction（抽象类）
└── + implementor: Implementor  // 桥：持有实现者引用
RefinedAbstraction（扩展抽象）

Implementor（实现接口）
└── ConcreteImplementorA
└── ConcreteImplementorB
```

### Demo

```java
// 场景：消息推送系统（消息类型 × 推送渠道，两个维度独立变化）
// 实现接口（推送渠道）
public interface PushChannel {
    void push(String recipient, String title, String content);

public class SmsChannel implements PushChannel {
```

## 六、组合模式（Composite）

### 定义与意图

将对象组合成**树形结构**来表示"部分-整体"层次，使客户端对单个对象和组合对象的操作方式保持一致。适合处理树形结构数据。

### 类图角色

```
Component（抽象组件）
├── Leaf（叶子，无子节点）
└── Composite（容器，包含子 Component 集合）
```

### Demo

```java
// 场景：公司组织架构（部门/员工树）
public abstract class OrgNode {
    protected String name;
    protected String position;

    public OrgNode(String name, String position) {
        this.name = name; this.position = position;
    }

    public abstract double getTotalSalary();
    public abstract int getHeadcount();
    public abstract void display(int depth);

    // 默认不支持子节点操作
    public void add(OrgNode node) { throw new UnsupportedOperationException(); }
    public void remove(OrgNode node) { throw new UnsupportedOperationException(); }
    public List<OrgNode> getChildren() { return List.of(); }
}

// 叶子节点：员工
public class Employee extends OrgNode {
    private double salary;

    public Employee(String name, String position, double salary) {
        super(name, position);
        this.salary = salary;
    }

    @Override
    public double getTotalSalary() { return salary; }

    @Override
    public int getHeadcount() { return 1; }

    @Override
    public void display(int depth) {
        System.out.printf("%s👤 %s（%s）- ¥%.0f%n",
                "  ".repeat(depth), name, position, salary);
    }
}

// 容器节点：部门
public class Department extends OrgNode {
    private final List<OrgNode> members = new ArrayList<>();

    public Department(String name) { super(name, "部门"); }

    @Override
    public void add(OrgNode node) { members.add(node); }

    @Override
    public void remove(OrgNode node) { members.remove(node); }

    @Override
    public List<OrgNode> getChildren() { return Collections.unmodifiableList(members); }

    @Override
    public double getTotalSalary() {
        return members.stream().mapToDouble(OrgNode::getTotalSalary).sum(); // 递归汇总
    }

    @Override
    public int getHeadcount() {
        return members.stream().mapToInt(OrgNode::getHeadcount).sum(); // 递归计数
    }

    @Override
    public void display(int depth) {
        System.out.printf("%s📁 %s（共%d人，薪酬¥%.0f）%n",
                "  ".repeat(depth), name, getHeadcount(), getTotalSalary());
        members.forEach(m -> m.display(depth + 1));
    }
}

// 使用
Department company = new Department("科技公司");
Department techDept = new Department("技术部");
Department bizDept = new Department("业务部");

techDept.add(new Employee("张三", "架构师", 35000));
techDept.add(new Employee("李四", "高级工程师", 28000));
techDept.add(new Employee("王五", "工程师", 18000));

Department frontendTeam = new Department("前端组");
frontendTeam.add(new Employee("赵六", "前端工程师", 20000));
frontendTeam.add(new Employee("钱七", "前端工程师", 18000));
techDept.add(frontendTeam);

bizDept.add(new Employee("孙八", "产品经理", 25000));
bizDept.add(new Employee("周九", "运营", 15000));

company.add(techDept);
company.add(bizDept);

company.display(0);
System.out.printf("公司总人数: %d，总薪酬: ¥%.0f%n", company.getHeadcount(), company.getTotalSalary());
```

## 七、享元模式（Flyweight）

### 定义与意图

通过**共享**来有效支持大量细粒度对象。把对象状态分成**内在状态**（不变可共享）和**外在状态**（变化不共享），只共享内在状态，外在状态由客户端传入。

### 类图角色

```
FlyweightFactory（享元工厂，维护对象池）
Flyweight（享元接口）
└── ConcreteFlyweight（内在状态）

Client（持有外在状态，调用时传入）
```

### Demo

```java
// 场景：在线文档编辑器，每个字符对象共享字体/大小/颜色
// 享元对象（内在状态：字体属性，多字符共享）
public class CharacterStyle {
    private final String fontFamily;
    private final int fontSize;
    private final String color;
    private final boolean bold;
    private final boolean italic;

    public CharacterStyle(String fontFamily, int fontSize, String color, boolean bold, boolean italic) {
        this.fontFamily = fontFamily; this.fontSize = fontSize;
        this.color = color; this.bold = bold; this.italic = italic;
    }

    // 渲染时，外在状态（位置/字符本身）从外部传入
    public void render(char character, int row, int col) {
        System.out.printf("渲染字符 '%c' 在(%d,%d)，字体=%s，大小=%d，颜色=%s，粗体=%s%n",
                character, row, col, fontFamily, fontSize, color, bold);
    }

    @Override
    public String toString() {
        return String.format("%s-%d-%s-%b-%b", fontFamily, fontSize, color, bold, italic);
    }
}

// 享元工厂
public class CharacterStyleFactory {
    private static final Map<String, CharacterStyle> pool = new HashMap<>();
    private static int createCount = 0;

    public static CharacterStyle getStyle(String fontFamily, int fontSize, String color,
                                          boolean bold, boolean italic) {
        String key = String.format("%s-%d-%s-%b-%b", fontFamily, fontSize, color, bold, italic);
        return pool.computeIfAbsent(key, k -> {
            createCount++;
            System.out.println("创建新样式对象: " + key);
            return new CharacterStyle(fontFamily, fontSize, color, bold, italic);
        });
    }

    public static int getPoolSize() { return pool.size(); }
    public static int getCreateCount() { return createCount; }
}

// 字符对象（只存储外在状态：字符本身）
public class DocumentCharacter {
    private final char ch;           // 外在状态：具体字符
    private final int row;           // 外在状态：行
    private final int col;           // 外在状态：列
    private final CharacterStyle style; // 内在状态：共享的样式对象

    public DocumentCharacter(char ch, int row, int col, CharacterStyle style) {
        this.ch = ch; this.row = row; this.col = col; this.style = style;
    }

    public void render() { style.render(ch, row, col); }
}

// 使用：10000个字符，但只有少量样式对象
public class Document {
    public static void main(String[] args) {
        List<DocumentCharacter> chars = new ArrayList<>();
        String text = "Hello World! This is a sample document with many characters.";
        String[] fonts = {"宋体", "微软雅黑", "楷体"};
        int[] sizes = {12, 14, 16};
        String[] colors = {"black", "red", "blue"};

        // 创建大量字符，但样式对象复用
        for (int i = 0; i < 10000; i++) {
            char c = text.charAt(i % text.length());
            CharacterStyle style = CharacterStyleFactory.getStyle(
                fonts[i % 3], sizes[i % 3], colors[i % 3], i % 5 == 0, i % 7 == 0
            );
            chars.add(new DocumentCharacter(c, i / 80, i % 80, style));
        }

        System.out.printf("字符对象数: %d，样式对象数: %d（节省内存约%.1f%%）%n",
                chars.size(), CharacterStyleFactory.getPoolSize(),
                (1 - (double) CharacterStyleFactory.getPoolSize() / chars.size()) * 100);
    }
}
```

## 八、七种结构型模式对比

| 模式 | 解决的问题 | 关键区别 | Spring 应用 |
|------|-----------|---------|------------|
| 代理 | 控制访问，增强与业务无关的逻辑 | 接口不变，加控制 | AOP，@Transactional |
| 装饰器 | 动态增强对象功能 | 同接口，叠加功能 | HttpServletRequestWrapper |
| 适配器 | 转换不兼容接口 | 接口转换，解决兼容 | HandlerAdapter |
| 门面 | 简化复杂子系统访问 | 多系统统一入口 | Service 层门面 |
| 桥接 | 抽象和实现独立变化 | 组合解决双维度变化 | JDBC DriverManager |
| 组合 | 统一处理树形结构 | 叶子/容器一致操作 | AWT Component |
| 享元 | 减少大量相似对象内存 | 共享内在状态 | 连接池，Integer缓存 |
