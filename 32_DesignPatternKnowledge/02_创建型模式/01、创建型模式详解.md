# 创建型模式详解（5种）

> 创建型模式专注于**对象的创建机制**，目标是把创建过程从使用过程中解耦，让系统不依赖具体类如何被实例化。GoF 定义 5 种创建型模式。本文通过生动的小故事来引入每个模式。

---

## 故事引入

### 1. 单例模式 —— 《小伙子，新娘到底叫啥名啊？》

相传很早以前，有一个小镇上住着一个年轻人，他各方面都很优秀，却一直没有找到合适的对象结婚。有一天，一位大娘给他介绍了一个姑娘，年轻人家人着急想看看姑娘长什么样，便问大娘："小伙子，新娘到底叫啥名啊？"

大娘笑着说："镇上只有一个叫那个名字的姑娘，你们不用问，直接去娶就好！"

这就是**单例模式**——在整个系统中只有一个实例，就像镇上只有一个"那位姑娘"。

### 2. 原型模式 —— 《小王的学习秘籍》

小王成绩很差学习方法不对，后来他发现学霸的学习笔记非常有效，于是借来复印了一份。他不需要自己重新整理知识点，直接复制学霸的笔记，在此基础上修改成适合自己的版本，就能快速提升成绩。

**原型模式**正是如此——通过克隆已有对象来创建新对象，不需要从零开始初始化。

### 3. 简单工厂/工厂方法/抽象工厂 —— 《没有中间商赚差价？》

从前有个 Pizza 店，老板娘直接面对顾客。顾客要什么 Pizza 她就做什么，后来订单太多了，老板娘忙不过来，就开始培养徒弟：
- 大徒弟只会做水果Pizza
- 二徒弟只会做肉类Pizza
- 三徒弟只会做蔬菜Pizza

顾客来了，老板娘根据顾客需求让不同徒弟去做——这就是**工厂方法**。

后来镇上开了连锁店，每家店都要配置会做各种Pizza的师傅，成本太高。于是总部制定了一套标准：每家店只要会"按照总部配方做Pizza"就行——这就是**抽象工厂**。

### 4. 建造者模式 —— 《网红小王买车记》

网红小王想买一辆车，他对销售说："我要发动机强劲的、车身要骚红色的、内饰要真皮的、轮毂要21寸的、音响要哈曼卡顿的……"

销售说："好的，我帮您定制！"

销售把小王的要求一一记录，然后让工厂按照这些要求组装——这就是**建造者模式**：将一个复杂对象的构建过程和它的表示分离。

## 一、单例模式（Singleton）

### 定义与意图

确保一个类**只有一个实例**，并提供一个全局访问点。适用于需要全局唯一实例的场景：数据库连接池、线程池、配置管理、日志记录器、ID 生成器。

### 类图角色

```
Singleton
├── - instance: Singleton      // 私有静态实例
├── - Singleton()              // 私有构造方法
└── + getInstance(): Singleton // 公共静态访问方法
```

### 五种实现方式

**1. 饿汉式（推荐，简单）**

```java
public class ConfigManager {
    // 类加载时立即初始化，JVM 类加载机制保证线程安全
    private static final ConfigManager INSTANCE = new ConfigManager();
    private Properties config;

    private ConfigManager() {
        config = new Properties();
        // 加载配置文件
        try (InputStream is = getClass().getResourceAsStream("/application.properties")) {
            config.load(is);
        } catch (Exception e) {
            throw new RuntimeException("配置加载失败", e);
        }
    }

    public static ConfigManager getInstance() {
        return INSTANCE;
    }

    public String get(String key) { return config.getProperty(key); }
    public String get(String key, String defaultValue) { return config.getProperty(key, defaultValue); }
}
```

**2. 懒汉式（双重检查锁 DCL，延迟加载推荐）**

```java
public class DatabaseConnectionPool {
    // volatile 防止指令重排序
    private static volatile DatabaseConnectionPool instance;
    private final List<Connection> pool = new ArrayList<>();
    private static final int POOL_SIZE = 10;

    private DatabaseConnectionPool() {
        // 初始化连接池（代价高，应该懒加载）
        for (int i = 0; i < POOL_SIZE; i++) {
            pool.add(createConnection());
        }
    }

    public static DatabaseConnectionPool getInstance() {
        if (instance == null) {                    // 第一次检查（无锁，高性能）
            synchronized (DatabaseConnectionPool.class) {
                if (instance == null) {            // 第二次检查（防重复创建）
                    instance = new DatabaseConnectionPool();
                }
            }
        }
        return instance;
    }

    public synchronized Connection getConnection() {
        if (pool.isEmpty()) throw new RuntimeException("连接池已满");
        return pool.remove(0);
    }

    public synchronized void returnConnection(Connection conn) {
        pool.add(conn);
    }

    private Connection createConnection() { return null; /* 实际创建DB连接 */ }
}
```

**3. 静态内部类（延迟加载 + 线程安全，最优雅）**

```java
public class IdGenerator {
    private final AtomicLong counter = new AtomicLong(0);

    private IdGenerator() {}

    // Holder 类只有在 getInstance() 第一次被调用时才会加载，实现延迟初始化
    // JVM 类加载的串行机制天然保证线程安全，无需 synchronized
    private static class Holder {
        private static final IdGenerator INSTANCE = new IdGenerator();
    }

    public static IdGenerator getInstance() {
        return Holder.INSTANCE;
    }

    public long nextId() {
        return System.currentTimeMillis() * 1000 + counter.incrementAndGet() % 1000;
    }
}
```

**4. 枚举（最强防护，防反射+防序列化）**

```java
public enum AppConfig {
    INSTANCE;

    private final Map<String, String> config = new HashMap<>();

    // 枚举构造方法（只执行一次）
    AppConfig() {
        config.put("app.name", "MyApp");
        config.put("app.version", "1.0.0");
        config.put("server.port", "8080");
    }

    public String get(String key) { return config.getOrDefault(key, ""); }
    public int getInt(String key, int defaultValue) {
        String val = config.get(key);
        return val != null ? Integer.parseInt(val) : defaultValue;
    }
}

// 使用
String appName = AppConfig.INSTANCE.get("app.name");
```

**5. 防反射破坏**

```java
public class SafeSingleton {
    private static final SafeSingleton INSTANCE = new SafeSingleton();

    private SafeSingleton() {
        // 防止反射攻击：如果实例已存在，拒绝通过反射创建第二个
        if (INSTANCE != null) {
            throw new IllegalStateException("请使用 getInstance() 方法获取实例");
        }
    }

    public static SafeSingleton getInstance() { return INSTANCE; }

    // 防止序列化攻击：反序列化时返回已有实例
    protected Object readResolve() { return INSTANCE; }
}
```

```java
// 场景：全局唯一的事件总线
public class EventBus {
    private static final EventBus INSTANCE = new EventBus();
    private final Map<Class<?>, List<Consumer<Object>>> listeners = new ConcurrentHashMap<>();

    private EventBus() {}

    public static EventBus getInstance() { return INSTANCE; }

    @SuppressWarnings("unchecked")
    public <T> void subscribe(Class<T> eventType, Consumer<T> listener) {
        listeners.computeIfAbsent(eventType, k -> new ArrayList<>())
                 .add((Consumer<Object>) listener);
    }

    public void publish(Object event) {
        List<Consumer<Object>> eventListeners = listeners.get(event.getClass());
        if (eventListeners != null) {
            eventListeners.forEach(l -> l.accept(event));
        }
    }
}
```

## 二、工厂方法模式（Factory Method）

### 定义与意图

定义一个创建对象的接口，但让子类决定实例化哪个类。工厂方法把对象的实例化推迟到子类。核心思想：把"**创建什么**"延迟到子类，父类只定义"**何时创建**"的流程。

### 类图角色

```
Creator（抽象工厂）
├── + factoryMethod(): Product   // 工厂方法（抽象）
└── + someOperation()            // 使用工厂方法的业务逻辑

ConcreteCreatorA extends Creator
└── + factoryMethod(): ConcreteProductA

ConcreteCreatorB extends Creator
└── + factoryMethod(): ConcreteProductB

Product（产品接口）
ConcreteProductA implements Product
ConcreteProductB implements Product
```

### Demo

```java
// 产品接口
public interface Logger {
    void info(String message);
    void error(String message, Throwable e);
    void warn(String message);
}

// 具体产品：控制台日志
public class ConsoleLogger implements Logger {
    private final String name;
    public ConsoleLogger(String name) { this.name = name; }

    public void info(String message) {
        System.out.printf("[INFO][%s] %s%n", name, message);
    }
    public void error(String message, Throwable e) {
        System.err.printf("[ERROR][%s] %s: %s%n", name, message, e.getMessage());
    }
    public void warn(String message) {
        System.out.printf("[WARN][%s] %s%n", name, message);
    }
}

// 具体产品：文件日志
public class FileLogger implements Logger {
    private final String name;
    private final String filePath;
    public FileLogger(String name, String filePath) { this.name = name; this.filePath = filePath; }

    public void info(String message) { writeToFile("[INFO] " + message); }
    public void error(String message, Throwable e) { writeToFile("[ERROR] " + message); }
    public void warn(String message) { writeToFile("[WARN] " + message); }
    private void writeToFile(String content) { /* 写文件 */ }
}

// 抽象工厂：定义工厂方法
public abstract class LoggerFactory {
    // 工厂方法（子类决定创建哪种 Logger）
    protected abstract Logger createLogger(String name);

    // 模板方法：工厂创建后统一做初始化
    public Logger getLogger(String name) {
        Logger logger = createLogger(name);
        // 可以在这里统一做些初始化（如注册到管理中心）
        return logger;
    }
}

// 具体工厂：控制台日志工厂
public class ConsoleLoggerFactory extends LoggerFactory {
    @Override
    protected Logger createLogger(String name) {
        return new ConsoleLogger(name);
    }
}

// 具体工厂：文件日志工厂
public class FileLoggerFactory extends LoggerFactory {
    private final String logDir;
    public FileLoggerFactory(String logDir) { this.logDir = logDir; }

    @Override
    protected Logger createLogger(String name) {
        return new FileLogger(name, logDir + "/" + name + ".log");
    }
}

// 使用
LoggerFactory factory = new ConsoleLoggerFactory();
// 切换到文件日志只需换工厂，业务代码不变
// LoggerFactory factory = new FileLoggerFactory("/var/logs");

Logger logger = factory.getLogger("OrderService");
logger.info("订单创建成功");
```

## 三、抽象工厂模式（Abstract Factory）

### 定义与意图

提供一个创建**一系列相关或相互依赖对象**的接口，而无需指定它们的具体类。解决产品族的一致性问题——一个抽象工厂创建出来的所有产品，保证是同一个产品家族，互相兼容。

### 类图角色

```
AbstractFactory
├── + createProductA(): AbstractProductA
└── + createProductB(): AbstractProductB

ConcreteFactory1 extends AbstractFactory  // 产品族1
ConcreteFactory2 extends AbstractFactory  // 产品族2

AbstractProductA
├── ConcreteProductA1
└── ConcreteProductA2

AbstractProductB
├── ConcreteProductB1
└── ConcreteProductB2
```

### Demo

```java
// 数据访问层的抽象工厂（不同数据库实现）
public interface UserDao {
    User findById(Long id);
    void save(User user);
}

public interface OrderDao {

    List<Order> findByUserId(Long userId);

}

public interface ProductDao {
    Product findById(Long id);
    List<Product> findAll(int page, int size);
}

// 抽象工厂：创建一套完整的 DAO 产品族
public interface DaoFactory {

    OrderDao createOrderDao();
    ProductDao createProductDao();
}

// MySQL 实现族
public class MySQLUserDao implements UserDao {

    public User findById(Long id) {
        return jdbcTemplate.queryForObject("SELECT * FROM users WHERE id=?", new UserRowMapper(), id);
    }
    public void save(User user) { jdbcTemplate.update("INSERT INTO users ...", user.getUsername()); }
}

// 具体工厂：MySQL 完整实现族
public class MySQLDaoFactory implements DaoFactory {

    public UserDao createUserDao()       { return new MySQLUserDao(jdbcTemplate); }
    public OrderDao createOrderDao()     { return new MySQLOrderDao(jdbcTemplate); }
    public ProductDao createProductDao() { return new MySQLProductDao(jdbcTemplate); }
}
```

## 四、建造者模式（Builder）

### 定义与意图

将复杂对象的**构建过程**与其**表示分离**，使得同样的构建过程可以创建不同的表示。适合：参数多（特别是可选参数多）、需要分步骤构建、需要不可变对象。

### 类图角色

```
Builder（建造者接口）
├── + buildPartA()
├── + buildPartB()
└── + getResult(): Product

ConcreteBuilder implements Builder
Director（指挥者，可选）
└── + construct(Builder)
Product（最终产品）
```

### Demo

```java
// 电商平台的查询条件构建器
public class ProductQueryCondition {
    // 所有字段都是 final，保证对象不可变
    private final String keyword;
    private final Long categoryId;
    private final BigDecimal minPrice;
    private final BigDecimal maxPrice;
    private final Integer minStock;
    private final String sortField;
    private final String sortOrder;
    private final int pageNum;
    private final int pageSize;
    private final boolean includeOffShelf;

    private ProductQueryCondition(Builder builder) {
        this.keyword = builder.keyword;
        this.categoryId = builder.categoryId;
        this.minPrice = builder.minPrice;
        this.maxPrice = builder.maxPrice;
        this.minStock = builder.minStock;
        this.sortField = builder.sortField;
        this.sortOrder = builder.sortOrder;
        this.pageNum = builder.pageNum;
        this.pageSize = builder.pageSize;
        this.includeOffShelf = builder.includeOffShelf;
    }

    // getters...
    public String getKeyword() { return keyword; }
    public Long getCategoryId() { return categoryId; }
    public BigDecimal getMinPrice() { return minPrice; }
    public BigDecimal getMaxPrice() { return maxPrice; }

    public static class Builder {
        // 必填参数
        private final int pageNum;
        private final int pageSize;
        // 可选参数，有默认值
        private String keyword;
        private Long categoryId;
        private BigDecimal minPrice;
        private BigDecimal maxPrice;
        private Integer minStock;
        private String sortField = "created_time";
        private String sortOrder = "DESC";
        private boolean includeOffShelf = false;

        public Builder(int pageNum, int pageSize) {
            if (pageNum < 1) throw new IllegalArgumentException("页码不能小于1");
            if (pageSize < 1 || pageSize > 100) throw new IllegalArgumentException("每页数量1-100");
            this.pageNum = pageNum;
            this.pageSize = pageSize;
        }

        public Builder keyword(String keyword) { this.keyword = keyword; return this; }
        public Builder category(Long categoryId) { this.categoryId = categoryId; return this; }
        public Builder priceRange(BigDecimal min, BigDecimal max) {
            if (min != null && max != null && min.compareTo(max) > 0) {
                throw new IllegalArgumentException("最低价不能大于最高价");
            }
            this.minPrice = min; this.maxPrice = max; return this;
        }
        public Builder minStock(int minStock) { this.minStock = minStock; return this; }
        public Builder sortBy(String field, String order) {
            this.sortField = field; this.sortOrder = order; return this;
        }
        public Builder includeOffShelf() { this.includeOffShelf = true; return this; }

        public ProductQueryCondition build() {
            return new ProductQueryCondition(this);
        }
    }

    @Override
    public String toString() {
        return String.format("查询条件[关键词=%s, 分类=%s, 价格区间=[%s,%s], 页码=%d, 每页=%d]",
                keyword, categoryId, minPrice, maxPrice, pageNum, pageSize);
    }
}

// 使用：链式调用，一目了然
ProductQueryCondition condition = new ProductQueryCondition.Builder(1, 20)
        .keyword("iPhone")
        .category(100L)
        .priceRange(new BigDecimal("5000"), new BigDecimal("15000"))
        .minStock(10)
        .sortBy("sales", "DESC")
        .build();

System.out.println(condition);
```

```java
// 场景：HTTP 请求构建器（类似 OkHttp 的 Request.Builder）
public class ApiRequest {
    private final String url;
    private final String method;
    private final Map<String, String> headers;
    private final Map<String, Object> params;
    private final Object body;
    private final int timeoutMs;
    private final int retries;

    private ApiRequest(Builder builder) { /* 赋值 */
        this.url = builder.url; this.method = builder.method;
        this.headers = Collections.unmodifiableMap(builder.headers);
        this.params = Collections.unmodifiableMap(builder.params);
        this.body = builder.body; this.timeoutMs = builder.timeoutMs;
        this.retries = builder.retries;
    }

    public static class Builder {
        private final String url;
        private String method = "GET";
        private final Map<String, String> headers = new LinkedHashMap<>();
        private final Map<String, Object> params = new LinkedHashMap<>();
        private Object body;
        private int timeoutMs = 5000;
        private int retries = 0;

        public Builder(String url) { this.url = url; }
        public Builder post() { this.method = "POST"; return this; }
        public Builder put() { this.method = "PUT"; return this; }
        public Builder delete() { this.method = "DELETE"; return this; }
        public Builder header(String key, String value) { headers.put(key, value); return this; }
        public Builder param(String key, Object value) { params.put(key, value); return this; }
        public Builder bearerToken(String token) { return header("Authorization", "Bearer " + token); }
        public Builder json(Object body) {
            this.body = body;
            return header("Content-Type", "application/json");
        }
        public Builder timeout(int ms) { this.timeoutMs = ms; return this; }
        public Builder retries(int n) { this.retries = n; return this; }
        public ApiRequest build() { return new ApiRequest(this); }
    }
}
```

## 五、原型模式（Prototype）

### 定义与意图

通过**克隆已有对象**来创建新对象，而不是通过 new 重新初始化。适合：对象创建成本高（复杂初始化）；需要创建与已有对象差别不大的新对象；避免反复执行相同的初始化逻辑。

### 类图角色

```
Prototype
└── + clone(): Prototype

ConcretePrototype implements Prototype
└── + clone(): ConcretePrototype
```

### Demo

```java

    // 深拷贝：复制自己，独立的技能/装备/背包列表
    @Override
    public GameCharacter clone() {
        try {
            GameCharacter copy = (GameCharacter) super.clone();
            copy.skills = new ArrayList<>(this.skills);           // 独立的 List
            copy.equipment = new HashMap<>(this.equipment);       // 独立的 Map
            copy.inventory = new ArrayList<>(this.inventory);     // 独立的背包
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new AssertionError("不应该发生", e);
        }
    }

    // 设置玩家个性化信息（在克隆的副本上修改）
    public GameCharacter withName(String name) {
        this.name = name;
        return this;
    }

    private List<String> loadSkillsByClass(String cls) {
        Map<String, List<String>> skillMap = Map.of(
            "warrior", List.of("重击", "盾墙", "战吼"),
            "mage",    List.of("火球术", "冰封", "魔法护盾"),
            "archer",  List.of("穿心箭", "急速射", "毒箭")
        );
        return new ArrayList<>(skillMap.getOrDefault(cls, List.of()));
    }

    private Map<String, Integer> loadDefaultEquipment(String cls) {
        Map<String, Map<String, Integer>> equipMap = Map.of(
            "warrior", Map.of("剑", 50, "盾", 30),
            "mage",    Map.of("法杖", 80, "法袍", 20),
            "archer",  Map.of("弓", 70, "箭", 200)
        );
        return new HashMap<>(equipMap.getOrDefault(cls, Map.of()));
    }

    @Override
    public String toString() {
        return String.format("角色[%s, 职业=%s, Lv.%d, HP=%d, 技能=%s]",
                name, characterClass, level, health, skills);
    }
}

// 原型注册表：管理各类原型
public class CharacterPrototypeRegistry {
    private static final Map<String, GameCharacter> prototypes = new HashMap<>();

    static {
        // 每种职业只初始化一次（复杂初始化只执行3次）
        prototypes.put("warrior", new GameCharacter("warrior"));
        prototypes.put("mage",    new GameCharacter("mage"));
        prototypes.put("archer",  new GameCharacter("archer"));
    }

    public static GameCharacter createCharacter(String type, String playerName) {
        GameCharacter prototype = prototypes.get(type);
        if (prototype == null) throw new IllegalArgumentException("未知职业: " + type);
        // 克隆原型，设置玩家名称，不需要重新初始化职业数据

    }
```

```java
// 场景：报表模板克隆（避免重复构建复杂的报表配置）
public class ReportTemplate implements Cloneable {
    private String templateName;
    private List<ReportColumn> columns;    // 列定义
    private List<ReportFilter> filters;    // 过滤条件
    private ChartConfig chartConfig;       // 图表配置
    private StyleConfig styleConfig;       // 样式配置

    // 深拷贝
    @Override
    public ReportTemplate clone() {
        try {
            ReportTemplate copy = (ReportTemplate) super.clone();
            copy.columns = columns.stream()
                    .map(ReportColumn::clone)
                    .collect(Collectors.toList());
            copy.filters = new ArrayList<>(filters);
            copy.chartConfig = chartConfig.clone();
            copy.styleConfig = styleConfig.clone();
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new AssertionError(e);
        }
    }

    public ReportTemplate withName(String name) {
        this.templateName = name;
        return this;
    }

    public ReportTemplate addFilter(ReportFilter filter) {
        this.filters.add(filter);
        return this;
    }
}
```

## 六、五种创建型模式对比

| 模式 | 解决的问题 | 核心机制 | 适用场景 |
|------|-----------|---------|---------|
| 单例 | 保证全局唯一实例 | 静态变量 + 私有构造 | 连接池、配置、日志 |
| 工厂方法 | 解耦对象创建，子类决定创建什么 | 抽象工厂方法 | 一种产品，多种实现 |
| 抽象工厂 | 创建一族兼容产品 | 工厂接口含多个工厂方法 | 多种产品，多个产品族 |
| 建造者 | 构建有多个复杂步骤的对象 | 链式方法 + build() | 参数多的复杂对象 |
| 原型 | 克隆已有对象，避免重复初始化 | clone() 方法 | 创建代价高的相似对象 |
