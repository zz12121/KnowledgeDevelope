## 模块一：设计原则

📄 **文件**：`01_设计原则/01、SOLID七大设计原则.md`

| 原则 | 缩写 | 一句话总结 |
|------|------|----------|
| 单一职责原则 | SRP | 一个类只负责一件事 |
| 开闭原则 | OCP | 对扩展开放，对修改关闭 |
| 里氏替换原则 | LSP | 子类必须能替换父类 |
| 接口隔离原则 | ISP | 接口应当细化，不强迫实现不需要的方法 |
| 依赖倒转原则 | DIP | 依赖抽象，不依赖具体实现 |
| 迪米特法则 | LoD | 只与直接朋友通信 |
| 合成复用原则 | CRP | 优先用组合，而非继承 |

## 模块二：创建型模式（5种）

📄 **文件**：`02_创建型模式/01、创建型模式详解.md`

### 单例模式（Singleton）
- **意图**：确保一个类只有一个实例，并提供全局访问点
- **五种实现**：饿汉式 / 懒汉式 / DCL双检锁 / 静态内部类（推荐）/ 枚举（最安全）
- **防破坏**：防反射（构造器判断）+ 防序列化（`readResolve()`）
- **Spring应用**：`DefaultSingletonBeanRegistry` 三级缓存

```java
// 静态内部类（推荐写法）
public class IdGenerator {
    private IdGenerator() {}
    private static class Holder {
        private static final IdGenerator INSTANCE = new IdGenerator();
    }
    public static IdGenerator getInstance() { return Holder.INSTANCE; }
}
```

### 工厂方法模式（Factory Method）
- **意图**：定义创建对象的接口，由子类决定实例化哪个类
- **角色**：Product（产品接口）+ ConcreteProduct + Creator（抽象工厂）+ ConcreteCreator
- **Spring应用**：`BeanFactory`、`FactoryBean`

### 抽象工厂模式（Abstract Factory）
- **意图**：提供创建一系列相关产品的接口，而不指定具体类
- **核心概念**：产品族（同一主题的一组产品）
- **典型场景**：DAO层切换（MySQL整套 vs MongoDB整套）

### 建造者模式（Builder）
- **意图**：分步构建复杂对象，同样的构建过程可生成不同的表示
- **链式Builder**：必填参数放构造器 + 可选参数链式调用 + `build()` 整体校验
- **典型场景**：查询条件构建、API请求对象、`ProcessBuilder`

```java
// 链式Builder核心结构
Order order = new Order.Builder(userId, items)
    .couponCode("SAVE20")
    .deliveryAddress("北京市...")
    .remark("请尽快发货")
    .build(); // build() 内做整体校验
```

### 原型模式（Prototype）
- **意图**：通过克隆现有对象创建新对象，避免重复初始化
- **浅拷贝 vs 深拷贝**：引用字段是否独立复制
- **原型注册表**：`PrototypeRegistry` 管理多种原型，按需克隆

## 模块三：结构型模式（7种）

📄 **文件**：`03_结构型模式/01、结构型模式详解.md`

### 代理模式（Proxy）
- **意图**：为目标对象提供代理，控制访问并可增强功能
- **三种实现**：
  - 静态代理（编译期）
  - JDK动态代理（需实现接口，`InvocationHandler`）
  - CGLIB代理（无需接口，字节码增强，`MethodInterceptor`）
- **Spring应用**：AOP核心，`JdkDynamicAopProxy` / `ObjenesisCglibAopProxy`
- **MyBatis应用**：`MapperProxy`

### 装饰器模式（Decorator）
- **意图**：动态给对象添加功能，且可以叠加（优于继承）
- **与代理的区别**：装饰器增强功能（用户主动套），代理控制访问（透明代理）
- **典型场景**：`java.io`流体系、数据处理管道（压缩→加密→日志）

### 适配器模式（Adapter）
- **意图**：将一个类的接口转换为客户期望的另一个接口
- **类适配器**（继承）vs **对象适配器**（组合，推荐）
- **Spring应用**：`HandlerAdapter`（适配不同类型的Controller）

### 门面模式（Facade）
- **意图**：为子系统提供一个统一的简化接口
- **典型场景**：`PlaceOrderFacade`（协调库存/支付/物流/通知多个子系统）

### 桥接模式（Bridge）
- **意图**：将抽象部分与实现部分分离，使两者可独立变化
- **解决类爆炸**：N种消息类型 × M种推送渠道 → N+M个类，而非N×M
- **两个维度**：消息类型（抽象层）+ 推送渠道（实现层）

### 组合模式（Composite）
- **意图**：将对象组合成树形结构，统一处理叶子和容器
- **关键**：叶子和容器实现同一接口
- **典型场景**：组织架构树、菜单树、MyBatis `SqlNode` 动态SQL树

### 享元模式（Flyweight）
- **意图**：用共享技术支持大量细粒度对象，节省内存
- **内在状态**（可共享，存在享元中）vs **外在状态**（不可共享，每次传入）
- **典型场景**：字符样式池、`Integer.valueOf()` [-128,127] 缓存、`String.intern()`

## 模块四：行为型模式（11种）

📄 **文件**：`04_行为型模式/01、行为型模式详解.md`

### 模板方法模式（Template Method）
- **意图**：定义算法骨架，将某些步骤延迟到子类
- **关键**：模板方法用 `final` 修饰，不可被重写
- **钩子方法**：提供默认空实现，子类可选择性重写
- **Spring应用**：`JdbcTemplate.execute()`、`AbstractApplicationContext.refresh()`

### 策略模式（Strategy）
- **意图**：定义一系列算法，使它们可互相替换
- **消除if-else**：Spring Map注入（`List<Strategy>` → `Map<String, Strategy>`）
- **Spring应用**：`HandlerMapping` 选择处理器

### 观察者模式（Observer）
- **意图**：一对多依赖，当对象状态改变时通知所有依赖者
- **Spring实现**：`ApplicationEvent` + `@EventListener` + `@Async`
- **@Order控制顺序**：同步监听器按 `@Order` 顺序执行

### 责任链模式（Chain of Responsibility）
- **意图**：请求沿链传递，每个节点决定处理或传递
- **Spring应用**：`Filter`链（`FilterChain`）、拦截器链
- **@Order自动排序**：`List<Filter>` 注入时按 `@Order` 自动排序

### 命令模式（Command）
- **意图**：将请求封装为对象，支持撤销、排队、日志记录
- **关键接口**：`execute()` + `undo()`
- **撤销栈**：`Deque<Command>` 实现撤销/重做

### 迭代器模式（Iterator）
- **意图**：提供统一的遍历接口，不暴露集合内部结构
- **分页懒加载**：`PagedIterator<T>` 按需请求下一页

### 中介者模式（Mediator）
- **意图**：将网状多对多通信转为星形，集中到中介者
- **典型场景**：机场塔台、聊天室、GUI事件总线

### 备忘录模式（Memento）
- **意图**：捕获并恢复对象内部状态，不破坏封装
- **撤销栈 + 重做栈**：两个 `Deque<Snapshot>` 实现完整撤销/重做

### 状态模式（State）
- **意图**：对象行为随状态改变而改变，状态转移封装在状态类中
- **状态机实现**：每个状态类持有 Context 引用，通过 `context.setState()` 转移
- **典型场景**：订单状态机、支付状态机、审批流

### 访问者模式（Visitor）
- **意图**：对稳定的数据结构定义多种操作，操作可独立变化
- **双分派**：`element.accept(visitor)` → `visitor.visit(element)`
- **适用条件**：数据结构稳定，但操作频繁变化

### 解释器模式（Interpreter）
- **意图**：为简单语言定义文法和解释器
- **实现结构**：终结符表达式（叶子）+ 非终结符表达式（组合）= 递归文法树
- **典型场景**：权限规则DSL、表达式引擎、SQL解析

## 高频面试考点速查

### 必会：代理三种实现对比

| 实现方式 | 要求 | 原理 | 性能 | 场景 |
|---------|------|------|------|------|
| 静态代理 | 编译期手写 | 实现同接口 | 最快 | 简单场景 |
| JDK动态代理 | 必须实现接口 | `Proxy` + `InvocationHandler` | 较快 | Spring AOP接口代理 |
| CGLIB | 无需接口，类不能final | 字节码继承子类 | 稍慢（首次） | Spring AOP类代理 |

### 必会：策略 vs 模板方法

| 维度 | 策略模式 | 模板方法 |
|------|---------|---------|
| 变化方式 | 组合（has-a） | 继承（is-a） |
| 灵活性 | 运行时切换 | 编译期固定 |
| 推荐程度 | ✅ 更推荐 | 适合骨架稳定时 |

### 必会：单例五种写法排名

```
安全性：枚举 > 静态内部类 = DCL > 饿汉式 > 懒汉式（无锁）
推荐度：静态内部类（日常）/ 枚举（序列化场景）
```

### 必会：Spring中用到的模式

```
单例 → DefaultSingletonBeanRegistry（三级缓存）
工厂 → BeanFactory / FactoryBean
模板方法 → JdbcTemplate / refresh()
代理 → JdkDynamicAopProxy / CglibAopProxy
观察者 → ApplicationEvent / @EventListener
责任链 → FilterChain / HandlerInterceptorChain
适配器 → HandlerAdapter
装饰器 → HttpServletRequestWrapper / BeanWrapper
```
