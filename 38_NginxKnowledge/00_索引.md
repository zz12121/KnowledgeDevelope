## 核心知识点速记

### 架构
- **Master-Worker 多进程**：Master 管理，Worker 单线程 + epoll 事件驱动
- **worker_processes auto**：等于 CPU 核数；**worker_connections 10240**：生产推荐值
- **惊群问题**：多个 Worker 同时被唤醒争抢连接 → `accept_mutex on`（旧版）/内核级锁（Nginx 1.9.1+）
- **热重载**：`nginx -s reload` → Master 优雅替换 Worker，不中断现有连接

### location 匹配优先级（高→低）
```
= 精确  >  ^~ 前缀阻断正则  >  ~/~* 正则（按顺序）  >  普通前缀（最长）
```

### 负载均衡 5 种策略
| 策略 | 指令 | 适用场景 |
|------|------|---------|
| 轮询（默认） | - | 无状态服务 |
| 加权轮询 | `weight=N` | 服务器性能不均 |
| IP Hash | `ip_hash` | Session 会话保持 |
| 最少连接 | `least_conn` | 长连接/响应时间差异大 |
| 哈希 | `hash $key` | 缓存穿透优化 |

### 性能三件套
- **sendfile on**：零拷贝传输静态文件（内核直接 socket → 磁盘，不走用户空间）
- **gzip on**：压缩文本响应（建议 `gzip_min_length 1k`，图片不压缩）
- **keepalive**：upstream keepalive 保持后端长连接，减少 TCP 握手开销

### 限流两板斧
- `limit_req_zone`：令牌桶/漏桶，控制**请求速率**（QPS）
- `limit_conn_zone`：控制**并发连接数**

### 高可用标准架构
```
DNS/GSLB → LVS+Keepalived（四层VIP）→ Nginx集群（七层SSL/限流/路由）→ 后端服务
```
