## 一、索引类型速查

| 索引类型 | 创建语法 | 适用场景 |
|---------|---------|---------|
| 单字段 | `{field: 1}` / `{field: -1}` | 最基础，等值/范围/排序 |
| 复合索引 | `{a: 1, b: 1}` | 多字段查询，遵循最左前缀 |
| 多键索引 | 数组字段自动创建 | 数组内容查询 |
| 文本索引 | `{title: "text"}` | 全文搜索（`$text`） |
| 地理空间 | `{loc: "2dsphere"}` | `$near`/`$geoWithin` |
| 哈希索引 | `{field: "hashed"}` | 分片键，等值查询 |
| TTL 索引 | `{date:1}, {expireAfterSeconds:N}` | 自动过期删除 |
| 稀疏索引 | `{field:1}, {sparse:true}` | 可选字段 + 唯一约束 |
| 部分索引 | `{field:1}, {partialFilterExpression:{...}}` | 只索引部分文档，节省空间 |

## 二、复合索引 ESR 原则

**E**quality → **S**ort → **R**ange（等值字段最前，排序字段中间，范围字段最后）

```javascript
// 查询：status="active" AND age BETWEEN 18 AND 30 ORDER BY createdAt DESC
// ESR 设计的索引：
db.users.createIndex({ status: 1, createdAt: -1, age: 1 })
//                     E（等值）  S（排序）        R（范围）

// 使用：
db.users.find({ status: "active", age: { $gte: 18, $lte: 30 } })
         .sort({ createdAt: -1 })
```

## 三、执行计划分析

```javascript
db.collection.find({...}).explain("executionStats")
```

**关键字段**：
- `stage: "IXSCAN"` — 走索引（好）
- `stage: "COLLSCAN"` — 全集合扫描（差，需加索引）
- `stage: "FETCH"` — 回表读文档（覆盖查询可以消除此阶段）
- `totalDocsExamined / nReturned ≈ 1` — 索引效率高；比值大 → 索引选择性差

## 四、聚合管道优化原则

1. **`$match` 越早越好** — 减少后续处理的数据量，且能用上索引
2. **`$project` 早减字段** — 减少文档大小，降低后续阶段内存占用
3. **`$sort` + `$limit`** — 若排序字段有索引，顺序 Top N 效率极高
4. **避免内存限制** — 单个聚合操作默认 100MB 内存上限；超出需 `allowDiskUse: true`
5. **`$lookup` 的外键建索引** — 确保 `foreignField` 上有索引

## 五、查询优化清单

```javascript
// ✅ 好：投影只取需要的字段（减少网络传输）
db.users.find({ status: "active" }, { name: 1, email: 1, _id: 0 })

// ✅ 好：$regex 加前缀锚点（可以用索引）
db.users.find({ name: /^张/ })  // 前缀匹配，走索引
// ❌ 差：中间匹配，全集合扫描
db.users.find({ name: /张/ })

// ✅ 好：分页用游标（大数据集）
const nextPage = db.articles.find({ _id: { $gt: lastId } }).limit(20)
// ❌ 差：大 skip（越到后面越慢）
db.articles.find().skip(100000).limit(20)

// ✅ 好：hint 强制使用指定索引
db.orders.find({ status: "pending" }).hint({ status: 1, createdAt: -1 })
```
