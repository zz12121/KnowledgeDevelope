# 常见实战场景实现

## 1. 限流

### 1.1 为什么用 Redis 限流

限流的本质是**计数 + 比较 + 拒绝**，这三步必须是原子的，否则并发下会有竞态。Redis 的单线程特性加上 Lua 脚本，天然满足这个需求，而且 Redis 的 QPS 比数据库高几个数量级，用它做限流不会成为系统瓶颈。

### 1.2 四种限流算法

**固定窗口**：用 `INCR` + `EXPIRE` 计数，超过阈值拒绝。实现简单，缺点是窗口临界点可以瞬间通过 2 倍请求量。

**滑动窗口**：用 ZSet 存每次请求的时间戳，每次请求时删除窗口外的元素再计数。精确但内存消耗随请求量线性增长。

```lua
-- 滑动窗口 Lua 脚本
local key = KEYS[1]
local windowStart = ARGV[1]   -- 当前时间 - 窗口大小
local now = ARGV[2]
local limit = tonumber(ARGV[3])

redis.call('ZREMRANGEBYSCORE', key, 0, windowStart)
local count = redis.call('ZCARD', key)

if count >= limit then
    return 0
else
    redis.call('ZADD', key, now, now .. math.random())
    redis.call('EXPIRE', key, tonumber(ARGV[4]))
    return 1
end
```

**令牌桶**：允许一定程度的突发流量，适合接口有自然流量波峰的场景。可以用 Redis + Lua 自己实现，生产上直接用 Redisson 的 `RRateLimiter`。

**漏桶**：严格控制处理速率，输出平稳，适合对下游有保护需求的场景（例如调用第三方 API）。

### 1.3 生产推荐方案

```java
// Redisson 令牌桶（内部基于 Lua 脚本实现，分布式安全）
RRateLimiter limiter = redisson.getRateLimiter("api:limit:user:" + userId);
// 每分钟 100 个令牌（首次调用才需要初始化）
limiter.trySetRate(RateType.PER_CLIENT, 100, 1, RateIntervalUnit.MINUTES);

if (limiter.tryAcquire()) {
    // 请求通过
} else {
    throw new RateLimitException("请求过于频繁，请稍后再试");
}
```

## 2. 排行榜

ZSet 是天生的排行榜数据结构，score 存分数，member 存用户 ID，自动排序，`ZREVRANGE` 取前 N 名，`ZREVRANK` 查某用户排名，复杂度都是 O(logN)。

```java
String leaderboardKey = "game:leaderboard:daily";

// 更新分数（不存在则新增，已存在则更新）
jedis.zadd(leaderboardKey, score, userId);

// 增量加分
jedis.zincrby(leaderboardKey, deltaScore, userId);

// 取前 10 名（含分数）
Set<Tuple> top10 = jedis.zrevrangeWithScores(leaderboardKey, 0, 9);

// 查询某用户排名（从 0 开始，+1 转为人类排名）
Long rank = jedis.zrevrank(leaderboardKey, userId);
long humanRank = rank != null ? rank + 1 : -1;

// 查询某分数段的用户
Set<String> users = jedis.zrevrangeByScore(leaderboardKey, maxScore, minScore);
```

**多榜单方案**：日榜用 `leaderboard:daily:20260315`，周榜用 `leaderboard:week:2026-11`，总榜用 `leaderboard:all`。日榜设置 25 小时 TTL，自动清理历史数据。

**分数相同排名处理**：ZSet 默认分数相同时按 member 字典序排，如果要做到"同分先到的排前面"，可以把 score 设计成 `原始分数 * 10^10 + (MAX_TIME - 发生时间戳)`，把时间因素编码进 score 里。

## 3. 签到功能

Bitmap 是签到的最优方案。一个用户一个月的签到数据只需要 31 位，远不到一个字节，1000 万用户一个月才 40MB 不到。

```java
// key 格式：sign:{userId}:{year}-{month}
String key = "sign:" + userId + ":" + year + "-" + String.format("%02d", month);

// 签到（offset 为日期 0-30，注意月份第一天对应 offset=0）
jedis.setbit(key, dayOfMonth - 1, true);

// 查询当月签到总天数
long signCount = jedis.bitcount(key);

// 查询今天是否已签到
boolean signedToday = jedis.getbit(key, LocalDate.now().getDayOfMonth() - 1);

// 获取连续签到天数（从今天往前找，遇到 0 停止）
int continuous = 0;
for (int i = 0; i < 30; i++) {
    LocalDate date = LocalDate.now().minusDays(i);
    String dateKey = "sign:" + userId + ":" + date.getYear() + "-" + date.getMonthValue();
    if (jedis.getbit(dateKey, date.getDayOfMonth() - 1)) {
        continuous++;
    } else {
        break;  // 断签了，停止往前找
    }
}
```

**全站签到统计**：`BITOP AND result sign:all:20260315 sign:all:20260314`，结果 key 中值为 1 的 bit 表示连续两天都签到的用户 ID。

## 4. 点赞功能

点赞有两个核心需求：**防重复点赞** 和 **快速查询点赞数**。

```java
// 点赞数量：用 Hash 存 → like:post:123 → {count: 45}
// 用户点赞记录：用 Set 存 → user_liked:user456 → {"post:123", "comment:789"}

public boolean toggleLike(String userId, String postId) {
    String likeCountKey = "like:post:" + postId;
    String userLikedKey = "user_liked:" + userId;

    boolean alreadyLiked = jedis.sismember(userLikedKey, postId);
    
    // 用事务保证原子性
    Transaction tx = jedis.multi();
    if (alreadyLiked) {
        tx.hincrBy(likeCountKey, "count", -1);
        tx.srem(userLikedKey, postId);
    } else {
        tx.hincrBy(likeCountKey, "count", 1);
        tx.sadd(userLikedKey, postId);
    }
    tx.exec();
    
    return !alreadyLiked;
}
```

**注意**：Redis 中的点赞数是缓存，需要**异步定期同步到数据库**（例如每分钟将变更的 key 批量写入 MySQL）。Redis 宕机重启后，用数据库的值重新加载缓存，不能只依赖 Redis 做永久存储。

## 5. 共同好友 / 共同关注

这是 Set 集合运算的经典场景，`SINTER` 直接求交集，语义上天然贴合"共同"的概念。

```java
String key1 = "friends:" + userId1;
String key2 = "friends:" + userId2;

// 共同好友
Set<String> common = jedis.sinter(key1, key2);

// 好友推荐（userId1 关注但 userId2 没关注的）
Set<String> recommendations = jedis.sdiff(key1, key2);

// 求两人好友的并集
Set<String> union = jedis.sunion(key1, key2);
```

**性能注意**：`SINTER` 的时间复杂度是 O(N × M)，N 是最小集合的大小，M 是集合个数。当用户好友数量很大（比如大 V 有几百万粉丝）时，直接用 `SINTER` 会很慢。实际处理方法是对集合大小设上限，超过阈值时截断或改用离线计算。

## 6. 附近的人

GEO 命令封装了 Geohash 编码和 ZSet 范围查询，用起来非常简单。

```java
String locationKey = "user:locations";

// 更新用户位置
jedis.geoadd(locationKey, longitude, latitude, userId);

// 查找半径 5km 内的用户，按距离升序，最多返回 20 个
List<GeoRadiusResponse> nearby = jedis.georadius(
    locationKey, myLongitude, myLatitude, 5, GeoUnit.KM,
    GeoRadiusParam.geoRadiusParam()
        .withDist()    // 返回距离
        .withCoord()   // 返回坐标
        .sortAscending()
        .count(20)
);

// 获取两人之间的距离
Double dist = jedis.geodist(locationKey, userId1, userId2, GeoUnit.KM);
```

**隐私保护**：真实应用中不能把精确位置直接存 Redis，通常会做**位置模糊化**处理，比如把坐标精度降低到街道级别（保留 3-4 位小数），或者只存"区域 ID"而非精确经纬度。

**数据过期**：用户可能长时间不更新位置，`GEORADIUS` 返回的可能是几天前的陈旧位置。需要在 GEO 之外维护一个 `user:location:time:{userId}` 记录最后更新时间，在结果过滤时剔除太久没更新的数据。

## 7. 购物车

购物车用 Hash 存储：一个用户一个 Hash，field 是 productId，value 是数量。

```java
String cartKey = "cart:" + userId;

// 添加/修改商品数量
jedis.hset(cartKey, productId, String.valueOf(quantity));
jedis.expire(cartKey, 30 * 24 * 3600);  // 30 天有效期

// 删除某商品
jedis.hdel(cartKey, productId);

// 获取所有商品
Map<String, String> cart = jedis.hgetAll(cartKey);

// 清空购物车
jedis.del(cartKey);

// 用户登录时合并游客购物车
Pipeline pipeline = jedis.pipelined();
for (Map.Entry<String, Integer> entry : tempCart.entrySet()) {
    pipeline.hincrBy(cartKey, entry.getKey(), entry.getValue());
}
pipeline.expire(cartKey, 30 * 24 * 3600);
pipeline.sync();
```

**注意细节**：商品价格不要存在购物车里，只存数量。下单时实时从商品服务取价格，避免价格变动后购物车里的金额不一致。

## 8. 秒杀系统

秒杀核心是**原子地扣减库存并防止超卖**。用 Lua 脚本把"检查库存 → 扣减 → 记录用户"三步封装成原子操作。

```lua
-- 秒杀 Lua 脚本
local stockKey = KEYS[1]   -- 库存 key
local userKey = KEYS[2]    -- 已参与用户 Set
local userId = ARGV[1]

-- 防止重复秒杀
if redis.call('SISMEMBER', userKey, userId) == 1 then
    return -1  -- 重复购买
end

-- 检查库存
local stock = tonumber(redis.call('GET', stockKey))
if not stock or stock <= 0 then
    return 0  -- 库存不足
end

-- 原子扣减
redis.call('DECR', stockKey)
redis.call('SADD', userKey, userId)
return 1  -- 秒杀成功
```

```java
// Java 调用
String result = jedis.eval(luaScript, 2,
    "seckill:stock:" + productId,
    "seckill:users:" + productId,
    userId).toString();
// 1: 成功, 0: 库存不足, -1: 重复购买
```

**完整架构**：秒杀成功后不要同步写数据库，而是往 MQ 发消息，消费者异步创建订单、扣减 MySQL 库存。Redis 只负责"是否抢到"的快速判断，数据库只负责最终落库。

## 9. 分布式 Session

用 Redis 存 Session，解决分布式部署下多实例间 Session 不共享的问题。

```java
// 创建 Session
String sessionId = UUID.randomUUID().toString().replace("-", "");
String sessionKey = "session:" + sessionId;

Map<String, String> sessionData = new HashMap<>();
sessionData.put("userId", user.getId());
sessionData.put("username", user.getUsername());
sessionData.put("loginTime", String.valueOf(System.currentTimeMillis()));

jedis.hset(sessionKey, sessionData);
jedis.expire(sessionKey, 30 * 60);  // 30 分钟过期

// 读取时刷新过期时间（滑动过期）
Map<String, String> data = jedis.hgetAll(sessionKey);
if (data != null && !data.isEmpty()) {
    jedis.expire(sessionKey, 30 * 60);
}
```

**Spring Session 集成**：实际项目里更推荐直接用 Spring Session + Redis，加一个注解就搞定，不需要手写。

```xml
<dependency>
    <groupId>org.springframework.session</groupId>
    <artifactId>spring-session-data-redis</artifactId>
</dependency>
```

```java
@EnableRedisHttpSession(maxInactiveIntervalInSeconds = 1800)
public class SessionConfig {}
```

## 10. 分布式唯一 ID

用 Redis 的 `INCR` 命令生成自增 ID，天然全局唯一，性能极高。

```java
// 简单全局自增 ID
long id = jedis.incr("id:order");

// 带日期前缀的 ID（格式：20260315000001）
String dateStr = LocalDate.now().format(DateTimeFormatter.BASIC_ISO_DATE); // 20260315
String key = "id:order:" + dateStr;
long seq = jedis.incr(key);
jedis.expire(key, 25 * 3600);  // 第二天自动过期
String orderId = dateStr + String.format("%06d", seq);
```

**号段模式（减少 Redis 访问）**：

```lua
-- 一次性申请 1000 个 ID
local key = KEYS[1]
local step = tonumber(ARGV[1])
local maxId = redis.call('INCRBY', key, step)
return {maxId - step + 1, maxId}  -- 返回 [起始ID, 结束ID]
```

应用层拿到号段后在内存里自增，用完再请求 Redis，大幅降低 Redis 压力。

## 11. 推荐系统（协同过滤）

Redis 的 Set 和 ZSet 可以快速实现基于物品的协同过滤。

```java
// 记录用户行为（用 ZSet 存最近 N 个行为，score 为时间戳）
String userBehaviorKey = "user:behavior:" + userId;
jedis.zadd(userBehaviorKey, System.currentTimeMillis(), itemId);
jedis.zremrangeByRank(userBehaviorKey, 0, -1001);  // 只保留最近 1000 条

// 更新物品相似度（当两个物品被同一用户浏览时，相似度 +1）
String simKey = "item:sim:" + itemId1;
jedis.zincrby(simKey, 1, itemId2);
jedis.zincrby("item:sim:" + itemId2, 1, itemId1);

// 推荐：取用户最近浏览的物品，找它们的相似物品
Set<String> recentItems = jedis.zrevrange(userBehaviorKey, 0, 9);

Pipeline pipeline = jedis.pipelined();
for (String item : recentItems) {
    pipeline.zrevrange("item:sim:" + item, 0, 19);
}
// 合并结果，去掉用户已看过的，按相似度排序
```

## 12. 搜索自动补全

用 ZSet 的 `ZRANGEBYLEX` 命令实现前缀匹配，score 全部设为 0，用字典序排列，然后另一个 ZSet 按热度排序。

```java
// 索引词条（将词条所有前缀都加进去）
public void addSearchTerm(String term) {
    for (int i = 1; i <= term.length(); i++) {
        jedis.zadd("suggest:index", 0, term.substring(0, i));
    }
    jedis.zincrby("suggest:score", 1, term);  // 热度 +1
}

// 自动补全
public List<String> suggest(String prefix, int limit) {
    // 用 ZRANGEBYLEX 找所有以 prefix 开头的词
    Set<String> candidates = jedis.zrangeByLex("suggest:index",
        "[" + prefix, "[" + prefix + "\xff");

    // 按热度排序，返回 top N
    return jedis.zrevrangeWithScores("suggest:score", 0, -1)
        .stream()
        .map(Tuple::getElement)
        .filter(term -> term.startsWith(prefix))
        .limit(limit)
        .collect(Collectors.toList());
}
```

## 13. 社交关系链（关注/粉丝）

用 ZSet 存关系，score 为关注时间戳，可以按关注时间分页。

```java
// 关注
public void follow(String userId, String targetId) {
    long now = System.currentTimeMillis();
    Transaction tx = jedis.multi();
    tx.zadd("following:" + userId, now, targetId);    // 我的关注列表
    tx.zadd("followers:" + targetId, now, userId);    // 对方的粉丝列表
    tx.exec();
}

// 取消关注
public void unfollow(String userId, String targetId) {
    Transaction tx = jedis.multi();
    tx.zrem("following:" + userId, targetId);
    tx.zrem("followers:" + targetId, userId);
    tx.exec();
}

// 获取关注列表（按时间倒序，支持分页）
public List<String> getFollowing(String userId, int page, int size) {
    int start = (page - 1) * size;
    return new ArrayList<>(jedis.zrevrange("following:" + userId, start, start + size - 1));
}

// 共同关注
public Set<String> getCommonFollowing(String uid1, String uid2) {
    String tempKey = "temp:common:" + uid1 + ":" + uid2;
    jedis.zinterstore(tempKey, "following:" + uid1, "following:" + uid2);
    Set<String> result = jedis.zrange(tempKey, 0, -1);
    jedis.del(tempKey);
    return result;
}

// 是否互相关注（双向关注 = 朋友关系）
public boolean isMutualFollow(String uid1, String uid2) {
    return jedis.zscore("following:" + uid1, uid2) != null
        && jedis.zscore("following:" + uid2, uid1) != null;
}
```
