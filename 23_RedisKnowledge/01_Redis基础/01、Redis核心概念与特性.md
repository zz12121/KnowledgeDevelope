# Redis 核心概念与特性

## 1. Redis 是什么

Redis（Remote Dictionary Server）是一个**开源的内存数据结构存储系统**，可作为数据库、缓存和消息中间件使用。

**核心设计哲学**：
- **内存优先**：数据主要存储在内存，读写直接操作内存数据结构，这是高性能的根本
- **数据持久化**：通过 RDB 快照和 AOF 日志将内存数据持久化到磁盘
- **键值存储模型**：键是字符串，值支持多种数据结构，操作语义由值的数据类型决定

**源码视角**：所有数据存储在全局的 `redisDb` 结构中，其核心是哈希表，存储键到值的映射。值通过 `redisObject` 结构体表示：
- `type` 字段标识数据类型（`REDIS_STRING`、`REDIS_LIST` 等）
- `encoding` 字段标识底层编码方式
- `ptr` 指针指向实际数据结构（SDS、字典、跳跃表等）

---

## 2. Redis 的主要特点

| 特性 | 说明 |
|------|------|
| **高性能** | 内存操作，读写速度可达 10 万+/秒 |
| **数据结构丰富** | String、Hash、List、Set、ZSet、BitMap、HyperLogLog、Geo、Stream |
| **持久化** | RDB 快照 + AOF 日志 + 混合模式 |
| **高可用** | 主从复制、哨兵模式（Sentinel）、集群模式（Cluster） |
| **原子性** | 所有命令原子执行，支持事务和 Lua 脚本 |

---

## 3. Redis 为什么这么快

**五个核心原因**：

1. **内存存储**：数据在内存，避免磁盘 I/O 瓶颈，读写延迟在微秒级

2. **单线程命令处理（6.0 前）**：避免多线程上下文切换和锁竞争，命令顺序执行无需加锁

3. **I/O 多路复用**：通过 `epoll`/`kqueue` 实现单线程管理成千上万并发连接

4. **高效数据结构**：SDS、字典、跳跃表、ziplist 等均经过精心设计，时间复杂度低

5. **简单的协议**：RESP 协议简单高效，解析开销小

**源码角度（事件循环）**：Redis 启动后进入 `ae.c` 的 `aeMain` 函数事件循环，通过 `aeApiPoll`（封装 epoll 等）监听所有 socket 事件。命令到达时，文件事件处理器读入缓冲区、解析、调用命令函数（如 `setCommand`）执行，最后写回客户端。整个过程在单线程内顺序执行，高效且无锁。

---

## 4. Redis 单线程还是多线程

**6.0 之前**：网络 I/O 和命令处理是**单线程**，但持久化（fork 子进程）、异步删除（UNLINK）、集群同步等由后台线程/子进程执行。

**Redis 6.0 多线程模型**：
- **引入原因**：万兆网卡普及后，网络 I/O 处理开销成为瓶颈，单线程在发送大量数据时受限
- **多线程范围**：仅用于**网络 I/O**（解析请求、发送响应），命令执行仍是单线程
- **启用配置**：默认关闭，需设置 `io-threads-do-reads yes` 和 `io-threads N`（建议 CPU 核心数的 3/4）
- **适用场景**：对大结果集操作（`LRANGE`、`HGETALL`）效果显著，简单 GET/SET 提升不明显

## 5. Redis 与 Memcached 的区别

| 维度 | Redis | Memcached |
|------|-------|-----------|
| **数据类型** | 支持 String、Hash、List、Set、ZSet 等 | 仅支持字符串键值对 |
| **持久化** | 支持 RDB 和 AOF | 不提供持久化 |
| **架构模型** | 单线程处理命令（6.0 前） | 多线程，可利用多核 |
| **内存管理** | 多种淘汰策略，支持内存优化编码 | LRU 算法，相对简单 |
| **功能特性** | 发布订阅、事务、Lua 脚本等高级功能 | 功能聚焦于缓存 |
| **值大小限制** | 最大 512MB | 单个值通常限制 1MB |

**选择建议**：需要复杂数据结构、持久化或高级功能选 Redis；纯缓存场景且无需持久化可考虑 Memcached。

---

## 6. Redis 的典型应用场景

- **缓存**：数据库查询结果、热点数据，设置过期时间
- **会话存储**：分布式 Session 集中管理
- **消息队列**：基于 List/Stream 实现生产消费
- **排行榜**：ZSet 按分数排序，`ZREVRANGE` 获取 TopN
- **分布式锁**：`SET lock_key unique_value NX PX 30000`
- **实时统计**：HyperLogLog 统计 UV，BitMap 记录签到
- **地理位置**：GEO 命令实现附近的人、距离计算
- **限流**：Lua 脚本 + INCR 实现固定窗口/滑动窗口限流
