# 五大基本数据类型

Redis 提供五种基本数据类型，每种类型根据数据特征动态选择最优的底层编码，通过 `OBJECT ENCODING key` 命令可查看当前编码。

---

## 1. String（字符串）

String 是最基础的类型，通过 `redisObject` 系统封装，根据存储内容动态选择三种编码：

| 编码 | 条件 | 说明 |
|------|------|------|
| **int** | 可表示为 64 位有符号整数 | 直接将整数值存入 ptr 指针位置，零额外开销 |
| **embstr** | 字符串 ≤ 44 字节 | redisObject 与 SDS 连续分配，只需一次 malloc，内存局部性好，只读（修改即转为 raw） |
| **raw** | 字符串 > 44 字节 | redisObject 和 SDS 分开存储，支持动态修改 |

**44 字节阈值由来**：64（jemalloc 最小分配单元）- 16（redisObject）- 3（sdshdr8）- 1（`\0`）= 44 字节

**编码转换规则**：
- `int` 编码执行非数值命令（如 APPEND）→ `raw`
- `embstr` 被修改 → `raw`（单向，不可逆）

**常用命令**：`SET/GET/MSET/MGET/INCR/INCRBY/SETEX/SETNX`

**典型场景**：缓存、计数器（INCR 原子性）、分布式锁（SET NX PX）

---

## 2. Hash（哈希）

Hash 根据数据规模在两种编码间切换：

| 编码 | 条件 | 特点 |
|------|------|------|
| **listpack**（Redis 7+） / **ziplist** | 所有键值长度 < 64 字节 且 键值对数量 < 128 | 内存紧凑，O(N) 查询 |
| **hashtable** | 超出上述阈值 | O(1) 查询，内存开销较大 |

**hashtable 实现**：字典包含两个哈希表（`ht[0]` 和 `ht[1]`），采用链地址法解决冲突，支持**渐进式 rehash**——当负载因子超过阈值时，分批迁移数据，避免单次 rehash 导致性能抖动。

**常用命令**：`HSET/HGET/HMSET/HMGET/HGETALL/HDEL/HLEN/HKEYS/HVALS`

**典型场景**：存储对象（如用户信息），避免序列化开销，支持单字段更新

---

## 3. List（列表）

List 经历了显著的结构演进：

- **3.2 前**：linkedlist（双向链表，内存碎片多）或 ziplist（元素少时使用）
- **3.2+ 至 7.0 前**：**quicklist**——ziplist 的双向链表，每个节点是一个 ziplist，通过 `list-max-ziplist-size` 控制节点大小
- **7.0+**：节点改为 **listpack**，彻底解决 ziplist 的连锁更新问题

**quicklist 的设计权衡**：将大列表分割为多个小块，限制连锁更新影响范围，兼顾内存效率和操作性能。

**常用命令**：`LPUSH/RPUSH/LPOP/RPOP/LRANGE/LLEN/LINDEX/BRPOP`

**典型场景**：消息队列（LPUSH + BRPOP）、最新列表（LPUSH + LRANGE 取最新 N 条）

---

## 4. Set（集合）

Set 根据元素内容和数量选择编码：

| 编码 | 条件 | 特点 |
|------|------|------|
| **intset** | 所有元素均为整数 且 数量 ≤ `set-max-intset-entries`（默认 512） | 有序整数数组，二分查找，极高内存密度 |
| **hashtable** | 包含非整数元素 或 超出数量阈值 | O(1) 操作，字典键存元素，值为 NULL |

**intset 编码升级**：当添加超出当前编码范围的整数时，自动从 16 位 → 32 位 → 64 位升级，不可逆。

**常用命令**：`SADD/SREM/SMEMBERS/SISMEMBER/SCARD/SINTER/SUNION/SDIFF`

**典型场景**：标签系统、好友关系（SINTER 求共同好友）、去重统计

---

## 5. Sorted Set（有序集合/ZSet）

每个元素包含一个分数（score），按分数排序，支持范围查询。

| 编码 | 条件 | 特点 |
|------|------|------|
| **listpack**（Redis 7+） / **ziplist** | 所有元素长度 < 64 字节 且 数量 < 128 | 内存紧凑，按分数排序存储 |
| **skiplist + dict** | 超出上述阈值 | 跳跃表支持 O(logN) 范围查询，字典支持 O(1) 单键查找 |

**skiplist + dict 双重结构**：以空间换时间，两种结构共享元素节点，确保不同操作场景下的最优性能：
- 跳跃表（O(logN)）：`ZRANGE`、`ZREVRANGE`、`ZRANGEBYSCORE` 等范围查询
- 字典（O(1)）：`ZSCORE`、`ZRANK` 等单键操作

**常用命令**：`ZADD/ZRANGE/ZREVRANGE/ZRANGEBYSCORE/ZRANK/ZREVRANK/ZSCORE/ZINCRBY/ZCARD/ZREM`

**典型场景**：排行榜、延迟队列（score 为执行时间戳）、带权重的任务调度
