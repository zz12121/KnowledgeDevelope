# 集合框架总览

> **核心关键词**：Collection、Map、Iterator、比较器、线程安全、选型指南

---

## 一、集合框架体系结构

```
java.util 集合体系

Collection（接口）
  ├── List（有序、可重复）
  │     ├── ArrayList       动态数组，随机访问 O(1)
  │     ├── LinkedList      双向链表，插入/删除 O(1)，也实现了 Deque
  │     ├── Vector          同步的动态数组（不推荐使用）
  │     └── CopyOnWriteArrayList  写时复制，读不加锁（并发包）
  │
  ├── Set（不重复）
  │     ├── HashSet         无序，基于 HashMap 实现
  │     ├── LinkedHashSet   维护插入顺序，基于 LinkedHashMap
  │     ├── TreeSet         自然排序/自定义排序，基于 TreeMap（红黑树）
  │     └── CopyOnWriteArraySet  写时复制（并发包）
  │
  └── Queue（队列）
        ├── LinkedList      也实现了 Deque（双端队列）
        ├── ArrayDeque      基于数组的双端队列，比 LinkedList 更高效
        ├── PriorityQueue   堆（小顶堆），自然排序或自定义比较器
        └── BlockingQueue（并发包）
              ├── LinkedBlockingQueue
              ├── ArrayBlockingQueue
              ├── SynchronousQueue
              └── PriorityBlockingQueue

Map（接口，键值对，键不重复）
  ├── HashMap             无序，允许 null 键/值，非线程安全
  ├── LinkedHashMap       维护插入/访问顺序，基于 HashMap + 双向链表
  ├── TreeMap             按键有序，红黑树实现
  ├── Hashtable           过时，同步，不允许 null（不推荐）
  ├── Properties          继承 Hashtable，用于配置文件
  ├── WeakHashMap         弱键引用，键可被 GC 回收
  ├── IdentityHashMap     用 == 而非 equals 比较键
  └── ConcurrentHashMap   高性能并发 Map（并发包）
```

## 二、Collection 与 Map 接口核心 API

### 2.1 Collection 通用方法

```java
// 增
boolean add(E e);
boolean addAll(Collection<? extends E> c);

// 删
boolean remove(Object o);
boolean removeAll(Collection<?> c);
void clear();

// 查
boolean contains(Object o);
boolean containsAll(Collection<?> c);
int size();
boolean isEmpty();

// 迭代
Iterator<E> iterator();

// 转数组
Object[] toArray();
<T> T[] toArray(T[] a);

// JDK 8+
boolean removeIf(Predicate<? super E> filter);
void forEach(Consumer<? super E> action);
Stream<E> stream();
```

### 2.2 List 特有方法

```java
// 按索引操作
E get(int index);
E set(int index, E element);
void add(int index, E element);
E remove(int index);

// 查找
int indexOf(Object o);       // 第一次出现的位置
int lastIndexOf(Object o);   // 最后一次出现的位置

// 子列表（视图，修改会影响原列表）
List<E> subList(int fromIndex, int toIndex);

// 排序（JDK 8+）
void sort(Comparator<? super E> c);
```

### 2.3 Map 通用方法

```java
// 增/改
V put(K key, V value);
void putAll(Map<? extends K, ? extends V> m);

// 删
V remove(Object key);
boolean remove(Object key, Object value);  // JDK 8+
void clear();

// 查
V get(Object key);
boolean containsKey(Object key);
boolean containsValue(Object value);
int size();

// 视图
Set<K> keySet();
Collection<V> values();
Set<Map.Entry<K, V>> entrySet();

// JDK 8+ 新增（非常常用）
V getOrDefault(Object key, V defaultValue);
V putIfAbsent(K key, V value);
V computeIfAbsent(K key, Function<? super K, ? extends V> mappingFunction);
V computeIfPresent(K key, BiFunction<? super K, ? super V, ? extends V> remappingFunction);
V merge(K key, V value, BiFunction<? super V, ? super V, ? extends V> remappingFunction);
void forEach(BiConsumer<? super K, ? super V> action);
```

## 三、迭代器模式

### 3.1 Iterator 接口

```java
public interface Iterator<E> {
    boolean hasNext();
    E next();
    default void remove() { throw new UnsupportedOperationException(); }
    default void forEachRemaining(Consumer<? super E> action) { ... }
}
```

### 3.2 for-each 的本质

```java
// for-each 语法糖，编译器会将其翻译为迭代器写法
for (String s : list) {
    System.out.println(s);
}

// 等价于：
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String s = it.next();
    System.out.println(s);
}
```

### 3.3 ListIterator（List 特有）

```java
ListIterator<String> lit = list.listIterator();
// 支持双向遍历
while (lit.hasPrevious()) {
    String s = lit.previous();
}
// 支持在迭代中修改
lit.add("newElement");   // 在当前位置插入
lit.set("updated");      // 替换上次 next/previous 返回的元素
```

## 四、排序与比较

### 4.1 Comparable（自然排序）

实现 `Comparable` 接口，让对象本身具有可比性：

```java
class Student implements Comparable<Student> {
    String name;
    int age;
    
    @Override
    public int compareTo(Student other) {
        // 按年龄升序
        return Integer.compare(this.age, other.age);
        // 负数：this < other
        // 0：   this == other
        // 正数：this > other
    }
}

// 使用场景：TreeSet/TreeMap 的自然排序、Collections.sort()
List<Student> students = ...;
Collections.sort(students);  // 使用 Comparable
```

### 4.2 Comparator（外部比较器）

```java
// 匿名类写法（传统）
Comparator<Student> byName = new Comparator<Student>() {
    @Override
    public int compare(Student s1, Student s2) {
        return s1.name.compareTo(s2.name);
    }
};

// Lambda 写法（JDK 8+，推荐）
Comparator<Student> byAge = (s1, s2) -> Integer.compare(s1.age, s2.age);

// 方法引用 + 链式比较（JDK 8+，最简洁）
Comparator<Student> complex = Comparator
    .comparingInt(Student::getAge)    // 主排序：按年龄
    .thenComparing(Student::getName)  // 次排序：按姓名
    .reversed();                       // 反转

// 使用场景
students.sort(byAge);
TreeSet<Student> set = new TreeSet<>(byName);
```

## 五、Collections 工具类

```java
// 排序
Collections.sort(list);                    // 自然排序
Collections.sort(list, comparator);        // 指定比较器
Collections.reverse(list);                 // 反转
Collections.shuffle(list);                 // 随机打乱
Collections.shuffle(list, random);         // 用指定 Random

// 查找
int idx = Collections.binarySearch(list, key);      // 二分查找（必须已排序）
T min = Collections.min(collection);                // 最小值
T max = Collections.max(collection);                // 最大值

// 填充和复制
Collections.fill(list, obj);               // 用 obj 填充整个 list
Collections.copy(dest, src);               // 复制（dest.size() >= src.size()）
Collections.nCopies(n, obj);               // 返回 n 个 obj 的不可变 List

// 频率和不相交
int freq = Collections.frequency(collection, obj);  // obj 出现次数
boolean disjoint = Collections.disjoint(c1, c2);   // 两集合是否无公共元素

// 包装为线程安全
List<String> syncList = Collections.synchronizedList(list);
Map<String, Object> syncMap = Collections.synchronizedMap(map);
// 注意：迭代时仍需手动同步！

// 包装为不可变（视图）
List<String> immutableList = Collections.unmodifiableList(list);
Map<K, V> immutableMap = Collections.unmodifiableMap(map);
// 推荐用 JDK 9+ 的 List.of() / Map.of() 更安全
```

## 六、JDK 9+ 不可变集合

```java
// JDK 9+：List.of() / Set.of() / Map.of()
// 特点：真正不可变（任何修改都抛 UnsupportedOperationException），不允许 null

List<String> list = List.of("a", "b", "c");
Set<String> set = Set.of("x", "y", "z");
Map<String, Integer> map = Map.of("a", 1, "b", 2, "c", 3);

// JDK 10+：copyOf（可变集合 → 不可变副本）
List<String> copy = List.copyOf(mutableList);

// JDK 9+：Map.entry() / Map.ofEntries()
Map<String, Integer> bigMap = Map.ofEntries(
    Map.entry("a", 1),
    Map.entry("b", 2),
    Map.entry("c", 3)
    // ... 超过10个键值对时使用这个
);
```

## 七、集合选型指南

### 7.1 List 选型

| 场景 | 推荐 | 原因 |
|------|------|------|
| 默认/读多写少 | **ArrayList** | 随机访问 O(1)，内存紧凑 |
| 频繁头部插入/删除 | LinkedList | O(1) 操作 |
| 多线程读，偶尔写 | CopyOnWriteArrayList | 读无锁 |
| 多线程频繁读写 | Collections.synchronizedList | 全量锁 |

### 7.2 Set 选型

| 场景 | 推荐 | 原因 |
|------|------|------|
| 不关心顺序，快速查找 | **HashSet** | O(1) 操作 |
| 需要维护插入顺序 | LinkedHashSet | 有序 + O(1) |
| 需要排序 | TreeSet | 自然/自定义排序 |

### 7.3 Map 选型

| 场景 | 推荐 | 原因 |
|------|------|------|
| 通用 | **HashMap** | O(1) 操作，最常用 |
| 需要维护插入/访问顺序 | LinkedHashMap | 有序（LRU 场景）|
| 需要按键排序 | TreeMap | 有序 Map |
| 多线程 | **ConcurrentHashMap** | 高并发下最佳 |
| 弱键（内存敏感缓存）| WeakHashMap | GC 可回收键 |

### 7.4 Queue 选型

| 场景 | 推荐 | 原因 |
|------|------|------|
| 普通队列/双端队列 | **ArrayDeque** | 比 LinkedList 性能更好 |
| 优先级队列 | PriorityQueue | 最小堆 |
| 生产者-消费者（有界）| ArrayBlockingQueue | 有界阻塞队列 |
| 生产者-消费者（高吞吐）| LinkedBlockingQueue | 分头尾锁，并发性更好 |
| 直接交付（无缓冲）| SynchronousQueue | CachedThreadPool 使用 |

## 八、面试要点速查

| 问题 | 要点 |
|------|------|
| ArrayList vs LinkedList | 随机访问用 ArrayList（O1）；频繁增删头尾用 LinkedList |
| HashMap vs TreeMap | HashMap 无序 O(1)；TreeMap 按键有序 O(log n) |
| HashMap vs ConcurrentHashMap | 线程安全用 ConcurrentHashMap，不允许 null 键值 |
| HashSet 如何保证不重复 | 底层 HashMap，插入时判断 hashCode + equals |
| Comparable vs Comparator | Comparable：类本身可比（implements），侵入性；Comparator：外部比较器，灵活 |
| fail-fast 机制 | modCount 快照，迭代期间结构修改抛 ConcurrentModificationException |
| 线程安全集合有哪些 | CopyOnWriteArrayList、ConcurrentHashMap、BlockingQueue 系列 |
| List.of() vs Collections.unmodifiableList() | List.of() 真不可变，不含 null；unmodifiable 是视图，原集合改了视图也变 |
