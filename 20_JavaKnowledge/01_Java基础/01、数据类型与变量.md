# 数据类型与变量

> **核心关键词**：基本类型、引用类型、自动装箱拆箱、常量池、类型转换、溢出

---

## 一、数据类型全景

Java 是强类型语言，所有变量必须先声明类型。数据类型分为两大类：

```
Java 数据类型
├── 基本类型（Primitive Type）—— 8种，存在栈/常量池
│   ├── 整数型：byte(1) short(2) int(4) long(8)
│   ├── 浮点型：float(4) double(8)
│   ├── 字符型：char(2)  ← UTF-16 编码单元
│   └── 布尔型：boolean  ← JVM 实现相关，理论 1bit
└── 引用类型（Reference Type）—— 存引用地址（4或8字节）
    ├── 类（class）
    ├── 接口（interface）
    ├── 数组（array）
    └── 枚举（enum）/ 注解（annotation）
```

### 1.1 基本类型详细规格

| 类型      | 字节 | 位数 | 取值范围                                        | 默认值    | 包装类      |
| ------- | -- | -- | ------------------------------------------- | ------ | -------- |
| byte    | 1  | 8  | -128 \~ 127                                 | 0      | Byte     |
| short   | 2  | 16 | -32768 \~ 32767                             | 0      | Short    |
| int     | 4  | 32 | -2^31 \~ 2^31-1（约±21亿）                     | 0      | Integer  |
| long    | 8  | 64 | -2^63 \~ 2^63-1                             | 0L     | Long     |
| float   | 4  | 32 | ±3.4×10^38（精度约7位有效数字）                       | 0.0f   | Float    |
| double  | 8  | 64 | ±1.7×10^308（精度约15位有效数字）                     | 0.0d   | Double   |
| char    | 2  | 16 | 0 \~ 65535（'\u0000' \~ '\uFFFF'）            | \u0000 | Character |
| boolean | -  | -  | true / false（JVM 中通常用 int 0/1 表示，数组用 byte） | false  | Boolean  |

> ⚠️ **float 精度陷阱**：`0.1f + 0.2f ≠ 0.3f`，浮点数采用 IEEE 754 二进制表示，无法精确表示大多数十进制小数。**金融场景必须用 `BigDecimal`**。

## 二、自动装箱与拆箱（Autoboxing / Unboxing）

### 2.1 本质

编译器的语法糖，本质是调用包装类的静态方法：

```java
// 装箱：编译器将其转换为
Integer a = 100;
// → Integer a = Integer.valueOf(100);

// 拆箱：编译器将其转换为
int b = a;
// → int b = a.intValue();
```

### 2.2 整数缓存池（Integer Cache）

`Integer.valueOf()` 对 **-128 ~ 127** 范围内的值使用缓存，直接返回同一对象。

```java
Integer a = 127;
Integer b = 127;
System.out.println(a == b);  // true  ← 同一缓存对象

Integer c = 128;
Integer d = 128;
System.out.println(c == d);  // false ← 超出缓存范围，new 了两个对象
System.out.println(c.equals(d)); // true ← equals 比较值
```

**缓存范围可通过 JVM 参数调整**：
```
-XX:AutoBoxCacheMax=<size>  // 只能调大上限，不能调整下限 -128
```

**源码实现**（JDK 8）：
```java
// java.lang.Integer
public static Integer valueOf(int i) {
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}
```

### 2.3 装箱拆箱的性能陷阱

```java
// ❌ 性能陷阱：循环中频繁装箱，产生大量 Integer 对象
Long sum = 0L;
for (long i = 0; i < Integer.MAX_VALUE; i++) {
    sum += i;  // 每次都触发自动装箱！
}

// ✅ 正确做法：使用基本类型
long sum = 0L;
for (long i = 0; i < Integer.MAX_VALUE; i++) {
    sum += i;
}
```

**NullPointerException 陷阱**：
```java
Integer count = null;
int c = count;  // 拆箱时调用 count.intValue() → NullPointerException！

// 三元表达式中的隐式拆箱也会触发
Integer x = null;
int result = (x != null) ? x : 0;  // 安全
int bad    = true ? x : 0;         // x 被拆箱 → NPE！（因为 0 是 int，触发拆箱）
```

## 三、类型转换

### 3.1 隐式转换（宽化转换，Widening）

小范围 → 大范围，编译器自动完成，不会丢失精度（int → long 安全，int → float 可能丢精度）：

```
byte → short → int → long → float → double
                          ↗
               char → int
```

### 3.2 显式转换（强制转换，Narrowing）

大范围 → 小范围，可能丢失数据：

```java
int i = 300;
byte b = (byte) i;  // 300 的二进制低8位 → 44（截断高位）

double d = 3.99;
int n = (int) d;    // → 3（直接截断小数，不是四舍五入）
```

### 3.3 字符串与数值互转

```java
// String → 数值
int i = Integer.parseInt("123");
double d = Double.parseDouble("3.14");

// 数值 → String（三种方式，性能从低到高：+""  < String.valueOf() ≈ Integer.toString()）
String s1 = 123 + "";              // 隐式调用 StringBuilder
String s2 = String.valueOf(123);   // 推荐
String s3 = Integer.toString(123); // 推荐
```

## 四、变量分类与内存位置

| 变量类型 | 声明位置    | 存储位置          | 生命周期          |
| ---- | ------- | ------------- | ------------- |
| 局部变量 | 方法/代码块内 | 栈帧（Stack Frame） | 方法调用期间        |
| 实例变量 | 类中、方法外  | 堆（对象内）        | 对象存活期间        |
| 类变量  | static 修饰 | 方法区/元空间      | 类加载到类卸载       |
| 常量   | final static | 方法区/常量池    | 类加载到类卸载       |

```java
public class VariableDemo {
    static int classVar = 10;     // 类变量，存方法区
    int instanceVar = 20;          // 实例变量，存堆
    
    public void method() {
        int localVar = 30;         // 局部变量，存栈
        final int CONST = 40;      // 常量（局部），存栈
    }
}
```

## 五、整数溢出

Java 整数运算溢出**不会抛异常**，直接截断高位，结果"回绕"：

```java
int max = Integer.MAX_VALUE;  // 2147483647
System.out.println(max + 1);  // → -2147483648（溢出变负数）

// 检测溢出的安全方式（JDK 8+）
int result = Math.addExact(max, 1);  // → 抛 ArithmeticException

// 大数运算用 BigInteger
BigInteger big = BigInteger.valueOf(Long.MAX_VALUE).add(BigInteger.ONE);
```

## 六、实战场景与踩坑

### 场景1：== 与 equals 的混用

```java
// 包装类比较必须用 equals，== 比较的是引用地址
Integer a = 200, b = 200;
System.out.println(a == b);      // false（超出缓存范围）
System.out.println(a.equals(b)); // true

// 基本类型 == 比较值本身，没问题
int x = 200, y = 200;
System.out.println(x == y); // true
```

### 场景2：switch 支持的类型

```java
// Java 支持 switch 的类型：
// byte short int char String（JDK7+）Enum 以及对应包装类
// ❌ 不支持：long float double boolean
switch (someInt) { ... }     // ✅
switch (someLong) { ... }    // ❌ 编译错误
```

### 场景3：char 的运算

```java
char c = 'A';          // Unicode 值 65
int n = c + 1;         // → 66（char 参与运算自动提升为 int）
char next = (char)(c + 1); // → 'B'（需要强转）

// 汉字也是 char
char ch = '中';         // U+4E2D，十进制 20013
System.out.println((int)ch); // 20013
```

## 七、面试要点速查

| 问题 | 要点 |
|------|------|
| `int` 和 `Integer` 的区别 | 基本类型 vs 包装类；栈 vs 堆；不能为 null vs 可以为 null；缓存池 |
| 为什么用 `BigDecimal` 而不用 `double` | IEEE 754 无法精确表示十进制小数，`0.1 + 0.2 ≠ 0.3` |
| Integer 缓存范围 | -128 ~ 127，上限可通过 JVM 参数调整 |
| 自动拆箱的 NPE 风险 | 包装类为 null 时拆箱触发 NPE，三元表达式中尤要注意 |
| 整数溢出是否报错 | 不报错，直接截断。可用 `Math.addExact` 等方法检测 |
| `float` 和 `double` 的精度 | float 约7位有效数字，double 约15位 |
