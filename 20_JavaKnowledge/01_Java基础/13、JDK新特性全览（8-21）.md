## 一、JDK 8（2014，里程碑版本）

### 1.1 Lambda 表达式

```java
// 函数式接口：只有一个抽象方法的接口
@FunctionalInterface
interface Comparator<T> { int compare(T o1, T o2); }

// Lambda 语法：(参数) -> 表达式 或 (参数) -> { 代码块 }
List<String> list = Arrays.asList("banana", "apple", "cherry");

// 旧方式
Collections.sort(list, new Comparator<String>() {
    @Override
    public int compare(String a, String b) { return a.compareTo(b); }
});

// Lambda
list.sort((a, b) -> a.compareTo(b));

// 方法引用（Lambda 的简写）
list.sort(String::compareTo);            // 实例方法引用
list.forEach(System.out::println);       // 实例方法引用
list.stream().map(String::toUpperCase);  // 实例方法引用
Stream.of(1,2,3).map(Integer::valueOf);  // 静态方法引用
```

### 1.2 Stream API

```java
// 流：不存储数据，描述计算操作，惰性求值，可并行
List<Integer> result = list.stream()
    .filter(s -> s.startsWith("a"))   // 中间操作（惰性）
    .map(String::toUpperCase)          // 中间操作
    .sorted()                          // 中间操作
    .collect(Collectors.toList());     // 终止操作（触发计算）

// 常用收集器
Collectors.toList()
Collectors.toSet()
Collectors.toMap(User::getId, User::getName)
Collectors.groupingBy(User::getDept)            // 分组
Collectors.partitioningBy(u -> u.getAge() > 18) // 分区（true/false）
Collectors.counting()
Collectors.joining(", ", "[", "]")

// 数值流（避免装箱）
IntStream.range(0, 10).sum();
Arrays.stream(new int[]{1,2,3}).average();

// 并行流（底层 ForkJoin，适合 CPU 密集型无状态操作）
list.parallelStream().filter(...).collect(...);
```

### 1.3 Optional

```java
// 避免 NullPointerException 的容器类
Optional<String> opt = Optional.ofNullable(getValue());  // 可能为 null
Optional<String> present = Optional.of("value");          // 不能为 null
Optional<String> empty = Optional.empty();

// 常用方法
opt.isPresent()                          // 是否有值
opt.ifPresent(v -> System.out.println(v)) // 有值时执行
opt.orElse("default")                    // 无值时返回默认值
opt.orElseGet(() -> computeDefault())    // 无值时调用 Supplier
opt.orElseThrow(IllegalStateException::new) // 无值时抛异常
opt.map(String::toUpperCase)             // 变换值
opt.filter(s -> s.length() > 3)         // 过滤

// 链式调用（替代嵌套 null 检查）
String city = Optional.ofNullable(user)
    .map(User::getAddress)
    .map(Address::getCity)
    .orElse("未知");
```

### 1.4 新日期时间 API（java.time）

```java
// 不可变、线程安全，取代 Date/Calendar
LocalDate date = LocalDate.now();           // 2026-03-14
LocalTime time = LocalTime.now();           // 20:13:00
LocalDateTime dt = LocalDateTime.now();     // 2026-03-14T20:13:00
ZonedDateTime zdt = ZonedDateTime.now(ZoneId.of("Asia/Shanghai"));
Instant instant = Instant.now();            // 时间戳

// 日期计算
LocalDate tomorrow = date.plusDays(1);
LocalDate lastMonth = date.minusMonths(1);
Period period = Period.between(date1, date2);

// 格式化
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String str = dt.format(fmt);
LocalDateTime parsed = LocalDateTime.parse("2026-03-14 20:13:00", fmt);
```

### 1.5 其他 JDK 8 特性

```java
// 接口默认方法和静态方法
interface Vehicle {
    default void start() { System.out.println("Starting..."); }
    static Vehicle create() { return new Car(); }

// CompletableFuture（异步编排）
CompletableFuture.supplyAsync(() -> fetchData())

map.getOrDefault(key, defaultVal);

```

## 二、JDK 9（2017）

```java
// 1. 模块化系统（JPMS）
// module-info.java
module com.example.app {
    requires java.sql;
    requires com.example.utils;
    exports com.example.api;          // 对外暴露的包
    opens com.example.model;          // 允许反射访问
}

// 2. 集合工厂方法（不可变集合）
List<String> list = List.of("a", "b", "c");
Set<Integer> set = Set.of(1, 2, 3);
Map<String, Integer> map = Map.of("a", 1, "b", 2);
Map<String, Integer> map2 = Map.ofEntries(
    Map.entry("a", 1), Map.entry("b", 2));

// 3. Stream 新方法
Stream.iterate(0, n -> n < 100, n -> n + 1)  // 带条件的 iterate
Stream.ofNullable(nullableValue)              // 空安全
stream.takeWhile(n -> n < 5)                  // 取满足条件的前缀
stream.dropWhile(n -> n < 5)                  // 丢弃满足条件的前缀

// 4. Optional 新方法
opt.ifPresentOrElse(v -> use(v), () -> handleEmpty());
opt.or(() -> Optional.of(fallback));
opt.stream();  // Optional → Stream（方便流操作）

// 5. try-with-resources 改进（可以用 effectively final 变量）
InputStream in = new FileInputStream("file.txt");
try (in) { /* 使用 */ }  // JDK 9+：不再需要重新声明变量
```

## 三、JDK 10 ~ 11

```java
// JDK 10：var 局部变量类型推断
var list = new ArrayList<String>();   // 编译器推断类型
var map = new HashMap<String, List<Integer>>();

// 注意：var 只能用于局部变量，不能用于字段、方法参数、返回值
// var 不是动态类型，编译后就是强类型

// JDK 11：String 新方法
" hello ".strip()          // 去除首尾空白（支持 Unicode 空白，比 trim 更准确）
"  ".isBlank()             // true（全是空白）
"a\nb\nc".lines()          // 按行分割 → Stream<String>
"ab".repeat(3)             // "ababab"

// JDK 11：HttpClient（内置 HTTP 客户端，支持 HTTP/2）
HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.example.com/users"))
    .GET()
    .build();
HttpResponse<String> response = client.send(request, 
    HttpResponse.BodyHandlers.ofString());

// JDK 11：Lambda 参数的 var
list.stream().filter((@NotNull var s) -> !s.isEmpty());  // 注解配合 var
```

## 四、JDK 14 ~ 16

```java
// JDK 14：Switch 表达式（正式版）
// 旧 switch（语句，容易漏 break）
int days;
switch (month) {
    case 1: case 3: case 5: days = 31; break;
    default: days = 30;
}

// 新 switch（表达式，有返回值，用 -> 消除 fall-through）
int days = switch (month) {
    case 1, 3, 5, 7, 8, 10, 12 -> 31;
    case 4, 6, 9, 11 -> 30;
    case 2 -> 28;
    default -> throw new IllegalArgumentException();
};

// yield 用于多行 case 块的返回值
int result = switch (op) {
    case "add" -> a + b;
    case "mul" -> {
        System.out.println("Multiplying");
        yield a * b;  // 多行 case 用 yield 返回
    }
    default -> 0;
};

// JDK 15：文本块（Text Blocks）
String json = """
        {
            "name": "张三",
            "age": 25
        }
        """;
// 自动处理缩进，\\n 换行，无需转义内部引号

// JDK 16：Record（不可变数据类）
// 编译器自动生成：构造器、getter、equals、hashCode、toString
public record Point(int x, int y) {
    // 紧凑构造器（可做参数校验）
    Point {
        if (x < 0 || y < 0) throw new IllegalArgumentException();
    }
    // 可添加自定义方法
    public double distanceTo(Point other) {
        return Math.hypot(x - other.x, y - other.y);
    }
}

Point p = new Point(3, 4);
p.x();          // getter（不是 getX()）
p.y();
```

## 五、JDK 17（LTS，2021）

```java
// 1. 密封类（Sealed Classes）—— 限制继承范围
public sealed class Shape
    permits Circle, Rectangle, Triangle { }  // 只允许这三个子类

public final class Circle extends Shape {
    double radius;
}
public non-sealed class Rectangle extends Shape { }  // non-sealed = 可被任意继承

// 2. instanceof 模式匹配（Pattern Matching）
// 旧方式
if (obj instanceof String) {
    String s = (String) obj;
    System.out.println(s.length());
}

// JDK 16+ 新方式
if (obj instanceof String s) {  // 自动 cast 并赋值给 s
    System.out.println(s.length());
}

// 结合 && 条件
if (obj instanceof String s && s.length() > 5) {
    System.out.println(s.toUpperCase());
}
```

## 六、JDK 21（LTS，2023）

### 6.1 虚拟线程（Virtual Threads）— 最重磅特性

```java
// 虚拟线程：JVM 管理的轻量级线程（不直接映射 OS 线程）
// 目标：让 BIO 代码也能达到 NIO 的并发吞吐量

// 创建虚拟线程
Thread vThread = Thread.ofVirtual().start(() -> {
    System.out.println("运行在虚拟线程中");
});

// 虚拟线程执行器（每个任务一个虚拟线程）
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    IntStream.range(0, 100_000).forEach(i -> {
        executor.submit(() -> handleRequest(i));  // 轻松10万并发
    });
}

// 阻塞不再是问题：虚拟线程阻塞时只挂起虚拟线程，底层 OS 线程继续执行其他任务
// Spring Boot 3.2+ 可通过配置一键启用虚拟线程
spring.threads.virtual.enabled=true
```

### 6.2 Switch 模式匹配（正式版）

```java
// 结合 sealed class 和 pattern matching，完美替代 if-instanceof 链
String describe(Shape shape) {
    return switch (shape) {
        case Circle c    -> "圆形，半径 " + c.radius();
        case Rectangle r -> "矩形，宽 " + r.width() + " 高 " + r.height();
        case Triangle t  -> "三角形";
        // sealed class 使 switch 可以穷举，无需 default
    };
}

// 守卫模式（when 子句）
String classify(Object obj) {
    return switch (obj) {
        case Integer i when i > 0 -> "正整数";
        case Integer i when i < 0 -> "负整数";
        case Integer i            -> "零";
        case String s             -> "字符串: " + s;
        case null                 -> "null";
        default                   -> "其他";
    };
}
```

### 6.3 SequencedCollection（JDK 21）

```java
// 新接口：SequencedCollection，提供有序集合的统一访问方式
List<String> list = new ArrayList<>(List.of("a", "b", "c"));
list.getFirst();   // "a"（不用 list.get(0)）
list.getLast();    // "c"（不用 list.get(list.size()-1)）
list.addFirst("x");
list.addLast("z");
list.reversed();   // 返回反向视图
```

## 八、面试要点速查

| 问题 | 要点 |
|------|------|
| Stream 和 Collection 区别 | Stream 不存储数据、惰性求值、不可重用；Collection 存储数据 |
| 并行流注意事项 | 适合无状态 CPU 密集操作；有状态操作（排序）需注意；共享可变状态不安全 |
| Optional 能解决所有 NPE 吗 | 不能，Optional 本身也可能为 null，应作为返回值使用而非字段 |
| var 是动态类型吗 | 不是，是编译时类型推断，运行时是强类型 |
| Record 和普通类区别 | Record 不可变（字段 final）、自动生成 getter/equals/hashCode/toString、不能继承 |
| 虚拟线程原理 | JVM 管理的轻量级线程，M个虚拟线程复用N个OS线程（M>>N）；阻塞时挂起虚拟线程 |
| 密封类的作用 | 限制继承层次，使 switch 可以穷举所有子类型，提升代码安全性 |
| JDK 8 → 17 → 21 哪些是 LTS | 8、11、17、21 是长期支持版 |
