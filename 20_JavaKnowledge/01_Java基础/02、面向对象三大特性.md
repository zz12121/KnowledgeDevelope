# 面向对象三大特性

> **核心关键词**：封装、继承、多态、重写、重载、向上转型、动态分派、抽象类 vs 接口

---

## 一、封装（Encapsulation）

### 1.1 本质

将对象的**属性（状态）**隐藏起来，只通过**方法（行为）**对外暴露操作接口。

**好处**：
- 隐藏实现细节，降低耦合
- 控制数据访问权限，防止非法修改
- 便于内部逻辑修改而不影响外部调用

### 1.2 访问修饰符

| 修饰符         | 同一类 | 同一包 | 子类  | 其他包 |
| ----------- | --- | --- | --- | --- |
| `private`   | ✅   | ❌   | ❌   | ❌   |
| 默认（无修饰符）    | ✅   | ✅   | ❌   | ❌   |
| `protected` | ✅   | ✅   | ✅   | ❌   |
| `public`    | ✅   | ✅   | ✅   | ✅   |

> ⚠️ **子类继承 protected**：子类可以访问父类的 protected 成员，但**在不同包的子类中，只能通过子类引用访问，不能通过父类引用访问**。

### 1.3 标准 JavaBean 封装

```java
public class User {
    private String name;
    private int age;

    // 构造器
    public User(String name, int age) {
        this.name = name;
        setAge(age);  // 调用 setter，统一走验证逻辑
    }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) {
        if (age < 0 || age > 150) throw new IllegalArgumentException("年龄非法: " + age);
        this.age = age;
    }
}
```

## 二、继承（Inheritance）

### 2.1 本质与机制

子类继承父类，获得父类的**非 private** 成员（字段和方法），并可以扩展或覆盖它们。

```
Object
  └── Animal
        ├── Dog
        └── Cat
```

**Java 单继承**：一个类只能有一个直接父类（`extends`），但可以实现多个接口（`implements`）。

### 2.2 构造器调用链

子类构造器**必须**（显式或隐式）调用父类构造器，否则编译报错：

```java
class Animal {
    String name;
    Animal(String name) {
        this.name = name;
        System.out.println("Animal 构造: " + name);
    }
}

class Dog extends Animal {
    String breed;
    Dog(String name, String breed) {
        super(name);  // 必须是第一行！调用父类构造器
        this.breed = breed;
        System.out.println("Dog 构造: " + breed);
    }
}
```

### 2.3 方法重写（Override）

子类重写父类方法的规则（**两同两小一大**原则）：
- **两同**：方法名相同、参数列表相同
- **两小**：返回值类型 ≤ 父类（协变返回类型），抛出异常 ≤ 父类
- **一大**：访问权限 ≥ 父类

```java
class Animal {
    protected Object produce() throws Exception { return null; }
}

class Dog extends Animal {
    @Override
    public Dog produce() throws RuntimeException {  // ✅ 返回值更具体，异常更小，权限更大
        return new Dog("puppy", "柴犬");
    }
}
```

**`@Override` 注解的价值**：让编译器帮你检查是否真的在重写，避免因拼写错误等导致方法没生效（变成重载）。

### 2.4 `final` 与继承

```java
final class Immutable { }        // ❌ 无法被继承（String、Integer 都是 final）
class Parent {
    final void doSomething() {}  // ❌ 子类无法重写此方法
}
```

### 2.5 继承的问题：脆弱基类

父类的内部实现变化可能意外破坏子类行为（Josh Bloch：**优先组合，而非继承**）：

```java
// 经典案例：继承 HashSet，统计添加次数
class InstrumentedHashSet<E> extends HashSet<E> {
    private int addCount = 0;

    @Override public boolean add(E e) { addCount++; return super.add(e); }
    @Override public boolean addAll(Collection<? extends E> c) {
        addCount += c.size(); return super.addAll(c);  // ❌ 问题：super.addAll 内部调用 add，导致 addCount 双倍计数
    }
}
// 解决方案：用组合（持有 HashSet 引用）而非继承
```

## 三、多态（Polymorphism）

### 3.1 多态的类型

| 类型 | 实现方式 | 绑定时机 |
|------|---------|---------|
| 编译时多态（静态分派）| 方法重载（Overload）| 编译期 |
| 运行时多态（动态分派）| 方法重写（Override）+ 向上转型 | 运行期 |

### 3.2 运行时多态原理：虚方法表（vtable）

JVM 为每个类维护一张**虚方法表（Virtual Method Table）**，存放该类所有虚方法（非 static、非 private、非 final）的实际地址。

```
Animal 的 vtable：
  [0] Object.toString()  → Object::toString
  [1] Animal.eat()       → Animal::eat
  [2] Animal.sleep()     → Animal::sleep

Dog 的 vtable（继承并覆盖）：
  [0] Object.toString()  → Object::toString（未重写，继承）
  [1] Animal.eat()       → Dog::eat         （重写，指向 Dog 的实现）
  [2] Animal.sleep()     → Animal::sleep    （未重写，继承）
  [3] Dog.fetch()        → Dog::fetch       （Dog 新增）
```

**动态分派**就是在运行时查这张表，根据对象**实际类型**找到对应方法：

```java
Animal animal = new Dog("旺财", "拉布拉多");
animal.eat();  // 编译期：看 Animal，运行期：查 Dog 的 vtable → 执行 Dog::eat
```

### 3.3 向上转型与向下转型

```java
// 向上转型（隐式，安全）：子类引用赋给父类变量
Animal a = new Dog("旺财", "拉布拉多");

// 向下转型（强制，有风险）：父类变量强转为子类
if (a instanceof Dog) {        // 先判断，再转换
    Dog d = (Dog) a;           // 安全
    d.fetch();
}

// JDK 16+：模式匹配 instanceof，更简洁
if (a instanceof Dog d) {      // 判断 + 转换一步完成
    d.fetch();
}

// ❌ ClassCastException：类型不匹配
Animal cat = new Cat("咪咪");
Dog d2 = (Dog) cat;            // 运行时抛 ClassCastException
```

### 3.4 多态的陷阱：字段不多态

**字段（Field）是静态绑定的**，只由引用类型决定，不会发生多态：

```java
class Parent {
    int x = 10;
    void print() { System.out.println("Parent x = " + x); }
}
class Child extends Parent {
    int x = 20;  // 隐藏父类字段
    @Override void print() { System.out.println("Child x = " + x); }
}

Parent p = new Child();
System.out.println(p.x);   // 10  ← 编译期看引用类型，访问 Parent.x
p.print();                 // Child x = 20  ← 运行时多态，调用 Child::print
```

## 四、重载（Overload）vs 重写（Override）

| 对比项 | 重载（Overload）| 重写（Override）|
|--------|----------------|----------------|
| 定义位置 | 同一个类中 | 子类中 |
| 方法名 | 相同 | 相同 |
| 参数列表 | **必须不同** | 必须相同 |
| 返回值 | 无要求 | 子类返回值 ≤ 父类 |
| 访问权限 | 无要求 | 子类权限 ≥ 父类 |
| 绑定时机 | 编译期（静态分派）| 运行期（动态分派）|
| `@Override` | 不能用 | 推荐使用 |

### 重载的静态分派

编译器根据参数的**静态类型**（引用类型）选择方法，而不是运行时类型：

```java
void say(Object o)  { System.out.println("Object"); }
void say(String s)  { System.out.println("String"); }
void say(Integer i) { System.out.println("Integer"); }

Object obj = "Hello";  // 静态类型是 Object
say(obj);              // → "Object"（编译期看 Object 类型）
say("Hello");          // → "String"（静态类型是 String）
```

## 五、抽象类 vs 接口

### 5.1 对比

| 特性 | 抽象类（abstract class）| 接口（interface）|
|------|------------------------|----------------|
| 实例化 | ❌ 不能直接实例化 | ❌ 不能直接实例化 |
| 构造器 | ✅ 有构造器 | ❌ 没有构造器 |
| 继承/实现 | 单继承（extends）| 多实现（implements）|
| 字段 | 任意类型字段 | 只有 `public static final` 常量 |
| 方法 | 可有抽象方法和具体方法 | 抽象方法 + default/static/private（JDK 8+）|
| 访问修饰符 | 任意 | 方法默认 public |
| 适用场景 | **is-a** 关系，共享实现代码 | **has-a/can-do** 能力约定 |

### 5.2 接口的演变（JDK 各版本）

```java
// JDK 7：只有抽象方法和常量
interface OldStyle {
    int MAX = 100;              // 隐式 public static final
    void doWork();              // 隐式 public abstract
}

// JDK 8：新增 default 和 static 方法
interface JDK8Style {
    default void log(String msg) {     // 默认实现，子类可选择性重写
        System.out.println("[LOG] " + msg);
    }
    static JDK8Style of() {            // 工厂方法，通过接口名调用
        return new Impl();
    }
}

// JDK 9：新增 private 方法（抽取公共逻辑，不对外暴露）
interface JDK9Style {
    default void methodA() { helper(); }
    default void methodB() { helper(); }
    private void helper() { System.out.println("共用逻辑"); }
}
```

### 5.3 接口冲突解决

当一个类实现了两个接口，这两个接口有同名 default 方法：

```java
interface A { default void hello() { System.out.println("A"); } }
interface B { default void hello() { System.out.println("B"); } }

class C implements A, B {
    @Override
    public void hello() {
        A.super.hello();  // 显式指定调用哪个接口的 default 实现
    }
}
```

## 六、设计原则与实战场景

### 场景1：模板方法模式（抽象类的典型应用）

```java
// 定义算法骨架，具体步骤延迟到子类
abstract class DataMigrator {
    // 模板方法（final 防止被重写，保证流程固定）
    public final void migrate() {
        connect();
        readData();
        transformData();  // 每个子类各自实现
        writeData();
        disconnect();
    }

    private void connect()    { /* 通用连接逻辑 */ }
    private void disconnect() { /* 通用断开逻辑 */ }
    protected abstract void readData();
    protected abstract void transformData();
    protected abstract void writeData();
}

class MySQLMigrator extends DataMigrator {
    @Override protected void readData()      { /* 从 MySQL 读 */ }
    @Override protected void transformData() { /* 转换 */ }
    @Override protected void writeData()     { /* 写入目标 */ }
}
```

### 场景2：策略模式（接口的典型应用）

```java
// 接口定义能力
@FunctionalInterface
interface PayStrategy {
    void pay(BigDecimal amount);
}

// 多种实现
class AlipayStrategy    implements PayStrategy { ... }
class WechatPayStrategy implements PayStrategy { ... }

// 上下文持有策略接口
class Order {
    private PayStrategy strategy;
    Order(PayStrategy strategy) { this.strategy = strategy; }
    void checkout(BigDecimal amount) { strategy.pay(amount); }
}

// 使用（Lambda 简化）
Order order = new Order(amount -> System.out.println("支付宝支付: " + amount));
order.checkout(new BigDecimal("99.00"));
```

## 八、内部类（Inner Class）

### 8.1 四种内部类总览

Java 有四种内部类，各有适用场景：

| 类型 | 位置 | 关键特点 |
|------|------|---------|
| **成员内部类** | 类体中，非static | 隐式持有外部类实例引用，可访问外部类所有成员 |
| **静态内部类** | 类体中，加 static | 独立于外部类实例，只能访问外部类静态成员 |
| **局部内部类** | 方法/代码块内 | 作用域最窄，不能有访问修饰符 |
| **匿名内部类** | 任意表达式位置 | 没有类名，声明与实例化合一，Java 8 后多被 Lambda 替代 |

### 8.2 成员内部类（Member Inner Class）

```java
public class Outer {
    private int x = 10;

    class Inner {                         // 非 static 成员内部类
        void print() {
            System.out.println(x);        // 直接访问外部类私有字段
            System.out.println(Outer.this.x); // 明确指定外部类实例
        }
    }

    void createInner() {
        Inner inner = new Inner();        // 外部类内部直接 new
    }
}

// 外部创建：必须先有外部类实例
Outer outer = new Outer();
Outer.Inner inner = outer.new Inner();
```

**内存含义**：非静态内部类对象在堆上持有一个指向外部类实例的引用（`this$0`），编译器自动生成。这是非静态内部类造成内存泄漏的根本原因——如果内部类实例生命周期比外部类长（比如把内部类传出去），外部类就无法被 GC。

### 8.3 静态内部类（Static Nested Class）

```java
public class Outer {
    private static int staticX = 10;
    private int x = 20;

    static class StaticInner {
        void print() {
            System.out.println(staticX);  // ✅ 可以访问外部类静态成员
            // System.out.println(x);     // ❌ 不能访问外部类实例成员
        }
    }
}

// 可以直接创建，不需要外部类实例
Outer.StaticInner inner = new Outer.StaticInner();
```

**常见设计模式**：
- **Builder 模式**：`StringBuilder` 不需要 `String` 实例就能工作，`Map.Entry<K,V>` 也是静态内部类
- **单例模式（Holder 模式）**：利用类加载机制实现懒加载的线程安全单例

```java
public class Singleton {
    private Singleton() {}

    private static class Holder {
        static final Singleton INSTANCE = new Singleton();  // 类加载时初始化，线程安全
    }

    public static Singleton getInstance() {
        return Holder.INSTANCE;
    }
}
```

### 8.4 匿名内部类与 effectively final

匿名内部类的局部变量捕获机制：

```java
void demo() {
    int count = 10;           // effectively final（未被修改）
    // count = 20;            // 一旦加上这行，下面就编译报错

    Runnable r = new Runnable() {
        @Override
        public void run() {
            System.out.println(count);  // 编译器将 count 的值复制进来
        }
    };
}
```

**为什么要 effectively final**：
- 局部变量在栈上，方法结束后消失；匿名内部类对象在堆上，生命周期更长
- Java 的解决方案是**值复制**：把局部变量的值 copy 一份存到内部类对象里
- 为保证一致性（不出现"外面改了，里面没变"的混乱），要求原始值不能变

Java 8 放宽到 **effectively final**（不需要显式声明 `final`，只要事实上没有被修改即可）。

### 8.5 字节码层面

编译器会为内部类生成独立的 `.class` 文件：
- 成员内部类：`Outer$Inner.class`，带有 `this$0` 合成字段
- 静态内部类：`Outer$StaticInner.class`，无合成字段
- 匿名内部类：`Outer$1.class`、`Outer$2.class`（数字序号）
- 局部内部类：`Outer$1LocalClass.class`

## 九、面试要点速查

| 问题 | 要点 |
|------|------|
| 多态的实现原理 | vtable（虚方法表）+ 动态分派，运行时根据对象实际类型查表 |
| 字段有没有多态 | 没有，字段是静态绑定，由引用类型决定 |
| 重载是编译期还是运行期 | 编译期（静态分派），看参数的静态类型 |
| 抽象类和接口的核心区别 | 抽象类可以有构造器、普通字段、非 public 方法；接口是纯能力约定，可多实现 |
| 什么时候用抽象类、什么时候用接口 | is-a 关系 + 共享实现 → 抽象类；has-a/can-do 能力 + 多实现 → 接口 |
| 接口 default 方法冲突怎么解决 | 实现类必须重写该方法，可通过 `接口名.super.方法()` 选择调用哪个 |
| `instanceof` 的现代写法 | JDK 16+ 模式匹配：`if (a instanceof Dog d) { d.fetch(); }` |
