# 序列化与反序列化

> **核心关键词**：Serializable、serialVersionUID、transient、Externalizable、JSON、Protobuf、Kryo、性能对比

---

## 一、序列化概述

**序列化**：将 Java 对象转换为字节流（或其他格式）的过程，便于网络传输或持久化存储。  
**反序列化**：将字节流还原为 Java 对象的过程。

```
使用场景：
  1. 网络传输     → RPC（Dubbo、gRPC）传递参数与返回值
  2. 持久化存储   → 写入文件、数据库 BLOB 字段
  3. 分布式缓存   → Redis 存储 Java 对象
  4. 消息队列     → Kafka/RocketMQ 消息体序列化
  5. 深拷贝       → 序列化后反序列化得到全新对象
```

## 二、Java 原生序列化（Serializable）

### 2.1 基本使用

```java
// 实现 Serializable 接口（标记接口，无方法）
public class User implements Serializable {
    // 强烈建议显式定义，避免类结构变化导致反序列化失败
    private static final long serialVersionUID = 1L;

    private String name;
    private int age;
    private transient String password;  // transient：不参与序列化

    // static 字段也不参与序列化（属于类，不属于对象实例）
    private static String appName = "MyApp";
}

// 序列化
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("user.ser"))) {
    oos.writeObject(user);
}

// 反序列化
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("user.ser"))) {
    User user = (User) ois.readObject();
}
```

### 2.2 serialVersionUID

```java
// JVM 根据 serialVersionUID 判断序列化字节流与当前类是否兼容
// 若不显式定义，JVM 会根据类结构自动生成（类结构一变就不兼容！）

// 兼容性修改（可以继续反序列化旧数据）：
//   新增字段（反序列化时新字段取默认值）
//   删除字段（反序列化时忽略）
// 不兼容修改（必须修改 serialVersionUID）：
//   修改字段类型、修改类名/包名
```

### 2.3 自定义序列化钩子

```java
public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    private String encryptedPassword;  // 存加密后的密码

    // 自定义序列化逻辑
    private void writeObject(ObjectOutputStream oos) throws IOException {
        oos.defaultWriteObject();  // 先执行默认序列化
        oos.writeObject(encrypt(encryptedPassword));  // 额外写加密数据
    }

    // 自定义反序列化逻辑
    private void readObject(ObjectInputStream ois) 
            throws IOException, ClassNotFoundException {
        ois.defaultReadObject();  // 先执行默认反序列化
        this.encryptedPassword = decrypt((String) ois.readObject());
    }

    // 替换对象（单例模式防止反序列化破坏）
    private Object readResolve() {
        return INSTANCE;  // 返回现有单例实例
    }
}
```

### 2.4 Externalizable（完全自定义）

```java
public class User implements Externalizable {
    private String name;
    private int age;

    // 必须有无参构造器！反序列化时先调用无参构造器
    public User() {}

    @Override
    public void writeExternal(ObjectOutput out) throws IOException {
        out.writeUTF(name);
        out.writeInt(age);
    }

    @Override
    public void readExternal(ObjectInput in) throws IOException {
        this.name = in.readUTF();
        this.age = in.readInt();
    }
}
// 优点：完全控制，性能比 Serializable 好
// 缺点：需手动处理每个字段，维护成本高
```

## 三、主流序列化框架对比

### 3.1 JSON 序列化（Jackson / Gson / FastJSON）

```java
// Jackson（Spring Boot 默认）
ObjectMapper mapper = new ObjectMapper();

// 序列化
String json = mapper.writeValueAsString(user);
// → {"name":"张三","age":25}

// 反序列化
User user = mapper.readValue(json, User.class);

// List 泛型反序列化
List<User> users = mapper.readValue(json, 
    new TypeReference<List<User>>() {});

// 常用注解
@JsonProperty("user_name")   // 自定义 JSON 字段名
@JsonIgnore                  // 忽略该字段
@JsonFormat(pattern = "yyyy-MM-dd")  // 日期格式
@JsonInclude(NON_NULL)       // 为 null 的字段不序列化
```

### 3.2 Protobuf（Google，高性能二进制）

```protobuf
// user.proto
syntax = "proto3";
message User {
    string name = 1;
    int32 age = 2;
    repeated string tags = 3;
}
```

```java
// 编译生成 Java 代码后使用
User user = User.newBuilder()
    .setName("张三")
    .setAge(25)
    .addTags("java")
    .build();

byte[] bytes = user.toByteArray();           // 序列化
User decoded = User.parseFrom(bytes);        // 反序列化
```

### 3.3 Kryo（高性能 Java 序列化）

```java
// Kryo（Spark、Dubbo 可选）
Kryo kryo = new Kryo();
kryo.register(User.class);  // 注册类（提升性能，减少类名写入）

// 序列化
Output output = new Output(new FileOutputStream("user.bin"));
kryo.writeObject(output, user);
output.close();

// 反序列化
Input input = new Input(new FileInputStream("user.bin"));
User decoded = kryo.readObject(input, User.class);
input.close();
```

### 3.4 性能对比（参考数据）

```
序列化框架     序列化大小    序列化速度    反序列化速度    跨语言
Java原生       大            慢            慢             ❌
JSON（Jackson） 中           中            中             ✅
Protobuf       小（最小）    快            快             ✅
Kryo           小            快（最快）     快             ❌
Hessian        中            中            中             ✅（多语言）
```

## 四、序列化常见问题

### 4.1 序列化破坏单例

```java
// 问题：反序列化会创建新对象，破坏单例
Singleton s1 = Singleton.getInstance();
byte[] bytes = serialize(s1);
Singleton s2 = deserialize(bytes);  // s1 != s2 !!!

// 解决：实现 readResolve()
public class Singleton implements Serializable {
    private static final Singleton INSTANCE = new Singleton();
    private Singleton() {}
    
    // 反序列化时，JVM 调用此方法获取返回值，而非直接使用反序列化的对象
    private Object readResolve() {
        return INSTANCE;
    }
}
```

### 4.2 父类不可序列化时的子类序列化

```java
// 父类没有实现 Serializable，子类实现了
// 父类的字段不会被序列化！
// 反序列化时会调用父类的无参构造器初始化父类字段
// → 父类必须有无参构造器，否则抛 InvalidClassException
```

### 4.3 反序列化安全漏洞

```
Java 原生反序列化是著名的安全漏洞来源：
  攻击者构造恶意字节流 → readObject() 时触发任意代码执行
  Apache Commons Collections 反序列化漏洞（2015）
  WebLogic、JBoss、Jenkins 等大量中间件受影响

防御措施：
  1. 不接受来自不可信来源的序列化数据
  2. 使用 ObjectInputFilter（JDK 9+）白名单过滤
  3. 升级依赖版本，修复已知漏洞
  4. 优先使用 JSON/Protobuf 等安全格式
```

## 五、深拷贝的序列化实现

```java
// 利用序列化实现深拷贝（简单但性能较差）
public static <T extends Serializable> T deepCopy(T obj) throws Exception {
    ByteArrayOutputStream bos = new ByteArrayOutputStream();
    new ObjectOutputStream(bos).writeObject(obj);
    ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
    return (T) new ObjectInputStream(bis).readObject();
}

// 更高性能的深拷贝：Kryo 或 Apache Commons Lang 的 SerializationUtils
T copy = SerializationUtils.clone(obj);
```

## 六、面试要点速查

| 问题 | 要点 |
|------|------|
| serialVersionUID 作用 | 版本控制；JVM 用它判断字节流与当前类是否兼容；不写会自动生成（类改变就不兼容） |
| transient 作用 | 标记的字段不参与序列化，反序列化后取默认值（null/0/false） |
| 如何防止反序列化破坏单例 | 实现 `readResolve()` 返回现有实例 |
| Serializable vs Externalizable | Serializable 自动处理，简单；Externalizable 完全手动，性能好 |
| 父类不可序列化时注意什么 | 父类字段不被序列化；父类必须有无参构造器 |
| static 字段参与序列化吗 | 不参与，static 属于类不属于对象 |
| 反序列化安全问题 | 恶意字节流触发 RCE；使用 ObjectInputFilter 白名单、或改用安全格式 |
| Protobuf 比 JSON 快的原因 | 二进制格式（非字符串）、字段用编号而非名称、varint 压缩整数 |
