# 反射机制

> **核心关键词**：Class 对象、Method/Field/Constructor、动态代理、性能优化、模块系统

---

## 一、反射概述

**反射（Reflection）**：在**运行时**动态获取类的信息（类名、字段、方法、构造器、注解等）并操作它们的能力。

```
正常调用（编译期确定）：
  User user = new User();
  user.setName("张三");

反射调用（运行时确定）：
  Class<?> clazz = Class.forName("com.example.User");
  Object user = clazz.getDeclaredConstructor().newInstance();
  Method method = clazz.getMethod("setName", String.class);
  method.invoke(user, "张三");
```

**反射的核心价值**：框架的基石（Spring IoC、MyBatis、Jackson、JUnit 等框架大量使用）

## 二、获取 Class 对象的三种方式

```java
// 方式1：Class.forName()（最常用，可以加载任意类）
// 会触发类的初始化（执行 static 代码块）
Class<?> c1 = Class.forName("java.lang.String");

// 方式2：类名.class（编译期已知类型时，性能最好）
// 不会触发类的初始化
Class<String> c2 = String.class;

// 方式3：对象.getClass()（已有对象实例时使用）
String s = "hello";
Class<? extends String> c3 = s.getClass();

// 三者获取的是同一个 Class 对象（JVM 中每个类只有一个 Class 实例）
System.out.println(c1 == c2);  // true
System.out.println(c2 == c3);  // true
```

## 三、Class 对象的常用 API

### 3.1 获取类的基本信息

```java
Class<?> c = User.class;

c.getName()              // "com.example.User"（全限定名）
c.getSimpleName()        // "User"（简单类名）
c.getCanonicalName()     // "com.example.User"（规范名，数组有区别）
c.getPackageName()       // "com.example"（JDK 9+）
c.getSuperclass()        // 父类 Class 对象
c.getInterfaces()        // 实现的接口 Class[]
c.getModifiers()         // 修饰符（int 表示，可用 Modifier 工具类解析）
c.isInterface()          // 是否是接口
c.isEnum()               // 是否是枚举
c.isAnnotation()         // 是否是注解
c.isArray()              // 是否是数组
c.isPrimitive()          // 是否是基本类型
```

### 3.2 获取字段

```java
// getFields()：获取 public 字段（含父类）
Field[] publicFields = c.getFields();

// getDeclaredFields()：获取本类所有字段（含 private，不含父类）
Field[] allFields = c.getDeclaredFields();

// 获取特定字段
Field field = c.getDeclaredField("name");

// 读写字段
field.setAccessible(true);  // 突破访问权限限制（private 字段必须）
Object value = field.get(userInstance);       // 读
field.set(userInstance, "李四");               // 写
```

### 3.3 获取方法

```java
// getMethods()：获取 public 方法（含父类和接口的方法）
Method[] publicMethods = c.getMethods();

// getDeclaredMethods()：获取本类所有方法（不含父类，含 private）
Method[] allMethods = c.getDeclaredMethods();

// 获取特定方法（需指定参数类型，解决重载问题）
Method method = c.getMethod("setName", String.class);         // public
Method pvtMethod = c.getDeclaredMethod("privateHelper");       // private

// 调用方法
method.setAccessible(true);
Object result = method.invoke(instance, "参数值");
// 静态方法：method.invoke(null, args);
// 无参方法：method.invoke(instance);
```

### 3.4 获取构造器

```java
// 获取所有 public 构造器
Constructor<?>[] constructors = c.getConstructors();

// 获取所有构造器（含 private）
Constructor<?>[] allConstructors = c.getDeclaredConstructors();

// 获取特定构造器
Constructor<User> constructor = c.getConstructor(String.class, int.class);
Constructor<User> pvtConstructor = c.getDeclaredConstructor();  // 无参私有构造器（单例）

// 创建实例
User user = constructor.newInstance("张三", 25);

// 快捷方式（无参公共构造器）
Object instance = c.newInstance();  // JDK 9 已废弃
Object instance2 = c.getDeclaredConstructor().newInstance();  // 推荐
```

## 四、反射与泛型

### 4.1 泛型擦除与 Type

运行时泛型类型信息被擦除，但通过反射的 `Type` 体系可以获取部分泛型信息：

```java
class UserService extends BaseService<User, Long> { ... }

// 通过 getGenericSuperclass() 获取带泛型的父类类型
Type genericSuperclass = UserService.class.getGenericSuperclass();
// → ParameterizedType: BaseService<User, Long>

ParameterizedType pt = (ParameterizedType) genericSuperclass;
Type[] typeArguments = pt.getActualTypeArguments();
// typeArguments[0] = class com.example.User
// typeArguments[1] = class java.lang.Long

Class<?> entityClass = (Class<?>) typeArguments[0];  // → User.class
```

**Spring 和 MyBatis** 大量使用这个技术来获取泛型参数类型。

### 4.2 获取方法的泛型参数

```java
// 获取方法的泛型返回类型
Method method = UserService.class.getMethod("findList");
Type returnType = method.getGenericReturnType();
// 如果返回类型是 List<User>
if (returnType instanceof ParameterizedType) {
    Type[] types = ((ParameterizedType)returnType).getActualTypeArguments();
    // types[0] = User.class
}
```

## 五、反射与注解

```java
// 自定义注解
@Retention(RetentionPolicy.RUNTIME)  // 必须是 RUNTIME，运行时才能被反射读取
@Target({ElementType.FIELD, ElementType.METHOD})
public @interface Column {
    String name() default "";
    boolean nullable() default true;
}

// 使用注解
class User {
    @Column(name = "user_name", nullable = false)
    private String name;
}

// 反射读取注解
Field field = User.class.getDeclaredField("name");
Column column = field.getAnnotation(Column.class);
if (column != null) {
    System.out.println(column.name());     // "user_name"
    System.out.println(column.nullable()); // false
}

// 获取所有注解
Annotation[] annotations = field.getAnnotations();
```

## 六、反射的性能问题与优化

### 6.1 反射慢的原因

1. **安全检查**：每次调用 `invoke` 都进行权限检查（调用 `setAccessible(true)` 可跳过）
2. **装箱拆箱**：基本类型参数需要装箱（`int` → `Integer`），返回值需要拆箱
3. **可变参数**：参数打包成 `Object[]`，有额外开销
4. **JIT 优化受限**：反射调用难以被 JIT 内联优化

### 6.2 优化手段

```java
// 优化1：setAccessible(true)，跳过安全检查（最重要的优化）
method.setAccessible(true);

// 优化2：缓存 Method/Field 对象（避免重复查找）
// ❌ 每次都查找
public void process(Object obj) {
    Method m = obj.getClass().getMethod("doWork");  // 每次都有开销
    m.invoke(obj);
}

// ✅ 缓存
private static final Map<Class<?>, Method> METHOD_CACHE = new ConcurrentHashMap<>();
public void process(Object obj) {
    Method m = METHOD_CACHE.computeIfAbsent(
        obj.getClass(), 
        c -> c.getMethod("doWork")
    );
    m.invoke(obj);
}

// 优化3：使用 MethodHandle（JDK 7+），比反射快，可被 JIT 优化
MethodHandles.Lookup lookup = MethodHandles.lookup();
MethodHandle mh = lookup.findVirtual(User.class, "getName", 
                                     MethodType.methodType(String.class));
String name = (String) mh.invoke(user);

// 优化4：使用字节码生成框架（ASM/Javassist/ByteBuddy/ReflectASM）
// 动态生成调用代理类，完全避免反射，性能接近直接调用
```

### 6.3 性能对比（参考）

```
方式                     相对性能
直接调用                  1x（基准）
MethodHandle（热身后）    ~1x（JIT 内联后接近直接调用）
反射（setAccessible）     ~2-5x 慢
反射（未setAccessible）   ~5-20x 慢
```

## 七、反射在框架中的典型应用

### 7.1 Spring IoC 容器

```java
// Spring 扫描 @Component 注解，通过反射创建 Bean
Class<?> beanClass = Class.forName("com.example.UserService");
// 查找 @Autowired 字段，注入依赖
for (Field field : beanClass.getDeclaredFields()) {
    if (field.isAnnotationPresent(Autowired.class)) {
        field.setAccessible(true);
        field.set(beanInstance, getBean(field.getType()));
    }
}
```

### 7.2 MyBatis 结果映射

```java
// ResultSet → Java 对象的映射
// 根据列名找到 JavaBean 的 setter，通过反射设置值
String columnName = resultSet.getMetaData().getColumnName(i);
Method setter = beanClass.getMethod("set" + capitalize(columnName), columnType);
setter.invoke(instance, resultSet.getObject(i));
```

### 7.3 JUnit 测试框架

```java
// 扫描测试类，找到 @Test 注解的方法，通过反射调用
for (Method method : testClass.getMethods()) {
    if (method.isAnnotationPresent(Test.class)) {
        method.invoke(testInstance);  // 无参，无返回值
    }
}
```

## 八、模块系统对反射的限制（JDK 9+）

```java
// JDK 9 引入模块系统（JPMS），强封装：
// 默认情况下，模块内 non-exported 包的类无法被外部反射访问
// 即使调用 setAccessible(true) 也会抛出 InaccessibleObjectException

// 解决方案：
// 1. 在 module-info.java 中 opens 相关包
module com.example.app {
    opens com.example.internal to framework.module;  // 允许特定模块反射访问
    opens com.example.model;                          // 允许所有模块反射访问
}

// 2. JVM 启动参数（临时方案，用于未适配模块的框架）
// --add-opens java.base/java.lang=ALL-UNNAMED
// --add-opens java.base/java.util=ALL-UNNAMED
```

## 九、面试要点速查

| 问题 | 要点 |
|------|------|
| 反射的三种获取 Class 方式 | `Class.forName()`（触发初始化）/ `.class`（不触发）/ `obj.getClass()` |
| `getFields` vs `getDeclaredFields` | `getFields`：public+父类；`getDeclaredFields`：本类所有（含private） |
| 反射为什么慢 | 安全检查 + 装箱拆箱 + 参数打包 + JIT 优化受限 |
| 如何优化反射性能 | `setAccessible(true)` + 缓存 Method 对象 + 用 MethodHandle |
| 反射能获取到泛型参数吗 | 运行时泛型擦除，但通过 `getGenericSuperclass()` 等可获取声明处的泛型类型 |
| 注解要用 RUNTIME 保留策略才能被反射读取 | `@Retention(RetentionPolicy.RUNTIME)` |
| JDK 9 对反射的影响 | 模块封装，non-exported 包需要在 `module-info.java` 中 opens |
