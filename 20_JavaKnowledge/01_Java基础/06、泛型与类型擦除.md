# 泛型与类型擦除

> **核心关键词**：泛型方法、通配符、上下界、类型擦除、桥接方法、PECS原则

---

## 一、泛型的本质

泛型是 **编译期的类型检查机制**，让编译器帮你在编译期发现类型错误，而不是在运行期抛 `ClassCastException`。

```java
// 没有泛型的时代（JDK 1.4 以前）
List list = new ArrayList();
list.add("hello");
list.add(123);          // 可以放任何类型
String s = (String) list.get(1);  // 运行时：ClassCastException！

// 有了泛型
List<String> list = new ArrayList<>();
list.add("hello");
list.add(123);          // ❌ 编译错误，类型不符
String s = list.get(0); // ✅ 不需要强转，编译器保证类型
```

## 二、泛型类、泛型方法、泛型接口

### 2.1 泛型类

```java
public class Pair<A, B> {
    private A first;
    private B second;
    
    public Pair(A first, B second) {
        this.first = first;
        this.second = second;
    }
    
    public A getFirst() { return first; }
    public B getSecond() { return second; }
    
    @Override
    public String toString() {
        return "(" + first + ", " + second + ")";
    }
}

Pair<String, Integer> pair = new Pair<>("hello", 42);
String s = pair.getFirst();   // 无需强转
Integer n = pair.getSecond(); // 无需强转
```

### 2.2 泛型方法

泛型方法的类型参数声明在**返回类型之前**：

```java
// 泛型方法（可以在非泛型类中使用）
public static <T> T firstElement(List<T> list) {
    if (list.isEmpty()) throw new NoSuchElementException();
    return list.get(0);
}

// 调用时类型自动推断
String first = firstElement(Arrays.asList("a", "b", "c"));
Integer num = firstElement(Arrays.asList(1, 2, 3));
```

### 2.3 泛型接口

```java
public interface Repository<T, ID> {
    T findById(ID id);
    void save(T entity);
    List<T> findAll();
}

// 实现时指定具体类型
public class UserRepository implements Repository<User, Long> {
    @Override
    public User findById(Long id) { ... }
}

// 或者保持泛型
public class GenericRepository<T, ID> implements Repository<T, ID> { ... }
```

## 三、类型边界（Bounded Type Parameters）

### 3.1 上界（extends）

```java
// T 必须是 Number 或其子类
public <T extends Number> double sum(List<T> list) {
    double total = 0;
    for (T item : list) {
        total += item.doubleValue();  // 可以调用 Number 的方法
    }
    return total;
}

sum(Arrays.asList(1, 2, 3));       // ✅ Integer extends Number
sum(Arrays.asList(1.0, 2.0));      // ✅ Double extends Number
sum(Arrays.asList("a", "b"));      // ❌ String 不是 Number 的子类

// 多个边界（接口用 & 连接，类必须放第一个）
<T extends Comparable<T> & Serializable>
```

### 3.2 通配符（Wildcard）

```java
// ? 表示"某个未知类型"
List<?> list = new ArrayList<String>();  // 只知道是 List，不知道元素类型
Object obj = list.get(0);  // 只能当 Object 处理
list.add("hello");  // ❌ 不能添加，因为不知道确切类型

// 上界通配符：? extends T（生产者，只读）
List<? extends Number> nums = new ArrayList<Integer>();
Number n = nums.get(0);  // ✅ 可以读（保证是 Number 或子类）
nums.add(1);             // ❌ 不能写（不知道确切是 Integer 还是 Double）

// 下界通配符：? super T（消费者，可写）
List<? super Integer> ints = new ArrayList<Number>();
ints.add(1);             // ✅ 可以写（保证能放 Integer 或 Integer 的父类）
Integer i = ints.get(0); // ❌ 不能读为 Integer（只能当 Object）
Object o = ints.get(0);  // ✅ 可以读为 Object
```

## 四、PECS 原则

**Producer Extends, Consumer Super**：

```java
// 从集合中读取（生产者）→ 用 ? extends T
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    //                             消费者（写入）          生产者（读取）
    for (T element : src) {
        dest.add(element);
    }
}

// 实际使用
List<Number> dest = new ArrayList<>();
List<Integer> src = Arrays.asList(1, 2, 3);
copy(dest, src);  // ✅ Integer extends Number
```

**记忆口诀**：
- 往外取（读）→ `? extends`（上界通配符）
- 往里放（写）→ `? super`（下界通配符）
- 又读又写 → 用具体的类型参数 `T`

## 五、类型擦除（Type Erasure）

### 5.1 什么是类型擦除？

Java 的泛型是**伪泛型**，泛型信息只存在于**编译期**，运行时被擦除：

```java
List<String> stringList = new ArrayList<>();
List<Integer> intList = new ArrayList<>();

// 运行时，两者的 Class 对象完全相同！
System.out.println(stringList.getClass() == intList.getClass()); // true
System.out.println(stringList.getClass()); // class java.util.ArrayList（没有泛型信息）
```

### 5.2 类型擦除的规则

```java
// 编译前（源码）
public class Box<T> {
    T value;
    T get() { return value; }
}

// 编译后（运行时，擦除为上界，没有上界用 Object）
public class Box {
    Object value;          // T → Object
    Object get() { return value; }
}

// 有上界的情况
public class NumBox<T extends Number> {
    T value;
}
// 擦除后
public class NumBox {
    Number value;          // T extends Number → Number
}
```

### 5.3 类型擦除引发的问题

```java
// 1. 不能用基本类型作为泛型参数
List<int> list;     // ❌ 编译错误
List<Integer> list; // ✅

// 2. 不能对泛型进行 instanceof 判断
if (list instanceof List<String>) { ... }  // ❌ 编译错误

// 3. 不能创建泛型类型的数组
T[] arr = new T[10];  // ❌ 运行时 T 是 Object，类型信息丢失

// 4. 不能直接用 new T()
T t = new T();  // ❌ T 被擦除，不知道调用哪个构造器
// ✅ 解决方案：传入 Class<T> 然后反射创建
public <T> T create(Class<T> clazz) throws Exception {
    return clazz.getDeclaredConstructor().newInstance();
}
```

### 5.4 桥接方法（Bridge Method）

为了保证多态性，编译器在类型擦除后会自动生成桥接方法：

```java
// 源码
class Node<T> {
    T data;
    void setData(T data) { this.data = data; }
}

class StringNode extends Node<String> {
    @Override
    void setData(String data) { super.setData(data.toUpperCase()); }
}

// 编译后（类型擦除后）
class Node {
    Object data;
    void setData(Object data) { this.data = data; }  // 擦除后是 Object
}

class StringNode extends Node {
    void setData(String data) { super.setData(data.toUpperCase()); }  // 覆盖的方法
    
    // 编译器自动生成的桥接方法（保证多态）
    @Override
    void setData(Object data) {         // 桥接方法，签名匹配父类
        setData((String) data);         // 调用真正的实现
    }
}
```

## 六、运行时获取泛型信息

类型擦除后，方法的局部变量泛型信息丢失，但**类/接口声明处**的泛型信息保留在字节码中（通过 `Signature` 属性）：

```java
// 通过 getGenericSuperclass() 获取父类的泛型参数
class UserService extends BaseService<User, Long> {}

Type genericSuperclass = UserService.class.getGenericSuperclass();
ParameterizedType pt = (ParameterizedType) genericSuperclass;
Type[] typeArgs = pt.getActualTypeArguments();
Class<?> entityClass = (Class<?>) typeArgs[0];  // User.class
Class<?> idClass     = (Class<?>) typeArgs[1];  // Long.class

// Spring、MyBatis、Jackson 大量使用这个技术
// Jackson 的 TypeReference（获取复杂嵌套泛型类型）
TypeReference<List<User>> typeRef = new TypeReference<List<User>>() {};
// 内部类持有了泛型信息，通过 getGenericSuperclass() 可以获取到 List<User>
```

## 📚 PDF补充：泛型擦除详解与桥接方法

### 补充一：类型擦除详解

> 泛型是提供给 javac 编译器使用的"语法糖"，编译完成后会被擦除。

**擦除规则：**

```
1. 无上界的泛型 → 擦除为 Object
   class Box<T> → class Box<Object>

2. 有上界的泛型 → 擦除为上界类型
   class Box<T extends Number> → class Box<Number>

3. 泛型方法 → 擦除返回类型和参数类型
   static <T> T getValue() → static Object getValue()

4. 泛型数组 → 不能直接创建 new T[]
   T[] arr = new T[10]; // ❌ 编译错误
   T[] arr = (T[]) new Object[10]; // ✅ 但有警告
```

**类型擦除示例：**

```java
// 源代码
public class GenericType<T> {
    private T value;
    public T getValue() { return value; }
    public void setValue(T value) { this.value = value; }
}

// 编译后（字节码中）
public class GenericType {
    private Object value;  // T 被擦除为 Object
    public Object getValue() { return value; }  // 返回类型被擦除
    public void setValue(Object value) { this.value = value; }  // 参数被擦除
}
```

### 补充二：桥接方法（Bridge Method）

> 桥接方法是 JDK 1.5 引入泛型后，为使泛型字节码与 JDK 1.5 之前版本兼容，由编译器自动生成的方法。

**为什么需要桥接方法？**

```java
// 源代码
class Parent<T> {
    T value;
    T getValue() { return value; }
}

class Child extends Parent<String> {
    @Override
    String getValue() { return "hello"; }  // 重写父类方法
}

// 编译后（编译器自动生成桥接方法）
class Child extends Parent {
    // 编译器生成的桥接方法
    public Object getValue() { return this.getValue(); }  // 调用 String 版本
    
    // 用户定义的原始方法
    public String getValue() { return "hello"; }
}
```

**如何判断桥接方法？**

```java
Method method = ...;
if (method.isBridge()) {
    System.out.println("这是一个桥接方法");
}
```

### 补充三：泛型与SPI结合

> Spring、Dubbo 等框架大量使用"泛型 + SPI"来实现灵活的扩展机制。

```java
// 示例：Spring 的 GenericApplicationContext
GenericApplicationContext ctx = new GenericApplicationContext();
ctx.registerBean(User.class);
ctx.refresh();

// 底层原理：利用 getGenericSuperclass() 获取泛型类型信息
Type type = this.getClass().getGenericSuperclass(); // 获取父类的泛型参数
ParameterizedType paramType = (ParameterizedType) type;
Type[] typeArgs = paramType.getActualTypeArguments(); // 获取具体的泛型类型
```
