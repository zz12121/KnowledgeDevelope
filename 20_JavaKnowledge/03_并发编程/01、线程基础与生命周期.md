# 线程基础与生命周期

> **核心关键词**：Thread、Runnable、Callable、线程状态机、上下文切换、守护线程、线程优先级、interrupt、ThreadGroup

---

## 一、Java 线程的创建方式

### 1.1 四种创建方式对比

```java
// ── 方式1：继承 Thread ──────────────────────────────────────
public class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread: " + Thread.currentThread().getName());
    }
}
new MyThread().start();

// ── 方式2：实现 Runnable（推荐，解耦任务与线程）──────────────
Runnable task = () -> System.out.println("Runnable: " + Thread.currentThread().getName());
new Thread(task, "my-thread").start();

// ── 方式3：实现 Callable + FutureTask（有返回值，可抛异常）──
Callable<Integer> callable = () -> {
    Thread.sleep(100);
    return 42;
};
FutureTask<Integer> future = new FutureTask<>(callable);
new Thread(future, "callable-thread").start();
Integer result = future.get();  // 阻塞等待结果

// ── 方式4：线程池（生产环境推荐）──────────────────────────────
ExecutorService pool = Executors.newFixedThreadPool(4);
pool.submit(() -> System.out.println("Pool thread"));
pool.execute(() -> System.out.println("Pool execute"));
pool.shutdown();
```

**选择建议**：
- 生产代码：**线程池**（资源可控，复用线程）
- 需要返回值/异常传递：**Callable + Future**
- 简单任务：**Runnable Lambda**
- 避免继承 Thread（Java 单继承限制）

## 二、线程的六种状态（状态机）

### 2.1 状态定义

Java 线程状态定义在 `Thread.State` 枚举中，共 **6 种**：

```
NEW ──────────────────────────────────┐
                                      ▼
                                   start()
                                      ▼
RUNNABLE ←──── 获得CPU时间片 ─── WAITING/TIMED_WAITING/BLOCKED
    │                                 ▲
    │ 等待同步锁                        │
    ▼                                 │
 BLOCKED ──── 获得锁 ─────────────────┘
    
    │ wait() / join() / LockSupport.park()
    ▼
 WAITING ──── notify() / interrupt() ─── RUNNABLE

    │ wait(t) / sleep(t) / parkNanos(t)
    ▼
TIMED_WAITING ──── 超时 / 唤醒 ──────── RUNNABLE

    │ run() 结束 / 异常退出
    ▼
TERMINATED
```

### 2.2 各状态说明

| 状态 | 说明 | 典型触发 |
|------|------|---------|
| `NEW` | 线程对象创建，未调用 start() | `new Thread(...)` |
| `RUNNABLE` | 可运行（含正在 CPU 执行 + 等待 CPU 调度） | `start()` 后 |
| `BLOCKED` | 等待 **monitor 锁**（synchronized 锁） | 争抢 `synchronized` 块入口 |
| `WAITING` | 无限期等待，需他线程显式唤醒 | `Object.wait()`, `Thread.join()`, `LockSupport.park()` |
| `TIMED_WAITING` | 有超时限制的等待 | `Thread.sleep(n)`, `wait(n)`, `parkNanos(n)`, `join(n)` |
| `TERMINATED` | 执行完毕或异常退出 | `run()` 结束 |

### 2.3 BLOCKED vs WAITING — 核心区别

```java
// BLOCKED：等待进入 synchronized 块（被动阻塞，JVM 控制）
synchronized (lock) {
    // 若锁被占用，线程进入 BLOCKED 状态
}

// WAITING：主动挂起自己，等待特定条件（需配合 notify/signal）
synchronized (lock) {
    lock.wait();  // 释放锁，进入 WAITING；需 notify() 唤醒
}

// TIMED_WAITING：带超时的等待（不持有任何锁）
Thread.sleep(1000);  // 不释放锁
LockSupport.parkNanos(TimeUnit.SECONDS.toNanos(1));
```

> **面试关键点**：`sleep()` 不释放锁；`wait()` 释放锁。`BLOCKED` 是等待锁，`WAITING` 是等待条件。

## 三、线程状态转换代码实验

```java
public class ThreadStateDemo {
    private static Object lock = new Object();

    public static void main(String[] args) throws Exception {
        // --- NEW 状态 ---
        Thread t = new Thread(() -> {
            try {
                Thread.sleep(5000);  // TIMED_WAITING
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
            }
            synchronized (lock) {
                try {
                    lock.wait();     // WAITING
                } catch (InterruptedException e) {
                    Thread.currentThread().interrupt();
                }
            }
        });
        System.out.println("1. " + t.getState()); // NEW

        t.start();
        Thread.sleep(100);
        System.out.println("2. " + t.getState()); // TIMED_WAITING

        // 用 jstack 或如下方式观察 BLOCKED
        Thread t2 = new Thread(() -> {
            synchronized (lock) {}  // 若 t 持有 lock，t2 进入 BLOCKED
        });
    }
}
```

## 四、线程中断机制（Interrupt）

### 4.1 中断的本质

Java 线程中断是**协作式**的（非强制终止），只是设置一个中断标志位，线程需自行响应。

```java
Thread t = new Thread(() -> {
    while (!Thread.currentThread().isInterrupted()) {
        // 正常工作
        doWork();
    }
    // 收到中断信号，优雅退出
    cleanup();
});

// 发出中断请求
t.interrupt();
```

### 4.2 三个核心方法

| 方法 | 说明 |
|------|------|
| `t.interrupt()` | 设置线程 t 的中断标志为 true（不强制停止） |
| `Thread.interrupted()` | 静态方法，返回当前线程中断状态，**并清除标志** |
| `t.isInterrupted()` | 实例方法，返回线程中断状态，**不清除标志** |

### 4.3 中断与阻塞状态的交互

```java
// sleep/wait/join 遇到中断：抛出 InterruptedException，并清除中断标志
Thread t = new Thread(() -> {
    try {
        Thread.sleep(10000);
    } catch (InterruptedException e) {
        // 标志被清除了！如果需要传播，重新设置
        Thread.currentThread().interrupt();
        System.out.println("被中断，优雅退出");
    }
});
t.start();
Thread.sleep(100);
t.interrupt();  // 触发 InterruptedException
```

> **最佳实践**：捕获 `InterruptedException` 后，要么继续向上抛出，要么调用 `Thread.currentThread().interrupt()` 恢复中断标志，**不要吞掉中断**！

### 4.4 使用 volatile 标志 vs interrupt()

```java
// 方式1：volatile 标志（简单场景）
class Worker implements Runnable {
    private volatile boolean cancelled = false;
    
    @Override
    public void run() {
        while (!cancelled) {
            doWork();
        }
    }
    
    public void cancel() { cancelled = true; }
}

// 方式2：interrupt()（阻塞场景必用）
class BlockingWorker implements Runnable {
    @Override
    public void run() {
        while (!Thread.currentThread().isInterrupted()) {
            try {
                BlockingQueue.take();  // 可能阻塞，volatile 标志无法打断
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
}
// 若任务涉及阻塞操作（sleep/wait/IO），必须用 interrupt()
```

## 五、守护线程（Daemon Thread）

```java
// 守护线程：JVM 中所有用户线程结束时，守护线程自动终止（不阻止 JVM 退出）
// 典型守护线程：GC 线程、Finalizer 线程、定时任务线程

Thread daemon = new Thread(() -> {
    while (true) {
        monitorSystem();
        Thread.sleep(1000);
    }
});
daemon.setDaemon(true);  // 必须在 start() 前设置！
daemon.start();

// 注意事项
// 1. 守护线程中的 finally 块不保证执行（JVM 可能直接退出）
// 2. 不要在守护线程中做资源清理（文件关闭、DB 提交等）
// 3. 线程池中的线程默认是用户线程；若需守护线程池，需自定义 ThreadFactory
ThreadFactory daemonFactory = r -> {
    Thread t = new Thread(r);
    t.setDaemon(true);
    return t;
};
ExecutorService daemonPool = Executors.newFixedThreadPool(4, daemonFactory);
```

## 六、线程优先级

```java
// Java 线程优先级：1（MIN）~ 10（MAX），默认 5（NORM）
// 本质：提示 OS 调度器，不保证执行顺序！
Thread t = new Thread(task);
t.setPriority(Thread.MAX_PRIORITY); // 10

// ⚠️ 优先级的问题：
// 1. 平台相关：Linux 默认忽略 Java 线程优先级（NPTL 实现）
// 2. 饥饿风险：低优先级线程可能长期得不到调度
// 3. 生产代码不应依赖线程优先级控制逻辑顺序，应用 CountDownLatch/Semaphore 等
```

## 七、上下文切换（Context Switch）

```
CPU 调度原理：
  时间片（Time Slice）轮转：每个线程执行一个时间片（~10-20ms），然后切换
  
上下文切换的代价：
  保存当前线程的 CPU 寄存器、程序计数器、栈指针 → 存入内存
  加载下一个线程的上下文 → 载入寄存器

代价衡量：
  1次上下文切换 ≈ 几微秒 ~ 几十微秒（含 CPU 缓存失效代价）
  高并发场景：上下文切换开销可占总时间 10%~30%

减少上下文切换的策略：
  1. 无锁并发（CAS、ThreadLocal 分桶）
  2. 协程/虚拟线程（JDK 21 Virtual Thread：百万级协程，调度开销极低）
  3. 减少线程数（线程池大小 ≠ 线程越多越好）
  4. 适当自旋（自旋锁避免挂起/唤醒）
```

## 八、Thread 重要方法速查

### 8.1 sleep vs wait vs yield vs join

| 方法 | 释放锁？ | 能被中断？ | 用途 |
|------|---------|----------|------|
| `Thread.sleep(ms)` | ❌ 不释放 | ✅ 抛 InterruptedException | 当前线程暂停指定时间 |
| `object.wait()` | ✅ **释放锁** | ✅ 抛 InterruptedException | 等待条件（必须在 synchronized 内） |
| `Thread.yield()` | ❌ 不释放 | ❌ 不可中断 | 提示让出 CPU，同优先级线程竞争 |
| `thread.join()` | ❌（等待目标线程）| ✅ 抛 InterruptedException | 等待目标线程执行完毕 |

```java
// join() 的典型用法：主线程等待子线程完成
Thread worker = new Thread(heavyTask);
worker.start();
worker.join();  // 主线程阻塞，直到 worker 完成
System.out.println("worker 完成，继续主线程逻辑");

// join(timeout) 带超时
worker.join(5000);  // 最多等 5 秒，超时继续主线程
if (worker.isAlive()) {
    System.out.println("worker 超时未完成");
    worker.interrupt();
}
```

### 8.2 Thread 常用 API

```java
Thread t = Thread.currentThread();

t.getName()            // 线程名称
t.getId()              // 线程 ID（JDK 19+ 改为 threadId()）
t.getState()           // 获取 Thread.State 枚举值
t.isAlive()            // 是否还活着（start 后到 terminated 前）
t.isDaemon()           // 是否守护线程
t.getPriority()        // 优先级 1~10
t.getStackTrace()      // 当前调用栈（StackTraceElement[]）
Thread.activeCount()   // 当前线程组活跃线程数
Thread.getAllStackTraces() // 所有线程的调用栈（用于诊断）
```

## 九、虚拟线程（Virtual Thread，JDK 21）

```java
// JDK 21 正式 GA：虚拟线程（协程级别的轻量线程）
// 原理：平台线程（OS 线程）作为"载体"，虚拟线程挂载其上运行
// 阻塞时：虚拟线程 unmount，载体线程去执行其他虚拟线程
// 代价：每个虚拟线程仅几百字节，可创建百万级

// 创建方式
Thread vt = Thread.ofVirtual().name("virtual-1").start(() -> {
    System.out.println("running in: " + Thread.currentThread());
});

// 虚拟线程池（推荐）
ExecutorService vtPool = Executors.newVirtualThreadPerTaskExecutor();
vtPool.submit(() -> doBlockingWork());  // 阻塞操作完全无感知

// 适用场景：I/O 密集型（数据库、HTTP 调用、文件操作）
// 不适用：CPU 密集型（无阻塞，虚拟线程优势不存在）
// 注意：synchronized 持有时阻塞，虚拟线程仍会钉住载体线程（pinning 问题）
//       → 用 ReentrantLock 替代 synchronized 在虚拟线程中更合适
```

## 十、ThreadGroup 与 UncaughtExceptionHandler

```java
// UncaughtExceptionHandler：捕获线程未处理异常
Thread t = new Thread(() -> {
    throw new RuntimeException("未捕获异常！");
});
t.setUncaughtExceptionHandler((thread, ex) -> {
    System.err.println("线程 " + thread.getName() + " 抛出异常：" + ex.getMessage());
    // 记录日志、告警等
});
t.start();

// 线程池中设置全局 UncaughtExceptionHandler
ThreadFactory factory = r -> {
    Thread thread = new Thread(r);
    thread.setUncaughtExceptionHandler((t2, e) -> log.error("线程异常", e));
    return thread;
};

// ThreadGroup（不推荐直接使用，了解即可）
// 批量操作一组线程（interrupt、enumerate）
// JDK 19+ 已标记为不推荐
```

## 十一、面试要点速查

| 问题 | 要点 |
|------|------|
| Java 线程有几种状态 | 6种：NEW/RUNNABLE/BLOCKED/WAITING/TIMED_WAITING/TERMINATED |
| BLOCKED 和 WAITING 区别 | BLOCKED 等待 synchronized 锁；WAITING 调用 wait/park/join 主动挂起 |
| sleep 和 wait 区别 | sleep 不释放锁，不需要在同步块中；wait 释放锁，必须在 synchronized 中 |
| 如何优雅停止线程 | interrupt() 设置中断标志，线程自行检查 isInterrupted() 退出 |
| 守护线程的特点 | 用户线程全部结束后自动终止，不阻止 JVM 退出，finally 不保证执行 |
| JDK 21 虚拟线程 | 轻量级线程，可创建百万级，I/O 阻塞时自动挂起让出载体线程 |
| start() 和 run() 区别 | start() 创建新线程执行 run()；直接调 run() 是在当前线程同步执行 |
| 上下文切换代价 | 保存/恢复寄存器 + CPU 缓存失效，约几微秒~几十微秒 |
