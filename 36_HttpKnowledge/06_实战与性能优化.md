## 6.1 RESTful API 设计

### URL 设计原则

```
# 名词复数表示资源集合
GET    /users              # 查询用户列表
POST   /users              # 创建用户
GET    /users/1            # 查询单个用户
PUT    /users/1            # 全量更新用户
PATCH  /users/1            # 局部更新用户
DELETE /users/1            # 删除用户

# 资源层级嵌套
GET    /users/1/orders     # 查询用户1的订单

# 过滤/排序/分页用查询参数
GET    /users?status=active&page=1&size=20&sort=name,desc

# 版本控制
GET    /api/v1/users       # URL 版本（推荐）
GET    /users（Accept: application/vnd.api+json;version=2）  # Header 版本
```

### 幂等性 + 防重复提交

```java
// 创建接口添加幂等 Key，防止网络超时重试创建重复数据
POST /api/orders
Idempotency-Key: "a1b2c3d4-uuid"  // 客户端生成，服务端用 Redis 去重

// 服务端处理
String result = redisTemplate.opsForValue().get("idempotent:" + idempotencyKey);
if (result != null) return JSON.parse(result);   // 直接返回缓存的响应
// 执行业务逻辑
// 存入 Redis（TTL = 24h）
```

## 6.2 断点续传

### 核心机制

1. **HEAD 请求**：探测文件大小和是否支持断点续传（`Accept-Ranges: bytes`）
2. **Range 请求**：分片下载
3. **206 Partial Content**：服务器返回部分内容

```
# 获取文件信息
HEAD /large-file.zip
← Content-Length: 104857600
← Accept-Ranges: bytes
← ETag: "abc123"

# 分片请求（多线程）
Range: bytes=0-26214399          # 第1片（0-25MB）
Range: bytes=26214400-52428799   # 第2片（25-50MB）
Range: bytes=52428800-104857599  # 第3片（50-100MB）

# 服务器响应
HTTP/1.1 206 Partial Content
Content-Range: bytes 0-26214399/104857600
Content-Length: 26214400
```

## 6.3 WebSocket

### 握手升级过程

```
客户端:
GET /ws HTTP/1.1
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
Sec-WebSocket-Version: 13

服务器:
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Accept: s3pPLMBiTxaQ9kYGzzhZRbK+xOo=
```

握手完成后，HTTP 连接升级为 WebSocket，实现全双工通信。

### HTTP vs WebSocket vs SSE

| 维度 | HTTP | WebSocket | SSE |
|------|------|-----------|-----|
| 通信方向 | 请求-响应 | 全双工 | 服务器→客户端 |
| 连接 | 无状态（Keep-Alive 复用） | 持久连接 | 持久连接 |
| 适用场景 | API 调用 | 聊天/游戏/协作 | 行情/AI流式输出/推送 |
| 实现复杂度 | 低 | 中 | 低 |

### Spring Boot WebSocket（STOMP）

```java
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {
    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws").withSockJS();
    }
    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.enableSimpleBroker("/topic", "/queue");
        registry.setApplicationDestinationPrefixes("/app");
    }
}

@MessageMapping("/chat")
@SendTo("/topic/messages")       // 广播给所有订阅者
public ChatMessage chat(ChatMessage msg) { return msg; }
```

## 6.4 HTTP 性能优化清单

### 减少请求次数

- [x] 启用浏览器缓存（强缓存 + 协商缓存）
- [x] 静态资源使用 contenthash 文件名 + `max-age=31536000, immutable`
- [x] 图片懒加载（`loading="lazy"` / Intersection Observer）
- [x] 路由级代码分割（动态 import）
- [x] 升级 HTTP/2（多路复用，不需要多条连接）
- [x] 减少重定向链（HTTP→HTTPS 一步到位，不要多跳）

### 减少传输体积

- [x] 开启 GZIP/Brotli 压缩（文本资源）
- [x] 图片格式升级为 WebP（比 JPEG/PNG 小 25-35%）
- [x] 响应式图片（`srcset`，不同分辨率加载不同大小图片）
- [x] Tree Shaking 删除死代码
- [x] HTTP/2 HPACK 自动压缩 Header

### 减少延迟

- [x] CDN 就近加速（静态资源/全站 CDN）
- [x] DNS 预解析：`<link rel="dns-prefetch" href="//api.example.com">`
- [x] 提前建连：`<link rel="preconnect" href="https://api.example.com">`
- [x] 关键资源预加载：`<link rel="preload" href="/font.woff2" as="font">`
- [x] 升级 TLS 1.3（1-RTT 握手）+ OCSP Stapling
- [x] 启用 HSTS（消除 HTTP→HTTPS 重定向往返）

### CDN 最佳实践

```nginx
# Nginx 配置压缩
gzip on;
gzip_types text/plain application/json application/javascript text/css;
gzip_min_length 1024;
gzip_comp_level 6;

# 静态资源长缓存
location ~* \.(js|css|png|jpg|woff2)$ {
    expires 365d;
    add_header Cache-Control "public, max-age=31536000, immutable";
}

# HTML 文件不缓存或协商缓存
location ~* \.html$ {
    add_header Cache-Control "no-cache";
}
```

## 6.6 常见性能问题排查

### 页面加载慢

```
Chrome DevTools Network → 查看 Timing 面板
  - TTFB（Time to First Byte）长 → 服务端响应慢，查后端接口/SQL
  - Content Download 长 → 响应体太大，开启压缩/减少体积
  - 请求瀑布图串行 → 未开启 HTTP/2，或存在 JavaScript 阻塞渲染
```

### 接口响应慢（502/504）

- 502 → 后端服务是否正常运行，查后端进程和日志
- 504 → 后端处理超时，查慢 SQL/耗时接口
- 先看 Nginx access log，再看应用日志，再看 DB slow log

### HTTPS 握手失败

常见原因：证书过期、域名不匹配、证书链不完整、TLS 版本不兼容

```bash
# 快速检查
curl -v https://example.com 2>&1 | grep -E "SSL|TLS|cert"
openssl s_client -connect example.com:443 -servername example.com
# 或用 SSL Labs: https://www.ssllabs.com/ssltest/
```
