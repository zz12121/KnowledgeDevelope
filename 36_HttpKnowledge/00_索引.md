## 核心知识点速记

### HTTP 状态码记忆

```
2xx 成功：200(OK) / 201(创建) / 204(无内容) / 206(分块)
3xx 重定向：301(永久) / 302(临时) / 304(缓存命中) / 307(临时保持方法) / 308(永久保持方法)
4xx 客户端错误：400(参数) / 401(未认证) / 403(无权限) / 404(不存在) / 429(限流)
5xx 服务端错误：500(通用) / 502(网关无效响应) / 503(不可用) / 504(网关超时)
```

### 缓存决策口诀

```
强缓存未过期 → 直接用（不发请求，200 from cache）
强缓存过期 → 发条件请求
  ↓
ETag 未变 → 304（不返回响应体，用本地缓存）
ETag 变了 → 200（返回新响应体）
```

### Cache-Control 常用组合

```
no-store          # 敏感数据，完全不缓存
no-cache          # 动态内容，每次验证
max-age=86400     # 普通静态资源，缓存1天
max-age=31536000, immutable  # 带Hash的静态资源，永久缓存
```

### TLS 握手要点

```
TLS 1.2：2-RTT（Client Hello → Server Hello + 证书 → 验证 + 密钥交换 → 完成）
TLS 1.3：1-RTT（合并多步），0-RTT 支持会话恢复
混合加密：非对称（ECDHE）协商密钥 + 对称（AES）加密数据
```

### HTTP 版本演进要点

```
1.1：Keep-Alive 持久连接，但请求串行（应用层队头阻塞）
2.0：二进制分帧 + 多路复用（解决应用层阻塞），HPACK 压缩 Header
3.0：QUIC(UDP) 替换 TCP，彻底消除队头阻塞，0-RTT，连接迁移
```

### CORS 核心

```
触发预检（OPTIONS）：自定义 Header / PUT DELETE / Content-Type:application/json
allowCredentials(true) 时：allowedOrigins 不能用 * ，必须指定具体域名
Access-Control-Max-Age：缓存预检结果，减少重复 OPTIONS 请求
```

### WebSocket vs SSE

```
WebSocket：全双工，二进制/文本，适合聊天/游戏/实时协作
SSE：单向推送（服务器→客户端），基于 HTTP，自动重连，适合行情/AI流式输出
```
