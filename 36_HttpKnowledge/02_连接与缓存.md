# 02 连接与缓存

## 2.1 HTTP 连接管理

### 连接演进历史

| 版本 | 连接方式 | 问题 |
|------|---------|------|
| HTTP/1.0 | 短连接（每次建立/断开） | 大量 TCP 握手开销 |
| HTTP/1.1 | 持久连接（Keep-Alive，默认） | 请求串行，应用层队头阻塞 |
| HTTP/2 | 单连接多路复用 | TCP 层队头阻塞 |
| HTTP/3 | QUIC 多路复用（UDP） | 彻底消除队头阻塞 |

### 持久连接（Keep-Alive）

```nginx
keepalive_timeout 65;      # 空闲连接保持时间
keepalive_requests 1000;   # 单连接最大请求数
```

HTTP/1.1 默认持久连接，不想复用才加 `Connection: close`。

### 管道化（Pipelining）

理论上可以不等响应就发下一个请求，但响应必须顺序返回，慢请求阻塞后续（队头阻塞），现代浏览器默认禁用。

### Connection 头的常见取值

```
Connection: keep-alive    # 复用连接（HTTP/1.1 默认）
Connection: close         # 本次完事关闭连接
Connection: Upgrade       # 协议升级（WebSocket 握手）
```

### 超时参数（Spring HttpClient）

```java
RequestConfig config = RequestConfig.custom()
    .setConnectTimeout(3000)          // TCP 连接超时
    .setSocketTimeout(30000)          // 读取超时
    .setConnectionRequestTimeout(1000) // 从连接池获取超时
    .build();
```

## 2.2 HTTP 缓存

### 缓存决策流程

```
发起请求
  ↓
检查强缓存（Cache-Control max-age / Expires）
  ├── 未过期 → 200 from cache（不发请求）
  └── 已过期 → 发条件请求
        ↓
检查协商缓存（If-None-Match ETag / If-Modified-Since）
  ├── 未修改 → 304 Not Modified（不返回响应体）
  └── 已修改 → 200 + 新响应体
```

### Cache-Control 指令速查

| 指令 | 含义 |
|------|------|
| `max-age=N` | N 秒内直接用缓存（相对时间） |
| `s-maxage=N` | CDN/共享缓存的有效期（覆盖 max-age） |
| `no-cache` | 每次使用前必须向服务器验证（不是不缓存！） |
| `no-store` | 完全不缓存，每次从服务器拿 |
| `public` | 可被任何节点缓存（CDN、代理） |
| `private` | 只能被浏览器缓存（默认） |
| `must-revalidate` | 缓存过期后必须向服务器验证 |
| `immutable` | 文件永远不变，跳过到期后的重新验证 |

**常用组合**：

```
# 敏感数据（不缓存）
Cache-Control: no-store

# 动态接口（每次验证）
Cache-Control: no-cache

# 静态资源（带 Hash，永久缓存）
Cache-Control: public, max-age=31536000, immutable

# CDN 缓存 7 天，浏览器 1 天
Cache-Control: public, max-age=86400, s-maxage=604800
```

### 协商缓存对比

| 字段 | 优先级 | 精度 | 缺点 |
|------|--------|------|------|
| ETag + If-None-Match | 高 | 内容级精确 | 计算开销 |
| Last-Modified + If-Modified-Since | 低 | 秒级（1 秒内多次修改无法区分） | 误判（内容未变但时间变了） |

**两者同时存在时，ETag 优先**。

### 最优静态资源缓存策略

```
index.html    → no-cache（每次验证，保证拿到最新的引用）
app.abc123.js → max-age=31536000, immutable（文件名含 Hash，永久缓存）
```

构建工具（Vite/webpack）默认支持 `contenthash` 文件名，配合 CDN 可最大化缓存效果。

### 启发式缓存

响应中没有 `Cache-Control` 也没有 `Expires` 时，浏览器自行估算缓存时间：

```
缓存时长 = (Date - Last-Modified) × 10%
```

建议：**始终显式设置 Cache-Control**，不要依赖启发式缓存。

## 2.3 Vary 响应头

`Vary` 告诉缓存服务器，除了 URL 外，还需要把哪些请求头加入缓存 Key：

```
Vary: Accept-Encoding    # 不同压缩算法分开缓存
Vary: Accept-Language    # 不同语言分开缓存
Vary: Accept             # 不同格式（JSON/XML）分开缓存
```

几乎所有支持 gzip/br 压缩的服务都应该设置 `Vary: Accept-Encoding`。
