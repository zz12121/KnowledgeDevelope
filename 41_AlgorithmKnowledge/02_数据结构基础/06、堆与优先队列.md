# 堆与优先队列 - 高效获取极值

## 🎯 什么是堆？

### 1.1 直观理解
堆像一个"懒人版"的排序：它不保证所有元素有序，只保证**你最想要的那个（最大或最小）始终在顶部**，随时可以 O(1) 获取，插入和删除也只需 O(log n)。

### 1.2 堆的定义

**堆（Heap）** 是一棵**完全二叉树**，满足堆性质：
- **大顶堆（Max-Heap）**：任意节点的值 ≥ 其子节点的值（根节点最大）
- **小顶堆（Min-Heap）**：任意节点的值 ≤ 其子节点的值（根节点最小）

```
大顶堆示例：              小顶堆示例：
        90                      1
       /   \                   / \
      80    70                3   2
     / \   / \               / \ / \
    60  50 40  30           7  5 6  4
```

### 1.3 堆的存储 —— 用数组实现

由于堆是完全二叉树，可以用数组紧凑存储，无需额外指针：

```
索引:   0   1   2   3   4   5   6
数组: [90, 80, 70, 60, 50, 40, 30]

关键公式：
  父节点索引 = (i - 1) / 2
  左子节点   = 2 * i + 1
  右子节点   = 2 * i + 2
```

## 🛠️ 堆的核心操作

### 2.1 上浮（Sift Up / Bubble Up）—— 用于插入

新元素插入数组末尾，然后不断与父节点比较，如果比父节点大（大顶堆）就交换，直到不违反堆性质。

```java
private void siftUp(int[] heap, int i) {
    while (i > 0) {
        int parent = (i - 1) / 2;
        if (heap[parent] < heap[i]) {  // 大顶堆：父节点比我小，需要交换
            swap(heap, parent, i);
            i = parent;
        } else {
            break;
        }
    }
}
```

### 2.2 下沉（Sift Down / Heapify Down）—— 用于删除

堆顶元素移除后，将末尾元素移到堆顶，然后不断与**较大的子节点**比较，如果小于子节点（大顶堆）就交换，向下沉降直到满足堆性质。

```java
private void siftDown(int[] heap, int i, int size) {
    while (true) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;

        if (left < size && heap[left] > heap[largest]) largest = left;
        if (right < size && heap[right] > heap[largest]) largest = right;

        if (largest != i) {
            swap(heap, i, largest);
            i = largest;
        } else {
            break;
        }
    }
}
```

### 2.3 完整大顶堆实现

```java
public class MaxHeap {
    private int[] data;
    private int size;
    private int capacity;

    public MaxHeap(int capacity) {
        this.capacity = capacity;
        this.data = new int[capacity];
        this.size = 0;
    }

    // 插入：O(log n)
    public void offer(int val) {
        if (size >= capacity) throw new RuntimeException("Heap is full");
        data[size] = val;
        siftUp(size);
        size++;
    }

    // 取出堆顶（最大值）：O(log n)
    public int poll() {
        if (size == 0) throw new RuntimeException("Heap is empty");
        int max = data[0];
        data[0] = data[--size];  // 末尾元素移到堆顶
        siftDown(0);
        return max;
    }

    // 查看堆顶：O(1)
    public int peek() {
        if (size == 0) throw new RuntimeException("Heap is empty");
        return data[0];
    }

    private void siftUp(int i) {
        while (i > 0) {
            int parent = (i - 1) / 2;
            if (data[parent] < data[i]) {
                swap(i, parent);
                i = parent;
            } else break;
        }
    }

    private void siftDown(int i) {
        while (2 * i + 1 < size) {
            int largest = i;
            int left = 2 * i + 1, right = 2 * i + 2;
            if (left < size && data[left] > data[largest]) largest = left;
            if (right < size && data[right] > data[largest]) largest = right;
            if (largest != i) { swap(i, largest); i = largest; }
            else break;
        }
    }

    private void swap(int i, int j) {
        int tmp = data[i]; data[i] = data[j]; data[j] = tmp;
    }

    public boolean isEmpty() { return size == 0; }
    public int size() { return size; }

    // O(n) 建堆（比逐个插入 O(n log n) 更快）
    public static int[] buildHeap(int[] arr) {
        int n = arr.length;
        // 从最后一个非叶节点开始，依次对每个节点执行 siftDown
        for (int i = n / 2 - 1; i >= 0; i--) {
            siftDownStatic(arr, i, n);
        }
        return arr;
    }

    private static void siftDownStatic(int[] arr, int i, int size) {
        while (2 * i + 1 < size) {
            int largest = i;
            int left = 2 * i + 1, right = 2 * i + 2;
            if (left < size && arr[left] > arr[largest]) largest = left;
            if (right < size && arr[right] > arr[largest]) largest = right;
            if (largest != i) { int t = arr[i]; arr[i] = arr[largest]; arr[largest] = t; i = largest; }
            else break;
        }
    }
}
```

## 📊 堆排序

### 3.1 算法思想

1. **建堆**：将原数组调整为大顶堆，O(n)
2. **排序**：每次将堆顶（最大值）与末尾元素交换，堆大小减一，再对新堆顶 siftDown，重复 n-1 次

```java
public static void heapSort(int[] arr) {
    int n = arr.length;

    // 第一步：建大顶堆，O(n)
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDown(arr, i, n);
    }

    // 第二步：逐步将堆顶放到末尾，O(n log n)
    for (int i = n - 1; i > 0; i--) {
        // 堆顶（最大值）与末尾交换
        int tmp = arr[0]; arr[0] = arr[i]; arr[i] = tmp;
        // 对新堆顶 siftDown（堆大小变为 i）
        siftDown(arr, 0, i);
    }
}

private static void siftDown(int[] arr, int i, int size) {
    while (2 * i + 1 < size) {
        int largest = i;
        int left = 2 * i + 1, right = 2 * i + 2;
        if (left < size && arr[left] > arr[largest]) largest = left;
        if (right < size && arr[right] > arr[largest]) largest = right;
        if (largest != i) {
            int t = arr[i]; arr[i] = arr[largest]; arr[largest] = t;
            i = largest;
        } else break;
    }
}
```

**堆排序特性：**
- 时间复杂度：O(n log n)，最好/最坏/平均均如此
- 空间复杂度：O(1)（原地排序）
- 不稳定（相等元素可能换序）

## 🎯 优先队列（PriorityQueue）

Java 的 `PriorityQueue` 就是堆的官方实现（默认小顶堆）。

### 4.1 基本用法

```java
// 小顶堆（默认）：最小值优先出队
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(5); minHeap.offer(1); minHeap.offer(3);
System.out.println(minHeap.poll());  // 1

// 大顶堆：最大值优先出队
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
maxHeap.offer(5); maxHeap.offer(1); maxHeap.offer(3);
System.out.println(maxHeap.poll());  // 5

// 自定义比较器：按第二个元素（频率）降序
PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> b[1] - a[1]);
pq.offer(new int[]{1, 10}); // [值, 频率]
pq.offer(new int[]{2, 20});
System.out.println(pq.poll()[0]);  // 2（频率20最高）
```

## 🔥 高频算法应用

### 应用1：数组中的第K大元素（LeetCode 215）

用**大小为 K 的小顶堆**维护最大的 K 个元素：

```java
public int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();  // 小顶堆

    for (int num : nums) {
        minHeap.offer(num);
        if (minHeap.size() > k) {
            minHeap.poll();  // 超过K个时移除最小值
        }
    }
    return minHeap.peek();  // 堆顶就是第K大
    // 时间复杂度：O(n log k)，空间：O(k)
}
```

### 应用2：前K个高频元素（LeetCode 347）

```java
public int[] topKFrequent(int[] nums, int k) {
    // 统计频率
    Map<Integer, Integer> freqMap = new HashMap<>();
    for (int num : nums) freqMap.merge(num, 1, Integer::sum);

    // 小顶堆（按频率排序），保留频率最高的 K 个
    PriorityQueue<int[]> minHeap = new PriorityQueue<>(Comparator.comparingInt(a -> a[1]));
    for (Map.Entry<Integer, Integer> entry : freqMap.entrySet()) {
        minHeap.offer(new int[]{entry.getKey(), entry.getValue()});
        if (minHeap.size() > k) minHeap.poll();
    }

    return minHeap.stream().mapToInt(a -> a[0]).toArray();
}
```

### 应用3：合并K个升序链表（LeetCode 23）

```java
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>(Comparator.comparingInt(n -> n.val));

    // 将所有链表的头节点加入堆
    for (ListNode node : lists) {
        if (node != null) pq.offer(node);
    }

    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (!pq.isEmpty()) {
        ListNode node = pq.poll();
        curr.next = node;
        curr = curr.next;
        if (node.next != null) pq.offer(node.next);
    }
    return dummy.next;
    // 时间复杂度：O(n log k)，n 是总节点数，k 是链表数
}
```

### 应用4：数据流的中位数（LeetCode 295）

用**一个大顶堆（存较小的一半）+ 一个小顶堆（存较大的一半）**，始终维持两个堆大小相差不超过1：

```java
class MedianFinder {
    PriorityQueue<Integer> maxHeap;  // 存较小的一半，堆顶是较小一半最大值
    PriorityQueue<Integer> minHeap;  // 存较大的一半，堆顶是较大一半最小值

    public MedianFinder() {
        maxHeap = new PriorityQueue<>(Collections.reverseOrder());
        minHeap = new PriorityQueue<>();
    }

    public void addNum(int num) {
        maxHeap.offer(num);
        minHeap.offer(maxHeap.poll());  // 确保 minHeap 中的数都 >= maxHeap 中的数
        if (maxHeap.size() < minHeap.size()) {
            maxHeap.offer(minHeap.poll());  // 保持 maxHeap 数量 >= minHeap 数量
        }
    }

    public double findMedian() {
        if (maxHeap.size() > minHeap.size()) {
            return maxHeap.peek();
        }
        return (maxHeap.peek() + minHeap.peek()) / 2.0;
    }
}
```

### 应用5：任务调度器（LeetCode 621）

```java
public int leastInterval(char[] tasks, int n) {
    int[] freq = new int[26];
    for (char task : tasks) freq[task - 'A']++;

    // 大顶堆：剩余次数最多的任务优先执行
    PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
    for (int f : freq) if (f > 0) maxHeap.offer(f);

    int time = 0;
    Queue<int[]> cooldown = new LinkedList<>();  // [剩余次数, 可执行时间]

    while (!maxHeap.isEmpty() || !cooldown.isEmpty()) {
        time++;
        if (!maxHeap.isEmpty()) {
            int remain = maxHeap.poll() - 1;
            if (remain > 0) cooldown.offer(new int[]{remain, time + n});
        }
        if (!cooldown.isEmpty() && cooldown.peek()[1] == time) {
            maxHeap.offer(cooldown.poll()[0]);
        }
    }
    return time;
}
```

## 📊 堆的复杂度总结

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 查看堆顶 | O(1) | 直接返回根节点 |
| 插入 | O(log n) | 上浮操作 |
| 删除堆顶 | O(log n) | 下沉操作 |
| 建堆 | O(n) | 从底部开始 siftDown |
| 堆排序 | O(n log n) | 建堆 + n 次取堆顶 |

---

## 🚀 总结

堆是解决"**动态数据集合中快速获取极值**"问题的利器：

- **大顶堆**：快速获取最大值
- **小顶堆**：快速获取最小值，是解决 Top K 问题的标准工具
- **双堆技巧**：一大一小堆维护动态中位数
- **优先队列**：Java PriorityQueue 默认小顶堆，传 `Collections.reverseOrder()` 变大顶堆
- 堆排序虽然时间复杂度最优，但实际比快速排序慢（缓存不友好），工程中不常用

记住这个口诀：**Top K 用小顶堆，最大值用大顶堆，中位数用双堆。**
