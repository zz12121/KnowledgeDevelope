# 哈希表 - O(1) 查找的秘密

## 🎯 哈希表是什么？

### 1.1 直观理解
想象一个超级大的衣柜，每件衣服（数据）根据颜色（键的哈希值）被放进对应的格子（桶）里。找衣服时直接算出格子编号，不用一件件翻——这就是哈希表的核心思想。

### 1.2 正式定义
**哈希表（Hash Table / Hash Map）** 是通过哈希函数将键（Key）映射到数组索引，实现平均 O(1) 时间复杂度的增删查操作的数据结构。

### 1.3 核心三要素
```
Key → 哈希函数 → 数组索引 → 存/取 Value
```

## 📊 哈希函数

### 2.1 什么是好的哈希函数？
1. **确定性**：同一个 Key 每次得到的哈希值相同
2. **均匀性**：尽量将 Key 均匀分布在数组各个位置
3. **高效性**：计算速度快

### 2.2 常见哈希函数

```java
// 整数：取模法（最简单）
int hash(int key, int capacity) {
    return key % capacity;
}

// 字符串：多项式滚动哈希
int hash(String key, int capacity) {
    int hash = 0;
    for (char c : key.toCharArray()) {
        hash = (hash * 31 + c) % capacity;
        // 31 是质数，能减少哈希碰撞，Java String.hashCode() 就用 31
    }
    return hash;
}

// Java 内置 hashCode 使用
int index = (key.hashCode() ^ (key.hashCode() >>> 16)) & (capacity - 1);
// HashMap 用异或高位和低位来混合，让分布更均匀
// capacity 必须是 2 的幂，& (capacity-1) 等效于 % capacity 但更快
```

## 💥 哈希冲突（Hash Collision）

两个不同的 Key 算出相同的数组索引，称为**哈希冲突**。这不可避免，关键是如何处理。

### 3.1 链地址法（Chaining）——Java HashMap 采用此方案

每个桶存一个链表（Java 8 以后超过阈值转为红黑树）。

```
index 0: → [A:1] → [F:6] → null
index 1: → [B:2] → null
index 2: → [C:3] → [G:7] → null
index 3: → null
index 4: → [D:4] → null
```

```java
// 简化版链地址法 HashMap 实现
public class SimpleHashMap<K, V> {
    private static final int DEFAULT_CAPACITY = 16;
    private static final float LOAD_FACTOR = 0.75f;

    private LinkedList<Entry<K, V>>[] table;
    private int size;
    private int capacity;

    @SuppressWarnings("unchecked")
    public SimpleHashMap() {
        this.capacity = DEFAULT_CAPACITY;
        this.table = new LinkedList[capacity];
    }

    private int hash(K key) {
        if (key == null) return 0;
        int h = key.hashCode();
        return (h ^ (h >>> 16)) & (capacity - 1);
    }

    public void put(K key, V value) {
        int index = hash(key);
        if (table[index] == null) {
            table[index] = new LinkedList<>();
        }
        for (Entry<K, V> entry : table[index]) {
            if (Objects.equals(entry.key, key)) {
                entry.value = value;  // 覆盖
                return;
            }
        }
        table[index].add(new Entry<>(key, value));
        size++;
        if (size > capacity * LOAD_FACTOR) resize();
    }

    public V get(K key) {
        int index = hash(key);
        if (table[index] == null) return null;
        for (Entry<K, V> entry : table[index]) {
            if (Objects.equals(entry.key, key)) return entry.value;
        }
        return null;
    }

    private void resize() {
        capacity *= 2;
        LinkedList<Entry<K, V>>[] newTable = new LinkedList[capacity];
        for (LinkedList<Entry<K, V>> bucket : table) {
            if (bucket != null) {
                for (Entry<K, V> entry : bucket) {
                    int index = hash(entry.key);
                    if (newTable[index] == null) newTable[index] = new LinkedList<>();
                    newTable[index].add(entry);
                }
            }
        }
        table = newTable;
    }

    static class Entry<K, V> {
        K key;
        V value;
        Entry(K key, V value) { this.key = key; this.value = value; }
    }
}
```

### 3.2 开放寻址法（Open Addressing）

冲突时，在数组中找下一个可用位置。

```java
// 线性探测：冲突时往后找（+1, +2, +3...）
int findSlot(K key) {
    int index = hash(key);
    while (table[index] != null && !table[index].key.equals(key)) {
        index = (index + 1) % capacity;  // 线性探测
    }
    return index;
}
```

**两种方案对比：**

| 维度 | 链地址法 | 开放寻址法 |
|------|----------|------------|
| 空间 | 额外链表开销 | 紧凑，无额外指针 |
| 缓存 | 不友好（链表跳转） | 友好（数组连续） |
| 负载因子 | 可超过 1 | 必须小于 1 |
| 删除 | 简单 | 复杂（需要标记） |
| 代表 | Java HashMap | Python dict、ThreadLocalMap |

## 🏗️ Java HashMap 深度解析

### 4.1 核心属性
```java
// HashMap 关键参数
DEFAULT_INITIAL_CAPACITY = 16    // 初始容量（必须是2的幂）
DEFAULT_LOAD_FACTOR = 0.75f      // 负载因子
TREEIFY_THRESHOLD = 8            // 链表转红黑树的阈值（链表长度 >= 8）
UNTREEIFY_THRESHOLD = 6          // 红黑树退化为链表的阈值（树节点 <= 6）
MIN_TREEIFY_CAPACITY = 64        // 转红黑树还要求总容量 >= 64
```

### 4.2 put() 流程
```
put(key, value)
  ↓
计算 hash(key)
  ↓
定位桶（index = hash & (n-1)）
  ↓
桶为空？→ 直接放入
  ↓
桶不为空？→ 链表/红黑树查找相同 key
  ├─ 找到相同 key？→ 覆盖 value
  └─ 没找到？→ 追加到链表末尾
               ↓
               链表长度 >= 8 且总容量 >= 64？→ 转红黑树
               ↓
               size > capacity * 0.75？→ 扩容（容量×2，重新哈希）
```

### 4.3 为什么负载因子是 0.75？
- 过低（如 0.5）：空间浪费大，频繁扩容
- 过高（如 1.0）：冲突多，链表变长，查询退化
- 0.75 是时间与空间的最优折衷（来自泊松分布统计）

### 4.4 HashMap vs LinkedHashMap vs TreeMap

| 类型 | 底层结构 | 顺序 | 查询 | 场景 |
|------|----------|------|------|------|
| HashMap | 数组+链表/红黑树 | 无序 | O(1) 平均 | 通用Map |
| LinkedHashMap | HashMap + 双向链表 | 插入顺序/访问顺序 | O(1) | LRU缓存 |
| TreeMap | 红黑树 | 按key自然排序 | O(log n) | 需要排序的Map |

### 4.5 HashSet 底层
HashSet 底层就是一个 HashMap，Key 是 Set 的元素，Value 是一个固定的 Object 常量。

## 🎯 哈希表的经典应用

### 应用1：两数之和（LeetCode 1）
```java
public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> map = new HashMap<>();  // value → index
    for (int i = 0; i < nums.length; i++) {
        int complement = target - nums[i];
        if (map.containsKey(complement)) {
            return new int[]{map.get(complement), i};
        }
        map.put(nums[i], i);
    }
    return new int[0];
}
// 时间复杂度：O(n)，空间复杂度：O(n)
```

### 应用2：字母异位词分组（LeetCode 49）
```java
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String s : strs) {
        char[] chars = s.toCharArray();
        Arrays.sort(chars);
        String key = new String(chars);  // 排序后相同 → 是异位词
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    return new ArrayList<>(map.values());
}
```

### 应用3：LRU 缓存（LeetCode 146）
```java
class LRUCache {
    private final int capacity;
    // LinkedHashMap 维护访问顺序（accessOrder=true）
    private final LinkedHashMap<Integer, Integer> cache;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        this.cache = new LinkedHashMap<>(capacity, 0.75f, true) {
            @Override
            protected boolean removeEldestEntry(Map.Entry<Integer, Integer> eldest) {
                return size() > capacity;
            }
        };
    }

    public int get(int key) {
        return cache.getOrDefault(key, -1);
    }

    public void put(int key, int value) {
        cache.put(key, value);
    }
}
```

### 应用4：无重复字符的最长子串（LeetCode 3）
```java
public int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> map = new HashMap<>();  // 字符 → 最新出现索引
    int maxLen = 0;
    int left = 0;

    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (map.containsKey(c) && map.get(c) >= left) {
            left = map.get(c) + 1;  // 窗口左边界跳到重复字符的下一位
        }
        map.put(c, right);
        maxLen = Math.max(maxLen, right - left + 1);
    }
    return maxLen;
}
```

### 应用5：前缀和 + 哈希表（和为K的子数组）
```java
public int subarraySum(int[] nums, int k) {
    Map<Integer, Integer> prefixCount = new HashMap<>();
    prefixCount.put(0, 1);  // 前缀和为0出现1次（空数组）
    int sum = 0, count = 0;

    for (int num : nums) {
        sum += num;
        // 如果 sum - k 曾经出现过，说明找到了一段和为 k 的子数组
        count += prefixCount.getOrDefault(sum - k, 0);
        prefixCount.merge(sum, 1, Integer::sum);
    }
    return count;
}
// LeetCode 560. 和为 K 的子数组
```

## ⚡ 复杂度分析

| 操作 | 平均情况 | 最坏情况（全部冲突） |
|------|----------|----------------------|
| 查找 | O(1) | O(n) |
| 插入 | O(1) | O(n) |
| 删除 | O(1) | O(n) |
| 空间 | O(n) | O(n) |

Java 8 后加入红黑树，最坏情况从 O(n) 优化到 O(log n)。

## 📝 常见面试题与考点

### Q1：HashMap 是线程安全的吗？
不是。多线程下 `put` 操作可能导致数据丢失、死循环（Java 7 头插法扩容时）。
- 线程安全替代：`ConcurrentHashMap`（分段锁/CAS，高并发首选）
- 简单场景：`Collections.synchronizedMap(new HashMap<>())`

### Q2：HashMap 的 key 可以是 null 吗？
可以，但只能有一个 null key（hash 为 0，放在 index 0 的桶）。
`Hashtable` 不允许 null key/value；`ConcurrentHashMap` 也不允许。

### Q3：为什么 HashMap 容量必须是 2 的幂？
方便用位运算 `hash & (capacity - 1)` 代替取模 `hash % capacity`，性能更好。
同时扩容时，元素要么留在原位，要么移动到原位 + 旧容量，重新分布均匀。

### Q4：equals 和 hashCode 为什么必须同时重写？
HashMap 先用 `hashCode` 定位桶，再用 `equals` 找具体节点。
- 若只重写 `equals`：两个"相等"的对象可能哈希值不同，被放到不同桶，`get` 找不到
- 若只重写 `hashCode`：所有对象都进同一个桶，性能退化为 O(n)

## 🚀 总结

哈希表是面试中最高频的数据结构之一，掌握这几个关键点：

1. **O(1) 查找**的核心是哈希函数 + 冲突处理
2. **Java HashMap** = 数组 + 链表 + 红黑树，负载因子 0.75，扩容×2
3. **常见用途**：计数（词频统计）、去重（Set）、缓存（LRU）、前缀和优化
4. **多线程**用 `ConcurrentHashMap`，不要用 `HashMap`
5. 自定义 key 类型必须同时重写 `hashCode()` 和 `equals()`
