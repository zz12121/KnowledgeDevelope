# 复杂度分析详解 - 大O表示法完全指南

## 🎯 为什么需要复杂度分析？

### 1.1 直观感受
假设有两个算法解决同一个问题：
- 算法A：处理1000个数据需要1秒
- 算法B：处理1000个数据需要10秒

看起来算法A更好，但如果：
- 算法A：时间复杂度 O(n²)
- 算法B：时间复杂度 O(n)

当数据量增加到10000时：
- 算法A需要：100秒 (10000² / 1000² = 100倍)
- 算法B需要：100秒 (10000 / 1000 = 10倍)

数据量越大，算法B的优势越明显！

## 📊 大O表示法（Big O Notation）

### 2.1 什么是大O表示法？
大O表示法描述算法的**最坏情况**下，时间或空间需求随输入规模增长的**上限**。

### 2.2 常见复杂度对比

| 复杂度 | 名称 | n=10 | n=100 | n=1000 | 例子 |
|--------|------|------|-------|--------|------|
| O(1) | 常数时间 | 1 | 1 | 1 | 数组访问 |
| O(log n) | 对数时间 | 3.3 | 6.6 | 10 | 二分查找 |
| O(n) | 线性时间 | 10 | 100 | 1000 | 遍历数组 |
| O(n log n) | 线性对数 | 33 | 664 | 9966 | 快速排序 |
| O(n²) | 平方时间 | 100 | 10000 | 1000000 | 冒泡排序 |
| O(2ⁿ) | 指数时间 | 1024 | 1.26e30 | 1.07e301 | 子集问题 |
| O(n!) | 阶乘时间 | 3628800 | 9.3e157 | 4.0e2567 | 排列问题 |

## 🔍 如何分析复杂度？

### 3.1 时间复杂度的分析步骤

**步骤1：找出基本操作**
找出算法中执行次数最多的操作。

**步骤2：计算执行次数**
用数学表达式表示执行次数与输入规模n的关系。

**步骤3：保留最高阶项**
忽略常数和低阶项，只保留增长最快的项。

### 3.2 实例分析

**例1：简单循环**
```java
void example1(int n) {
    for (int i = 0; i < n; i++) {    // 执行n次
        System.out.println(i);        // 执行n次
    }
}
```
总操作次数：2n
时间复杂度：O(n)

**例2：嵌套循环**
```java
void example2(int n) {
    for (int i = 0; i < n; i++) {        // 执行n次
        for (int j = 0; j < n; j++) {    // 执行n次
            System.out.println(i + j);   // 执行n²次
        }
    }
}
```
总操作次数：n²
时间复杂度：O(n²)

**例3：混合复杂度**
```java
void example3(int n) {
    // 常数操作
    int a = 10;                         // O(1)
    
    // 线性操作
    for (int i = 0; i < n; i++) {       // O(n)
        System.out.println(i);
    }
    
    // 平方操作
    for (int i = 0; i < n; i++) {       // O(n²)
        for (int j = 0; j < n; j++) {
            System.out.println(i + j);
        }
    }
}
```
总复杂度：O(1) + O(n) + O(n²) = O(n²)

## 🧮 数学基础

### 4.1 常用数学公式

**求和公式：**
- 1 + 2 + 3 + ... + n = n(n+1)/2 = O(n²)
- 1 + 2 + 4 + ... + 2ⁿ = 2ⁿ⁺¹ - 1 = O(2ⁿ)

**对数公式：**
- log₂n = 以2为底n的对数
- 在复杂度分析中，通常省略底数，因为不同底数之间只差常数倍

### 4.2 递归算法的复杂度

**递归复杂度公式：**
T(n) = a * T(n/b) + f(n)

可以使用**主定理（Master Theorem）** 求解。

## 🔧 实际代码分析练习

### 5.1 练习1：数组求和
```java
int sumArray(int[] arr) {
    int sum = 0;                       // O(1)
    for (int num : arr) {              // 执行n次
        sum += num;                    // O(1) * n
    }
    return sum;                        // O(1)
}
```
总复杂度：O(1) + O(n) + O(1) = O(n)

### 5.2 练习2：矩阵乘法
```java
void matrixMultiply(int[][] A, int[][] B, int n) {
    int[][] C = new int[n][n];         // O(1) 空间分配
    
    for (int i = 0; i < n; i++) {              // n次
        for (int j = 0; j < n; j++) {          // n次
            C[i][j] = 0;                      // n²次
            for (int k = 0; k < n; k++) {      // n³次
                C[i][j] += A[i][k] * B[k][j]; // n³次
            }
        }
    }
}
```
总时间复杂度：O(n³)
总空间复杂度：O(n²) （存储结果矩阵）

### 5.3 练习3：二分查找
```java
int binarySearch(int[] arr, int target) {
    int left = 0;                      // O(1)
    int right = arr.length - 1;        // O(1)
    
    while (left <= right) {            // 最多执行log₂n次
        int mid = left + (right - left) / 2;  // O(1)
        
        if (arr[mid] == target) {      // O(1)
            return mid;                // O(1)
        } else if (arr[mid] < target) { // O(1)
            left = mid + 1;            // O(1)
        } else {
            right = mid - 1;           // O(1)
        }
    }
    return -1;                         // O(1)
}
```
总时间复杂度：O(log n)

## 🎯 复杂度分析技巧

### 7.1 快速判断技巧
1. **单循环**：通常是 O(n)
2. **嵌套循环**：通常是 O(n²) 或 O(n*m)
3. **二分**：通常是 O(log n)
4. **递归**：需要分析递归深度和每层工作量

### 7.2 注意事项
1. **关注最坏情况**：大O表示法描述最坏情况
2. **忽略常数**：O(2n) = O(n)，O(n/2) = O(n)
3. **关注主导项**：O(n² + n) = O(n²)
4. **不同输入**：可能有多个复杂度

## 📝 实战练习

分析以下代码的时间复杂度：

```java
// 练习1
void func1(int n) {
    for (int i = 0; i < n; i += 2) {
        System.out.println(i);
    }
}
// 答案：O(n)

// 练习2  
void func2(int n) {
    for (int i = 1; i < n; i *= 2) {
        System.out.println(i);
    }
}
// 答案：O(log n)

// 练习3
void func3(int n) {
    for (int i = 0; i < n; i++) {
        for (int j = 1; j < n; j *= 2) {
            System.out.println(i + j);
        }
    }
}
// 答案：O(n log n)
```
