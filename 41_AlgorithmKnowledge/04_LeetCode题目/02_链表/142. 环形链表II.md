# 142. 环形链表 II

> 难度：🟡 Medium | 标签：`链表` `双指针` `数学`

## 题目描述

给定一个链表的头节点 `head`，返回链表开始入环的第一个节点。如果链表无环，返回 `null`。

## 示例

```
输入：head = [3, 2, 0, -4]，pos = 1
输出：返回下标为 1 的节点（节点值为 2）

输入：head = [1, 2]，pos = 0
输出：返回下标为 0 的节点（节点值为 1）
```

## 解题思路

### Floyd 判圈算法找入口 ⭐

**第一步**：快慢指针找相遇点（同 141 题）

**第二步**：找环的入口
- 设：链表头到环入口的距离为 `a`，环入口到相遇点的距离为 `b`，相遇点到环入口的距离为 `c`
- 慢指针走了 `a + b` 步，快指针走了 `a + b + n(b+c)` 步（n圈）
- 快指针步数 = 2 × 慢指针步数：`a + b + n(b+c) = 2(a + b)`
- 化简得：`a = (n-1)(b+c) + c`，即 `a = c`（当n=1时）

**结论**：相遇后，将 `fast` 重置到 `head`，两个指针以相同速度前进，再次相遇处即为环的入口！

## 代码

```java
public ListNode detectCycle(ListNode head) {
    ListNode slow = head, fast = head;
    
    // 第一步：找相遇点
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;
    }
    
    // 无环
    if (fast == null || fast.next == null) return null;
    
    // 第二步：找环的入口
    // 将 fast 重置到 head，两者同速前进
    fast = head;
    while (fast != slow) {
        slow = slow.next;
        fast = fast.next;
    }
    return fast; // 再次相遇，即为入口
}
```

## 数学推导

```
设：
  a = head 到环入口的距离
  b = 环入口到相遇点的距离  
  c = 相遇点到环入口的距离（绕回去）
  
相遇时：
  slow 走了 a + b 步
  fast 走了 a + b + k*(b+c) 步（k ≥ 1 圈）
  
因为 fast = 2 * slow：
  a + b + k*(b+c) = 2*(a + b)
  k*(b+c) = a + b
  a = k*(b+c) - b = (k-1)*(b+c) + c
  
当 k=1 时：a = c
→ 从 head 和相遇点同时走，走 a 步后在环入口相遇！
```
