# 贪心算法

## 什么是贪心算法

贪心算法（Greedy Algorithm）是一种在每一步都选择**当前看起来最优的选择**的策略，期望通过局部最优解达到全局最优解。

### 贪心 vs 动态规划

| | 贪心 | 动态规划 |
|---|---|---|
| 选择方式 | 每步选局部最优 | 考虑所有可能性 |
| 适用条件 | 贪心选择性质 + 最优子结构 | 最优子结构 + 重叠子问题 |
| 时间复杂度 | 通常更低 | 通常较高 |
| 证明难度 | 难（需证明贪心有效） | 较易（状态转移清晰） |

**什么时候贪心有效？** 当**贪心选择性质**成立时：即局部最优的贪心选择不会影响到以后的最优选择。

## 经典贪心问题

### 问题一：区间调度（活动选择问题）

**问题**：给定若干活动，每个活动有开始时间和结束时间，活动不能重叠，最多可以参加多少个活动？（LeetCode 435 变形）

**贪心策略**：优先选择**结束时间最早**的活动

**直觉**：结束越早，给后续活动留下的时间越多

```java
public int maxActivities(int[][] intervals) {
    // 按结束时间排序
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]);
    
    int count = 1;
    int lastEnd = intervals[0][1];
    
    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] >= lastEnd) {  // 开始时间 >= 上一个结束时间
            count++;
            lastEnd = intervals[i][1];
        }
    }
    return count;
}
```

### 问题二：无重叠区间（LeetCode 435）

**问题**：给定一组区间，找出需要移除区间的最小数量，使剩余区间互不重叠。

**贪心策略**：尽可能保留更多区间 → 等价于活动选择问题（找最多不重叠区间）

```java
public int eraseOverlapIntervals(int[][] intervals) {
    if (intervals.length == 0) return 0;
    
    // 按结束时间排序
    Arrays.sort(intervals, (a, b) -> a[1] - b[1]);
    
    int keep = 1;  // 能保留的最多不重叠区间数
    int lastEnd = intervals[0][1];
    
    for (int i = 1; i < intervals.length; i++) {
        if (intervals[i][0] >= lastEnd) {
            keep++;
            lastEnd = intervals[i][1];
        }
    }
    return intervals.length - keep;  // 需要删除的数量
}
```

### 问题三：跳跃游戏（LeetCode 55）

**问题**：数组中每个元素代表从该位置最多能跳多远，判断能否到达最后一个位置。

**贪心策略**：维护当前能到达的**最远距离** maxReach，如果当前位置超出 maxReach 则无法到达

```java
public boolean canJump(int[] nums) {
    int maxReach = 0;
    
    for (int i = 0; i < nums.length; i++) {
        if (i > maxReach) return false;  // 当前位置不可达
        maxReach = Math.max(maxReach, i + nums[i]);
    }
    return true;
}
```

### 问题四：跳跃游戏 II（LeetCode 45）

**问题**：假设总是能到达最后位置，求最少跳跃次数。

**贪心策略**：每次跳跃都选择能**到达最远位置**的跳法

```java
public int jump(int[] nums) {
    int jumps = 0;
    int curEnd = 0;    // 当前跳跃能到达的最远位置
    int farthest = 0;  // 下一跳能到达的最远位置
    
    for (int i = 0; i < nums.length - 1; i++) {
        farthest = Math.max(farthest, i + nums[i]);
        if (i == curEnd) {  // 到达当前跳跃边界，必须再跳一次
            jumps++;
            curEnd = farthest;
        }
    }
    return jumps;
}
```

### 问题五：加油站（LeetCode 134）

**问题**：环形公路上 n 个加油站，gas[i] 是可加油量，cost[i] 是到下一站的消耗，求出发点。

**贪心结论**：
1. 若总 gas >= 总 cost，一定有解
2. 从 gas - cost 总和开始变负的下一个位置出发

```java
public int canCompleteCircuit(int[] gas, int[] cost) {
    int totalTank = 0;
    int currTank = 0;
    int startStation = 0;
    
    for (int i = 0; i < gas.length; i++) {
        int net = gas[i] - cost[i];
        totalTank += net;
        currTank += net;
        
        if (currTank < 0) {
            // 从 i+1 重新开始
            startStation = i + 1;
            currTank = 0;
        }
    }
    return totalTank >= 0 ? startStation : -1;
}
```

### 问题六：分发糖果（LeetCode 135）

**问题**：n 个孩子站成一排，评分更高的孩子比相邻孩子获得更多糖果，最少需要多少糖果？

**贪心策略**：两次遍历
1. 从左向右：右边比左边评分高，则右边多一颗
2. 从右向左：左边比右边评分高，则左边取 max(当前, 右边+1)

```java
public int candy(int[] ratings) {
    int n = ratings.length;
    int[] candy = new int[n];
    Arrays.fill(candy, 1);
    
    // 从左到右
    for (int i = 1; i < n; i++) {
        if (ratings[i] > ratings[i - 1]) {
            candy[i] = candy[i - 1] + 1;
        }
    }
    
    // 从右到左
    for (int i = n - 2; i >= 0; i--) {
        if (ratings[i] > ratings[i + 1]) {
            candy[i] = Math.max(candy[i], candy[i + 1] + 1);
        }
    }
    
    int total = 0;
    for (int c : candy) total += c;
    return total;
}
```

## 贪心算法证明思路

贪心算法的正确性通常需要证明，常用方法：

### 1. 反证法（Exchange Argument）

假设存在一个最优解不使用贪心选择，通过"交换"操作将其改造成使用贪心选择的解，并证明改造后的解不会变差。

### 2. 数学归纳法

证明贪心选择后，剩余的子问题仍然具有最优子结构。

## 贪心不适用的情况

**硬币问题**（面值为 1, 3, 4，凑 6 元）：
- 贪心：4 + 1 + 1 = 3 枚
- 最优：3 + 3 = 2 枚

面值非标准时，贪心不适用，应使用动态规划。
