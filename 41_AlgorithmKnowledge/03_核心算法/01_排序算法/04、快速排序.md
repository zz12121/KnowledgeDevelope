# 快速排序 - 实践中最快的排序算法

## 🎯 算法思想

### 1.1 核心思想：分区（Partition）

选取一个**基准元素（Pivot）**，将数组分成两部分：
- 左边：所有元素 ≤ pivot
- 右边：所有元素 ≥ pivot

然后递归地对左右两部分进行同样操作。当每个区间只有一个元素时，整个数组有序。

```
原始: [3, 6, 8, 10, 1, 2, 1]，选 pivot = 3

分区后: [1, 2, 1, 3, 10, 8, 6]
                  ↑ pivot 落在索引 3

递归左: [1, 2, 1] → [1, 1, 2]
递归右: [10, 8, 6] → [6, 8, 10]

最终: [1, 1, 2, 3, 6, 8, 10]
```

## 🛠️ Java 实现

### 2.1 Lomuto 分区（简洁易理解）

```java
public static void quickSort(int[] arr, int low, int high) {
    if (low >= high) return;
    int pivotIdx = partition(arr, low, high);
    quickSort(arr, low, pivotIdx - 1);   // 递归排左边
    quickSort(arr, pivotIdx + 1, high);  // 递归排右边
}

// Lomuto 分区：以最后一个元素为 pivot
private static int partition(int[] arr, int low, int high) {
    int pivot = arr[high];  // 选最后一个作为基准
    int i = low - 1;        // i 指向已处理的小于 pivot 区间的末尾

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr, i, j);  // 将小于等于 pivot 的元素放到左边
        }
    }
    swap(arr, i + 1, high);  // 将 pivot 放到正确位置
    return i + 1;            // 返回 pivot 的索引
}

private static void swap(int[] arr, int i, int j) {
    int temp = arr[i]; arr[i] = arr[j]; arr[j] = temp;
}
```

### 2.2 Hoare 分区（更高效，双指针）

```java
// Hoare 分区：从两端向中间夹逼
private static int partitionHoare(int[] arr, int low, int high) {
    int pivot = arr[low + (high - low) / 2];  // 选中间元素为 pivot
    int i = low - 1;
    int j = high + 1;

    while (true) {
        do { i++; } while (arr[i] < pivot);   // 左指针找到 >= pivot 的
        do { j--; } while (arr[j] > pivot);   // 右指针找到 <= pivot 的
        if (i >= j) return j;
        swap(arr, i, j);
    }
}
```

### 2.3 三路快排（处理大量重复元素）

当数组有大量重复元素时，普通快排退化。三路快排将数组分成 `< pivot`、`== pivot`、`> pivot` 三部分：

```java
public static void quickSort3Way(int[] arr, int low, int high) {
    if (low >= high) return;

    int lt = low;     // arr[low..lt-1] < pivot
    int gt = high;    // arr[gt+1..high] > pivot
    int pivot = arr[low];
    int i = low + 1;  // arr[lt..i-1] == pivot

    while (i <= gt) {
        if (arr[i] < pivot) {
            swap(arr, lt++, i++);
        } else if (arr[i] > pivot) {
            swap(arr, i, gt--);
        } else {
            i++;
        }
    }
    // 现在 arr[low..lt-1] < pivot = arr[lt..gt] < arr[gt+1..high]
    quickSort3Way(arr, low, lt - 1);
    quickSort3Way(arr, gt + 1, high);
}
```

### 2.4 随机化快排（防止最坏情况）

```java
private static final Random RANDOM = new Random();

public static void quickSortRandom(int[] arr, int low, int high) {
    if (low >= high) return;
    // 随机选择 pivot，避免有序数组导致 O(n²)
    int randomIdx = low + RANDOM.nextInt(high - low + 1);
    swap(arr, randomIdx, high);  // 将随机选择的元素与末尾交换
    int pivotIdx = partition(arr, low, high);
    quickSortRandom(arr, low, pivotIdx - 1);
    quickSortRandom(arr, pivotIdx + 1, high);
}
```

## 📊 复杂度分析

| 指标 | 值 | 条件 |
|------|----|----|
| 时间（最好） | O(n log n) | 每次分区均匀 |
| 时间（平均） | O(n log n) | 随机数据 |
| 时间（最坏） | **O(n²)** | 已排序/全相同/每次选最值 |
| 空间 | O(log n) | 递归栈（平均），最坏 O(n) |
| 稳定性 | **不稳定** | 交换会打乱相等元素顺序 |

### 为什么最坏是 O(n²)？

若每次选的 pivot 都是最大或最小值（如数组已排序），分区极不均衡：

```
[1, 2, 3, 4, 5]，每次选末尾作 pivot
第1次分区：pivot=5，左4个，右0个 → n-1 次比较
第2次分区：pivot=4，左3个，右0个 → n-2 次比较
...
总计：(n-1) + (n-2) + ... + 1 = O(n²)
```

**解决方案**：随机化选 pivot，期望情况下接近 O(n log n)。

## 🔥 快排的核心应用

### 3.1 数组中的第 K 大元素（QuickSelect）

利用分区思想，只需要一侧递归，平均 O(n)：

```java
public int findKthLargest(int[] nums, int k) {
    return quickSelect(nums, 0, nums.length - 1, nums.length - k);
}

// 找第 k 小（0-indexed）的元素
private int quickSelect(int[] arr, int low, int high, int k) {
    if (low == high) return arr[low];

    int pivotIdx = partition(arr, low, high);

    if (pivotIdx == k) {
        return arr[pivotIdx];
    } else if (pivotIdx < k) {
        return quickSelect(arr, pivotIdx + 1, high, k);  // 只递归右边
    } else {
        return quickSelect(arr, low, pivotIdx - 1, k);   // 只递归左边
    }
}
// LeetCode 215. 数组中的第K个最大元素
// 平均时间复杂度 O(n)，最坏 O(n²)
```

## 📝 快排的工程优化

现代工程中的排序（如 Java `Arrays.sort` 对基本类型采用 DualPivot QuickSort）：

1. **双轴快排（Dual-Pivot QuickSort）**：选两个 pivot，将数组分三段，减少递归深度
2. **小数组用插入排序**：当子数组长度 < 47 时，插入排序更快
3. **三数取中**：选 low、mid、high 三者中的中间值作 pivot，避免最坏情况

```java
// 三数取中优化
private static int medianOfThree(int[] arr, int low, int high) {
    int mid = low + (high - low) / 2;
    if (arr[low] > arr[mid]) swap(arr, low, mid);
    if (arr[low] > arr[high]) swap(arr, low, high);
    if (arr[mid] > arr[high]) swap(arr, mid, high);
    // 现在 arr[low] <= arr[mid] <= arr[high]
    swap(arr, mid, high - 1);  // 将中间值放到 high-1
    return arr[high - 1];
}
```

## 🔄 快排 vs 归并排序

| 维度 | 快速排序 | 归并排序 |
|------|----------|----------|
| 时间（平均） | O(n log n) | O(n log n) |
| 时间（最坏） | O(n²) | O(n log n) |
| 空间 | O(log n) | O(n) |
| 稳定性 | ❌ | ✅ |
| 缓存友好 | ✅（局部访问） | ❌（跨区间访问） |
| 实际速度 | 更快（常数小） | 稳定但慢一些 |
| 适用场景 | 基本类型数组 | 对象排序、链表排序、需要稳定 |

---

**记住**：快排是平均性能最优的排序算法。实际工程中配合随机化、三路分区、小数组插入排序，可以应对各种输入场景。面试中要能手写基础版并分析 O(n log n) 均摊复杂度。
