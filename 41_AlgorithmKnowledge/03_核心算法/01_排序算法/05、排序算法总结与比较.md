# 排序算法总结与比较

## 📊 所有排序算法一览表

| 算法 | 时间（最好） | 时间（平均） | 时间（最坏） | 空间 | 稳定 | 特点 |
|------|------------|------------|------------|------|------|------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ | 简单，教学用 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | ❌ | 交换次数最少 |
| 插入排序 | **O(n)** | O(n²) | O(n²) | O(1) | ✅ | 近乎有序时最快 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ | 最坏保证，外排 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ | 实际最快 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ | 原地+最坏保证 |
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | O(k) | ✅ | 整数，范围小 |
| 桶排序 | O(n+k) | O(n+k) | O(n²) | O(n+k) | ✅ | 均匀分布数据 |
| 基数排序 | O(d·n) | O(d·n) | O(d·n) | O(n+k) | ✅ | 多关键字排序 |

其中 k 是数据范围，d 是数字位数。

## 🔢 线性时间排序算法

### 计数排序（Counting Sort）

**适用条件**：整数，且范围 k 不大（如 0~100）

```java
public static void countingSort(int[] arr) {
    if (arr.length <= 1) return;

    int max = Arrays.stream(arr).max().getAsInt();
    int min = Arrays.stream(arr).min().getAsInt();
    int range = max - min + 1;

    int[] count = new int[range];
    for (int num : arr) count[num - min]++;

    // 重建数组
    int idx = 0;
    for (int i = 0; i < range; i++) {
        while (count[i]-- > 0) arr[idx++] = i + min;
    }
}
```

### 基数排序（Radix Sort）

**适用条件**：非负整数，按每位数字排序

```java
public static void radixSort(int[] arr) {
    int max = Arrays.stream(arr).max().getAsInt();

    // 从个位开始，按每位进行计数排序
    for (int exp = 1; max / exp > 0; exp *= 10) {
        countByDigit(arr, exp);
    }
}

private static void countByDigit(int[] arr, int exp) {
    int n = arr.length;
    int[] output = new int[n];
    int[] count = new int[10];

    for (int num : arr) count[(num / exp) % 10]++;
    for (int i = 1; i < 10; i++) count[i] += count[i - 1];

    // 从右到左填充保证稳定性
    for (int i = n - 1; i >= 0; i--) {
        int digit = (arr[i] / exp) % 10;
        output[--count[digit]] = arr[i];
    }
    System.arraycopy(output, 0, arr, 0, n);
}
```

## 🎯 如何选择排序算法？

```
数据量 n < 50？
  → 插入排序（简单快速）

需要稳定排序？
  → 归并排序（稳定 + O(n log n) 保证）

内存严格限制（原地）？
  → 堆排序（O(1) 空间 + O(n log n) 保证）
  → 快速排序（O(log n) 空间，实际更快）

数据是整数且范围小？
  → 计数排序 O(n+k)

数据几乎有序？
  → 插入排序（接近 O(n)）

一般情况（对象排序）？
  → 归并排序（Java Arrays.sort 对象类型用 TimSort）

一般情况（基本类型）？
  → 快速排序（Java Arrays.sort 基本类型用 DualPivot QuickSort）
```

## 💡 Java 中的排序

```java
// 基本类型数组：DualPivot QuickSort（不稳定，但对基本类型无所谓）
int[] arr = {5, 3, 1, 4, 2};
Arrays.sort(arr);

// 对象数组：TimSort（稳定，归并+插入）
Integer[] objArr = {5, 3, 1, 4, 2};
Arrays.sort(objArr);

// 自定义比较器
Arrays.sort(objArr, (a, b) -> b - a);  // 降序

// List 排序
List<Integer> list = Arrays.asList(5, 3, 1, 4, 2);
Collections.sort(list);
list.sort(Comparator.reverseOrder());  // 降序

// 二维数组按某列排序
int[][] matrix = {{3, 1}, {1, 2}, {2, 3}};
Arrays.sort(matrix, (a, b) -> a[0] - b[0]);  // 按第一列升序
```

## 📝 面试常见问题

### Q1：为什么快排实际上比堆排序快？
虽然两者都是 O(n log n)，但快排的常数更小：
- 快排访问连续内存（缓存友好）
- 堆排序频繁访问不连续的堆节点（缓存不友好）
- 实测快排约是堆排序的 2-3 倍速度

### Q2：什么情况下快排退化为 O(n²)？
- 数组已排序（升序或降序）时选末尾/首位作 pivot
- 所有元素相同时（不使用三路分区）
- **解决**：随机化选 pivot，或使用三路分区

### Q3：归并排序和快排哪个更好？
取决于场景：
- 需要**稳定性**：归并排序
- **空间有限**：快速排序（O(log n) vs O(n)）
- **链表排序**：归并排序（不需要随机访问）
- 一般**数组排序**：快速排序（缓存友好，实际更快）

### Q4：手写快排的注意点
1. 终止条件：`low >= high`
2. pivot 选取：建议随机化或三数取中
3. 分区后递归：`[low, pivotIdx-1]` 和 `[pivotIdx+1, high]`
4. 有大量重复元素时用三路分区
