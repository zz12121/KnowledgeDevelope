# 归并排序 - 分治思想的经典实现

## 🎯 算法思想

### 1.1 分治策略
**"分而治之，合而成之"**——将大问题分解为小问题，递归解决，再合并结果。

```
               [38, 27, 43, 3, 9, 82, 10]
                     /            \
           [38, 27, 43]        [3, 9, 82, 10]
             /      \           /       \
         [38, 27]  [43]      [3, 9]  [82, 10]
          /    \              /  \    /    \
        [38]  [27]          [3] [9] [82]  [10]
          \   /              \  /     \   /
         [27, 38]            [3, 9]  [10, 82]
              \                  \   /
           [27, 38, 43]        [3, 9, 10, 82]
                      \       /
              [3, 9, 10, 27, 38, 43, 82]
```

三步走：
1. **分（Divide）**：将数组从中间劈成两半
2. **治（Conquer）**：递归地对两半分别排序
3. **合（Combine）**：将两个有序子数组合并成一个有序数组

## 🛠️ Java 实现

### 2.1 经典递归版本

```java
public static void mergeSort(int[] arr, int left, int right) {
    if (left >= right) return;  // 终止条件：只有一个或零个元素

    int mid = left + (right - left) / 2;  // 防止 (left+right) 溢出

    mergeSort(arr, left, mid);      // 递归排序左半部分
    mergeSort(arr, mid + 1, right); // 递归排序右半部分
    merge(arr, left, mid, right);   // 合并两个有序部分
}

// 合并两个有序区间 [left, mid] 和 [mid+1, right]
private static void merge(int[] arr, int left, int mid, int right) {
    // 复制到临时数组
    int[] temp = new int[right - left + 1];

    int i = left;      // 左半部分指针
    int j = mid + 1;   // 右半部分指针
    int k = 0;         // 临时数组指针

    // 两部分都有元素时，比较后取较小的
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {  // <= 保证稳定性
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
        }
    }

    // 复制左半部分剩余元素
    while (i <= mid) temp[k++] = arr[i++];
    // 复制右半部分剩余元素
    while (j <= right) temp[k++] = arr[j++];

    // 将临时数组复制回原数组
    System.arraycopy(temp, 0, arr, left, temp.length);
}

// 调用方式
public static void main(String[] args) {
    int[] arr = {38, 27, 43, 3, 9, 82, 10};
    mergeSort(arr, 0, arr.length - 1);
    System.out.println(Arrays.toString(arr));  // [3, 9, 10, 27, 38, 43, 82]
}
```

### 2.2 自底向上（迭代版本）—— 避免递归栈溢出

```java
public static void mergeSortBottomUp(int[] arr) {
    int n = arr.length;
    // size 表示每个子数组的大小：1, 2, 4, 8, ...
    for (int size = 1; size < n; size *= 2) {
        // 对每对相邻子数组进行合并
        for (int left = 0; left < n - size; left += 2 * size) {
            int mid = left + size - 1;
            int right = Math.min(left + 2 * size - 1, n - 1);
            merge(arr, left, mid, right);
        }
    }
}
```

## 📊 复杂度分析

| 指标 | 值 | 原因 |
|------|----|----|
| 时间（最好） | O(n log n) | 每层合并 O(n)，共 log n 层 |
| 时间（最坏） | O(n log n) | 与输入无关，始终如此 |
| 时间（平均） | O(n log n) | 同上 |
| 空间 | **O(n)** | 合并时需要辅助数组 |
| 稳定性 | **稳定** | merge 时 `<=` 优先取左边 |

**为什么是 O(n log n)？**
```
递归树共 log₂n 层
每层合并的工作量合计 O(n)
总时间 = O(n) × log₂n = O(n log n)
```

## 🎯 归并排序的重要应用

### 3.1 计算逆序对数量（LeetCode 315）

**逆序对**：对于 i < j，若 arr[i] > arr[j]，则 (i, j) 是逆序对。

```java
// 在归并排序的 merge 过程中统计逆序对
private static int count = 0;

public static void mergeSortCount(int[] arr, int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSortCount(arr, left, mid);
    mergeSortCount(arr, mid + 1, right);
    mergeCount(arr, left, mid, right);
}

private static void mergeCount(int[] arr, int left, int mid, int right) {
    int[] temp = new int[right - left + 1];
    int i = left, j = mid + 1, k = 0;

    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            // arr[i] > arr[j]：左半部分从 i 到 mid 的所有元素都大于 arr[j]
            count += (mid - i + 1);  // 统计逆序对
            temp[k++] = arr[j++];
        }
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= right) temp[k++] = arr[j++];
    System.arraycopy(temp, 0, arr, left, temp.length);
}
```

### 3.2 链表排序（LeetCode 148）

归并排序天然适合链表（不需要随机访问）：

```java
public ListNode sortList(ListNode head) {
    if (head == null || head.next == null) return head;

    // 快慢指针找中点
    ListNode slow = head, fast = head.next;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
    ListNode mid = slow.next;
    slow.next = null;  // 断开链表

    ListNode left = sortList(head);
    ListNode right = sortList(mid);
    return mergeList(left, right);
}

private ListNode mergeList(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) { curr.next = l1; l1 = l1.next; }
        else { curr.next = l2; l2 = l2.next; }
        curr = curr.next;
    }
    curr.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

## 🔧 与快速排序的对比

| 维度 | 归并排序 | 快速排序 |
|------|----------|----------|
| 时间（平均） | O(n log n) | O(n log n) |
| 时间（最坏） | O(n log n) | O(n²) |
| 空间 | O(n) | O(log n) |
| 稳定性 | ✅ 稳定 | ❌ 不稳定 |
| 缓存友好 | 较差 | 较好 |
| 适用 | 链表排序、外部排序、需要稳定 | 内存排序，实际更快 |

---

**记住**：归并排序是**外部排序**（数据量超过内存时）的基础算法，也是 Java `Arrays.sort` 对对象数组（`TimSort`）的核心思想。其 **O(n log n) 最坏保证**是相对于快排的最大优势。
