# 选择排序与插入排序 - 简单但经典

## 🎯 选择排序（Selection Sort）

### 1.1 算法思想

每轮从**未排序区间**找到最小值，放到已排序区间末尾。

```
原始: [64, 25, 12, 22, 11]

第1轮: 未排序[64,25,12,22,11]，找最小值11，与64交换
       → [11, 25, 12, 22, 64]

第2轮: 未排序[25,12,22,64]，找最小值12，与25交换
       → [11, 12, 25, 22, 64]

第3轮: 未排序[25,22,64]，找最小值22，与25交换
       → [11, 12, 22, 25, 64]

第4轮: 未排序[25,64]，找最小值25，位置不变
       → [11, 12, 22, 25, 64]

完成！
```

### 1.2 Java 实现

```java
public static void selectionSort(int[] arr) {
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;  // 假设当前位置最小
        // 在未排序区间找最小值的索引
        for (int j = i + 1; j < n; j++) {
            if (arr[j] < arr[minIdx]) {
                minIdx = j;
            }
        }
        // 将最小值放到已排序区间末尾
        if (minIdx != i) {
            int temp = arr[i];
            arr[i] = arr[minIdx];
            arr[minIdx] = temp;
        }
    }
}
```

### 1.3 复杂度分析

| 指标 | 值 | 原因 |
|------|----|----|
| 时间（最好） | O(n²) | 每次必须遍历未排序区间 |
| 时间（最坏） | O(n²) | 同上 |
| 时间（平均） | O(n²) | 同上 |
| 空间 | O(1) | 原地排序 |
| 稳定性 | **不稳定** | 交换可能破坏相等元素顺序 |

> **不稳定示例**：`[3a, 3b, 1]`，第1轮将 `1` 和 `3a` 交换，结果 `[1, 3b, 3a]`，原始相对顺序被破坏。

### 1.4 适用场景

- **数据量小**（n < 100）
- **内存写入代价高**的场景：选择排序交换次数最多只有 O(n)，而冒泡排序最多 O(n²) 次交换

## 🎯 插入排序（Insertion Sort）

### 2.1 算法思想

想象**整理扑克牌**：每次拿到一张新牌，插入到手牌中正确的位置。

- 将数组分为**已排序（左）** 和**未排序（右）** 两部分
- 每次取未排序区间第一个元素，向左移动，插入到已排序区间的正确位置

```
原始: [5, 2, 4, 6, 1, 3]

i=1: 取2，与5比较，5>2，5右移 → [5,5,4,6,1,3]，插入2 → [2,5,4,6,1,3]
i=2: 取4，与5比较，5>4，5右移 → [2,5,5,6,1,3]，与2比较，2<4，插入4 → [2,4,5,6,1,3]
i=3: 取6，与5比较，5<6，插入6 → [2,4,5,6,1,3]
i=4: 取1，依次右移5,6,5,4,2 → [1,2,4,5,6,3]
i=5: 取3，依次右移6,5,4 → [1,2,3,4,5,6]

完成！
```

### 2.2 Java 实现

```java
public static void insertionSort(int[] arr) {
    int n = arr.length;
    for (int i = 1; i < n; i++) {
        int key = arr[i];   // 当前要插入的元素
        int j = i - 1;

        // 将大于 key 的元素向右移动一位
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j--;
        }
        // 插入到正确位置
        arr[j + 1] = key;
    }
}
```

### 2.3 复杂度分析

| 指标 | 值 | 原因 |
|------|----|----|
| 时间（最好） | **O(n)** | 数组已有序，每次只比较1次不移动 |
| 时间（最坏） | O(n²) | 数组完全逆序 |
| 时间（平均） | O(n²) | 平均移动 n/2 次 |
| 空间 | O(1) | 原地排序 |
| 稳定性 | **稳定** | 相等元素不交换 |

### 2.4 优化变体——二分插入排序

用二分查找找到插入位置（减少比较次数，但移动次数不变）：

```java
public static void binaryInsertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];
        int left = 0, right = i - 1;

        // 二分查找插入位置
        while (left <= right) {
            int mid = (left + right) / 2;
            if (arr[mid] > key) right = mid - 1;
            else left = mid + 1;
        }

        // 移动元素，left 就是插入位置
        System.arraycopy(arr, left, arr, left + 1, i - left);
        arr[left] = key;
    }
}
```

### 2.5 适用场景

- **数据量小**（n < 100）
- **数据几乎有序**时接近 O(n)
- **在线算法**：一边接收数据一边排序
- **实际工程优化**：Java/Python 内置排序（TimSort）在小段数组上用插入排序

## 📊 三种简单排序算法对比

| 算法 | 最好 | 平均 | 最坏 | 空间 | 稳定 | 特点 |
|------|------|------|------|------|------|------|
| 冒泡排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ | 简单，适合教学 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | ❌ | 交换次数少 |
| 插入排序 | O(n) | O(n²) | O(n²) | O(1) | ✅ | 近乎有序时最快 |

**结论**：三者中，**插入排序在实际中性能最好**（近乎有序时接近 O(n)，TimSort 使用它）。选择排序在写入代价高时有优势（交换次数最少）。

## 🧪 测试代码

```java
public static void main(String[] args) {
    int[] arr1 = {64, 25, 12, 22, 11};
    selectionSort(arr1);
    System.out.println("选择排序: " + Arrays.toString(arr1));  // [11, 12, 22, 25, 64]

    int[] arr2 = {5, 2, 4, 6, 1, 3};
    insertionSort(arr2);
    System.out.println("插入排序: " + Arrays.toString(arr2));  // [1, 2, 3, 4, 5, 6]

    // 验证近乎有序时的性能
    int[] nearSorted = {1, 2, 3, 5, 4};
    insertionSort(nearSorted);
    System.out.println("近乎有序: " + Arrays.toString(nearSorted));  // [1, 2, 3, 4, 5]
}
```

**记住**：在面试中，这两个算法本身不会直接考，但理解它们的思想有助于理解更复杂的算法。插入排序的思想在希尔排序（分组插入排序）中也有应用。
