# 动态规划入门

## 什么是动态规划

动态规划（Dynamic Programming，DP）是一种通过把原问题分解为相对简单的子问题的方式来求解复杂问题的方法。它的核心思想是：**记住已解决的子问题的答案，避免重复计算**。

### 动态规划的本质

```
动态规划 = 递归 + 记忆化（或者 = 填表法）
```

与分治算法的区别：
- 分治：子问题**相互独立**，互不重叠
- 动态规划：子问题**相互重叠**，需要记忆化避免重复求解

---

## 动态规划的三要素

### 1. 最优子结构

问题的最优解包含子问题的最优解。即：大问题的最优解可以由小问题的最优解推导出来。

**例**：最短路径中，若 A→B→C 是最短路径，则 A→B 也必须是从 A 到 B 的最短路径。

### 2. 重叠子问题

递归求解时，相同的子问题被多次计算。

**例**：斐波那契数列 fib(5) = fib(4) + fib(3)，而 fib(4) = fib(3) + fib(2)，fib(3) 被重复计算。

### 3. 无后效性

某阶段的状态一旦确定，不受此后各阶段决策的影响。即**未来的决策只能影响未来的状态，不能改变过去的状态**。

---

## 解题框架

### 五步法

```
1. 定义状态：dp[i] 表示什么？
2. 确定状态转移方程：dp[i] 和 dp[i-1] 等的关系
3. 确定初始值（边界条件）
4. 确定计算顺序（自底向上）
5. 根据题目要求确定最终答案
```

## 经典入门问题

### 问题一：斐波那契数列

**问题**：F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)，求 F(n)

#### 方法一：朴素递归（指数级时间）

```java
public int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}
// 时间复杂度：O(2^n)，大量重复计算
```

#### 方法二：记忆化递归（自顶向下）

```java
public int fib(int n) {
    int[] memo = new int[n + 1];
    Arrays.fill(memo, -1);
    return helper(n, memo);
}

private int helper(int n, int[] memo) {
    if (n <= 1) return n;
    if (memo[n] != -1) return memo[n];
    memo[n] = helper(n - 1, memo) + helper(n - 2, memo);
    return memo[n];
}
// 时间复杂度：O(n)，空间复杂度：O(n)
```

#### 方法三：动态规划（自底向上）

```java
public int fib(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[0] = 0;
    dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];
    }
    return dp[n];
}
// 时间复杂度：O(n)，空间复杂度：O(n)
```

#### 方法四：空间优化

```java
public int fib(int n) {
    if (n <= 1) return n;
    int prev2 = 0, prev1 = 1;
    for (int i = 2; i <= n; i++) {
        int curr = prev1 + prev2;
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
// 时间复杂度：O(n)，空间复杂度：O(1)
```

### 问题二：爬楼梯（LeetCode 70）

**问题**：爬 n 阶楼梯，每次可以爬 1 或 2 阶，有多少种方法？

**分析**：
- 到达第 n 阶，可以从第 n-1 阶爬 1 步，或从第 n-2 阶爬 2 步
- dp[n] = dp[n-1] + dp[n-2]
- 本质就是斐波那契数列！

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int dp1 = 1, dp2 = 2;
    for (int i = 3; i <= n; i++) {
        int dp = dp1 + dp2;
        dp1 = dp2;
        dp2 = dp;
    }
    return dp2;
}
```

### 问题三：最大子数组和（LeetCode 53）

**问题**：找出数组中具有最大和的连续子数组，返回其最大和。

**状态定义**：dp[i] = 以 nums[i] 结尾的连续子数组的最大和

**状态转移**：
- 如果 dp[i-1] > 0，则 dp[i] = dp[i-1] + nums[i]（前面的和有贡献，接着加）
- 如果 dp[i-1] <= 0，则 dp[i] = nums[i]（前面的和是负担，重新开始）

```java
public int maxSubArray(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n];
    dp[0] = nums[0];
    int maxSum = dp[0];
    
    for (int i = 1; i < n; i++) {
        dp[i] = Math.max(dp[i - 1] + nums[i], nums[i]);
        maxSum = Math.max(maxSum, dp[i]);
    }
    return maxSum;
}
// 时间复杂度：O(n)，空间复杂度：O(n)
```

**空间优化**：

```java
public int maxSubArray(int[] nums) {
    int maxSum = nums[0];
    int currSum = nums[0];
    
    for (int i = 1; i < nums.length; i++) {
        currSum = Math.max(currSum + nums[i], nums[i]);
        maxSum = Math.max(maxSum, currSum);
    }
    return maxSum;
}
// 时间复杂度：O(n)，空间复杂度：O(1)
```

### 问题四：打家劫舍（LeetCode 198）

**问题**：一排房子，不能连续偷两间相邻的房子，求最大金额。

**状态定义**：dp[i] = 偷前 i 间房子能获得的最大金额

**状态转移**：
- 偷第 i 间：dp[i] = dp[i-2] + nums[i]
- 不偷第 i 间：dp[i] = dp[i-1]
- dp[i] = max(dp[i-2] + nums[i], dp[i-1])

```java
public int rob(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    
    int[] dp = new int[n];
    dp[0] = nums[0];
    dp[1] = Math.max(nums[0], nums[1]);
    
    for (int i = 2; i < n; i++) {
        dp[i] = Math.max(dp[i - 2] + nums[i], dp[i - 1]);
    }
    return dp[n - 1];
}
```

**空间优化**：

```java
public int rob(int[] nums) {
    int n = nums.length;
    if (n == 1) return nums[0];
    
    int prev2 = nums[0];
    int prev1 = Math.max(nums[0], nums[1]);
    
    for (int i = 2; i < n; i++) {
        int curr = Math.max(prev2 + nums[i], prev1);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}
```

## 两种实现方式对比

| 方式 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 记忆化递归（自顶向下） | 递归 + 缓存 | 代码直观，只计算需要的子问题 | 有递归开销，可能栈溢出 |
| 动态规划（自底向上） | 迭代 + 填表 | 无递归开销，空间可优化 | 需要想清楚填表顺序 |

## 常见 DP 类型预览

| 类型 | 特征 | 代表题目 |
|------|------|----------|
| 线性 DP | 一维状态，线性推进 | 爬楼梯、打家劫舍、最大子数组 |
| 背包 DP | 物品选择 + 容量限制 | 0/1背包、完全背包 |
| 区间 DP | 状态是区间 [l, r] | 戳气球、矩阵链乘法 |
| 序列 DP | 两个序列对比 | LCS、编辑距离 |
| 树形 DP | 在树上做DP | 打家劫舍III |
| 状态压缩 DP | 用位运算表示状态 | 旅行商问题 |

---

## 总结

动态规划的关键：
1. **想清楚 dp 数组的含义**，这是最重要的一步
2. **找到状态转移方程**，即大问题与小问题的关系
3. **处理好边界条件**，初始值往往影响整个结果
4. **考虑空间优化**，很多 DP 可以压缩到 O(1) 空间

> 练习建议：先从线性 DP 入手（爬楼梯、打家劫舍系列），再挑战背包问题和序列 DP
