# 注解和反射的关系是什么？

## 一句话说明（白话）
反射是读取注解信息的主要手段。

## 它解决什么问题 / 为什么重要
让框架在运行时按注解配置行为。

## 核心原理（一步步讲清楚）
- 注解是元数据。
-反射 API读取注解并执行逻辑。

##典型使用场景
- Spring依赖注入。
- ORM 映射。

## 简单例子 /伪代码
```java
clazz.getAnnotation(MyAnno.class)
```

## 常见坑与误区
- Retention不是 RUNTIME 时读不到。
