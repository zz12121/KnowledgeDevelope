# finally 块中的代码一定会执行吗？

## 一句话说明（白话）
大多数情况下会执行，但遇到 JVM直接退出或崩溃时不会。

## 它解决什么问题 / 为什么重要
确保资源释放（关闭流、连接）不被遗漏。

## 核心原理（一步步讲清楚）
- try 无论正常还是异常退出都会进入 finally。
- 遇到 `System.exit()`、JVM 崩溃或断电，finally 不会执行。

##典型使用场景
-关闭文件/数据库连接。
-清理锁或临时资源。

## 简单例子 /伪代码
```java
try { ... } finally { close(); }
```

## 常见坑与误区
- 在 finally里 return 会覆盖 try 的返回值。
