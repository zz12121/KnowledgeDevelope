# SPI 的实现原理是什么？

## 一句话说明（白话）
SPI 是一套“约定配置 +运行时发现”的插件机制。

## 它解决什么问题 / 为什么重要
让接口与实现解耦，扩展功能时无需改主程序。

## 核心原理（一步步讲清楚）
-实现类在 `META-INF/services/接口全名`中声明。
-`ServiceLoader` 扫描 classpath 找到实现。
-使用反射实例化并返回实现列表。

##典型使用场景
- JDBC Driver 加载。
-日志框架、序列化框架的可插拔实现。

## 简单例子 /伪代码
```
META-INF/services/com.demo.Pay
com.demo.impl.AliPay
com.demo.impl.WeChatPay
```

## 常见坑与误区
-配置文件路径/命名错误导致加载失败。
-实现类无无参构造器。
