# equals 为 true，但 hashCode 不同，可能吗？

## 一句话说明（白话）
不可以。equals 为 true 时，hashCode 必须相同。

## 它解决什么问题 / 为什么重要
保证 HashMap/HashSet 正常工作。

## 核心原理（一步步讲清楚）
- Java规范要求：equals 相等 ⇒ hashCode 相等。
- 若违反，会导致集合查找失败或重复。

##典型使用场景
-重写 equals/hashCode 的实体类。

## 简单例子 /伪代码
```java
@Override public boolean equals(Object o) { ... }
@Override public int hashCode() { ... }
```

## 常见坑与误区
-只重写 equals，不重写 hashCode。
