# Java8-22 新特性总结

## 一句话说明（白话）
Java8之后的重点是**语法简化、并发模型升级、性能与安全**。

## 它解决什么问题 / 为什么重要
了解版本差异，方便选型升级与面试回答；避免错用版本特性。

## 核心原理（一步步讲清楚）
- **Java8**：Lambda、Stream、默认方法、`java.time`。
- **Java9-11**：模块系统（JPMS）、HTTP Client、var（11）。
- **Java14-17**：records、sealed classes、switch 表达式、文本块。
- **Java19-21**：虚拟线程（Loom）、结构化并发、Record Patterns。

##典型使用场景
-需要高并发时用虚拟线程（21+）。
-需要更清晰的数据结构时用 record（16+）。
- 字符串模板/多行文本处理用文本块。

## 简单例子 /伪代码
```java
record User(String name, int age) {}
var user = new User("Tom",18);
```

## 常见坑与误区
- 不区分 LTS版本导致线上 JDK 不匹配。
-误把预览特性当稳定特性使用。
