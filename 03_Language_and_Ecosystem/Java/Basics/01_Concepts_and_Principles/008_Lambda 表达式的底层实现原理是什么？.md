# Lambda表达式的底层实现原理是什么？

## 一句话说明（白话）
Lambda不是匿名内部类，而是运行时动态生成的函数对象。

## 它解决什么问题 / 为什么重要
理解性能与运行机制，避免错误的“匿名类等价”理解。

## 核心原理（一步步讲清楚）
- Lambda依赖 **`invokedynamic`** 指令。
-编译器会生成一个私有静态方法保存 Lambda逻辑。
-运行时通过 `LambdaMetafactory`生成实现函数式接口的类实例。

##典型使用场景
-集合处理与函数式编程。
-简化回调与事件处理。

## 简单例子 /伪代码
```java
Runnable r = () -> System.out.println("hi");
```

## 常见坑与误区
-以为 Lambda 就是匿名内部类（底层实现不同）。
-捕获变量过多导致额外开销。
