# Java代码初始化顺序是什么？

## 一句话说明（白话）
静态先于实例，父类先于子类。

## 它解决什么问题 / 为什么重要
避免初始化顺序导致的 bug。

## 核心原理（一步步讲清楚）
1. 父类静态变量/静态代码块
2. 子类静态变量/静态代码块
3. 父类实例变量/构造块
4. 父类构造函数
5. 子类实例变量/构造块
6. 子类构造函数

##典型使用场景
-复杂继承关系的初始化问题。

## 简单例子 /伪代码
- 写继承类观察打印顺序。

## 常见坑与误区
-误以为构造器最先执行。
