#3 *0.1 ==0.3 将会返回什么？true还是 false？

## 一句话说明（白话）
浮点数在计算机里是近似表示的，直接用 `==` 比较经常不相等。

## 它解决什么问题 / 为什么重要
避免把浮点近似当成精确值，导致比较出错或线上 bug。

## 核心原理（一步步讲清楚）
- `double/float` 使用 IEEE754 二进制表示，很多十进制小数无法精确表示。
- `0.1` 本身就是近似值，`3 *0.1` 的结果会是 `0.30000000000000004`之类的近似值。
- `==` 比较的是**精确二进制值**，因此通常返回 `false`。

##典型使用场景
- 金额/比例/精度要求高的计算场景。
-统计与数据处理时的数值比较。

## 简单例子 /伪代码
```java
double x =3 *0.1;
System.out.println(x ==0.3); // false

double eps =1e-10;
System.out.println(Math.abs(x -0.3) < eps); // true
```

## 常见坑与误区
- 用 `==` 比较浮点数；应该用**误差阈值**（epsilon）。
- 金融计算仍用 `double`；应优先 `BigDecimal`。
