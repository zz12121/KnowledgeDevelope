# 什么是多态机制？

## 一句话说明（白话）
多态就是“同一个接口，不同实现”，运行时决定调用哪一个。

## 它解决什么问题 / 为什么重要
提高扩展性和可维护性，减少 if/else 分支。

## 核心原理（一步步讲清楚）
-继承/接口 + 方法重写。
-运行时通过动态绑定选择实际实现。

##典型使用场景
-面向接口编程。
-策略模式、工厂模式。

## 简单例子 /伪代码
```java
Animal a = new Dog();
a.speak();
```

## 常见坑与误区
-以为多态只靠重载（重载是编译期绑定）。

##关联知识
-继承/接口
- 动态绑定

## 延伸阅读（后续补充）
- 面向对象三大特性
