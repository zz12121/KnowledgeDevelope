# volatile关键字的作用是什么？

## 一句话说明（白话）
volatile 保证可见性，禁止指令重排序，但不保证原子性。

## 它解决什么问题 / 为什么重要
解决多线程下“看不到最新值”的问题。

## 核心原理（一步步讲清楚）
-读写 volatile 会建立**内存屏障**。
- 写入立即刷新到主内存，读直接从主内存读取。

##典型使用场景
-状态标志、配置开关。

## 简单例子 /伪代码
```java
volatile boolean running = true;
```

## 常见坑与误区
- 用 volatile 实现计数器（会丢失更新）。
