# 元注解有哪些？各自的作用是什么？

## 一句话说明（白话）
元注解是“注解的注解”，用于约束自定义注解的行为。

## 它解决什么问题 / 为什么重要
定义注解能用在哪、保留多久、是否生成文档。

## 核心原理（一步步讲清楚）
- `@Target`：作用范围。
- `@Retention`：保留策略。
- `@Documented`：是否进 Javadoc。
- `@Inherited`：是否被子类继承。
- `@Repeatable`：可重复注解。

##典型使用场景
- 自定义注解时必配。

## 简单例子 /伪代码
```java
@Retention(RUNTIME)
@Target(TYPE)
public @interface MyAnno {}
```

## 常见坑与误区
- `Retention`选错导致运行期读不到。
