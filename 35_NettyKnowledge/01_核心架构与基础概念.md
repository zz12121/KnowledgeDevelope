# Netty 核心架构与基础概念

## 1. Netty 是什么？

Netty 是一个**基于 Java NIO 的、异步事件驱动的网络通信框架**，由 JBoss 团队开发（现由社区维护），目标是让高性能、高可靠的网络应用开发变得简单高效。

**核心特点：**

- **异步非阻塞**：基于 NIO 的 Selector 模型，少量线程处理大量并发连接
- **事件驱动**：所有 I/O 操作都是异步的，通过事件回调通知结果
- **高度可扩展**：通过 ChannelPipeline + ChannelHandler 责任链模式，灵活组合协议处理逻辑
- **开箱即用**：内置大量编解码器（HTTP、WebSocket、Protobuf、SSL/TLS），不需要从零开发

**主要应用场景：**

- **RPC 框架传输层**：Dubbo、gRPC-Java、Thrift 的底层通信
- **消息中间件**：RocketMQ、Kafka（部分组件）的节点通信
- **IM 系统**：微信、游戏等长连接实时通讯
- **API 网关**：高并发请求代理转发
- **IoT 物联网**：海量设备长连接管理

---

## 2. Netty vs 原生 NIO

原生 NIO 问题太多：Selector.select() 空轮询 Bug、epoll 触发机制复杂、ByteBuffer 读写模式切换繁琐、粘包/拆包需要自己处理……

Netty 封装了这一切：

| 对比维度 | 原生 NIO | Netty |
|---|---|---|
| 开发难度 | 极高，Selector/Channel/Buffer 手动管理 | 简单，核心关注业务 Handler |
| 粘包处理 | 手动实现 | 内置多种解码器 |
| 内存管理 | ByteBuffer API 繁琐 | ByteBuf 双指针设计，池化分配 |
| epoll Bug | 需要手动绕过 | 内置计数检测 + Selector 重建 |
| 线程模型 | 自己实现 Reactor | 成熟的主从 Reactor 模型 |

## 3. Netty 核心组件速览

**Channel**：对 NIO Channel 的封装，抽象了网络连接，提供统一的 I/O 操作 API（bind/connect/read/write/close）。

**EventLoop**：事件循环，Channel 生命周期内的所有 I/O 事件都由绑定的 EventLoop 线程处理，实现无锁化串行设计。

**ChannelPipeline**：Handler 责任链，入站事件从 Head → Tail，出站事件从 Tail → Head。

**ChannelHandler**：业务逻辑单元，分为 `ChannelInboundHandler`（处理入站）和 `ChannelOutboundHandler`（处理出站）。

**ByteBuf**：Netty 的字节容器，双指针（readerIndex/writerIndex）设计，支持池化和直接内存。

**Bootstrap / ServerBootstrap**：启动配置的建造者，串联所有组件。

## 4. 线程模型：主从 Reactor

```
         BossGroup（1个线程）
              |
        accept 新连接
              |
         WorkerGroup（CPU×2 线程）
         /    |    \
    EL-1   EL-2   EL-3
     |       |       |
  CH-1,2  CH-3,4  CH-5,6
```

每个 Channel 终身绑定一个 EventLoop，每个 EventLoop 可负责多个 Channel。EventLoop 线程不能阻塞——I/O 线程与业务线程必须严格隔离。

## 5. Channel 生命周期

```
channelRegistered → channelActive → channelRead（多次）→ channelReadComplete → channelInactive → channelUnregistered
```

| 状态 | 触发时机 |
|---|---|
| `channelRegistered` | Channel 注册到 EventLoop |
| `channelActive` | 连接建立（TCP 三次握手完成） |
| `channelRead` | 有数据到达 |
| `channelReadComplete` | 本次读循环结束 |
| `channelInactive` | 连接断开 |
| `channelUnregistered` | Channel 从 EventLoop 注销 |
