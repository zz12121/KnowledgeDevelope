| [[01_IoC容器源码/01、BeanFactory体系结构]] | BeanFactory/ApplicationContext 接口体系、DefaultListableBeanFactory、三级缓存 | ⭐⭐⭐ |
| [[01_IoC容器源码/02、ApplicationContext启动流程（refresh方法）]] | refresh() 12步全流程、ConfigurationClassPostProcessor、preInstantiateSingletons | ⭐⭐⭐⭐ |

### 核心知识点

- `DefaultListableBeanFactory` 是 Spring 容器的真正实现，几乎所有容器最终都委托给它
- `refresh()` 的 12 步是面试最高频的 Spring 知识点
- 三级缓存解决循环依赖：一级（成品）→ 二级（早期引用）→ 三级（ObjectFactory）

| 文件 | 核心内容 | 难度 |
|------|---------|------|
| [[03_AOP源码/01、AOP代理创建源码]] | AnnotationAwareAspectJAutoProxyCreator、wrapIfNecessary、JDK vs CGLIB 选择、拦截链执行 | ⭐⭐⭐⭐ |
| [[03_AOP源码/02、AOP切点与通知源码]] | AspectJExpressionPointcut、各种 Advice 实现、多切面排序 | ⭐⭐⭐ |

### AOP 核心要点

- 代理创建时机：Bean 初始化完成后，`BeanPostProcessor.postProcessAfterInitialization`
- JDK 代理：有接口时使用；CGLIB：无接口或 `proxyTargetClass=true`
- 内部调用不走代理，可用 `AopContext.currentProxy()` 解决

## 四、事务源码

| 文件 | 核心内容 | 难度 |
|------|---------|------|
| [[04_事务源码/01、@Transactional源码解析]] | TransactionInterceptor、invokeWithinTransaction、7种传播行为源码、回滚判断逻辑、ThreadLocal 事务绑定 | ⭐⭐⭐⭐ |

### 事务失效场景（必背）

| 场景 | 原因 |
|------|------|
| 同类内部调用 | 走 `this.method()`，不经过代理 |
| private 方法 | CGLIB 无法代理 |
| 异常被吞掉 | `completeTransactionAfterThrowing` 收不到异常 |
| checked 异常 | 默认只回滚 `RuntimeException`，需配置 `rollbackFor` |

## 八、源码阅读辅助工具

### 如何搭建 Spring 源码调试环境

```bash
# 克隆官方仓库（Spring 6.x）
git clone https://github.com/spring-projects/spring-framework.git
cd spring-framework
git checkout v6.1.x

# 使用 Gradle 构建（需要 Java 17+）
./gradlew :spring-context:jar
./gradlew :spring-webmvc:jar

# IDEA 导入：File → Open → 选择 build.gradle
```

### 断点调试建议

| 调试目标 | 推荐断点 |
|---------|---------|
| 容器启动流程 | `AbstractApplicationContext.refresh()` |
| Bean 创建过程 | `AbstractAutowireCapableBeanFactory.doCreateBean()` |
| 依赖注入 | `AutowiredAnnotationBeanPostProcessor.postProcessProperties()` |
| AOP 代理创建 | `AbstractAutoProxyCreator.wrapIfNecessary()` |
| 事务开启 | `AbstractPlatformTransactionManager.getTransaction()` |
| 请求处理 | `DispatcherServlet.doDispatch()` |
