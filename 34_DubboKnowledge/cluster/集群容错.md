# 集群容错

## 概述

Dubbo集群容错机制是确保分布式服务稳定性和可用性的关键组件，负责协调服务提供者与消费者之间的通信，处理节点故障，保证服务高可用。

## 核心组件

| 组件 | 说明 |
|------|------|
| **Cluster** | 封装集群容错策略，提供各种容错策略的Invoker代理 |
| **Directory** | 缓存和提供可用的服务提供者Invoker列表 |
| **Router** | 根据预设规则从Invoker列表中筛选符合条件的提供者 |
| **LoadBalance** | 从路由后的Invoker列表中选择一个进行调用 |
| **Invoker** | 最终执行远程调用的组件 |

## 调用流程

```
Cluster Invoker 
    → Directory.list() 
        → RouterChain.route() 
            → LoadBalance.select() 
                → DubboInvoker（网络传输）
```

## 集群容错策略

### Failover（失败自动切换）

默认策略。调用失败后，自动切换到其他提供者重试：

```java
// 配置
<dubbo:reference cluster="failover" retries="2"/>
```

适用于幂等操作，如读服务。

### Failfast（快速失败）

调用失败后立即报错，不重试：

```java
// 配置
<dubbo:reference cluster="failfast"/>
```

适用于非幂等操作，如新增订单。

### Failsafe（失败安全）

出现异常时直接忽略，仅记录日志：

```java
// 配置
<dubbo:reference cluster="failsafe"/>
```

适用于写入审计日志等场景。

### Failback（失败自动恢复）

失败请求记录到队列，定时重发：

```java
// 配置
<dubbo:reference cluster="failback"/>
```

适用于重要操作，不允许失败。

### Forking（并行调用）

并行调用多个提供者，谁先返回就使用谁的结果：

```java
// 配置
<dubbo:reference cluster="forking" forks="3"/>
```

适用于实时性要求高的场景。

## 核心组件详解

### Directory

**RegistryDirectory**是核心实现，动态监听注册中心变化，通过`routerChain.route()`获取经过路由规则过滤后的Invoker列表。

### Router

以**ConditionRouter**为例，解析`when`（条件判断）和`then`（目标选择）规则。

示例规则：
```
host = 1.1.1.* & host != 1.1.1.2 & method=sayHello
```

### LoadBalance

支持以下算法：
- **RandomLoadBalance**：基于权重的随机（默认）
- **RoundRobinLoadBalance**：基于权重的轮询
- **LeastActiveLoadBalance**：最少活跃数
- **ConsistentHashLoadBalance**：一致性哈希

### Cluster

`AbstractCluster`封装通用逻辑，通过`doJoin()`创建具体容错策略的Invoker。

## 总结

Dubbo集群容错机制通过Directory、Router、Cluster、LoadBalance、Invoker五大核心组件的协作，提供了高性能、高可用的服务调用能力。基于SPI的扩展性设计，使开发者可以自定义各个组件。
