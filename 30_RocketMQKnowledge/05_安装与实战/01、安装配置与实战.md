# RocketMQ 安装与实战

## 一、环境准备

### 1.1 JDK 安装

RocketMQ 基于 Java 开发，要求 JDK 1.8 及以上版本。

```bash
# 验证 Java 环境
java -version

# 推荐使用 JDK 8 或 JDK 11
```

### 1.2 下载 RocketMQ

```
官网：https://rocketmq.apache.org/
下载地址：https://rocketmq.apache.org/download/
```

**版本选择建议：**
- 生产环境推荐使用 4.9.x 稳定版
- 5.x 版本支持更多新特性（如延迟消息自定义、存算分离）

## 二、安装与启动

### 2.1 解压 RocketMQ

```bash
# 解压到指定目录
tar -xvf rocketmq-all-4.9.4-bin-release.zip -C /usr/local/

# 或者使用 tar.gz 版本
tar -xzf rocketmq-4.9.4.tar.gz -C /usr/local/
```

### 2.2 启动 NameServer

```bash
# Linux/Mac
cd /usr/local/rocketmq-4.9.4
./bin/mqnamesrv

# Windows
.\bin\mqnamesrv.cmd

# 后台启动
nohup ./bin/mqnamesrv > /dev/null 2>&1 &

# 验证启动成功
tail -f ~/logs/rocketmqlogs/namesrv.log
# 看到 "Name Server startup successful" 表示启动成功
```

**NameServer 默认端口：** 9876

### 2.3 启动 Broker

```bash
# Linux/Mac（默认内存较大，需要先调整）
# 编辑 runbroker.sh，修改 JVM 内存
vi bin/runbroker.sh

# 修改 JAVA_OPT 的内存配置（根据服务器配置调整）
JAVA_OPT="${JAVA_OPT} -Xms512m -Xmx512m"

# 启动 Broker
./bin/mqbroker -n localhost:9876

# 后台启动
nohup ./bin/mqbroker -n localhost:9876 > /dev/null 2>&1 &

# Windows
.\bin\mqbroker.cmd -n localhost:9876
```

**常见启动参数：**

| 参数 | 说明 | 示例 |
|------|------|------|
| -n | NameServer 地址 | -n localhost:9876 |
| -c | 配置文件路径 | -c confbroker.properties |
| -m | 打印配置信息 | -m |
| -h | 打印帮助 | -h |

### 2.4 测试消息发送与消费

```bash
# 设置环境变量
export NAMESRV_ADDR=localhost:9876

# 测试消息生产者
./bin/tools.sh org.apache.rocketmq.example.quickstart.Producer

# 测试消息消费者（新开终端）
./bin/tools.sh org.apache.rocketmq.example.quickstart.Consumer
```

### 2.5 关闭服务

```bash
# 关闭 Broker
./bin/mqshutdown broker

# 关闭 NameServer
./bin/mqshutdown namesrv
```

## 三、Broker 配置文件

### 3.1 基础配置

创建 `broker.conf` 配置文件：

```properties
# Broker 名称
brokerName = broker-a

# Broker ID，0 表示 Master，大于 0 表示 Slave
brokerId = 0

# NameServer 地址，多个用分号分隔
namesrvAddr = localhost:9876

# 监听端口
listenPort = 10911

# 存储路径
storePathRootDir = /usr/local/rocketmq/store
storePathCommitLog = /usr/local/rocketmq/store/commitlog

# 消息最大大小（字节）
maxMessageSize = 4194304

# 刷盘方式：ASYNC_FLUSH（异步）/SYNC_FLUSH（同步）
flushDiskType = ASYNC_FLUSH

# 删除消息时间（默认凌晨4点）
deleteWhen = 04

# 文件保留时间（48小时）
fileReservedTime = 48
```

### 3.2 主从配置

**Master Broker 配置：**

```properties
brokerName = broker-a
brokerId = 0
brokerRole = SYNC_MASTER
flushDiskType = SYNC_FLUSH
```

**Slave Broker 配置：**

```properties
brokerName = broker-a
brokerId = 1
brokerRole = SLAVE
flushDiskType = SYNC_FLUSH
```

### 3.3 启用 SQL 过滤

```properties
# 启用 SQL 过滤功能
enablePropertyFilter = true
```

## 四、RocketMQ Dashboard

### 4.1 下载与构建

```bash
# 克隆项目
git clone https://github.com/apache/rocketmq-dashboard.git

# 进入目录
cd rocketmq-dashboard

# 修改配置（可选）
# 编辑 src/main/resources/application.yml
# 配置 namesrvAddr

# Maven 构建
mvn clean package -DskipTests

# 打包成功后，在 target 目录找到 jar 包
```

### 4.2 配置 application.yml

```yaml
server:
  port: 8080

rocketmq:
  config:
    namesrvAddr: localhost:9876
    # 如果有 ACL 认证，配置如下：
    # accessKey: rocketmq
    # secretKey: 12345678
```

### 4.3 启动 Dashboard

```bash
# 直接运行
java -jar rocketmq-dashboard-1.0.0.jar

# 后台运行
nohup java -jar rocketmq-dashboard-1.0.0.jar > /dev/null 2>&1 &

# 指定配置文件
java -jar rocketmq-dashboard-1.0.0.jar --spring.config.addition-location=application.yml
```

### 4.4 访问 Dashboard

```
地址：http://localhost:8080
默认用户名：admin
默认密码：admin
```

**Dashboard 功能：**
- Topic 管理（创建、删除、查看）
- Producer/Consumer 监控
- 消息查询
- 消费者组状态
- 消息轨迹

## 五、Spring Boot 集成

### 5.1 添加依赖

```xml
<dependency>
    <groupId>org.apache.rocketmq</groupId>
    <artifactId>rocketmq-spring-boot-starter</artifactId>
    <version>2.2.3</version>
</dependency>
```

### 5.2 配置 application.yml

```yaml
rocketmq:
  # NameServer 地址
  name-server: localhost:9876
  # 生产者配置
  producer:
    group: my-producer-group
    # 发送超时时间（毫秒）
    send-message-timeout: 3000
    # 压缩消息阈值（字节）
    compress-message-body-threshold: 4096
    # 最大消息大小（字节）
    max-message-size: 4194304
    # 重试次数
    retry-times-when-send-failed: 2
    # 异步发送重试次数
    retry-times-when-send-async-failed: 2
  # 消费者配置
  consumer:
    group: my-consumer-group
    # 监听器类型：DEFAULT（并发）/ORDERLY（顺序）
    listener: DEFAULT
    # 消息并发数
    consume-thread-min: 20
    consume-thread-max: 64
    # 批量拉取消息数
    consume-message-batch-max-size: 1
    # 拉取超时时间
    pull-timeout: 15000
```

### 5.3 生产者示例

```java
@RestController
public class RocketMQController {

    @Autowired
    private RocketMQTemplate rocketMQTemplate;

    // 同步发送
    @GetMapping("/send/{message}")
    public String sendMessage(@PathVariable String message) {
        SendResult result = rocketMQTemplate.syncSend(
            "TopicTest", 
            MessageBuilder.withPayload(message).build()
        );
        return "发送成功: " + result.getMsgId();
    }

    // 异步发送
    @GetMapping("/send/async/{message}")
    public String sendAsync(@PathVariable String message) {
        rocketMQTemplate.asyncSend(
            "TopicTest",
            MessageBuilder.withPayload(message).build(),
            new SendCallback() {
                @Override
                public void onSuccess(SendResult sendResult) {
                    System.out.println("发送成功");
                }
                @Override
                public void onException(Throwable e) {
                    System.out.println("发送失败: " + e.getMessage());
                }
            }
        );
        return "异步发送中...";
    }

    // 单向发送
    @GetMapping("/send/oneway/{message}")
    public String sendOneway(@PathVariable String message) {
        rocketMQTemplate.sendOneWay(
            "TopicTest",
            MessageBuilder.withPayload(message).build()
        );
        return "单向发送完成";
    }

    // 发送延迟消息
    @GetMapping("/send/delay/{message}")
    public String sendDelay(@PathVariable String message) {
        // 延迟等级：1s 5s 10s 30s 1m 2m 3m 4m 5m 6m 7m 8m 9m 10m 20m 30m 1h 2h
        rocketMQTemplate.syncSend(
            "TopicTest",
            MessageBuilder.withPayload(message).build(),
            2000,  // 延迟2秒（对应延迟等级2）
            3      // 超时时间
        );
        return "延迟消息发送完成";
    }

    // 发送事务消息
    @GetMapping("/send/transaction/{message}")
    public String sendTransaction(@PathVariable String message) {
        TransactionSendResult result = rocketMQTemplate.sendMessageInTransaction(
            "TopicTest",
            MessageBuilder.withPayload(message).build(),
            "myTransactionArg"
        );
        return "事务消息发送完成: " + result.getLocalTransactionState();
    }
}
```

### 5.4 消费者示例

```java
@Component
public class RocketMQConsumer {

    // 监听普通消息（并发消费）
    @RocketMQMessageListener(
        topic = "TopicTest",
        consumerGroup = "my-consumer-group",
        selectorExpression = "TagA || TagB"  // Tag 过滤
    )
    public class MyConsumer implements RocketMQListener<String> {
        @Override
        public void onMessage(String message) {
            System.out.println("收到消息: " + message);
        }
    }

    // 监听顺序消息
    @RocketMQMessageListener(
        topic = "TopicTest",
        consumerGroup = "my-order-consumer-group",
        consumeMode = ConsumeMode.ORDERLY  // 顺序消费
    )
    public class OrderConsumer implements RocketMQListener<String> {
        @Override
        public void onMessage(String message) {
            System.out.println("收到顺序消息: " + message);
        }
    }

    // SQL 过滤消费
    @RocketMQMessageListener(
        topic = "TopicTest",
        consumerGroup = "my-sql-consumer-group",
        sqlFilter = true,
        sqlFilterExpression = "age > 18"  // SQL 条件
    )
    public class SqlConsumer implements RocketMQListener<String> {
        @Override
        public void onMessage(String message) {
            System.out.println("收到SQL过滤消息: " + message);
        }
    }
}
```

## 六、集群搭建

### 6.1 双 Master 双 Slave 集群

**机器规划：**

| 机器 | Broker 角色 | Broker Name |
|------|-------------|-------------|
| 192.168.1.10 | Master-1 | broker-a |
| 192.168.1.11 | Slave-1 | broker-a |
| 192.168.1.12 | Master-2 | broker-b |
| 192.168.1.13 | Slave-2 | broker-b |

**Master-1 配置（broker-a-master.properties）：**

```properties
brokerClusterName = RocketMQCluster
brokerName = broker-a
brokerId = 0
namesrvAddr = 192.168.1.10:9876;192.168.1.12:9876
listenPort = 10911
storePathRootDir = /usr/local/rocketmq/store
brokerRole = SYNC_MASTER
flushDiskType = SYNC_FLUSH
```

**Slave-1 配置（broker-a-slave.properties）：**

```properties
brokerClusterName = RocketMQCluster
brokerName = broker-a
brokerId = 1
namesrvAddr = 192.168.1.10:9876;192.168.1.12:9876
listenPort = 10911
storePathRootDir = /usr/local/rocketmq/store
brokerRole = SLAVE
```

### 6.2 启动集群

```bash
# 在各台机器上启动 Broker
./bin/mqbroker -c conf/broker-a-master.properties

./bin/mqbroker -c conf/broker-a-slave.properties

# 验证集群状态
./bin/mqadmin clusterList -n localhost:9876
```

## 七、常用运维命令

### 7.1 Topic 操作

```bash
# 创建 Topic
./bin/mqadmin updateTopic -n localhost:9876 -c DefaultCluster -t MyTopic

# 查看 Topic 列表
./bin/mqadmin topicList -n localhost:9876

# 查看 Topic 路由信息
./bin/mqadmin topicRoute -n localhost:9876 -t MyTopic

./bin/mqadmin topicStatus -n localhost:9876 -t MyTopic
```

```bash
# 查看消费组
./bin/mqadmin consumerList -n localhost:9876

# 查看消费进度
./bin/mqadmin consumerProgress -n localhost:9876 -g MyConsumerGroup

# 查看消息堆积
./bin/mqadmin consumerProgress -n localhost:9876 -g MyConsumerGroup
```

```bash

./bin/mqadmin queryMsgById -n localhost:9876 -i "MessageID"

./bin/mqadmin queryMsgByKey -n localhost:9876 -t MyTopic -k "myKey"
```

## 八、RocketMQ vs Kafka 对比

### 8.1 为什么 Kafka 不能满足需求

**Kafka 的局限性：**

| 问题 | 说明 |
|------|------|
| 分区限制 | 写入并行性受分区数限制，分区越多，磁盘随机写增加，性能下降 |
| 延迟较高 | 专为日志设计，对低延迟场景不友好 |
| 事务支持弱 | 与外部事务协调复杂 |
| 顺序消息 | 分区有序但主节点宕机可能短暂乱序 |

**阿里业务需求：**
- 电商交易、金融支付需要严格的事务消息
- 高可靠、低延迟的消息传递
- 海量队列支持能力

### 8.2 RocketMQ 的改进

| 特性 | Kafka | RocketMQ |
|------|-------|----------|
| 存储模型 | 多分区，每个分区独立文件 | 统一 CommitLog + 消费队列 |
| 队列数量 | 过多分区影响性能 | 支持数万级队列 |
| 事务消息 | 外部协调复杂 | 原生支持（半消息机制） |
| 延迟消息 | 不支持 | 18 级固定延迟 |
| 消息追踪 | 需外部组件 | 原生消息轨迹 |
