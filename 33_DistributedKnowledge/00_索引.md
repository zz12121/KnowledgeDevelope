## 知识图谱

```
分布式理论 ──────────────────────────────────────────────
  CAP (P必须保留，C vs A 二选一)
    ├── ZooKeeper = CP（牺牲可用性保证一致性）
    └── Eureka = AP（牺牲一致性保证可用性）
  BASE（AP 系统的工程延伸）
    ├── 基本可用（Basically Available）
    ├── 软状态（Soft State）
    └── 最终一致性（Eventually Consistent）
  PACELC（CAP 扩展，增加延迟 vs 一致性权衡）

分布式共识 ──────────────────────────────────────────────
  Paxos（理论正确但难实现）
  Raft（工程友好，强 Leader）─→ etcd / TiKV / Apache Ratis
  ZAB（ZooKeeper 专用，4阶段协议）

分布式存储 ──────────────────────────────────────────────
  一致性哈希（虚拟节点解决数据倾斜）
  分库分表 ─→ ShardingSphere（5步：解析→路由→改写→执行→归并）
  分布式 ID ─→ 雪花算法（1+41+5+5+12）/ 美团 Leaf

分布式事务 ──────────────────────────────────────────────
  2PC / TCC / Saga / 本地消息表
  Seata（AT=零侵入 / TCC / XA / Saga）

分布式服务治理 ──────────────────────────────────────────
  注册发现 ─→ [[27_SpringCloudKnowledge]]
  负载均衡 ─→ 四层（LVS）/ 七层（Nginx）/ 客户端（Ribbon）
  熔断限流 ─→ Sentinel / Hystrix
  配置中心 ─→ Apollo / Nacos

分布式缓存 ─→ [[23_RedisKnowledge]]
消息队列   ─→ [[28_RabbitMQKnowledge]] / [[30_RocketMQKnowledge]] / [[31_KafkaKnowledge]]
```
