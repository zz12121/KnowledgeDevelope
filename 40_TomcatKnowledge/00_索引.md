## 核心知识点速记

### Tomcat 组件层次（口诀：一服双器多应用）
```
Server（一个进程）
  Service（一个 Service，名叫 Catalina）
    Connector × N（连接器，不同端口/协议）
    Engine（一个引擎）
      Host × N（虚拟主机）
        Context × N（Web 应用）
          Wrapper（Servlet）
```

### NIO 线程模型（三类线程）
```
Acceptor（1个）  → 接受TCP连接，注册到Poller
Poller（CPU核数）→ Selector轮询就绪事件，提交到Worker
Worker（线程池）→ 实际解析HTTP并调用Servlet
```

### 关键参数速查
| 参数 | 默认值 | 说明 |
|------|--------|------|
| maxThreads | 200 | Worker线程池最大线程数 |
| maxConnections | 10000 | NIO下最大并发连接 |
| acceptCount | 100 | OS级别等待队列长度 |
| connectionTimeout | 20000ms | 等待HTTP请求头超时 |

### Tomcat 类加载器（打破双亲委派的原因）
- 每个 Context 独立 WebappClassLoader → 应用隔离
- WebappClassLoader 先加载自己的类（WEB-INF/classes 优先）→ 允许覆盖容器版本
- JVM 核心类（java.*）始终由 Bootstrap 加载 → 安全红线

### Spring Boot 嵌入式 Tomcat
```yaml
server:
  tomcat:
    threads:
      max: 400
      min-spare: 50
    max-connections: 10000
    accept-count: 200
  shutdown: graceful
  http2:
    enabled: true
```

### 502 vs 504 与 Tomcat 的关系
- 502 Bad Gateway：Nginx 收到了 Tomcat 的响应但格式有问题，或 Tomcat 直接拒绝了连接（端口没监听/线程池满返回连接重置）
- 504 Gateway Timeout：Nginx 等待 Tomcat 响应超时（Tomcat 在处理但太慢）
- Tomcat 线程池满 + `acceptCount` 队列也满 → Nginx 得到 Connection refused → 502
