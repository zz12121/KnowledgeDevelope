# Tomcat 核心架构与组件

## 一、整体架构概览

Tomcat 是 Apache 基金会开源的 Servlet 容器，实现了 Servlet 和 JSP 规范。它的架构采用**分层组件树**设计，每个组件实现 `Lifecycle` 接口，支持统一的生命周期管理。

```
┌─────────────────────────────────────────────────────────┐
│                     Server                              │
│  ┌──────────────────────────────────────────────────┐   │
│  │                   Service                        │   │
│  │  ┌────────────┐  ┌────────────────────────────┐  │   │
│  │  │ Connector  │  │         Container          │  │   │
│  │  │ (HTTP:8080)│  │  ┌─────────────────────┐  │  │   │
│  │  │ Connector  │  │  │       Engine        │  │  │   │
│  │  │ (AJP:8009) │  │  │  ┌───────────────┐  │  │  │   │
│  │  └─────┬──────┘  │  │  │     Host      │  │  │  │   │
│  │        │          │  │  │  ┌─────────┐  │  │  │  │   │
│  │  CoyoteAdapter    │  │  │  │ Context │  │  │  │  │   │
│  │        │          │  │  │  │ Context │  │  │  │  │   │
│  │        └──────────┼──┼──┘  └─────────┘  │  │  │  │   │
│  │                   │  └─────────────────────┘  │  │   │
│  │                   └────────────────────────────┘  │   │
│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

## 二、核心组件详解

### 2.1 Connector（连接器）

Connector 负责监听端口、接受连接、解析协议，将请求数据转换成 `HttpServletRequest` 对象：

```xml
<!-- HTTP/1.1 NIO Connector（默认，推荐）-->
<Connector port="8080" 
           protocol="org.apache.coyote.http11.Http11NioProtocol"
           maxThreads="400"
           minSpareThreads="50"
           maxConnections="10000"
           acceptCount="200"
           connectionTimeout="20000"
           keepAliveTimeout="15000"
           compression="on"
           compressionMinSize="1024"
           URIEncoding="UTF-8" />

<!-- HTTPS NIO Connector -->
<Connector port="8443"
           protocol="org.apache.coyote.http11.Http11NioProtocol"
           SSLEnabled="true">
    <UpgradeProtocol className="org.apache.coyote.http2.Http2Protocol" />
    <SSLHostConfig protocols="TLSv1.2+TLSv1.3">
        <Certificate certificateFile="conf/cert.pem"
                     certificateKeyFile="conf/key.pem" />
    </SSLHostConfig>
</Connector>
```

**三种 I/O 模式对比**：
- `Http11NioProtocol`（默认）：Java NIO，Acceptor+Poller+Worker 三类线程，高并发首选
- `Http11Nio2Protocol`：Java AIO（异步 I/O），理论最优，实际收益有限
- `Http11AprProtocol`：Native APR 库，SSL 性能最佳，需安装 tomcat-native

### 2.2 Container（容器）层次

```
Engine  — 顶层容器，名字 Catalina，是所有 Host 的父
  Host  — 虚拟主机，对应一个域名（默认 localhost）
    Context — 一个 Web 应用（对应一个 war 或目录）
      Wrapper — 封装一个 Servlet，管理 Servlet 生命周期
```

### 2.3 Pipeline + Valve（管道-阀门）

每个容器层次都有独立的 Pipeline，请求在 Pipeline 中经过 Valve 链处理：

```
Engine Pipeline
  StandardEngineValve（必须是最后一个）
    ↓
Host Pipeline  
  AccessLogValve → ErrorReportValve → StandardHostValve
    ↓
Context Pipeline
  StandardContextValve
    ↓
Wrapper Pipeline
  StandardWrapperValve → 创建 FilterChain → 调用 Servlet.service()
```

**常用内置 Valve**：
```xml
<!-- 访问日志（每个 Host 下配置）-->
<Valve className="org.apache.catalina.valves.AccessLogValve"
       directory="logs" prefix="access_log" suffix=".txt"
       pattern="%h %l %u %t &quot;%r&quot; %s %b %D" />

<!-- IP 白名单/黑名单 -->
<Valve className="org.apache.catalina.valves.RemoteAddrValve"
       allow="10\.\d+\.\d+\.\d+|192\.168\..*" />

<!-- 卡住请求检测 -->
<Valve className="org.apache.catalina.valves.StuckThreadDetectionValve"
       threshold="60" />

<!-- 速率限制 -->
<Valve className="org.apache.catalina.valves.RemoteIpValve"
       remoteIpHeader="x-forwarded-for"
       protocolHeader="x-forwarded-proto" />
```

## 三、类加载器体系

### 3.1 类加载器层次结构

```
Bootstrap ClassLoader（JVM 内置）
    ↑
Extension ClassLoader（jre/lib/ext/）
    ↑
System ClassLoader（CLASSPATH）
    ↑
CommonClassLoader（$CATALINA_HOME/lib/ 所有 jar）
    ↑────────┬──────────────┐
CatalinaClassLoader    SharedClassLoader
（Tomcat 内部类）      （应用共享库）
                            ↑
                     WebappClassLoader × N
                     （每个 Context 独立）
                     加载 WEB-INF/classes/ + WEB-INF/lib/
```

### 3.2 WebappClassLoader 的加载顺序（打破双亲委派）

```java
// WebappClassLoader.loadClass() 的核心逻辑（简化）
public Class<?> loadClass(String name) {
    // 1. 检查 JVM 缓存
    Class c = findLoadedClass(name);
    if (c != null) return c;
    
    // 2. JVM 核心类（java.*）委托 Bootstrap（不可打破）
    if (name.startsWith("java.") || name.startsWith("javax.")) {
        return parent.loadClass(name);
    }
    
    // 3. 先看 WEB-INF/classes/ 和 WEB-INF/lib/（优先自己！打破双亲委派）
    c = findClass(name);   // 在 webapp 中查找
    if (c != null) return c;
    
    // 4. 再委托父加载器（Common）
    return parent.loadClass(name);
}
```

**打破双亲委派的原因**：
- 应用隔离：同一 Tomcat 中的不同应用可以用不同版本的同一库
- Servlet 规范：应用的类应优先于容器类

## 四、配置速查

### server.xml 核心结构

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Server port="8005" shutdown="SHUTDOWN">
  <Listener className="org.apache.catalina.startup.VersionLoggerListener" />
  <Listener className="org.apache.catalina.core.AprLifecycleListener" SSLEngine="on" />
  
  <GlobalNamingResources>
    <Resource name="UserDatabase" auth="Container"
              type="org.apache.catalina.UserDatabase" ... />
  </GlobalNamingResources>

  <Service name="Catalina">
    <Connector port="8080" protocol="org.apache.coyote.http11.Http11NioProtocol"
               maxThreads="400" minSpareThreads="50"
               maxConnections="10000" acceptCount="200"
               connectionTimeout="20000" URIEncoding="UTF-8" />

    <Engine name="Catalina" defaultHost="localhost">
      <Realm className="org.apache.catalina.realm.LockOutRealm">
        <Realm className="org.apache.catalina.realm.UserDatabaseRealm"
               resourceName="UserDatabase"/>
      </Realm>

      <Host name="localhost" appBase="webapps" unpackWARs="true" autoDeploy="false">
        <Valve className="org.apache.catalina.valves.AccessLogValve"
               directory="logs" prefix="localhost_access_log" suffix=".txt"
               pattern="%h %l %u %t &quot;%r&quot; %s %b %D" />
      </Host>
    </Engine>
  </Service>
</Server>
```
