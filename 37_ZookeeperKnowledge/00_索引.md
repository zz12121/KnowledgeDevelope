## 核心知识点速记

### ZAB 协议两阶段

```
崩溃恢复：Leader选举（epoch→zxid→myid）→ 数据同步（DIFF/TRUNC/SNAP）→ 超半数完成→UPTODATE
消息广播：写请求→分配ZXID→广播Proposal→等半数ACK→提交→广播COMMIT
```

### FastLeaderElection 选举规则

```
优先级：epoch > zxid > myid
收到更优的票 → 更新自己的票并重新广播 → 直到某节点获得 >N/2 票
```

### ZNode 类型记忆

```
持久 × 无序 = PERSISTENT         → 配置存储
持久 × 有序 = PERSISTENT_SEQ     → 全局唯一ID
临时 × 无序 = EPHEMERAL          → 服务注册/Leader抢注
临时 × 有序 = EPHEMERAL_SEQ      → 分布式锁（最重要）
```

### 分布式锁原理（记这3步）

```
1. 创建临时顺序节点（/locks/lock-000000000X）
2. 判断自己是否是最小节点：是→获得锁；否→监听前一个节点
3. 前一个节点删除 → 重新判断 → 获得锁（天然公平，无羊群效应）
```

### CAP 选型口诀

```
强一致优先（分布式锁/选举）→ ZooKeeper（CP）
高可用优先（服务注册）      → Nacos（AP/CP可切换）
云原生/K8s                  → etcd（Raft，CNCF）
老系统 Dubbo                → ZooKeeper（默认）
```

### 集群奇数原则

```
偶数台 = 浪费（4台 ≈ 3台，容错能力相同）
推荐：3台（小规模）/ 5台（生产）/ 7台（高可用）
```

### 写性能瓶颈三板斧

```
单 Leader 写 → ZAB 广播 → fsync 磁盘
↑
解决：SSD 磁盘（fsync 快）/ Observer 扩读 / 减少不必要写 ZK
```
