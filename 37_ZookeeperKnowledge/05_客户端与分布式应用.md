## 一、Curator 客户端（推荐）

**引入依赖：**
```xml
<dependency>
    <groupId>org.apache.curator</groupId>
    <artifactId>curator-framework</artifactId>
    <version>5.5.0</version>
</dependency>
<dependency>
    <groupId>org.apache.curator</groupId>
    <artifactId>curator-recipes</artifactId>
    <version>5.5.0</version>
</dependency>
```

**创建客户端：**
```java
CuratorFramework client = CuratorFrameworkFactory.builder()
    .connectString("zoo1:2181,zoo2:2181,zoo3:2181")
    .retryPolicy(new ExponentialBackoffRetry(1000, 3))
    .sessionTimeoutMs(30000)
    .connectionTimeoutMs(5000)
    .namespace("myapp")  // 等效于 Chroot，所有操作都在 /myapp 下
    .build();
client.start();

// 监听连接状态
client.getConnectionStateListenable().addListener((c, state) -> {
    if (state == ConnectionState.LOST) {
        log.error("Session 超时！重新初始化 ZK 资源");
        reinitializeZkResources();
    }
});
```

## 二、Curator 常用 API

```java
// ===== 增删改查 =====
// 创建节点（自动创建父路径）
client.create().creatingParentsIfNeeded()
    .withMode(CreateMode.EPHEMERAL_SEQUENTIAL)
    .forPath("/locks/lock-", data);

// 读取数据
Stat stat = new Stat();
byte[] data = client.getData().storingStatIn(stat).forPath("/config/db");

// 更新（CAS 乐观锁）
client.setData().withVersion(stat.getVersion()).forPath("/config/db", newData);

// 删除（递归删除子节点）
client.delete().guaranteed()  // guaranteed：即使失败也会重试，确保删除成功
    .deletingChildrenIfNeeded()
    .forPath("/app");

// 检查存在
Stat exists = client.checkExists().forPath("/app/node");

// 获取子节点
List<String> children = client.getChildren().forPath("/app");

// ===== 批量原子操作 =====
client.transaction().forOperations(
    client.transactionOp().create().forPath("/a", "1".getBytes()),
    client.transactionOp().create().forPath("/b", "2".getBytes()),
    client.transactionOp().setData().forPath("/c", "3".getBytes())
);
```

## 三、分布式锁实现

### 3.1 Curator InterProcessMutex（推荐）

```java
InterProcessMutex lock = new InterProcessMutex(client, "/locks/order");

// 尝试获取锁，等待最多30秒
if (lock.acquire(30, TimeUnit.SECONDS)) {
    try {
        // 临界区业务逻辑
        processOrder(orderId);
    } finally {
        lock.release();  // 必须在 finally 中释放
    }
} else {
    throw new RuntimeException("获取分布式锁超时");
}
```

### 3.2 原理（临时顺序节点 + 前驱监听）

```
所有客户端在 /locks/ 下创建临时顺序节点：
/locks/lock-0000000001  (Client A)
/locks/lock-0000000002  (Client B)  ← 监听 lock-0000000001
/locks/lock-0000000003  (Client C)  ← 监听 lock-0000000002

Client A 持锁执行业务
Client A 释放（删除 lock-0000000001）
→ 只唤醒 Client B（监听前驱，无羊群效应）
→ Client B 成为最小节点，获取锁
```

### 3.3 可用的锁类型

| 类型 | 说明 |
|------|------|
| `InterProcessMutex` | 可重入互斥锁（最常用） |
| `InterProcessReadWriteLock` | 读写锁（读共享，写独占） |
| `InterProcessSemaphoreMutex` | 不可重入互斥锁 |
| `InterProcessMultiLock` | 多节点联合锁 |

## 四、服务注册与发现

```java
// ===== Provider：服务注册 =====
public void register(String serviceName, String address) throws Exception {
    String path = "/services/" + serviceName + "/" + address;
    // 临时节点：Provider 宕机后节点自动消失
    client.create().creatingParentsIfNeeded()
        .withMode(CreateMode.EPHEMERAL)
        .forPath(path, address.getBytes());
}

// ===== Consumer：服务发现（持续订阅） =====
public void subscribeService(String serviceName, Consumer<List<String>> onChange) {
    PathChildrenCache cache = new PathChildrenCache(
        client, "/services/" + serviceName, false);
    cache.getListenable().addListener((c, event) -> {
        if (event.getType() == PathChildrenCacheEvent.Type.CHILD_ADDED
            || event.getType() == PathChildrenCacheEvent.Type.CHILD_REMOVED) {
            List<String> addresses = client.getChildren()
                .forPath("/services/" + serviceName);
            onChange.accept(addresses);
        }
    });
    cache.start(StartMode.BUILD_INITIAL_CACHE);
}
```

## 五、配置中心

```java
// 发布配置
public void publishConfig(String key, String value) throws Exception {
    String path = "/config/" + key;
    if (client.checkExists().forPath(path) == null) {
        client.create().creatingParentsIfNeeded()
            .withMode(CreateMode.PERSISTENT).forPath(path, value.getBytes());
    } else {
        client.setData().forPath(path, value.getBytes());
    }
}

// 订阅配置（NodeCache，无间隙）
public void subscribeConfig(String key, BiConsumer<String, String> onChange) throws Exception {
    String path = "/config/" + key;
    NodeCache cache = new NodeCache(client, path);
    cache.getListenable().addListener(() -> {
        String newValue = new String(cache.getCurrentData().getData());
        onChange.accept(key, newValue);
    });
    cache.start(true);  // true：立即加载当前值
}
```

## 六、Leader 选举

```java
// Curator LeaderSelector（推荐）
LeaderSelector selector = new LeaderSelector(client, "/election/leader",
    new LeaderSelectorListenerAdapter() {
        @Override
        public void takeLeadership(CuratorFramework client) throws Exception {
            log.info("成为 Leader，开始执行 Leader 任务");
            // 此方法返回时，自动放弃 Leader
            // 通常在这里 block，直到需要放弃时抛出异常或外部中断
            doLeaderWork();
        }
    });
selector.autoRequeue();  // 放弃后自动重新排队，持续竞选
selector.start();

// 查询当前 Leader
Participant leader = selector.getLeader();
```

## 八、ZooKeeper vs Redis 分布式锁

| 维度 | ZooKeeper | Redis（Redisson） |
|------|-----------|-----------------|
| 锁自动释放 | Session 超时自动删除，天然防死锁 | 必须设置 TTL，存在误释放风险 |
| 可靠性 | 高（CP，多数派写入） | 中（主从切换可能锁丢失） |
| 性能 | 低（ZAB 两阶段） | 高（内存操作，10w+ QPS） |
| 公平性 | 天然公平（顺序节点） | 默认不公平 |
| 选型建议 | 金融/订单强一致场景 | 高并发/性能优先场景 |
