### 十、模块与扩展类

---

###### 1. Nginx 有哪些常用的第三方模块？

| 模块 | 功能 | 使用场景 |
|------|------|---------|
| `nginx-module-vts` | 流量状态统计（VirtualHost Traffic Status） | 监控每个 upstream/location 的请求量、响应时间 |
| `nginx_upstream_check_module` | upstream 主动健康检查 | 自动探测后端存活，比被动检查更快发现故障 |
| `ngx_http_lua_module`（OpenResty） | 在 Nginx 中运行 Lua 脚本 | 限流、鉴权、灰度发布、自定义路由逻辑 |
| `headers-more-nginx-module` | 修改/删除请求和响应头 | 完全自定义 Server 头、增删任意 Header |
| `nginx-sticky-module` | 基于 Cookie 的会话保持 | 有状态应用的负载均衡 |
| `ngx_http_geoip_module` | 基于客户端 IP 地理位置路由 | 按国家/地区提供不同内容 |
| `ModSecurity` | Web 应用防火墙（WAF） | 防 SQLi、XSS、路径遍历等攻击 |
| `ngx_cache_purge` | 主动清除 proxy_cache | 内容更新后立即清除指定缓存 |
| `nginx-rtmp-module` | RTMP 直播流服务 | 视频直播、HLS/DASH 切片 |

###### 2. 如何编译安装 Nginx 并添加第三方模块？

```bash
# 1. 下载 Nginx 源码
wget http://nginx.org/download/nginx-1.24.0.tar.gz
tar -zxvf nginx-1.24.0.tar.gz

# 2. 下载第三方模块
git clone https://github.com/openresty/lua-nginx-module.git
git clone https://github.com/yaoweibin/nginx_upstream_check_module.git

# 3. 安装依赖
yum install -y gcc pcre-devel zlib-devel openssl-devel

# 4. 配置编译选项
cd nginx-1.24.0
./configure \
    --prefix=/etc/nginx \
    --sbin-path=/usr/sbin/nginx \
    --modules-path=/usr/lib64/nginx/modules \
    --conf-path=/etc/nginx/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_gzip_static_module \
    --with-http_stub_status_module \
    --with-http_realip_module \
    --add-module=../lua-nginx-module \
    --add-module=../nginx_upstream_check_module

# 5. 编译安装
make -j$(nproc)
make install

# 6. 验证
nginx -V    # 查看编译参数
nginx -t    # 检查配置
```

**动态模块（Nginx 1.9.11+）：**
```bash
# 动态模块无需重新编译 Nginx 主体
--add-dynamic-module=../lua-nginx-module   # 编译为 .so 文件

# nginx.conf 中加载
load_module modules/ngx_http_lua_module.so;
```

###### 3. ngx_http_lua_module 的作用是什么？

`ngx_http_lua_module` 是 OpenResty 项目的核心模块，允许在 Nginx 请求处理生命周期的各个阶段**嵌入 Lua 脚本**，将 Nginx 变成一个完整的 Web 应用框架：

**Nginx 生命周期阶段与 Lua 指令对应：**

| 阶段 | Lua 指令 | 常用场景 |
|------|---------|---------|
| 重写/路由 | `rewrite_by_lua_block` | URL 路由、参数处理 |
| 访问控制 | `access_by_lua_block` | JWT 验证、IP 白名单 |
| 内容生成 | `content_by_lua_block` | 返回自定义响应 |
| 响应头过滤 | `header_filter_by_lua_block` | 修改响应头 |
| 响应体过滤 | `body_filter_by_lua_block` | 修改响应内容 |
| 日志记录 | `log_by_lua_block` | 异步上报日志、统计 |
| 初始化 | `init_by_lua_block` | 预加载配置、共享字典初始化 |

**能力边界：** 通过 `lua-resty-*` 库可以在 Nginx 中进行：非阻塞 Redis 操作、MySQL 查询、HTTP 请求、消息队列操作等，**全程非阻塞，不影响 Nginx 性能**。

###### 4. 如何使用 Nginx 的 Lua 脚本进行编程？

**环境：OpenResty（内置 LuaJIT + 各种 resty 库）**

**示例一：JWT Token 验证**
```nginx
location /api/ {
    access_by_lua_block {
        local jwt = require "resty.jwt"
        local token = ngx.req.get_headers()["Authorization"]
        if not token then
            ngx.status = 401
            ngx.say('{"code":401,"msg":"unauthorized"}')
            return ngx.exit(401)
        end
        token = token:match("Bearer%s+(.+)")
        local verified = jwt:verify("your-secret-key", token)
        if not verified["verified"] then
            ngx.status = 401
            ngx.say('{"code":401,"msg":"token invalid"}')
            return ngx.exit(401)
        end
        -- 将用户信息传递给后端
        ngx.req.set_header("X-User-Id", verified.payload.sub)
    }
    proxy_pass http://backend;
}
```

**示例二：基于 Redis 的限流**
```nginx
lua_shared_dict rate_limit 10m;

location /api/ {
    access_by_lua_block {
        local redis = require "resty.redis"
        local red = redis:new()
        red:connect("127.0.0.1", 6379)
        
        local key = "rate:" .. ngx.var.remote_addr
        local count, _ = red:incr(key)
        if count == 1 then
            red:expire(key, 1)  -- 1秒窗口
        end
        
        if count > 100 then  -- 每秒最多100次
            ngx.status = 429
            ngx.say('{"code":429,"msg":"too many requests"}')
            return ngx.exit(429)
        end
    }
    proxy_pass http://backend;
}
```

###### 5. ngx_http_geoip_module 模块的作用是什么？

`ngx_http_geoip_module` 根据客户端 IP 地址判断其**地理位置**，并将信息存入变量供配置使用：

```bash
# 1. 安装 GeoIP 数据库（MaxMind 免费版或商业版）
yum install GeoIP GeoIP-devel
# 下载 GeoLite2 数据库并转换为 Legacy 格式，或使用 ngx_http_geoip2_module
```

```nginx
# 2. 配置（需要编译时加入 --with-http_geoip_module）
http {
    geoip_country /usr/share/GeoIP/GeoIP.dat;   # 国家数据库
    geoip_city    /usr/share/GeoIP/GeoLiteCity.dat;  # 城市数据库
    
    server {
        # 可用变量
        # $geoip_country_code    ← 国家代码（CN/US/JP）
        # $geoip_country_name    ← 国家名称
        # $geoip_city            ← 城市名
        # $geoip_latitude/longitude  ← 经纬度
        
        # 按国家路由（中国用户访问国内节点）
        if ($geoip_country_code = "CN") {
            proxy_pass http://cn-backend;
        }
        
        # 阻断特定国家的访问
        if ($geoip_country_code ~ "^(KP|IR|SY)$") {
            return 403;
        }
        
        # 在响应头中暴露地理信息
        add_header X-Country $geoip_country_code;
    }
}
```

**现代替代方案：** MaxMind 已停止更新旧版 GeoIP 格式，推荐使用 `ngx_http_geoip2_module` + GeoLite2 数据库（MMDB 格式）。

###### 追问：如何实现 Nginx 的热升级（不中断服务替换 Nginx 二进制）？

Nginx 支持**零停机热升级**，不中断任何现有连接：

```bash
# 1. 编译新版本 Nginx（不要 make install，只 make）
make

# 2. 备份旧版本
cp /usr/sbin/nginx /usr/sbin/nginx.old

# 3. 替换二进制文件（注意权限和路径）
cp objs/nginx /usr/sbin/nginx

# 4. 向 Master 进程发送 USR2 信号（启动新版本 Master，与旧版共存）
kill -USR2 $(cat /var/run/nginx.pid)
# 此时同时存在新旧两个 Master 进程，新 Master 接管新连接

# 5. 向旧 Master 进程发送 WINCH 信号（优雅停止旧 Worker）
kill -WINCH $(cat /var/run/nginx.pid.oldbin)
# 旧 Worker 处理完现有连接后退出，新 Worker 接管所有流量

# 6. 确认新版本工作正常后，停止旧 Master
kill -QUIT $(cat /var/run/nginx.pid.oldbin)

# 7. 如果新版本有问题，可回滚（让旧 Master 重新拉起 Worker）
kill -HUP $(cat /var/run/nginx.pid.oldbin)
kill -QUIT $(cat /var/run/nginx.pid)
```

整个过程零停机，现有连接完整处理完毕，用户无感知。
