###### 1. Comparable 和 Comparator 接口的区别？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#四、排序与比较|📖]]
这两个接口都用于定义排序规则，但设计思路不同，用法也不一样。

**Comparable** 在 `java.lang` 包里，核心方法是 `compareTo(T o)`。实现方式是在**要排序的类内部**实现这个接口，相当于给这个类定义了"天生的"排序规则，只能有一种。想改排序规则就得改类的源码，侵入性强。`String`、`Integer` 等类自带 Comparable，所以可以直接排序。

**Comparator** 在 `java.util` 包里，核心方法是 `compare(T o1, T o2)`。实现方式是在**类的外部**创建一个独立的比较器，和被比较的类完全解耦。可以为同一个类创建多种不同的比较器，符合开闭原则。无法修改源码的类（比如第三方库的类）也能用 Comparator 为它定制排序。

**一句话总结**：Comparable 是"这个类自带排序能力"，Comparator 是"外部定制排序工具"。如果需要多种排序方式，或者不能改类的代码，就用 Comparator。

###### 2. 如何对集合进行排序？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#四、排序与比较|📖]]
对集合排序主要用 `Collections.sort(list)` 或 `list.sort(comparator)` 方法：

- 如果列表元素实现了 Comparable，直接 `Collections.sort(list)` 就行，按自然顺序排
- 如果要自定义排序规则，传入一个 Comparator：`list.sort((a, b) -> a.getAge() - b.getAge())`

Java 8 之后还可以用 `Comparator.comparing()` 方法链式构建比较器，语法更简洁。

###### 3. 如何对自定义对象进行排序？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#四、排序与比较|📖]]
**方式一：实现 Comparable（自然排序）**

适合给一个类定义默认的、最常用的排序方式：

```java
public class Person implements Comparable<Person> {
    private String name;
    private int age;

    @Override
    public int compareTo(Person other) {
        return Integer.compare(this.age, other.age); // 按年龄升序
    }
}

// 使用
List<Person> people = new ArrayList<>();
Collections.sort(people); // 直接排，按年龄
```

**方式二：使用 Comparator（定制排序）**

适合无法修改类，或者需要多种不同排序规则的情况：

```java
// 按姓名排
Collections.sort(people, (p1, p2) -> p1.getName().compareTo(p2.getName()));

// 更推荐的写法：方法引用
people.sort(Comparator.comparing(Person::getName));

// 多字段排序：先按年龄，年龄相同再按姓名
people.sort(Comparator.comparingInt(Person::getAge)
             .thenComparing(Person::getName));

// 倒序
people.sort(Comparator.comparingInt(Person::getAge).reversed());
```

**最佳实践**：
- 实现 Comparable 时，`compareTo` 的结果必须和 `equals` 一致（compareTo 返回0时，equals 也应该返回 true），在 TreeSet、TreeMap 里尤为重要
- 多字段排序优先用 `Comparator.comparing().thenComparing()` 链式写法，比自己写 if-else 清晰多了

###### 4. Collections.sort() 和 Arrays.sort() 的实现原理？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#四、排序与比较|📖]]
**Arrays.sort()**：

- 对**基本类型数组**（int[]、double[] 等）：使用**双基准快速排序（Dual-Pivot QuickSort）**，这是经典快速排序的改进版，通常比单基准更快。不稳定排序。
- 对**对象数组**：使用 **TimSort** 算法。这是 Tim Peters 为 Python 设计的混合排序算法，结合了归并排序的稳定性和插入排序对小数据的高效性。稳定排序。

**Collections.sort()**：底层是把 List 转成数组，调用 `Arrays.sort()`，排完再写回 List。所以本质上也是 TimSort。

**TimSort 的核心思路**：
1. 数据量很小（< 32）时直接用二分插入排序
2. 否则扫描数组，找出连续有序的段（称为"run"），太短的 run 用插入排序扩充到 minrun 长度
3. 把所有 run 按规则合并，最终得到完整有序序列

TimSort 对"基本有序"的数据特别友好，最好情况接近 O(n)，平均和最坏 O(n log n)，并且是稳定排序（相等元素不会换位置）。
