###### 1. Collections 工具类的常用方法有哪些？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#五、Collections 工具类|📖]]
`Collections` 是一个纯工具类，构造器私有，全是静态方法，干的都是对集合进行操作的活儿。

**排序与打乱**：
- `sort(list)` — 自然排序
- `sort(list, comparator)` — 自定义规则排序
- `reverse(list)` — 反转顺序
- `shuffle(list)` — 随机打乱（测试、游戏场景常用）

**查找与统计**：
- `binarySearch(list, key)` — 在**已排序的** List 中二分查找，O(log n)
- `max(collection)` / `min(collection)` — 找最大/最小值
- `frequency(collection, obj)` — 统计某个元素出现次数

**修改**：
- `swap(list, i, j)` — 交换两个位置的元素
- `replaceAll(list, old, new)` — 替换所有匹配元素
- `fill(list, obj)` — 用指定元素填充整个 List（会覆盖原有内容，注意）

**包装与创建**：
- `synchronizedList/Set/Map(...)` — 包装成线程安全版本（粗粒度锁，有性能问题）
- `unmodifiableList/Set/Map(...)` — 包装成只读版本，试图修改会抛 UnsupportedOperationException
- `emptyList()/emptySet()/emptyMap()` — 返回不可变空集合，**推荐用来代替返回 null**
- `singleton(obj)` / `singletonList(obj)` — 返回只含一个元素的不可变集合

```java
// 好习惯：空结果返回空集合而不是 null
public List<String> getData() {
    // 没数据时
    return Collections.emptyList(); // 比 return null 安全多了
}
```

###### 2. 如何实现集合的浅拷贝和深拷贝？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#六、JDK 9+ 不可变集合|📖]]
**浅拷贝**：只复制集合本身，集合里的对象还是同一批引用，改了元素里的属性，原集合和副本都会变。

```java
List<String> original = new ArrayList<>(Arrays.asList("A", "B"));
List<String> shallowCopy = new ArrayList<>(original); // 构造器浅拷贝
```

**深拷贝**：集合和集合里的每个对象都完全独立，改副本不影响原集合。

最常见的实现是**序列化/反序列化**（元素类需要实现 Serializable）：
```java
public static <T extends Serializable> List<T> deepCopy(List<T> src) throws Exception {
    ByteArrayOutputStream byteOut = new ByteArrayOutputStream();
    new ObjectOutputStream(byteOut).writeObject(src);
    ByteArrayInputStream byteIn = new ByteArrayInputStream(byteOut.toByteArray());
    return (List<T>) new ObjectInputStream(byteIn).readObject();
}
```

也可以用第三方库：Apache Commons Lang 的 `SerializationUtils.clone()`，或者 Jackson/Gson 序列化再反序列化。

**选哪种**：元素是不可变对象（String、Integer 等）就浅拷贝没问题；元素是可变对象，又不想两份集合互相影响，就必须深拷贝。

###### 3. 如何实现集合的去重？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#四、排序与比较|📖]]
主要三种方式：

**用 HashSet**：最简单直接，`new HashSet<>(list)` 一行搞定，但不保留顺序。要保留插入顺序就用 `LinkedHashSet`：
```java
List<String> unique = new ArrayList<>(new LinkedHashSet<>(list));
```

**用 Stream.distinct()**：函数式风格，默认保持原顺序：
```java
List<String> unique = list.stream().distinct().collect(Collectors.toList());
```

**手动遍历判断**：性能差，不推荐，除非有特殊逻辑需要在去重时做额外处理。

**建议优先用 Stream.distinct()**，语义清晰，代码简洁，保持顺序。

###### 4. 如何实现两个集合的交集、并集、差集？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#四、排序与比较|📖]]
Collections 工具类没有直接提供这些集合运算，但用 Set 的 API 可以轻松实现：

```java
Set<Integer> set1 = new HashSet<>(Arrays.asList(1, 2, 3, 4));
Set<Integer> set2 = new HashSet<>(Arrays.asList(3, 4, 5, 6));

// 并集（先拷贝一份，再 addAll）
Set<Integer> union = new HashSet<>(set1);
union.addAll(set2); // [1, 2, 3, 4, 5, 6]

// 交集（先拷贝，再 retainAll 保留共有的）
Set<Integer> intersection = new HashSet<>(set1);
intersection.retainAll(set2); // [3, 4]

// 差集（set1 有但 set2 没有的）
Set<Integer> difference = new HashSet<>(set1);
difference.removeAll(set2); // [1, 2]
```

注意：一定要先拷贝再操作，不要直接在原集合上调用这些方法，会改掉原来的数据。

###### 5. Arrays.asList() 方法有什么注意事项？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#五、Collections 工具类|📖]]
这个方法有三个坑，面试常考：

**坑一：返回的 List 是固定大小的**。它返回的是 Arrays 内部的一个包装类，不是真正的 `ArrayList`，**不支持 add 和 remove**，调用会抛 `UnsupportedOperationException`。要可变的 List 要包一层：`new ArrayList<>(Arrays.asList(array))`。

**坑二：修改会反映到原数组**。返回的 List 是原数组的视图，`set()` 操作会改原数组，反过来改原数组也会影响这个 List。

**坑三：传基本类型数组会整体变成一个元素**。比如 `Arrays.asList(new int[]{1, 2, 3})` 返回的是 `List<int[]>`，里面就一个元素——那个 int 数组，而不是 `List<Integer>`。要处理基本类型数组应该这样：
```java
int[] arr = {1, 2, 3};
List<Integer> list = Arrays.stream(arr).boxed().collect(Collectors.toList());
```

###### 6. 如何将集合转换为数组？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#五、Collections 工具类|📖]]
```java
List<String> list = new ArrayList<>(Arrays.asList("A", "B", "C"));

// 推荐方式：传一个空数组，JVM 自动决定大小
String[] arr = list.toArray(new String[0]);

// 也可以指定大小
String[] arr2 = list.toArray(new String[list.size()]);

// 不推荐：无参版本返回 Object[]，类型不安全
Object[] arr3 = list.toArray();
```

传 `new String[0]` 的写法是官方推荐的，反而更高效（JVM 内部优化了这种模式）。

###### 7. Stream API 对集合操作有什么优势？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#五、Collections 工具类|📖]]
Java 8 的 Stream API 让集合操作从"怎么做"转变为"做什么"，代码更声明式，更易读：

**声明式编程**：不用写 for + if，直接 `filter/map/collect` 表达意图，业务逻辑一目了然。

**链式操作**：多个操作串成管道，一气呵成，不需要中间变量。

**并行简单**：`stream()` 改成 `parallelStream()` 就能多核并行处理，适合数据量大且操作耗时的场景（注意线程安全问题）。

**延迟执行**：中间操作（filter、map 等）是懒执行的，只有遇到终端操作（collect、forEach）才真正触发，有短路优化。

```java
// 传统写法
Set<Integer> evenSet = new HashSet<>();
for (Integer n : numbers) {
    if (n % 2 == 0) evenSet.add(n);
}
List<Integer> result = new ArrayList<>();
for (Integer n : evenSet) result.add(n * n);

// Stream 写法：一行，清晰多了
List<Integer> result = numbers.stream()
    .filter(n -> n % 2 == 0)
    .distinct()
    .map(n -> n * n)
    .collect(Collectors.toList());
```

###### 8. 如何遍历集合？各种方式的优缺点是什么？

[[../../../20_JavaKnowledge/02_集合框架/01、集合框架总览#五、Collections 工具类|📖]]
**for-each（增强for循环）**：最常用，语法简洁，适用所有 Iterable。缺点是不能在遍历中增删元素（会抛 ConcurrentModificationException）。**大多数场景首选**。

**Iterator 迭代器**：语法稍繁琐，但是**遍历中安全删除元素的唯一方式**（通过 `iterator.remove()`）：
```java
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if ("B".equals(it.next())) it.remove(); // 安全删除
}
```

**forEach + Lambda（Java 8+）**：代码简洁，函数式风格，但同样不能修改集合结构。

**普通 for 循环（带索引）**：只适用于 List，需要用到索引时才用。

**Stream API**：适合需要过滤、转换、汇总等复杂操作的场景，简单遍历用它有点大材小用。

**口诀**：只读遍历用 for-each，遍历中要删元素用 Iterator，数据加工用 Stream。
