###### 1. 什么是 Java 中的动态代理？

[[../../../20_JavaKnowledge/01_Java基础/09、动态代理（JDK与CGLIB）#二、JDK 动态代理|📖]]
动态代理是在**运行时**动态创建代理对象的技术。可以把它理解成一个"智能替身"：替身实现了和真实对象一样的接口，客户端调用替身的方法时，不是直接到真实对象，而是先被一个"拦截器"截获，拦截器可以在调用前后插入自己的逻辑（日志、权限、事务等），然后再调到真实对象。

核心价值是**解耦**：把那些和业务逻辑无关的横切关注点（日志、事务、权限）从业务代码里剥离出来，这就是AOP（面向切面编程）的本质。

动态代理的"动态"体现在：代理类是在运行时生成的，不需要你手写代理代码，一个代理可以代理任何实现了接口的类。

###### 2. JDK 动态代理和 CGLIB 动态代理有什么区别？

[[../../../20_JavaKnowledge/01_Java基础/09、动态代理（JDK与CGLIB）#三、CGLIB 动态代理|📖]]
这是高频面试题，两者在实现原理和适用条件上有本质区别：

**JDK动态代理**：基于Java反射机制，要求被代理类**必须实现至少一个接口**。核心是`Proxy`类和`InvocationHandler`接口。代理类在运行时动态生成，实现了目标接口，方法调用通过`InvocationHandler.invoke()`路由处理，invoke内部用反射调目标方法。

```java
// 核心API
UserService proxy = (UserService) Proxy.newProxyInstance(
    target.getClass().getClassLoader(), // 类加载器
    target.getClass().getInterfaces(),  // 代理哪些接口
    new InvocationHandler() {
        public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
            System.out.println("前置逻辑");
            Object result = method.invoke(target, args); // 反射调目标方法
            System.out.println("后置逻辑");
            return result;
        }
    }
);
```

**CGLIB动态代理**：通过字节码技术（ASM库）动态生成目标类的**子类**作为代理，不需要目标类实现接口。重写父类的非final方法，通过`MethodInterceptor.intercept()`拦截。调用父类方法时通过`MethodProxy`直接调用，不需要反射，性能比JDK代理好。

**选择时机**：Spring AOP默认策略是：如果目标类实现了接口，用JDK代理；否则用CGLIB。Spring Boot 2.x后默认改为优先用CGLIB。

两者都不能代理**final方法**（JDK代理代理的是接口，final方法不影响；CGLIB因为是继承，final方法不能重写，所以无法拦截）。

###### 3. 什么是静态代理？

[[../../../20_JavaKnowledge/01_Java基础/09、动态代理（JDK与CGLIB）#一、代理模式|📖]]
静态代理是在**编译期**就确定代理关系的代理模式。你需要手动写一个代理类，实现和目标类相同的接口，内部持有目标对象的引用，在每个方法里调用目标方法并加入额外逻辑。

```java
public class UserServiceProxy implements UserService {
    private UserService target;
    
    public UserServiceProxy(UserService target) { this.target = target; }
    
    @Override
    public void saveUser() {
        System.out.println("开启事务");
        target.saveUser(); // 调目标方法
        System.out.println("提交事务");
    }
}
```

**静态代理的致命缺点**：接口每加一个方法，代理类也得同步修改；每个需要代理的接口都要写一个代理类，类数量爆炸。这就是动态代理被发明出来的原因。

###### 4. 动态代理的实现原理是什么？

[[../../../20_JavaKnowledge/01_Java基础/09、动态代理（JDK与CGLIB）#四、实现原理对比|📖]]
**JDK动态代理原理**：

调用`Proxy.newProxyInstance()`时，JVM在内存中动态生成一个代理类（类名类似`$Proxy0`）的字节码，这个类实现了你指定的所有接口。代理类里每个接口方法的实现都是：把调用路由到`InvocationHandler.invoke()`。你在`invoke()`里通过`Method.invoke(target, args)`用反射调目标方法。

**CGLIB动态代理原理**：

使用ASM字节码框架动态生成目标类的子类。子类重写父类所有非final方法，重写的方法实现是：调用`MethodInterceptor.intercept()`。在`intercept()`里，通过`MethodProxy.invokeSuper(obj, args)`直接调用父类方法（走的是FastClass机制，不走反射，所以性能好）。

**性能对比**：CGLIB通常比JDK代理快，因为它调用父类方法时不用反射；但JDK代理在新版JDK里也做了大量优化，差距在缩小。在频繁调用的场景，差别可以忽略不计。
