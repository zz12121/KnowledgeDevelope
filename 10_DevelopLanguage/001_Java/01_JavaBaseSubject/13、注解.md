###### 1. Java 中的注解的作用是什么？

[[../../../20_JavaKnowledge/01_Java基础/08、注解与APT#一、注解概述|📖]]
注解本质是代码的**元数据**（关于代码的描述信息），它不直接改变代码逻辑，而是作为标记附着在类、方法、字段等代码元素上，供编译器或运行时环境读取并处理。

主要有三大作用：

**编译检查**：`@Override`让编译器验证你确实重写了父类方法；`@Deprecated`标记废弃元素，调用时编译器给警告；`@SuppressWarnings`告诉编译器抑制某种警告。

**框架配置（最重要的用途）**：这是现代Java开发的核心。用注解替代繁琐的XML配置，直接写在代码上，直观且集中。`@Component`/`@Service`告诉Spring这是个Bean；`@Autowired`实现依赖注入；`@Entity`/`@Table`做ORM映射；`@Test`告诉JUnit这是个测试方法。

**代码生成**：Lombok的`@Data`/`@Builder`等注解在编译期自动生成getter、setter、构造器等模板代码，极大减少样板代码。

###### 2. Java注解的常见分类？

[[../../../20_JavaKnowledge/01_Java基础/08、注解与APT#二、注解分类|📖]]
**按来源分**：
- **内置注解**：JDK自带的，如`@Override`、`@Deprecated`、`@SuppressWarnings`、`@FunctionalInterface`
- **元注解**：用来定义其他注解的注解，有5个：`@Target`、`@Retention`、`@Documented`、`@Inherited`、`@Repeatable`
- **自定义注解**：业务需要自己定义的

**按保留策略分（`@Retention`指定）**：
- `SOURCE`：只在源码阶段存在，编译后丢弃（如`@Override`）
- `CLASS`：保留在.class文件里，但运行时JVM不加载（默认策略）
- `RUNTIME`：运行时也保留，可以通过反射读取。**框架配置注解基本都用这个**

###### 3. 如何自定义注解？

[[../../../20_JavaKnowledge/01_Java基础/08、注解与APT#三、自定义注解|📖]]
三步：定义注解 → 使用注解 → 通过反射处理注解。

**第一步，定义**：用`@interface`关键字，加上必要的元注解：
```java
@Retention(RetentionPolicy.RUNTIME) // 运行时保留，反射能读到
@Target(ElementType.METHOD)         // 只能标注在方法上
@Documented
public @interface MyLog {
    String value() default ""; // 注解的成员，用"方法"语法，可以有默认值
    String desc() default "";
}
```

**第二步，使用**：
```java
@MyLog(value = "用户登录", desc = "记录登录操作")
public void login(String username) { ... }
```

**第三步，处理**（通过反射读取并执行逻辑）：
```java
Method method = clazz.getMethod("login", String.class);
if (method.isAnnotationPresent(MyLog.class)) {
    MyLog annotation = method.getAnnotation(MyLog.class);
    System.out.println("操作：" + annotation.value());
    // 在这里可以做日志记录、权限检查等逻辑
}
```

注解本身只是标记，没有逻辑，**读取并处理注解的代码才是真正产生效果的地方**。

###### 4. 元注解有哪些？各自的作用是什么？

[[../../../20_JavaKnowledge/01_Java基础/08、注解与APT#二、注解分类|📖]]
Java有5个元注解，用来定义自定义注解的行为：

**`@Target`**：指定注解可以标注在哪些地方。常用值：`ElementType.TYPE`（类/接口/枚举）、`METHOD`（方法）、`FIELD`（字段）、`PARAMETER`（参数）、`CONSTRUCTOR`（构造方法）。不指定则任何地方都能用。

**`@Retention`**：指定注解的生命周期。`SOURCE`（源码期）、`CLASS`（编译期，默认值）、`RUNTIME`（运行时，反射能读到，框架配置必用）。

**`@Documented`**：标记注解，表示这个注解的信息应包含在Javadoc生成的API文档中。

**`@Inherited`**：标记注解，表示这个注解具有继承性——如果父类被这个注解标注，子类自动也有这个注解。**只对类的继承有效，对接口实现无效**。

**`@Repeatable`**：Java 8引入，允许同一个元素上多次使用同一个注解，需要配套一个"容器注解"来保存重复注解。

###### 5. 注解和反射的关系是什么？

[[../../../20_JavaKnowledge/01_Java基础/08、注解与APT#四、注解的运行时处理|📖]]
**注解提供元数据**：注解在代码上做标记，存储配置信息，但注解本身不执行任何逻辑。

**反射提供读取能力**：反射API可以在运行时检查类、方法、字段，并获取它们上面的注解信息，是读取RUNTIME级别注解的唯一手段。

两者结合起来，就能实现各种"魔法"：

Spring启动时扫描类路径，通过反射找到带`@Component`的类，自动注册为Bean；找到`@Autowired`的字段，通过反射注入依赖。

JUnit通过反射找到带`@Test`的方法，然后动态调用执行测试。

Jackson/Gson通过反射读取`@JsonProperty`注解，决定JSON字段和Java字段的映射关系。

标准流程是：**定义注解（指定保留策略为RUNTIME）→ 标注到代码元素 → 运行时通过反射读取注解 → 根据注解信息执行对应逻辑**。没有反射，RUNTIME注解只是废纸；没有注解，反射读不到任何配置信息。两者相辅相成。
