###### 1. final 有哪些用法？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#一、final 关键字|📖]]
`final`关键字就是"不可改变"的意思，可以修饰变量、方法、类，分别代表不同的约束：

**final变量**：基本类型的值不能改，引用类型的引用地址不能改（但引用指向的对象内部的内容可以改）。常和`static`配合用：`public static final double PI = 3.14;`，这就是Java里定义常量的标准写法。

**final方法**：不能被子类重写。用于保护核心逻辑不被修改，或者早期JVM版本用于性能优化（现在JIT已经很厉害了，这个优化效果不明显）。

**final类**：不能被继承，是最终形态。`String`类就是final的，这也是String不可变的一个保障之一。

###### 2. 说说 this 关键字的用法？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#三、this 关键字|📖]]
`this`是指向当前对象实例的引用，主要有四个用途：

**区分变量名冲突**：最常见的场景，当方法参数名和成员变量名相同时，用`this.变量名`来指定成员变量。
```java
public void setName(String name) {
    this.name = name; // 左边this.name是成员变量，右边name是参数
}
```

**构造方法间互调**：在构造方法里用`this(参数)`调用本类其他构造方法，实现代码复用，必须放在第一行。

**传当前对象引用**：需要把当前对象传给别人时，用`this`代表自己，比如事件回调注册`EventManager.register(this)`。

**链式调用**：方法返回`return this`，让调用者可以连续调方法：`builder.append("a").append("b").build()`。

**注意**：`this`不能在静态方法里用，因为静态方法属于类，不依赖对象实例。

###### 3. 说说 super 关键字的用法？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#四、super 关键字|📖]]
`super`是指向当前对象的直接父类的引用，主要三个用途：

**调父类构造方法**：`super()`或`super(参数)`，必须放在子类构造方法第一行。如果不写，编译器默认插入`super()`调父类无参构造。
```java
public Dog(String name) {
    super("Animal"); // 先调父类构造，必须第一行
    this.name = name;
}
```

**调父类被重写的方法**：子类重写了父类方法后，还想用父类原版的逻辑，用`super.方法名()`。

**访问父类被隐藏的变量**：子类和父类有同名成员变量时，用`super.变量名`明确访问父类那个。

###### 4. this() & super() 在构造方法中的区别？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#四、super 关键字|📖]]
两者都必须放在构造方法第一行，所以**同一个构造方法里只能用其中一个**，不能同时出现。

`this()`：调用**本类**的另一个构造方法，目的是减少代码重复，在本类的多个构造之间做链式调用。

`super()`：调用**父类**的构造方法，目的是正确初始化父类部分的状态。是继承机制中对象正确初始化的基础。

如果两者都不写，编译器默认在第一行插入`super()`调父类无参构造——所以如果父类没有无参构造，子类又不显式调`super(参数版本)`，编译就会报错。

###### 5. static 都有哪些用法？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#二、static 关键字|📖]]
`static`的核心含义是：该成员属于**类本身**，而不是某个具体实例：

**静态变量（类变量）**：内存中只有一份，类加载时初始化，所有实例共享。推荐通过`类名.变量名`访问（如`Math.PI`）。

**静态方法（类方法）**：可以直接`类名.方法名()`调用，不需要创建对象。限制是静态方法内部不能直接访问非静态成员，也不能用`this`和`super`。`main`方法必须是静态的就是因为启动时还没有任何对象。

**静态代码块**：`static { ... }`，类加载时执行一次，且只执行一次，先于构造方法执行。常用于复杂的静态变量初始化，比如加载数据库驱动。

**静态内部类**：不依赖外部类实例，可以独立创建。比非静态内部类少了一个对外部类的隐式引用，不容易造成内存泄漏，推荐优先使用。

###### 6. & 和 && 的区别？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#五、常见面试题解析|📖]]
**`&&`（短路与）**：如果第一个条件为false，直接返回false，**不会再计算第二个条件**。这叫短路特性。日常判断基本都用这个，性能更好，而且可以防止空指针：
```java
if (obj != null && obj.isValid()) { // obj为null时后面不执行，安全
```

**`&`（按位与/无短路逻辑与）**：两个条件都会计算，无论第一个结果如何。另外`&`还能做整数的**按位与**运算：
```java
int c = 5 & 3; // 0101 & 0011 = 0001，结果为1
```

**总结**：逻辑判断场景用`&&`（短路、安全、高效），只在需要两边都执行或做位运算时才用`&`。

###### 7. switch 是否能作用在 byte 上，是否能作用在 long 上，是否能作用在 String 上？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#五、常见面试题解析|📖]]
**byte：可以**。switch支持`byte`、`short`、`char`、`int`这些整数类型。

**long：不可以**。比int大的整数类型（`long`）不支持switch表达式。

**String：可以（Java 7开始）**。底层是编译器转换成`hashCode()` + `equals()`的int比较来实现的，所以本质上还是int。

除了这些，switch还支持枚举类型和对应的包装类（`Integer`、`Character`等）。

Java 14开始还有了switch表达式（不只是语句），语法更简洁，支持`->`箭头写法，感兴趣可以了解一下。

###### 8. a=a+b 与 a+=b 有什么区别吗？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#五、常见面试题解析|📖]]
在Java里，关键区别是**类型转换**。

当`a`是`byte`、`short`、`char`这种小类型时，`a + b`的结果会自动提升为`int`，如果用`a = a + b`赋回去，编译器报错说类型不兼容，需要你手动强转。

而`a += b`这种复合赋值运算符，编译器**自动做了隐式强制转换**，等价于`a = (a的类型)(a + b)`，所以不报错。

```java
byte a = 10, b = 20;
// a = a + b; // 编译错误！a+b结果是int，不能直接赋给byte
a += b;       // 正确！等价于 a = (byte)(a + b)
```

在大多数int类型的场景里，两者没有什么区别，效果完全一样。

###### 9. 熟悉 instanceof 关键字的作用吗？

[[../../../20_JavaKnowledge/01_Java基础/03、关键字深度解析（final-static-this-super）#五、常见面试题解析|📖]]
`instanceof`是个二元运算符，用来**在运行时检查一个对象是否是某个类（或接口）的实例**，返回布尔值。

最常见的用途是在向下转型前做安全检查，避免`ClassCastException`：
```java
if (animal instanceof Dog) {
    Dog dog = (Dog) animal; // 确认后再转，安全
}
```

两个要点：如果被检查的对象是`null`，`instanceof`返回`false`，不抛异常。`instanceof`考虑继承关系，子类实例`instanceof`父类也是`true`。

Java 16开始有了**模式匹配**增强，可以写成`if (animal instanceof Dog dog)`，检查和转型一步完成，不用再写一行转型代码了，更简洁。

###### 10. transient 关键字的作用是什么？

[[../../../20_JavaKnowledge/01_Java基础/11、序列化与反序列化#二、Java 原生序列化（Serializable）|📖]]
`transient`用来**标记一个成员变量不参与序列化**。

当类实现`Serializable`接口时，所有非静态、非transient的字段都会被序列化。被`transient`修饰的字段在序列化时被跳过，反序列化后该字段恢复为类型默认值（对象引用为null，int为0）。

典型使用场景：密码、银行卡号等敏感信息不想持久化；某些大型临时数据或可以推导出来的派生数据不必序列化；依赖JVM运行环境的对象（如线程句柄）序列化没有意义。

注意：静态变量不管加不加`transient`都不会被序列化，因为序列化针对的是对象实例状态，不是类状态。

###### 11. volatile 关键字的作用是什么？

[[../../../20_JavaKnowledge/03_并发编程/04、volatile深度解析#一、volatile 是什么|📖]]
`volatile`是个轻量级同步机制，它的作用是：**保证可见性 + 禁止指令重排序，但不保证原子性**。

**可见性**：一个线程修改了volatile变量，会立即刷新到主内存，其他线程读时会强制从主内存读，不能用CPU缓存里的旧值。

**禁止重排序**：通过插入内存屏障，阻止编译器和CPU对这个变量前后的指令进行重排序。DCL（双重检查锁）单例中的`instance`就必须加volatile，就是为了防止`new`操作的指令重排序。

**不保证原子性**：`count++`即使加了volatile也不是原子的，因为它是读取-加1-写回三步。要保证原子性用`synchronized`或原子类（`AtomicInteger`）。

volatile的典型场景：状态标志位（一个线程写，多个线程读）、DCL单例中的instance变量。
