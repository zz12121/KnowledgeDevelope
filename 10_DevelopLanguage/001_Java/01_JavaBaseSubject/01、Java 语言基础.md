###### 1. Java 语言有哪些特点？

[[../../../20_JavaKnowledge/04_JVM/01、JVM整体架构#一、JVM 整体架构图|📖]]
Java是一门非常成熟的面向对象语言，它的核心特点可以用几个关键词来概括：

**面向对象**是Java的根基，支持封装、继承、多态，代码可维护性和复用性很好。

**跨平台性**是Java最著名的特性，靠的是"编译成字节码，JVM来执行"这套机制，实现了"一次编写，到处运行"。

**健壮性**体现在强类型检查、异常处理机制、自动垃圾回收三个方面，大大降低了程序崩溃和内存泄漏的风险。

**安全性**有内置的沙箱模型和字节码验证器保驾护航，防止恶意代码攻击。

**多线程支持**是语言级别的，不需要依赖第三方库。

**高性能**靠的是JIT即时编译器，把热点代码编译成本地机器码，性能接近C++。

**简单易学**是相对C++而言的，去掉了指针、运算符重载、多重继承这些复杂特性，还有自动GC，降低了学习门槛。

###### 2. 说说你对 JDK、JRE、JVM 的理解？

[[../../../20_JavaKnowledge/04_JVM/01、JVM整体架构#一、JVM 整体架构图|📖]]
这三者是包含关系：**JDK ⊃ JRE ⊃ JVM**。

**JVM（Java虚拟机）** 是最底层的，负责把字节码翻译成具体平台的机器指令去执行。它是Java跨平台的关键，不同操作系统有对应的JVM实现。

**JRE（Java运行环境）** 包含JVM加上Java核心类库，是**运行**Java程序所必需的环境。普通用户只需要装JRE就能跑Java程序。

**JDK（Java开发工具包）** 包含JRE加上开发工具，比如编译器javac、调试器jdb、打包工具jar等。**开发者**必须装JDK。

简单记忆：想用Java就装JRE，想写Java就装JDK。

###### 3. Java 是如何实现跨平台的？

[[../../../20_JavaKnowledge/04_JVM/01、JVM整体架构#三、执行引擎|📖]]
两个关键：**字节码**和**JVM**。

Java源码经javac编译后，生成的不是某种CPU的机器码，而是与平台无关的**字节码**（.class文件）。字节码是JVM的"通用语言"。

各个平台（Windows、Linux、macOS）安装对应的JVM，由JVM负责把字节码转换为本地机器指令。这样同一份字节码就能到处运行。

关键点是：JVM本身不跨平台，但它屏蔽了底层差异，让上层字节码跨了平台。

###### 4. 简单说说你写的 Java 程序是如何执行的？

[[../../../20_JavaKnowledge/04_JVM/01、JVM整体架构#三、执行引擎|📖]]
分两大阶段：**编译**和**运行**。

**编译阶段**：javac把.java源文件编译成.class字节码文件，经过词法分析、语法分析、语义分析等步骤。

**运行阶段**主要有三步：

第一步，**类加载**：ClassLoader找到对应的.class文件加载进JVM，然后进行链接（验证合法性、为静态变量分配内存、解析符号引用），最后初始化（执行静态代码块和静态变量赋值）。

第二步，**执行引擎**：两种方式并行工作。解释器逐行把字节码翻译成机器码，启动快但效率偏低。JIT编译器发现热点代码后，一次性编译成本地机器码缓存起来，后续调用直接用机器码，性能大幅提升。

第三步，**自动GC**：堆中不再使用的对象会被垃圾回收器自动清理，无需手动管理内存。

###### 5. 标识符的命名规则？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#一、数据类型全景|📖]]
**硬性规则（违反就报错）**：
1. 只能用字母、数字、下划线`_`、美元符`$`组成
2. **不能以数字开头**
3. 区分大小写，`myVar`和`myvar`是不同的标识符
4. 不能用关键字，比如`class`、`public`这些
5. 长度理论上没限制

**特别提醒**：虽然Java支持Unicode，理论上可以用中文做标识符，但强烈不建议，可维护性很差。

###### 6. Java中的main方法为什么要声明为public static void？

[[../../../20_JavaKnowledge/04_JVM/01、JVM整体架构#一、JVM 整体架构图|📖]]
这是JVM启动规范决定的，三个关键字各有原因：

**`public`**：程序入口要对外可见，JVM需要从外部调用它，所以必须是public。

**`static`**：程序启动时还没有任何对象存在，JVM必须在没有实例的情况下调用main方法，所以必须是static，直接用类名调。

**`void`**：main方法执行完程序就结束了，不需要向JVM返回任何值，所以是void。

这是一个约定俗成的规范，方便JVM找到并执行程序入口。

###### 7. 什么是字节码？字节码的好处是什么？

[[../../../20_JavaKnowledge/04_JVM/10、字节码与执行引擎#一、Java 字节码概述|📖]]
字节码是Java源码编译后生成的、与平台无关的中间格式，保存在.class文件里。它是JVM的"机器语言"，不面向任何具体的CPU，而是面向JVM这个抽象层。

字节码本质上是二进制格式，结构严格，包含魔数（0xCAFEBABE，这个命名很Java圈子风格）、版本号、常量池、字段表、方法表等组成部分，基于操作数栈模型执行。

字节码带来的好处主要有几个：

**跨平台性**：一次编译，到处运行。只要有JVM就能执行，不用针对每个平台重新编译。

**安全性**：JVM在执行前会验证字节码，防止有害操作，提供了安全沙箱。

**JIT性能优化**：JVM监控代码执行频率，对热点代码用JIT编译器一次性编译成机器码缓存，使得Java长期运行性能接近本地代码。

**动态性**：支持运行时动态加载类，是实现热部署、AOP、动态代理的基础。

**跨语言支持**：Kotlin、Scala等语言也编译成字节码跑在JVM上，共享整个JVM生态。

查看字节码可以用JDK自带的`javap -c MyClass`命令，加`-verbose`可以看更详细的信息，包括常量池。
