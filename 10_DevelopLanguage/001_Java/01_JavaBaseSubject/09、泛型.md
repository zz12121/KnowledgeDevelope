###### 1. 为什么需要泛型？

[[../../../20_JavaKnowledge/01_Java基础/06、泛型与类型擦除#一、泛型的本质|📖]]
泛型出现之前，Java集合只能用`Object`来装任意类型，带来三个明显的问题：

**类型不安全**：什么类型都能放进去，取的时候强转，如果类型不对就在运行时抛`ClassCastException`，问题发现太晚。

**代码冗余**：每次从集合取元素都要强转，写起来繁琐还容易出错。

**可读性差**：看代码根本不知道这个集合里装的是什么类型。

泛型把这些问题都解决了：类型检查提前到**编译期**，一旦类型不匹配，编译就报错；编译器自动插入转换代码，不用手写；代码一目了然，`List<String>`就知道这是字符串集合。

泛型是"把类型当参数"，让类、接口、方法可以在定义时先用占位符，使用时再指定具体类型。

###### 2. 如何使用泛型？

[[../../../20_JavaKnowledge/01_Java基础/06、泛型与类型擦除#二、泛型的使用|📖]]
**泛型类/接口**：在类名或接口名后加`<T>`声明类型参数，`T`只是个占位符，可以换成任何字母：
```java
public class Box<T> {
    private T value;
    public T getValue() { return value; }
    public void setValue(T value) { this.value = value; }
}
Box<String> stringBox = new Box<>(); // 指定T为String
```

**泛型方法**：在返回类型前声明`<T>`，泛型方法可以在普通类里定义：
```java
public static <T> List<T> fromArray(T[] array) {
    return Arrays.asList(array);
}
// 调用时类型推断，不用显式指定
List<Integer> list = fromArray(new Integer[]{1, 2, 3});
```

###### 3. 泛型的上限和下限是什么？

[[../../../20_JavaKnowledge/01_Java基础/06、泛型与类型擦除#三、通配符与边界|📖]]
**上界通配符`<? extends T>`**：表示"某种T的子类型"。可以安全地**读取**（得到的是T类型），但不能往里**写**（不知道具体是哪个子类型）。适合"生产者"角色——从集合里取数据。

```java
public double sum(List<? extends Number> list) {
    // 可以安全读取为Number
    return list.stream().mapToDouble(Number::doubleValue).sum();
    // list.add(1); // 报错！不能写入
}
```

**下界通配符`<? super T>`**：表示"某种T的父类型"。可以安全地**写入**T类型（父类引用可以接受子类对象），但读出来只能作为Object（不知道具体父类型）。适合"消费者"角色——往集合里放数据。

```java
public void fillNumbers(List<? super Integer> list) {
    list.add(1); // 可以安全写入Integer
    // Integer i = list.get(0); // 报错！只能读成Object
}
```

记忆法则：**PECS原则（Producer Extends, Consumer Super）**——产出数据用extends，消费数据用super。

###### 4. Java中的泛型为什么是伪泛型？什么是类型擦除？

[[../../../20_JavaKnowledge/01_Java基础/06、泛型与类型擦除#四、类型擦除（Type Erasure）|📖]]
Java泛型是通过**类型擦除（Type Erasure）** 来实现的，这是它被称为"伪泛型"的根本原因。

**类型擦除**：编译器在生成字节码时，把所有泛型类型信息全部移除。`<T>`如果没有上界限定，会被替换成`Object`；有上界则替换成上界类型。必要的地方编译器自动插入强转代码。

```java
// 编译前
List<String> list = new ArrayList<>();
String s = list.get(0);

// 编译后（字节码里是这样的）
List list = new ArrayList(); // 类型信息擦除
String s = (String) list.get(0); // 编译器自动加了转型
```

所以`List<String>`和`List<Integer>`在运行时都是`List`，`list.getClass() == intList.getClass()`结果是true，根本区分不了——这就是"伪泛型"的含义。

**类型擦除带来的限制**：
- 不能用基本类型作为泛型参数（`List<int>`不行，必须`List<Integer>`）
- 不能`new T()`创建泛型类型的实例
- 不能创建泛型数组`new T[10]`
- 运行时`instanceof`检查不带泛型信息，`obj instanceof List<String>`会编译报错

###### 5. 泛型中 extends 和 super 的区别（PECS原则）？

[[../../../20_JavaKnowledge/01_Java基础/06、泛型与类型擦除#三、通配符与边界|📖]]
PECS（Producer-Extends, Consumer-Super）是Joshua Bloch在《Effective Java》里提出的原则，用来指导通配符的选择：

**`<? extends T>`（Producer用extends）**：从结构中**读取**T类型数据时使用。你把它当生产者，它给你生产T类型的数据。

**`<? super T>`（Consumer用super）**：往结构中**写入**T类型数据时使用。你把它当消费者，你往里塞T类型的数据。

经典例子是`Collections.copy`：
```java
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    // dest是消费者，接收写入 → super T
    // src是生产者，提供读取 → extends T
    for (T item : src) dest.add(item);
}
```

如果一个泛型参数既要读又要写，就别用通配符，直接用具体类型`<T>`。
