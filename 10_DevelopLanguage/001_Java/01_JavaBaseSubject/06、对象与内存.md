###### 1. 什么是构造函数？构造函数有什么特点？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#一、对象的创建过程|📖]]
构造函数是一种特殊的方法，**创建对象时自动调用**，主要用于初始化对象的成员变量。

几个关键特点：

**必须和类同名**，这是辨认构造函数的标志。

**没有返回类型**，连void都不写，写了就变成普通方法了。

**支持重载**，一个类可以有多个参数列表不同的构造函数，new时根据传参决定调哪个。

**默认构造函数**：如果你一个构造函数都没写，编译器自动提供无参构造；一旦你写了任何构造函数，默认无参构造就不再自动提供，如果还需要就得自己写。

**继承中的调用规则**：创建子类对象时，一定先调父类构造函数（默认调无参，可以用`super(参数)`显式调有参），然后才执行子类构造。

###### 2. 说说 Java 代码初始化顺序？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#一、对象的创建过程|📖]]
这是个经典题，记住一个原则：**静态优先于非静态，父类优先于子类，同级按代码顺序**。

完整顺序：
1. 父类静态变量和静态代码块（按代码出现顺序）
2. 子类静态变量和静态代码块（按代码出现顺序）
3. 父类实例变量和实例初始化块（按代码出现顺序）
4. 父类构造函数
5. 子类实例变量和实例初始化块（按代码出现顺序）
6. 子类构造函数

**静态初始化只在类首次加载时执行一次**，后续创建对象不再执行；实例初始化每次new对象都会执行。

###### 3. Java 创建对象有几种方式？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#一、对象的创建过程|📖]]
主要四种：

`new关键字`：最常用、最直接，`MyClass obj = new MyClass()`。

`反射`：通过`Constructor.newInstance()`动态创建，框架最爱用。

`clone()`：对象克隆，需要实现Cloneable接口，默认浅拷贝。

`反序列化`：从字节流恢复对象，反序列化不走构造方法，直接从字节流填充字段值。

###### 4. 如何实现对象克隆？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#三、对象的深克隆与浅克隆|📖]]
两步：实现`Cloneable`标记接口，重写`clone()`方法并改为public。

```java
class Person implements Cloneable {
    private String name;
    private Address address; // 引用类型
    
    @Override
    public Object clone() throws CloneNotSupportedException {
        return super.clone(); // 默认浅拷贝
    }
}
```

`Object.clone()`默认是**浅拷贝**：基本类型字段直接复制值，引用类型字段只复制引用（地址），不复制引用指向的对象。

要实现**深拷贝**，有两种方式：一是重写clone方法，对每个引用类型字段也递归调用clone；二是用序列化/反序列化（代码简单，但性能开销大）。

###### 5. 深拷贝和浅拷贝的区别是什么？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#三、对象的深克隆与浅克隆|📖]]
**浅拷贝**：只复制对象本身，引用类型的字段只复制引用（地址），原对象和副本**共享同一个引用对象**。修改副本里引用类型的内容，原对象也受影响。`Object.clone()`默认就是浅拷贝。

**深拷贝**：不只复制对象本身，还递归地复制所有引用对象，原对象和副本**完全独立**，互不影响。修改副本不影响原对象。实现比浅拷贝复杂，性能开销也更大。

一句话：浅拷贝是"一层复制"，深拷贝是"整颗树都复制"。

实际项目中深拷贝用序列化方式最省事，需要对象实现Serializable接口。

###### 6. 熟悉 Java 的对象引用类型吗？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#六、引用类型详解|📖]]
Java有四种不同强度的引用，垃圾回收器对它们的处理方式不同：

**强引用（Strong Reference）**：最普通的引用，`Object obj = new Object()`。只要强引用存在，GC永远不会回收这个对象。内存不够时抛OOM也不回收。

**软引用（Soft Reference）**：`SoftReference`包装。内存充足时不回收，**内存不足时才回收**。适合做内存敏感的缓存，比如图片缓存。

**弱引用（Weak Reference）**：`WeakReference`包装。只要发生GC，不管内存够不够，都会被回收。`WeakHashMap`的key就是弱引用，可以防止map造成内存泄漏。

**虚引用（Phantom Reference）**：`PhantomReference`包装。最弱的引用，无法通过虚引用获取对象。唯一作用是对象被回收时收到一个通知，通常配合`ReferenceQueue`用于跟踪GC活动或资源清理。

###### 7. 为什么 Java 中只有值传递？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#二、对象的内存布局|📖]]
这是个经典争议题，结论是：**Java只有值传递**。

传基本类型时，传的是值的副本，方法内修改副本不影响原值，这明显是值传递。

传对象（引用类型）时，传的是**引用的副本**（地址的拷贝），不是对象本身。所以：

- 通过这个引用副本修改对象内部的状态，原对象确实会改变（因为指向同一个堆对象）
- 但如果方法内把引用副本指向一个新对象（`param = new Object()`），原来的引用不受影响

```java
void change(String str, StringBuilder sb) {
    str = "Changed";      // 让副本指向新字符串，原str不变
    sb.append(" World");  // 通过副本修改共享对象，原sb变了
}
```

关键在于：Java传的是"地址的副本"，不是"地址的引用"，所以归根结底还是值传递。

###### 8. Java 对象的内存布局是怎样的？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#二、对象的内存布局|📖]]
HotSpot VM中，堆上的对象有三部分：

**对象头（Object Header）**：包含两块。Mark Word（8字节，64位VM）存哈希码、GC年龄、锁状态、线程ID等运行时数据；类型指针（4字节，开启指针压缩）指向方法区中的类元数据。

**实例数据（Instance Data）**：对象真正存储的有效字段，包括从父类继承来的。字段排列受JVM分配策略影响（通常按对象类型大小排序，以减少填充）。

**对齐填充（Padding）**：HotSpot要求对象起始地址必须是8字节整数倍，不够就补0填充。空Object = 8(MarkWord) + 4(类指针，压缩) = 12字节，补4字节对齐 = 16字节。

###### 9. 什么是对象的逃逸分析？

[[../../../20_JavaKnowledge/04_JVM/04、对象的创建与内存布局#四、逃逸分析（Escape Analysis）|📖]]
逃逸分析是JIT编译器的一种优化技术，分析对象的作用域是否"逃逸出"方法或线程。

如果一个对象在方法内创建后，不被外部方法引用、不被其他线程访问，就是"没有逃逸"，JVM可以做三种优化：

**栈上分配**：对象内存直接分配在栈帧上，方法返回自动销毁，不需要GC，减轻GC压力。

**标量替换**：把对象拆解成若干基本类型变量，分配在栈或寄存器上，完全不创建对象，省掉对象头的开销。

**同步消除**：如果对象只被一个线程访问，那么对它的synchronized锁可以安全地消除，不做同步开销。

逃逸分析默认开启（`-XX:+DoEscapeAnalysis`），是现代JVM重要的优化手段。你在代码里能感受到的好处就是：很多new出来的对象其实没有真正分配到堆上。
