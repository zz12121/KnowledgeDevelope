###### 1. 说说你对 Java 反射的理解？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#一、反射机制概述|📖]]
反射机制允许程序在**运行时**动态地获取任何类的结构信息（类名、方法、字段、构造器、注解等），并且可以动态地调用方法、访问字段、创建对象。

反射的核心价值是**动态性**：打破了"编译期绑定"的限制，让程序不必在编码时就写死要用哪个类，而是可以在运行时根据配置或需求动态决定。

这是Spring IOC容器、MyBatis、JUnit等众多主流框架的底层核心技术——框架不知道你会写什么类，但通过反射，它可以在运行时扫描、加载、操作你写的任何类。

###### 2. 说说反射机制的优缺点？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#二、反射的优缺点|📖]]
**优点**：高度灵活和动态，可以在运行时加载任意类、调用任意方法，是框架、插件化架构、热部署的基础，实现了代码的高度可扩展性。还能突破访问权限限制，访问私有成员，适合测试和序列化场景。

**缺点**主要三个：

**性能较低**：反射涉及动态类型解析，比直接方法调用慢几倍到几十倍，还会绕过JIT的某些优化。频繁调用时需要注意缓存Method/Field对象，避免重复获取。

**破坏封装**：可以访问private成员，破坏了类的封装设计，可能导致对象状态不一致。

**编译期无法检查**：代码行为在运行时才确定，IDE无法给出代码提示，调试难度更高，维护成本增加。

**最佳实践**：普通业务代码**不要用反射**，它的主战场是框架开发、工具类和系统集成。必须用时，及时缓存反射得到的对象，注意安全控制。

###### 3. 获取一个类 Class 对象的方式有哪些？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#三、获取 Class 对象|📖]]
反射的第一步都是获取Class对象，一个类在JVM中只有唯一一个Class对象，有三种主要方式：

**`类名.class`（类字面常量）**：编译期就确定，最安全最高效，不会触发类初始化，也不抛异常。
```java
Class<String> clazz = String.class;
```

**`对象.getClass()`**：通过已有的对象实例获取，可以拿到对象的运行时实际类型（多态下有用）。
```java
String s = "hello";
Class<?> clazz = s.getClass();
```

**`Class.forName("全限定类名")`**：最灵活，类名可以来自配置文件或数据库，框架开发最常用。会触发类的初始化，可能抛`ClassNotFoundException`。
```java
Class<?> clazz = Class.forName("com.example.MyClass");
```

此外还可以通过类加载器的`loadClass()`方法，提供更底层的控制。

###### 4. Class.forName 和 ClassLoader 有什么区别？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#三、获取 Class 对象|📖]]
两者都能加载类，关键区别是**是否执行初始化**：

**`Class.forName(className)`**：加载类的同时会执行类的**初始化**（静态代码块和静态变量赋值）。JDBC加载驱动就用这个：`Class.forName("com.mysql.cj.jdbc.Driver")`——驱动注册到DriverManager这个动作在驱动类的静态代码块里，必须执行初始化才会注册。

**`ClassLoader.loadClass(className)`**：只完成类加载（把字节码加载进JVM），**不执行初始化**，等到第一次主动使用时才初始化。适合提前加载类但不想立即执行初始化逻辑的场景，性能更好一些。

`Class.forName`还有重载版本`Class.forName(name, initialize, classLoader)`，第二个参数可以控制是否初始化，更灵活。

###### 5. 反射机制的应用场景有哪些？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#五、反射的应用场景|📖]]
**框架核心**：这是最重要的应用。Spring IOC通过反射创建Bean、注入依赖；MyBatis/Hibernate通过反射把数据库结果映射到Java对象字段；JUnit通过反射找到并调用`@Test`方法。

**动态代理**：JDK动态代理通过反射在`InvocationHandler.invoke`里调用目标方法，这是AOP（面向切面编程）的基础。

**注解处理**：框架在运行时通过反射读取类、方法、字段上的注解，然后根据注解信息执行相应逻辑（权限检查、数据校验等）。

**开发工具**：IDE的代码补全、调试器查看对象内部结构，都依赖反射。

###### 6. 如何通过反射创建对象？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#四、反射核心 API|📖]]
推荐用`Constructor.newInstance()`（现代Java标准做法），比`Class.newInstance()`更强大，可以调任意构造器：

```java
Class<?> clazz = Class.forName("com.example.Person");

// 调用无参构造
Object obj1 = clazz.getDeclaredConstructor().newInstance();

// 调用有参构造
Constructor<?> constructor = clazz.getDeclaredConstructor(String.class, int.class);
Object obj2 = constructor.newInstance("Alice", 25);

// 调用私有构造（如单例的私有构造）
Constructor<?> privateCtx = clazz.getDeclaredConstructor(String.class);
privateCtx.setAccessible(true); // 关键步骤：突破private限制
Object obj3 = privateCtx.newInstance("Admin");
```

`Class.newInstance()`已经在Java 9中标注为过时，不要再用了。

###### 7. 如何通过反射调用私有方法？

[[../../../20_JavaKnowledge/01_Java基础/07、反射机制#四、反射核心 API|📖]]
四步走：获取Class对象 → 用`getDeclaredMethod`获取私有方法 → `setAccessible(true)` → `invoke()`调用。

```java
SecretService service = new SecretService();
Class<?> clazz = service.getClass();

// getDeclaredMethod 可以获取包括私有在内的所有方法
// getMethod 只能获取public方法（包括父类的）
Method secretMethod = clazz.getDeclaredMethod("getSecretCode");

// 关键：关闭Java语言访问检查
secretMethod.setAccessible(true);

// 调用：第一个参数是对象实例，后面是方法参数（无参时不传）
String result = (String) secretMethod.invoke(service);
```

`setAccessible(true)`是反射的"破窗锤"，用了之后类的封装就被打穿了。应该**仅在必要场景谨慎使用**（比如单元测试私有方法、框架内部操作、序列化）。Java 9以后的模块化系统对这个有额外限制，跨模块访问可能需要额外的`opens`配置。
