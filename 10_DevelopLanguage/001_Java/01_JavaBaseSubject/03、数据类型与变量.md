###### 1. 说说八种基本数据类型的大小，以及他们的封装类？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#一、数据类型全景|📖]]
Java的8种基本数据类型，面试必背：

整型4种：`byte`（1字节，-128~127，封装类`Byte`）、`short`（2字节，封装类`Short`）、`int`（4字节，封装类`Integer`）、`long`（8字节，封装类`Long`）。

浮点2种：`float`（4字节，单精度，封装类`Float`）、`double`（8字节，双精度，封装类`Double`）。

字符1种：`char`（2字节，Unicode字符0~65535，封装类`Character`）。

布尔1种：`boolean`（大小未严格定义，通常是1位逻辑值，封装类`Boolean`）。

记住`int`是4字节，`long`是8字节，`double`是8字节，其他的参照这个往外推就行。

###### 2. char 型变量中能不能存贮一个中文汉字？为什么？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#一、数据类型全景|📖]]
**可以**。因为Java的`char`采用Unicode编码，占2字节（16位），可以表示0~65535范围内的字符。常用汉字都在Unicode的CJK统一编码区间内，每个汉字对应一个Unicode码点，所以可以直接存储。

```java
char ch = '中'; // 完全合法
```

但要注意：某些生僻字或emoji表情可能超出基本多语言平面（BMP），码点超过65535，这种情况需要用`String`来表示（实际上是两个`char`组成的字符对，叫做代理对）。

###### 3. Java 中包装类型和基本类型的区别是什么？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#二、自动装箱与拆箱（Autoboxing / Unboxing）|📖]]
基本类型和包装类型是Java里两套不同的东西，区别主要有以下几点：

**本质不同**：基本类型是直接存储数值的，比如`int`就是个数；包装类型是真正的对象，有方法可以调。

**存储位置不同**：基本类型的值存在栈上（作为局部变量时），读写更快；包装类型的对象在堆上，变量只是持有引用。

**默认值不同**：基本类型有默认值（`int`是0，`boolean`是false）；包装类型默认是`null`，这就是为什么用包装类型有时会出现空指针。

**泛型支持**：集合和泛型只能用引用类型，所以`List<int>`是不行的，必须用`List<Integer>`。这是包装类型存在的最重要原因之一。

**性能上**：基本类型更高效，包装类型涉及堆内存分配和GC，有额外开销。在大量计算的场景优先用基本类型。

###### 4. 说说 Java 自动装箱与拆箱？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#二、自动装箱与拆箱（Autoboxing / Unboxing）|📖]]
**自动装箱**是基本类型自动转成包装类对象，**自动拆箱**是包装类对象自动转回基本类型。这是Java 5引入的语法糖，本质上是编译器帮你插入了转换代码。

装箱底层调的是`Integer.valueOf()`，拆箱底层调的是`intValue()`。

```java
Integer num = 100;   // 装箱：编译器替换为 Integer.valueOf(100)
int value = num;     // 拆箱：编译器替换为 num.intValue()
```

**有三个坑要注意**：

第一个是**空指针陷阱**：包装类型为null时拆箱会抛`NullPointerException`，这是很常见的线上问题。

第二个是**性能问题**：循环里频繁装拆箱会产生大量临时对象，给GC造成压力。

第三个是**比较陷阱**：`==`比较包装类对象时，超出缓存范围（-128~127）就是比地址，不是比值，会出现意料之外的false。比较值要用`equals()`。

###### 5. Integer a = 127 与 Integer b = 127 相等吗？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#二、自动装箱与拆箱（Autoboxing / Unboxing）|📖]]
`a == b`结果是**true**。

原因是`Integer`有缓存机制：JVM默认缓存了-128到127之间的所有Integer对象。用自动装箱或`Integer.valueOf()`创建这个范围内的对象时，直接返回缓存中的同一个对象，不会新建。所以a和b指向的是同一个对象，`==`当然为true。

但如果换成128：`Integer c = 128; Integer d = 128;`，此时`c == d`是**false**，因为128超出缓存范围，每次都新建对象，两个引用指向不同对象。

这是个经典的坑，所以比较包装类型的值，**永远用`equals()`，不要用`==`**（除非你明确知道自己在比地址）。

另外`new Integer(127)`是强制新建对象，不走缓存，这个方法在JDK 9后被废弃了，用`Integer.valueOf(127)`代替。

###### 6. 成员变量与局部变量有什么区别？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#四、变量分类与内存位置|📖]]
**声明位置**：成员变量在类里方法外；局部变量在方法、构造方法或代码块里。

**生命周期**：实例成员变量随对象创建而生、随对象被回收而消亡；静态变量随类加载而生、随类卸载而消亡；局部变量随方法调用而生、方法执行结束就销毁。

**存储位置**：实例变量在堆内存的对象里；静态变量在方法区；局部变量在栈内存的方法栈帧里。

**默认值**：成员变量有默认值（数字是0，boolean是false，引用类型是null）；局部变量没有默认值，必须先显式初始化才能用，否则编译报错。

**访问权限**：成员变量可以用`public`、`private`等修饰；局部变量只能用`final`修饰，不能加访问权限。

###### 7. 静态变量和实例变量区别？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#四、变量分类与内存位置|📖]]
**静态变量（类变量）** 属于类，所有对象共享同一份，在类加载时分配一次内存，用类名就能访问，生命周期和类相同。

**实例变量** 属于对象，每个对象有自己独立的一份，每次new对象时随对象在堆内存分配，通过对象引用访问，生命周期和对象相同。

最典型的区别：`count++`如果count是静态变量，所有对象修改的是同一个值；如果是实例变量，每个对象修改自己的，互不干扰。

###### 8. 静态变量与局部变量区别？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#四、变量分类与内存位置|📖]]
**声明位置**：静态变量在类中方法外，加`static`；局部变量在方法或代码块里。

**内存位置**：静态变量在方法区，随类存在；局部变量在栈帧里，随方法执行而存在。

**初始化**：静态变量有默认值；局部变量没有，必须手动赋值。

**访问范围**：静态变量可以通过类名全局访问；局部变量只在定义它的作用域里有效。

**修饰符**：静态变量可以用`public`、`static`等修饰；局部变量只能用`final`。

###### 9. 3 * 0.1 == 0.3 将会返回什么？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#六、实战场景与踩坑|📖]]
返回 **`false`**。

原因是浮点数精度问题。计算机用二进制来存小数，但有些十进制小数（比如0.1）用二进制是无限循环的，必须截断，这就引入了误差。

`3 * 0.1`的实际结果是类似`0.30000000000000004`这样的近似值，跟`0.3`在二进制层面不完全一样，所以`==`比较结果为false。

正确的浮点数比较方式是判断差值是否在极小范围内：
```java
Math.abs(3 * 0.1 - 0.3) < 1e-10  // 这才是正确姿势
```

金融场景要精确计算，必须用`BigDecimal`，而且要用字符串构造（`new BigDecimal("0.1")`），不能用double构造（`new BigDecimal(0.1)`会把浮点误差带进去）。

###### 10. 什么是类型转换？自动类型转换和强制类型转换的区别？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#三、类型转换|📖]]
Java里两种类型转换：

**自动类型转换（隐式转换）**：小范围类型转大范围类型，编译器自动完成，安全，不丢数据。方向是`byte → short → int → long → float → double`。比如`int`赋给`long`变量，自动转，不需要写任何代码。

**强制类型转换（显式转换）**：大范围类型转小范围类型，必须你主动写`(目标类型)`，编译器警告你"后果自负"。风险是可能精度丢失（比如`(int) 10.9`得到10，小数直接截断不四舍五入）或数据溢出（128强转`byte`得到-128）。

```java
int i = (int) 10.9;   // 强转，结果是10，小数截断
```

特别注意：`byte`、`short`、`char`在运算时会自动提升为`int`，所以`byte a = 1; byte b = a + 1;`编译报错，因为`a + 1`已经是int了。

`+=`这种复合赋值运算符内部隐含了强制转换，所以`short s = 1; s += 1;`是合法的。

`boolean`类型和其他7种基本类型完全不兼容，不能互相转换。

###### 11. 为什么浮点数运算会有精度问题？

[[../../../20_JavaKnowledge/01_Java基础/01、数据类型与变量#六、实战场景与踩坑|📖]]
核心原因：**有些十进制小数在二进制下是无限循环的**，计算机存储空间有限，必须截断，截断就有误差。

比如0.1转二进制是`0.0001100110011...`无限循环，float只有23位尾数，double有52位，都必须截断，引入最初误差。

随后每次运算都可能再产生误差，多次运算下来，误差累积就明显了。

解决方案：
1. **不用`==`直接比较浮点数**，用差值判断法
2. **精确计算用`BigDecimal`**，用字符串构造，不要用double构造
3. **货币计算用整数**，比如金额用分（long类型）来存储，永远不用小数

BigDecimal的几个注意点：比较用`compareTo()`不用`equals()`（因为equals还比scale精度）；除法要指定精度和舍入模式，否则遇到无限小数会抛异常。
