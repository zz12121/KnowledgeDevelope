###### 1. 如何构建一个企业级搜索系统？需要考虑哪些因素？

**企业级搜索系统的核心挑战：搜索质量 + 性能 + 可用性。**

**整体架构：**

```
数据源（DB/文件/爬虫）
    ↓ 数据采集（Logstash / Canal / 自研同步服务）
    ↓ 数据清洗 & 预处理（Ingest Pipeline / Kafka）
    ↓ 索引写入（ES Bulk API）
    ↓ 搜索服务（Spring Boot + ES Client）
    ↓ 结果排序 & 过滤（业务层二次处理）
    ↓ 用户界面（前端 + Kibana）
```

**设计关键点：**

| 维度 | 考量点 |
|------|--------|
| **Mapping 设计** | 禁止 dynamic mapping；明确字段类型；keyword vs text；nested 谨慎用 |
| **分词策略** | 中文用 IK；支持同义词；多语言分字段 |
| **搜索质量** | BM25 调参；function_score 业务加权；pinned query 置顶 |
| **性能** | 合理分片数；查询用 filter；避免深分页；路由优化 |
| **数据同步** | 增量同步 vs 全量重建；Binlog + Canal；数据一致性保证 |
| **高可用** | 跨 AZ 部署；副本数≥1；ILM 热温冷归档 |
| **监控告警** | 集群健康、查询耗时、JVM 堆、磁盘水位 |

**搜索质量提升手段：**
- `function_score`：对商品销量、好评率、时效性加权
- `pinned query`：置顶特定文档（广告/运营干预）
- 同义词扩展：手机 = 移动电话
- 搜索词纠错：`fuzzy` 查询 + Suggest API
- 个性化排序：结合用户行为（点击率、购买转化）

###### 2. 如何设计一个日志分析系统的架构？

**标准 ELK Stack 架构：**

```
应用服务器
    ↓ Filebeat（轻量采集，无状态）
    ↓ Kafka（削峰，解耦，缓冲）
    ↓ Logstash（解析、过滤、转换）
    ↓ Elasticsearch（存储、索引、搜索）
    ↓ Kibana（可视化、Dashboard、告警）
```

**ES 索引设计（日志专用）：**

```json
PUT /_index_template/logs-template
{
  "index_patterns": ["logs-*-*"],
  "data_stream": {},
  "template": {
    "settings": {
      "number_of_shards": 2,
      "number_of_replicas": 1,
      "index.refresh_interval": "30s",
      "index.codec": "best_compression"
    },
    "mappings": {
      "properties": {
        "@timestamp": {"type": "date"},
        "level": {"type": "keyword"},
        "service": {"type": "keyword"},
        "trace_id": {"type": "keyword"},
        "message": {"type": "text", "index": false}
      }
    }
  }
}
```

**ILM 生命周期策略（日志按时间归档）：**

```json
PUT /_ilm/policy/logs-policy
{
  "policy": {
    "phases": {
      "hot": {
        "min_age": "0ms",
        "actions": {
          "rollover": {"max_size": "50gb", "max_age": "1d"},
          "set_priority": {"priority": 100}
        }
      },
      "warm": {
        "min_age": "7d",
        "actions": {
          "forcemerge": {"max_num_segments": 1},
          "shrink": {"number_of_shards": 1},
          "set_priority": {"priority": 50}
        }
      },
      "cold": {
        "min_age": "30d",
        "actions": {
          "freeze": {},
          "set_priority": {"priority": 0}
        }
      },
      "delete": {
        "min_age": "90d",
        "actions": {"delete": {}}
      }
    }
  }
}
```

**性能关键点：**
- `message` 字段设 `"index": false`，不需要全文搜索时节省大量空间
- 写入时批量 Bulk，size 设 5-15MB
- 查询统一用 `filter` 不影响评分缓存

###### 3. 如何设计一个电商搜索系统？

**电商搜索核心需求：相关性 + 排序 + 筛选 + 性能。**

**商品索引 Mapping 设计：**

```json
PUT /products
{
  "mappings": {
    "properties": {
      "id": {"type": "keyword"},
      "title": {
        "type": "text",
        "analyzer": "ik_max_word",
        "search_analyzer": "ik_smart",
        "copy_to": "all_text"
      },
      "brand": {"type": "keyword"},
      "category": {"type": "keyword"},
      "price": {"type": "double"},
      "sales_count": {"type": "integer"},
      "rating": {"type": "float"},
      "stock": {"type": "integer"},
      "is_active": {"type": "boolean"},
      "tags": {"type": "keyword"},
      "attrs": {
        "type": "nested",
        "properties": {
          "name": {"type": "keyword"},
          "value": {"type": "keyword"}
        }
      },
      "all_text": {"type": "text", "analyzer": "ik_max_word"}
    }
  }
}
```

**综合搜索查询（相关性 + 业务加权）：**

```json
GET /products/_search
{
  "query": {
    "function_score": {
      "query": {
        "bool": {
          "must": [
            {"match": {"title": "手机"}},
            {"term": {"is_active": true}},
            {"range": {"stock": {"gt": 0}}}
          ],
          "filter": [
            {"range": {"price": {"gte": 1000, "lte": 5000}}},
            {"term": {"category": "手机数码"}}
          ]
        }
      },
      "functions": [
        {
          "field_value_factor": {
            "field": "sales_count",
            "factor": 0.001,
            "modifier": "log1p"
          }
        },
        {
          "field_value_factor": {
            "field": "rating",
            "factor": 0.2,
            "modifier": "sqrt"
          }
        },
        {
          "gauss": {
            "price": {
              "origin": "2000",
              "scale": "500",
              "decay": 0.5
            }
          }
        }
      ],
      "score_mode": "sum",
      "boost_mode": "multiply"
    }
  },
  "aggs": {
    "brands": {"terms": {"field": "brand", "size": 20}},
    "price_range": {
      "range": {
        "field": "price",
        "ranges": [
          {"to": 1000}, {"from": 1000, "to": 3000},
          {"from": 3000, "to": 5000}, {"from": 5000}
        ]
      }
    }
  },
  "from": 0,
  "size": 20
}
```

**电商搜索关键优化点：**
- 用 `copy_to` 合并多字段，减少 `multi_match` 开销
- 筛选条件走 `filter`（可缓存），不参与相关性评分
- `function_score` 融合销量、评分、价格距离等业务因素
- 聚合用于左侧筛选栏（品牌、价格区间、属性）
- 避免深分页，搜索结果一般不超过 100 页

###### 4. 如何实现一个实时监控系统？

**架构：Metricbeat/自定义 Exporter → ES → Kibana Dashboard + Watcher 告警**

**写入指标数据：**

```json
POST /metrics-app-2024.03.15/_doc
{
  "@timestamp": "2024-03-15T10:30:00Z",
  "service": "order-service",
  "host": "192.168.1.10",
  "cpu_usage": 78.5,
  "memory_usage": 65.2,
  "request_count": 1250,
  "error_count": 15,
  "avg_response_time": 245.6
}
```

**实时聚合查询（近5分钟错误率）：**

```json
GET /metrics-*/_search
{
  "query": {
    "range": {
      "@timestamp": {"gte": "now-5m"}
    }
  },
  "aggs": {
    "by_service": {
      "terms": {"field": "service"},
      "aggs": {
        "error_rate": {
          "bucket_script": {
            "buckets_path": {
              "errors": "total_errors",
              "total": "total_requests"
            },
            "script": "params.errors / params.total * 100"
          }
        },
        "total_errors": {"sum": {"field": "error_count"}},
        "total_requests": {"sum": {"field": "request_count"}}
      }
    }
  },
  "size": 0
}
```

**Watcher 告警（错误率超阈值自动通知）：**

```json
PUT /_watcher/watch/high-error-rate-alert
{
  "trigger": {
    "schedule": {"interval": "1m"}
  },
  "input": {
    "search": {
      "request": {
        "indices": ["metrics-*"],
        "body": {
          "query": {"range": {"@timestamp": {"gte": "now-5m"}}},
          "aggs": {
            "error_rate": {
              "avg": {"script": "doc['error_count'].value / doc['request_count'].value * 100"}
            }
          },
          "size": 0
        }
      }
    }
  },
  "condition": {
    "compare": {
      "ctx.payload.aggregations.error_rate.value": {"gt": 5.0}
    }
  },
  "actions": {
    "send_email": {
      "email": {
        "to": ["ops@example.com"],
        "subject": "【告警】服务错误率超过 5%",
        "body": "当前错误率：{{ctx.payload.aggregations.error_rate.value}}%"
      }
    }
  }
}
```

###### 5. Elasticsearch 在推荐系统中有哪些应用？

**ES 在推荐系统中主要承担候选召回和排序两个角色。**

**应用场景：**

| 场景 | ES 实现方式 |
|------|------------|
| 相似内容推荐 | `more_like_this` 查询 |
| 协同过滤候选召回 | 向量 kNN 搜索 |
| 基于标签/兴趣推荐 | 用户画像 + bool 查询 |
| 热门推荐 | function_score + 时间衰减 |

**相似内容推荐（More Like This）：**

```json
GET /articles/_search
{
  "query": {
    "more_like_this": {
      "fields": ["title", "content", "tags"],
      "like": [{"_index": "articles", "_id": "1"}],
      "min_term_freq": 1,
      "max_query_terms": 20,
      "min_doc_freq": 2
    }
  }
}
```

**基于用户兴趣的个性化推荐：**

```json
GET /articles/_search
{
  "query": {
    "function_score": {
      "query": {
        "bool": {
          "should": [
            {"terms": {"tags": ["java", "spring", "microservice"], "boost": 2.0}},
            {"terms": {"category": ["后端", "架构"]}}
          ]
        }
      },
      "functions": [
        {
          "gauss": {
            "publish_time": {
              "origin": "now",
              "scale": "7d",
              "decay": 0.5
            }
          }
        }
      ]
    }
  }
}
```

**向量召回（基于 Embedding 的语义推荐）：**

```json
GET /products/_search
{
  "knn": {
    "field": "item_embedding",
    "query_vector": [0.1, -0.3, 0.5, ...],
    "k": 50,
    "num_candidates": 200
  }
}
```

###### 6. 如何使用 Elasticsearch 构建知识图谱？

**知识图谱核心：实体 + 关系 + 属性。ES 适合存储和查询实体，关系查询建议结合图数据库（Neo4j）。**

**实体存储（使用嵌套表示关系）：**

```json
PUT /knowledge_graph
{
  "mappings": {
    "properties": {
      "entity_id": {"type": "keyword"},
      "entity_type": {"type": "keyword"},
      "name": {"type": "text", "analyzer": "ik_max_word", "fields": {"keyword": {"type": "keyword"}}},
      "aliases": {"type": "keyword"},
      "description": {"type": "text"},
      "properties": {"type": "object", "dynamic": true},
      "relations": {
        "type": "nested",
        "properties": {
          "relation_type": {"type": "keyword"},
          "target_entity_id": {"type": "keyword"},
          "target_entity_name": {"type": "keyword"},
          "weight": {"type": "float"}
        }
      }
    }
  }
}
```

**查询某实体的所有关联实体：**

```json
GET /knowledge_graph/_search
{
  "query": {
    "nested": {
      "path": "relations",
      "query": {
        "bool": {
          "must": [
            {"term": {"relations.target_entity_id": "E001"}},
            {"term": {"relations.relation_type": "同事"}}
          ]
        }
      },
      "inner_hits": {}
    }
  }
}
```

**ES 在知识图谱中的优势**：全文搜索实体名称（别名、近义词）、高效属性过滤、结合向量搜索支持语义实体链接。

###### 7. 如何实现实时数据分析和可视化？

**架构：数据流 → ES（Data Stream）→ Kibana Dashboard / Grafana**

**实时写入（使用 Bulk API 批量高效写入）：**

```java
// Spring Boot + ES Java Client
BulkRequest.Builder bulkBuilder = new BulkRequest.Builder();
for (MetricEvent event : events) {
    bulkBuilder.operations(op -> op
        .index(idx -> idx
            .index("metrics-app")
            .document(event)
        )
    );
}
BulkResponse response = esClient.bulk(bulkBuilder.build());
```

**实时聚合（按分钟统计 QPS）：**

```json
GET /metrics-app/_search
{
  "query": {
    "range": {"@timestamp": {"gte": "now-1h"}}
  },
  "aggs": {
    "qps_over_time": {
      "date_histogram": {
        "field": "@timestamp",
        "fixed_interval": "1m"
      },
      "aggs": {
        "total_requests": {"sum": {"field": "request_count"}},
        "avg_latency": {"avg": {"field": "response_time"}}
      }
    }
  },
  "size": 0
}
```

**Kibana 可视化核心功能：**
- **Discover**：日志实时探索
- **Lens**：拖拽式图表（折线、饼图、热力图）
- **Dashboard**：多图表组合展示，支持时间范围联动
- **Canvas**：定制化报表展示
- **Maps**：地理位置可视化

**Grafana 集成 ES（使用 ES Data Source）：**
- 支持 Lucene 查询语法
- 支持 ES 聚合转折线图、柱状图
- 适合运维监控场景（与 Prometheus 面板统一）

###### 8. Elasticsearch 与 Logstash、Kibana 如何配合使用（ELK Stack）？

**ELK Stack 是日志分析领域最主流的开源方案，三个组件各司其职：**

```
Logstash（采集 & 处理）→ Elasticsearch（存储 & 搜索）→ Kibana（可视化）
```

**现代 ELK 架构（加入 Beats + Kafka）：**

```
应用服务器
    ↓ Filebeat（轻量，仅采集转发）
    ↓ Kafka（消息队列，削峰解耦）
    ↓ Logstash（过滤、解析、转换）
    ↓ Elasticsearch（存储、倒排索引）
    ↓ Kibana（Dashboard、Discover、告警）
```

**Logstash 配置示例（nginx 日志解析）：**

```ruby
input {
  kafka {
    bootstrap_servers => "kafka:9092"
    topics => ["nginx-logs"]
    codec => "json"
  }
}

filter {
  grok {
    match => { "message" => "%{COMBINEDAPACHELOG}" }
  }
  date {
    match => ["timestamp", "dd/MMM/yyyy:HH:mm:ss Z"]
    target => "@timestamp"
  }
  geoip {
    source => "clientip"
  }
  mutate {
    remove_field => ["message", "timestamp"]
    convert => { "bytes" => "integer" }
  }
}

output {
  elasticsearch {
    hosts => ["http://es:9200"]
    index => "nginx-logs-%{+YYYY.MM.dd}"
    user => "elastic"
    password => "${ES_PASSWORD}"
  }
}
```

**三者分工总结：**

| 组件 | 职责 | 关键能力 |
|------|------|---------|
| **Beats** | 轻量采集 | Filebeat/Metricbeat/Packetbeat，边车部署 |
| **Logstash** | 重度处理 | Grok 解析/GeoIP/多输入输出/插件生态 |
| **Elasticsearch** | 存储检索 | 倒排索引/聚合/全文搜索 |
| **Kibana** | 可视化 | Dashboard/Discover/Watcher/ML |

**记忆口诀**：Beats 采、Kafka 缓、Logstash 洗、ES 存、Kibana 看。

---

> **本章追问：ELK Stack 中 Filebeat 和 Logstash 各自的定位？什么时候只用 Filebeat 直接写 ES？**
>
> - **Filebeat**：轻量级，Go 编写，CPU/内存消耗极低，适合每台服务器部署。只能做简单过滤，复杂解析能力弱。
> - **Logstash**：重量级，JVM 运行，内存消耗大（默认 1G 堆），但支持 200+ 插件，Grok/Mutate/GeoIP 等解析能力强。
>
> **选择策略**：
> - 日志格式简单、量不大 → Filebeat + Ingest Pipeline（ES 内置处理）直写 ES，省去 Logstash
> - 日志格式复杂、需多源聚合、需输出到多个目标 → Filebeat → Kafka → Logstash → ES
