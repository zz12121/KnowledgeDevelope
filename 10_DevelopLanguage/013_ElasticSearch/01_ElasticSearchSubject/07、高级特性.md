###### 1. 嵌套查询和父子查询有什么区别？分别适用于什么场景？

**核心区别：存储方式不同，查询方式不同。**

| 对比项 | 嵌套（Nested） | 父子（Join） |
|--------|---------------|-------------|
| 存储方式 | 子文档与父文档存在同一个 Lucene 文档块中 | 父子文档是独立文档，靠路由关联 |
| 关系变动 | 修改子文档需要重建整个父文档 | 子文档可独立修改，不影响父文档 |
| 查询性能 | 较好（同 Segment，减少 I/O） | 较差（需要 global ordinals，内存开销大） |
| 适用场景 | 子文档不频繁变动（如订单明细） | 子文档频繁更新（如评论、标签） |

**嵌套查询示例（文章+标签）：**

```json
// Mapping
PUT /articles
{
  "mappings": {
    "properties": {
      "title": {"type": "text"},
      "tags": {
        "type": "nested",
        "properties": {
          "name": {"type": "keyword"},
          "weight": {"type": "float"}
        }
      }
    }
  }
}

// 查询
GET /articles/_search
{
  "query": {
    "nested": {
      "path": "tags",
      "query": {
        "bool": {
          "must": [
            {"term": {"tags.name": "elasticsearch"}},
            {"range": {"tags.weight": {"gte": 0.8}}}
          ]
        }
      }
    }
  }
}
```

**父子查询示例（商品+评论）：**

```json
// Mapping
PUT /products
{
  "mappings": {
    "properties": {
      "join_field": {
        "type": "join",
        "relations": {"product": "review"}
      }
    }
  }
}

// 查询有好评的商品（has_child）
GET /products/_search
{
  "query": {
    "has_child": {
      "type": "review",
      "query": {"term": {"rating": 5}}
    }
  }
}
```

**记忆口诀**：嵌套适合"一起改"，父子适合"独立改"。

###### 2. 如何实现跨集群搜索？有什么应用场景？

**跨集群搜索（Cross-Cluster Search，CCS）**允许一个集群查询另一个集群的数据，无需数据迁移。

**配置远程集群（elasticsearch.yml 或动态 API）：**

```json
PUT /_cluster/settings
{
  "persistent": {
    "cluster.remote.cluster_a.seeds": ["192.168.1.10:9300"],
    "cluster.remote.cluster_b.seeds": ["192.168.1.20:9300"],
    "cluster.remote.cluster_a.skip_unavailable": true
  }
}
```

**跨集群查询（语法：`远程集群名:索引名`）：**

```json
GET /cluster_a:logs-2024-*,cluster_b:logs-2024-*/_search
{
  "query": {
    "match": {"message": "error"}
  }
}
```

**应用场景：**
- **多数据中心部署**：每个机房一个集群，统一查询
- **按时间分集群**：冷热数据分离，历史数据在远程集群
- **多租户隔离**：每个业务单元一个集群，支持跨租户查询
- **灾备架构**：主集群故障时切换到备用集群查询

###### 3. 什么是 ES|QL？它的应用场景是什么？

**ES|QL（Elasticsearch Query Language）** 是 ES 8.11 引入的新一代查询语言，使用管道语法（类似 Unix 管道）。

**语法示例：**

```
POST /_query
{
  "query": """
    FROM logs-*
    | WHERE level == "ERROR"
    | STATS count = COUNT(*), avg_duration = AVG(duration) BY service
    | WHERE count > 100
    | SORT count DESC
    | LIMIT 10
  """
}
```

**核心操作符：**

| 操作符 | 说明 |
|--------|------|
| `FROM` | 指定数据源（索引） |
| `WHERE` | 过滤条件 |
| `STATS` | 聚合统计 |
| `SORT` | 排序 |
| `LIMIT` | 限制结果数 |
| `EVAL` | 计算新字段 |
| `KEEP/DROP` | 选择/丢弃字段 |

**应用场景：**
- 日志分析（比 KQL 更强大的聚合能力）
- 安全事件分析（SIEM）
- 实时监控指标计算
- Kibana Discover 快速数据探索

> **与 Query DSL 的区别**：ES|QL 更适合分析查询，DSL 更适合应用程序集成。

###### 4. Elasticsearch 有哪些机器学习功能？

**ES ML 功能分三类：**

**① 异常检测（Anomaly Detection）**：
- 自动学习数据规律，检测偏离正常的行为
- 适合：监控指标异常、用户行为异常、日志异常模式

**② 数据帧分析（Data Frame Analytics）**：
- 支持异常值检测（Outlier Detection）
- 回归分析（Regression）
- 分类（Classification）

**③ NLP 与推断（Inference）**：
- 支持导入 PyTorch 模型
- 命名实体识别（NER）
- 情感分析
- 文本分类
- 稀疏向量搜索（ELSER）

**使用 ML Job 示例：**

```json
PUT _ml/anomaly_detectors/high_error_rate
{
  "description": "高错误率检测",
  "analysis_config": {
    "bucket_span": "5m",
    "detectors": [{
      "detector_description": "high error count",
      "function": "high_count",
      "partition_field_name": "service"
    }]
  },
  "data_description": {
    "time_field": "@timestamp"
  }
}
```

**注意**：ML 功能需要白金（Platinum）许可证，部分功能（如 ELSER）在企业版可用。

###### 5. 什么是 Ingest Pipeline？如何使用它进行数据预处理？

**Ingest Pipeline 是 ES 内置的 ETL 工具**，文档写入前在 ingest 节点上执行一系列 Processor 进行预处理。

**常用 Processor：**

| Processor | 说明 |
|-----------|------|
| `grok` | 正则解析非结构化日志 |
| `date` | 解析日期格式 |
| `set` | 设置字段值 |
| `rename` | 重命名字段 |
| `remove` | 删除字段 |
| `convert` | 类型转换 |
| `split` | 字符串分割 |
| `geoip` | IP 地址转地理位置 |
| `user_agent` | 解析 User-Agent |

**创建 Pipeline：**

```json
PUT /_ingest/pipeline/nginx-log-pipeline
{
  "description": "解析 Nginx 访问日志",
  "processors": [
    {
      "grok": {
        "field": "message",
        "patterns": ["%{NGINXACCESS}"],
        "pattern_definitions": {
          "NGINXACCESS": "%{IPORHOST:client_ip} - %{USER:user} \\[%{HTTPDATE:timestamp}\\] \"%{WORD:method} %{URIPATHPARAM:path} HTTP/%{NUMBER:http_version}\" %{NUMBER:status_code} %{NUMBER:bytes}"
        }
      }
    },
    {
      "date": {
        "field": "timestamp",
        "formats": ["dd/MMM/yyyy:HH:mm:ss Z"]
      }
    },
    {
      "geoip": {
        "field": "client_ip"
      }
    },
    {
      "convert": {
        "field": "status_code",
        "type": "integer"
      }
    },
    {
      "remove": {
        "field": "message"
      }
    }
  ]
}
```

**使用 Pipeline 写入：**

```bash
POST /nginx-logs/_doc?pipeline=nginx-log-pipeline
{"message": "192.168.1.1 - alice [15/Mar/2024:10:30:00 +0800] \"GET /api/v1/users HTTP/1.1\" 200 1234"}
```

**在索引 Mapping 中设置默认 Pipeline：**

```json
PUT /nginx-logs
{
  "settings": {
    "default_pipeline": "nginx-log-pipeline"
  }
}
```

###### 6. 什么是 Painless 脚本？它有哪些应用场景？

**Painless 是 ES 内置的安全脚本语言**，语法类似 Java/Groovy，专为 ES 设计，性能好且安全（沙箱隔离）。

**应用场景 1：查询时动态计算字段（Script Query）**

```json
GET /products/_search
{
  "query": {
    "script_score": {
      "query": {"match_all": {}},
      "script": {
        "source": "doc['price'].value * params.discount",
        "params": {"discount": 0.8}
      }
    }
  }
}
```

**应用场景 2：更新文档时动态修改字段（Update by Script）**

```json
POST /products/_update/1
{
  "script": {
    "source": "ctx._source.stock -= params.quantity; if (ctx._source.stock < 0) { ctx.op = 'delete' }",
    "params": {"quantity": 5}
  }
}
```

**应用场景 3：聚合时动态计算（Scripted Metric Aggregation）**

```json
GET /orders/_search
{
  "aggs": {
    "total_revenue": {
      "scripted_metric": {
        "init_script": "state.revenues = []",
        "map_script": "state.revenues.add(doc['price'].value * doc['quantity'].value)",
        "combine_script": "double total = 0; for (r in state.revenues) { total += r } return total",
        "reduce_script": "double total = 0; for (a in states) { total += a } return total"
      }
    }
  }
}
```

**注意**：Painless 脚本会绕过 Mapping 类型检查，使用时需小心性能问题，建议用 `params` 传参而不是硬编码，提高脚本缓存命中率。

###### 7. 如何实现地理位置搜索？

**ES 支持两种地理类型：`geo_point`（点）和 `geo_shape`（形状）。**

**Mapping 定义：**

```json
PUT /restaurants
{
  "mappings": {
    "properties": {
      "name": {"type": "text"},
      "location": {"type": "geo_point"}
    }
  }
}
```

**写入数据：**

```json
POST /restaurants/_doc/1
{
  "name": "海底捞",
  "location": {"lat": 39.9042, "lon": 116.4074}
}
```

**地理查询类型：**

```json
// 1. 距离查询（某点附近 N 公里内）
GET /restaurants/_search
{
  "query": {
    "geo_distance": {
      "distance": "5km",
      "location": {"lat": 39.9, "lon": 116.4}
    }
  },
  "sort": [
    {
      "_geo_distance": {
        "location": {"lat": 39.9, "lon": 116.4},
        "order": "asc",
        "unit": "km"
      }
    }
  ]
}

// 2. 边界框查询（矩形区域内）
GET /restaurants/_search
{
  "query": {
    "geo_bounding_box": {
      "location": {
        "top_left": {"lat": 40.0, "lon": 116.0},
        "bottom_right": {"lat": 39.8, "lon": 116.8}
      }
    }
  }
}
```

###### 8. 什么是向量搜索？如何在 Elasticsearch 中实现？

**向量搜索（Vector Search）** 又叫语义搜索，通过将文本/图片转成高维向量，使用近似最近邻（ANN）算法找到语义相似的内容，而不是关键字匹配。

**使用 `dense_vector` 字段类型：**

```json
PUT /articles
{
  "mappings": {
    "properties": {
      "title": {"type": "text"},
      "content": {"type": "text"},
      "embedding": {
        "type": "dense_vector",
        "dims": 768,
        "index": true,
        "similarity": "cosine"
      }
    }
  }
}
```

**写入向量数据（embedding 由外部模型生成）：**

```json
POST /articles/_doc/1
{
  "title": "Elasticsearch 向量搜索",
  "embedding": [0.1, 0.2, -0.3, ...]  // 768维向量
}
```

**kNN 查询：**

```json
GET /articles/_search
{
  "knn": {
    "field": "embedding",
    "query_vector": [0.15, 0.18, -0.28, ...],
    "k": 10,
    "num_candidates": 100
  }
}
```

**混合检索（向量 + 关键字，Hybrid Search）：**

```json
GET /articles/_search
{
  "query": {
    "match": {"title": "向量搜索"}
  },
  "knn": {
    "field": "embedding",
    "query_vector": [...],
    "k": 10,
    "num_candidates": 100,
    "boost": 0.7
  }
}
```

**实践流程**：文本 → Sentence Transformers/OpenAI Embedding API → 向量 → 存入 ES → kNN 查询。

###### 9. 如何实现同义词搜索？

**同义词配置在分析器层面**，分词时自动将同义词等价处理。

**方式一：配置文件方式**

```json
PUT /products
{
  "settings": {
    "analysis": {
      "filter": {
        "my_synonyms": {
          "type": "synonym",
          "synonyms_path": "analysis/synonyms.txt"
        }
      },
      "analyzer": {
        "synonym_analyzer": {
          "tokenizer": "standard",
          "filter": ["lowercase", "my_synonyms"]
        }
      }
    }
  },
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "synonym_analyzer"
      }
    }
  }
}
```

**synonyms.txt 内容：**

```
手机, 移动电话, 手持设备
笔记本, 笔记本电脑, 便携电脑
ES, Elasticsearch, 弹性搜索
```

**方式二：内联方式（适合少量同义词）：**

```json
"my_synonyms": {
  "type": "synonym",
  "synonyms": [
    "手机, 移动电话, 手持设备",
    "ES => Elasticsearch"
  ]
}
```

**注意**：`=>` 是单向映射（搜索"ES"等于搜"Elasticsearch"），逗号分隔是双向等价。

###### 10. 什么是 Runtime Fields？它的优势是什么？

**Runtime Fields（运行时字段）** 是在查询时动态计算的字段，不存储在索引中。

**定义 Runtime Field（在 Mapping 中）：**

```json
PUT /orders
{
  "mappings": {
    "runtime": {
      "total_price": {
        "type": "double",
        "script": {
          "source": "emit(doc['price'].value * doc['quantity'].value)"
        }
      },
      "day_of_week": {
        "type": "keyword",
        "script": {
          "source": "emit(doc['@timestamp'].value.dayOfWeekEnum.getDisplayName(TextStyle.FULL, Locale.ROOT))"
        }
      }
    },
    "properties": {
      "price": {"type": "double"},
      "quantity": {"type": "integer"}
    }
  }
}
```

**查询时临时定义 Runtime Field：**

```json
GET /orders/_search
{
  "runtime_mappings": {
    "discount_price": {
      "type": "double",
      "script": "emit(doc['price'].value * 0.9)"
    }
  },
  "aggs": {
    "avg_discount": {
      "avg": {"field": "discount_price"}
    }
  }
}
```

**优势：**
- **Schema 灵活**：无需重建索引就能添加新字段
- **探索性分析**：数据分析阶段快速验证字段逻辑
- **减少存储**：计算字段不占磁盘空间
- **快速迭代**：业务逻辑变化时无需 Reindex

**劣势**：每次查询都要计算，性能不如存储字段，生产环境确认逻辑后建议转为存储字段。

###### 11. 如何实现多语言搜索？

**多语言搜索的核心是为每种语言配置对应的分析器。**

**方案一：每种语言一个字段（推荐）**

```json
PUT /articles
{
  "mappings": {
    "properties": {
      "title": {
        "type": "text",
        "analyzer": "standard"
      },
      "title_zh": {
        "type": "text",
        "analyzer": "ik_max_word"
      },
      "title_en": {
        "type": "text",
        "analyzer": "english"
      },
      "title_ja": {
        "type": "text",
        "analyzer": "kuromoji"
      }
    }
  }
}
```

**查询时按语言选择字段：**

```json
GET /articles/_search
{
  "query": {
    "multi_match": {
      "query": "elasticsearch",
      "fields": ["title_en^3", "title^1"]
    }
  }
}
```

**方案二：语言检测 + Ingest Pipeline 自动路由**

通过 `langdetect` 插件或外部服务检测语言，Pipeline 自动写入对应字段。

**中文分词推荐**：IK Analyzer（`ik_max_word` 精细分词 / `ik_smart` 智能分词）。

###### 12. 什么是 Data Stream？它适用于什么场景？

**Data Stream 是专为时序数据设计的索引管理方案**，由多个隐藏的后端索引（backing indices）组成，对外提供统一的虚拟索引名。

**核心特性：**
- 写入永远进入最新的索引（write index）
- 查询自动路由到所有后端索引
- 强制要求文档包含 `@timestamp` 字段
- 天然与 ILM（索引生命周期管理）集成

**创建 Data Stream：**

```json
// 1. 创建索引模板
PUT /_index_template/logs-template
{
  "index_patterns": ["logs-*"],
  "data_stream": {},
  "template": {
    "settings": {
      "number_of_shards": 1,
      "number_of_replicas": 1
    },
    "mappings": {
      "properties": {
        "@timestamp": {"type": "date"},
        "level": {"type": "keyword"},
        "message": {"type": "text"}
      }
    }
  }
}

// 2. 写入数据（自动创建 Data Stream）
POST /logs-app/_doc
{
  "@timestamp": "2024-03-15T10:30:00Z",
  "level": "ERROR",
  "message": "Connection refused"
}
```

**查看 Data Stream 状态：**

```json
GET /_data_stream/logs-app
```

**适用场景：**
- 应用日志（Logstash/Filebeat 写入）
- 监控指标（Metricbeat）
- 安全事件（Winlogbeat、Packetbeat）
- 任何只追加写入、按时间查询的场景

**记忆口诀**：Data Stream = 时序数据 + 自动 ILM + 统一入口，是 ES ECS（Elastic Common Schema）架构的基础。

> **本章追问：kNN 向量搜索和传统倒排索引搜索如何混合使用？**
>
> ES 8.x 支持原生 Hybrid Search，在同一个请求中同时执行 `query`（倒排）和 `knn`（向量），两个查询独立打分后通过 `boost` 参数加权合并。
> 更精细的融合可以使用 RRF（Reciprocal Rank Fusion）算法：
> ```json
> GET /articles/_search
> {
>   "retriever": {
>     "rrf": {
>       "retrievers": [
>         {"standard": {"query": {"match": {"title": "search"}}}},
>         {"knn": {"field": "embedding", "query_vector": [...], "k": 10}}
>       ],
>       "rank_window_size": 50,
>       "rank_constant": 20
>     }
>   }
> }
> ```
> RRF 不需要归一化两个分数，融合效果通常优于简单加权。
