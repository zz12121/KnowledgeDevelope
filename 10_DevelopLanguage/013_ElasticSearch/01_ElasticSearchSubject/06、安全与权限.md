###### 1. Elasticsearch 有哪些安全特性？如何配置？

**ES 的安全功能分三层：**

| 层次 | 功能 | 说明 |
|------|------|------|
| 传输层 | SSL/TLS | 节点间通信加密 |
| 认证层 | 用户名密码 / API Key / PKI / LDAP | 身份验证 |
| 授权层 | RBAC / 字段级安全 / 文档级安全 | 访问控制 |

**基础安全配置（elasticsearch.yml）：**

```yaml
xpack.security.enabled: true
xpack.security.transport.ssl.enabled: true
xpack.security.transport.ssl.verification_mode: certificate
xpack.security.transport.ssl.keystore.path: elastic-certificates.p12
xpack.security.transport.ssl.truststore.path: elastic-certificates.p12

xpack.security.http.ssl.enabled: true
xpack.security.http.ssl.keystore.path: http.p12
```

**快速启用：**

```bash
# 生成 CA 证书
bin/elasticsearch-certutil ca
# 生成节点证书
bin/elasticsearch-certutil cert --ca elastic-stack-ca.p12
# 设置内置用户密码（interactive 模式）
bin/elasticsearch-setup-passwords interactive
```

**记忆口诀**：传输加密 + 用户认证 + 角色授权 = ES 三层安全。

> **追问：ES 8.x 和 7.x 安全配置的区别？**
> 
> ES 8.x 默认开启安全功能（security enabled by default），无需手动配置 `xpack.security.enabled: true`。ES 7.x 需要明确开启。8.x 还默认启用了 HTTPS，首次启动会自动生成 enrollment token，更适合生产环境。

###### 2. 如何配置基于角色的访问控制（RBAC）？

**核心概念：用户 → 角色 → 权限**

**RBAC 三要素：**
- **User（用户）**：认证主体，可以是内置用户或自定义用户
- **Role（角色）**：权限集合，包含集群权限、索引权限、应用权限
- **Privilege（权限）**：具体操作，如 `read`、`write`、`create_index` 等

**创建角色（Kibana Dev Tools）：**

```json
PUT /_security/role/log_reader
{
  "cluster": [],
  "indices": [
    {
      "names": ["logs-*"],
      "privileges": ["read", "view_index_metadata"],
      "field_security": {
        "grant": ["@timestamp", "level", "message", "host"]
      },
      "query": "{\"match\": {\"level\": \"ERROR\"}}"
    }
  ]
}
```

**创建用户并分配角色：**

```json
POST /_security/user/alice
{
  "password": "SecurePass123!",
  "roles": ["log_reader"],
  "full_name": "Alice Smith",
  "email": "alice@example.com"
}
```

**常用内置角色：**

| 角色 | 权限 |
|------|------|
| `superuser` | 所有权限 |
| `kibana_admin` | Kibana 管理 |
| `monitor` | 集群监控（只读） |
| `read` | 索引读取 |
| `read_cross_cluster` | 跨集群读取 |

###### 3. 如何启用 SSL/TLS 加密？

**分两种：节点间通信（Transport TLS）+ HTTP 通信（HTTP TLS）**

**步骤一：生成证书**

```bash
# 生成 CA
bin/elasticsearch-certutil ca --out /etc/elasticsearch/certs/elastic-stack-ca.p12 --pass ""

# 为每个节点生成证书
bin/elasticsearch-certutil cert \
  --ca /etc/elasticsearch/certs/elastic-stack-ca.p12 \
  --ca-pass "" \
  --out /etc/elasticsearch/certs/elastic-certificates.p12 \
  --pass ""
```

**步骤二：配置 elasticsearch.yml**

```yaml
# Transport 层（节点间）
xpack.security.transport.ssl.enabled: true
xpack.security.transport.ssl.verification_mode: certificate
xpack.security.transport.ssl.keystore.path: certs/elastic-certificates.p12
xpack.security.transport.ssl.truststore.path: certs/elastic-certificates.p12

# HTTP 层（客户端）
xpack.security.http.ssl.enabled: true
xpack.security.http.ssl.keystore.path: certs/http.p12
```

**步骤三：验证**

```bash
curl --cacert /etc/elasticsearch/certs/ca.crt \
  -u elastic:password \
  https://localhost:9200/_cluster/health
```

**记忆口诀**：Transport 加密防节点窃听，HTTP 加密防客户端中间人。

###### 4. 如何配置 API 密钥认证？

**API Key 是推荐的程序访问方式**，比用户名密码更安全（可以限制权限范围、设置过期时间）。

**创建 API Key：**

```json
POST /_security/api_key
{
  "name": "my-app-key",
  "expiration": "30d",
  "role_descriptors": {
    "app-role": {
      "cluster": ["monitor"],
      "indices": [
        {
          "names": ["products-*"],
          "privileges": ["read", "write"]
        }
      ]
    }
  }
}
```

**响应：**

```json
{
  "id": "VuaCfGcBCdbkIjaNgtVm",
  "name": "my-app-key",
  "expiration": 1829460778001,
  "api_key": "ui2lp2axTNmsyakw9tvNnw",
  "encoded": "VnVhQ2ZHY0JDZGJrSWpxeUd0Vm06dWkybHAyYXhUTm1zeWFrdzl0dk5udw=="
}
```

**使用 API Key（HTTP Header）：**

```bash
curl -H "Authorization: ApiKey VnVhQ2ZHY0JDZGJrSWpxeUd0Vm06dWkybHAyYXhUTm1zeWFrdzl0dk5udw==" \
  https://localhost:9200/products-*/_search
```

**管理 API Key：**

```json
# 查看 API Key
GET /_security/api_key?name=my-app-key

# 使 API Key 失效
DELETE /_security/api_key
{
  "ids": ["VuaCfGcBCdbkIjaNgtVm"]
}
```

###### 5. 如何实现字段级和文档级安全？

**字段级安全（Field-Level Security，FLS）**：控制用户能看到哪些字段。
**文档级安全（Document-Level Security，DLS）**：控制用户能看到哪些文档。

**字段级安全配置：**

```json
PUT /_security/role/partial_reader
{
  "indices": [
    {
      "names": ["employee-*"],
      "privileges": ["read"],
      "field_security": {
        "grant": ["name", "department", "job_title"],
        "except": ["salary", "ssn", "bank_account"]
      }
    }
  ]
}
```

**文档级安全配置（基于 Query）：**

```json
PUT /_security/role/dept_reader
{
  "indices": [
    {
      "names": ["employee-*"],
      "privileges": ["read"],
      "query": {
        "term": {
          "department": "engineering"
        }
      }
    }
  ]
}
```

**注意事项：**
- FLS + DLS 组合使用时会有一定性能开销（每次查询额外过滤）
- DLS 使用的 query 支持所有标准 Query DSL 语法
- 如果字段被 FLS 排除，则聚合等操作也无法访问该字段
- FLS/DLS 需要 X-Pack 安全许可证（7.x+ 基础许可证免费包含）

###### 6. 如何进行审计日志配置？

**审计日志记录谁在什么时间做了什么操作**，满足合规要求。

**启用审计日志（elasticsearch.yml）：**

```yaml
xpack.security.audit.enabled: true
xpack.security.audit.logfile.events.include:
  - authentication_success
  - authentication_failed
  - access_denied
  - connection_denied
  - anonymous_access_denied
  - run_as_denied
  - index_event
xpack.security.audit.logfile.events.exclude:
  - "system_access_granted"
```

**审计日志输出（logs/elasticsearch_audit.json）：**

```json
{
  "@timestamp": "2024-01-15T10:30:00.000Z",
  "event.type": "access_denied",
  "user.name": "alice",
  "user.roles": ["log_reader"],
  "request.method": "DELETE",
  "url.path": "/products/_doc/1",
  "source.ip": "192.168.1.100"
}
```

**常用审计事件：**

| 事件 | 说明 |
|------|------|
| `authentication_success` | 认证成功 |
| `authentication_failed` | 认证失败 |
| `access_denied` | 授权拒绝 |
| `index_event` | 索引操作（增删改） |
| `search` | 搜索操作 |

###### 7. 如何与 LDAP/AD 集成进行身份验证？

**场景**：企业内部使用 Active Directory 或 LDAP 统一管理用户，ES 接入后用域账号登录。

**LDAP 集成配置（elasticsearch.yml）：**

```yaml
xpack.security.authc.realms.ldap.ldap1:
  order: 0
  url: "ldaps://ldap.example.com:636"
  bind_dn: "cn=es-service,ou=services,dc=example,dc=com"
  user_search:
    base_dn: "ou=users,dc=example,dc=com"
    filter: "(uid={0})"
    scope: sub_tree
  group_search:
    base_dn: "ou=groups,dc=example,dc=com"
    scope: sub_tree
  ssl.certificate_authorities: ["/etc/ssl/certs/ldap-ca.crt"]
```

**设置 LDAP 绑定密码（安全存储，不写 yml）：**

```bash
bin/elasticsearch-keystore add xpack.security.authc.realms.ldap.ldap1.secure_bind_password
```

**LDAP 组映射到 ES 角色：**

```json
PUT /_security/role_mapping/ldap_admin_mapping
{
  "roles": ["superuser"],
  "rules": {
    "field": {
      "groups": "cn=es-admins,ou=groups,dc=example,dc=com"
    }
  },
  "enabled": true
}
```

**Active Directory 类似，只是用 `active_directory` 域类型，配置稍有不同。**

###### 8. 如何防止 Elasticsearch 集群被未授权访问？

**生产环境 ES 安全加固清单：**

**网络层：**
```yaml
# 绑定具体 IP，不要绑定 0.0.0.0
network.host: 192.168.1.10
# 禁止 HTTP 对外暴露，通过 Nginx 反向代理
http.host: 127.0.0.1
```

**防火墙规则：**
```bash
# 只允许应用服务器访问 9200/9300
iptables -A INPUT -p tcp --dport 9200 -s 192.168.1.0/24 -j ACCEPT
iptables -A INPUT -p tcp --dport 9200 -j DROP
```

**启用安全功能：**
```yaml
xpack.security.enabled: true
xpack.security.http.ssl.enabled: true
xpack.security.transport.ssl.enabled: true
```

**其他加固手段：**
- 修改默认 `elastic` 超级用户密码为强密码
- 禁用不需要的内置用户（`apm_system`、`beats_system` 等）
- 设置 IP 过滤（`xpack.security.transport.filter`）
- 开启审计日志监控异常访问
- Kibana 放在内网，通过 VPN 访问

**记忆口诀**：网络隔离 + TLS 加密 + RBAC 认证 + 审计日志 = 四层防护。

> **本章追问：ES 基础许可证（Basic License）包含哪些安全功能？**
>
> 从 ES 7.1 开始，基础许可证（免费）已包含核心安全功能：
> - TLS/SSL 加密（Transport + HTTP）
> - 用户名密码认证
> - 基本 RBAC（角色权限管理）
> - 字段级安全（FLS）
> - 文档级安全（DLS）
> - API Key 认证
>
> 需要付费的白金许可证才包含：LDAP/AD 集成、SAML SSO、高级审计日志、IP 过滤等。
