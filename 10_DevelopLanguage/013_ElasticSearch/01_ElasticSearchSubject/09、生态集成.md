###### 1. Elasticsearch 如何与 Kafka 集成？

**ES + Kafka 是生产环境标准搭配：Kafka 作缓冲削峰，ES 负责存储检索。**

**集成方案一：Kafka Connector（推荐）**

使用 Confluent 官方的 `elasticsearch-sink-connector`，无需写代码：

```json
POST /connectors
{
  "name": "es-sink-connector",
  "config": {
    "connector.class": "io.confluent.connect.elasticsearch.ElasticsearchSinkConnector",
    "tasks.max": "4",
    "topics": "user-events,order-events",
    "connection.url": "http://elasticsearch:9200",
    "type.name": "_doc",
    "key.ignore": "true",
    "schema.ignore": "true",
    "transforms": "AddTimestamp",
    "transforms.AddTimestamp.type": "org.apache.kafka.connect.transforms.InsertField$Value",
    "transforms.AddTimestamp.timestamp.field": "@timestamp"
  }
}
```

**集成方案二：Logstash Kafka Input（灵活，支持复杂转换）**

```ruby
input {
  kafka {
    bootstrap_servers => "kafka1:9092,kafka2:9092"
    topics => ["app-logs"]
    group_id => "logstash-consumer"
    codec => json
    consumer_threads => 4
    auto_offset_reset => "latest"
  }
}

filter {
  date { match => ["event_time", "ISO8601"] target => "@timestamp" }
  mutate { remove_field => ["event_time"] }
}

output {
  elasticsearch {
    hosts => ["http://es:9200"]
    index => "app-logs-%{+YYYY.MM.dd}"
    document_id => "%{event_id}"
  }
}
```

**集成方案三：Java 代码直接消费写入**

```java
@KafkaListener(topics = "order-events", groupId = "es-writer")
public void consume(List<OrderEvent> events) {
    BulkRequest.Builder bulk = new BulkRequest.Builder();
    events.forEach(event -> bulk.operations(op ->
        op.index(i -> i.index("orders").id(event.getId()).document(event))
    ));
    esClient.bulk(bulk.build());
}
```

**数据流向：** Kafka → Consumer → ES Bulk → 索引完成。

**关键注意点：**
- 批量消费，不要每条消息单独写 ES（性能差 10x+）
- 设置合理的 `batch.size` 和 `linger.ms`
- 写入失败要做死信队列（DLQ）处理

###### 2. Elasticsearch 如何与 Spark 集成进行大数据分析？

**ES-Hadoop 库（`elasticsearch-hadoop`）提供 ES 和 Spark 的原生集成。**

**Spark 读取 ES 数据：**

```scala
import org.elasticsearch.spark._

val sc = new SparkContext(conf)

// 读取 ES 索引为 RDD
val rdd = sc.esRDD("orders/order", "?q=status:completed")
rdd.foreach(println)

// 使用 Spark SQL
val df = spark.read
  .format("org.elasticsearch.spark.sql")
  .option("es.nodes", "localhost:9200")
  .option("es.net.http.auth.user", "elastic")
  .option("es.net.http.auth.pass", "password")
  .load("orders")

df.createOrReplaceTempView("orders")
spark.sql("SELECT status, COUNT(*) as cnt FROM orders GROUP BY status").show()
```

**Spark 写入 ES：**

```scala
// DataFrame 写入 ES
df.write
  .format("org.elasticsearch.spark.sql")
  .option("es.nodes", "localhost:9200")
  .option("es.resource", "orders-analysis")
  .mode("append")
  .save()
```

**适用场景：**
- 历史数据批量分析（Spark 批处理 + ES 结果展示）
- 机器学习训练数据从 ES 读取
- 大规模数据 ETL（从 ES 读 → Spark 处理 → 写回 ES）

###### 3. Elasticsearch 如何与关系型数据库同步数据？

**核心挑战：保证 ES 与 DB 的数据最终一致性。**

**方案一：Logstash JDBC Input（全量 + 增量轮询）**

```ruby
input {
  jdbc {
    jdbc_driver_library => "/opt/mysql-connector-j-8.0.33.jar"
    jdbc_driver_class => "com.mysql.cj.jdbc.Driver"
    jdbc_connection_string => "jdbc:mysql://mysql:3306/shop"
    jdbc_user => "root"
    jdbc_password => "password"
    # 增量同步：记录上次同步的最大 update_time
    use_column_value => true
    tracking_column => "update_time"
    tracking_column_type => "timestamp"
    last_run_metadata_path => "/var/logstash/.logstash_jdbc_last_run"
    schedule => "*/1 * * * *"
    statement => "SELECT * FROM products WHERE update_time > :sql_last_value"
  }
}

output {
  elasticsearch {
    hosts => ["http://es:9200"]
    index => "products"
    document_id => "%{id}"
  }
}
```

**方案二：Canal（基于 MySQL Binlog，实时同步，推荐）**

```
MySQL Binlog → Canal Server → Canal Client → ES Bulk API
```

```java
// Canal Client 监听 Binlog 变更
CanalConnector connector = CanalConnectors.newSingleConnector(
    new InetSocketAddress("canal-server", 11111), "example", "", "");
connector.connect();
connector.subscribe("shop\\..*");

while (true) {
    Message message = connector.getWithoutAck(1000);
    for (CanalEntry.Entry entry : message.getEntries()) {
        if (entry.getEntryType() == CanalEntry.EntryType.ROWDATA) {
            CanalEntry.RowChange rowChange = CanalEntry.RowChange.parseFrom(entry.getStoreValue());
            switch (rowChange.getEventType()) {
                case INSERT:
                case UPDATE:
                    syncToES(entry.getHeader().getTableName(), rowChange.getRowDatasList());
                    break;
                case DELETE:
                    deleteFromES(entry.getHeader().getTableName(), rowChange.getRowDatasList());
                    break;
            }
        }
    }
    connector.ack(message.getId());
}
```

**方案三：双写（应用层同时写 DB 和 ES）**

适合新系统，但需要处理写 ES 失败的补偿逻辑（事务消息 / 重试队列）。

**方案对比：**

| 方案 | 实时性 | 侵入性 | 稳定性 |
|------|--------|--------|--------|
| JDBC 轮询 | 分钟级 | 无 | 高 |
| Canal Binlog | 秒级 | 低 | 高 |
| 双写 | 毫秒级 | 高 | 需补偿 |

###### 5. Elasticsearch 如何与 Spring Boot 集成？

**Spring Boot 集成 ES 有两种主流方式：Spring Data Elasticsearch 和 ES Java Client。**

**方式一：Spring Data Elasticsearch（推荐，简洁）**

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-elasticsearch</artifactId>
</dependency>
```

```yaml
# application.yml
spring:
  elasticsearch:
    uris: http://localhost:9200
    username: elastic
    password: changeme
```

```java
// 实体类
@Document(indexName = "products")
@Setting(shards = 2, replicas = 1)
public class Product {
    @Id
    private String id;
    
    @Field(type = FieldType.Text, analyzer = "ik_max_word", searchAnalyzer = "ik_smart")
    private String title;
    
    @Field(type = FieldType.Keyword)
    private String brand;
    
    @Field(type = FieldType.Double)
    private Double price;
    
    @Field(type = FieldType.Date, format = DateFormat.date_time)
    private LocalDateTime createTime;
}

// Repository
public interface ProductRepository extends ElasticsearchRepository<Product, String> {
    List<Product> findByBrandAndPriceBetween(String brand, Double min, Double max);
    
    @Query("{\"match\": {\"title\": {\"query\": \"?0\"}}}")
    List<Product> searchByTitle(String keyword, Pageable pageable);
}

// Service
@Service
public class ProductService {
    @Autowired
    private ElasticsearchOperations esOps;
    
    public SearchHits<Product> search(String keyword, String brand, Double minPrice, Double maxPrice) {
        Criteria criteria = new Criteria("title").matches(keyword);
        if (brand != null) criteria = criteria.and("brand").is(brand);
        if (minPrice != null) criteria = criteria.and("price").greaterThanEqual(minPrice);
        
        Query query = new CriteriaQuery(criteria)
            .setPageable(PageRequest.of(0, 20))
            .addSort(Sort.by(Sort.Direction.DESC, "price"));
        
        return esOps.search(query, Product.class);
    }
}
```

**方式二：ES Java Client（官方，灵活）**

```java
@Configuration
public class ElasticsearchConfig {
    @Bean
    public ElasticsearchClient elasticsearchClient() {
        RestClient restClient = RestClient.builder(
            new HttpHost("localhost", 9200, "http"))
            .setDefaultHeaders(new Header[]{
                new BasicHeader("Authorization", 
                    "Basic " + Base64.getEncoder().encodeToString("elastic:password".getBytes()))
            })
            .build();
        
        ElasticsearchTransport transport = new RestClientTransport(
            restClient, new JacksonJsonpMapper());
        return new ElasticsearchClient(transport);
    }
}

// 使用
@Service
public class EsService {
    @Autowired
    private ElasticsearchClient client;
    
    public SearchResponse<Product> search(String keyword) throws IOException {
        return client.search(s -> s
            .index("products")
            .query(q -> q
                .match(m -> m
                    .field("title")
                    .query(keyword)
                )
            )
            .size(20),
            Product.class
        );
    }
}
```

> **本章追问：Canal 同步 ES 如何保证数据一致性？**
>
> Canal 基于 MySQL Binlog 实现近实时同步，但存在以下一致性风险及解决方案：
> 1. **Canal 消费延迟**：Binlog 产生到 ES 写入有毫秒到秒级延迟 → 业务上接受最终一致性，实时性强的查询走 DB
> 2. **写 ES 失败**：Canal Client 写 ES 异常时，不要 ACK binlog 位点，让 Canal 重推（at-least-once 语义）→ ES 写入要幂等（用 `document_id = DB 主键`）
> 3. **全量初始化**：Canal 只同步增量，初始全量数据通过 Logstash JDBC 导入后，再启动 Canal 从对应 binlog 位点开始增量
> 4. **Schema 变更**：DB 加字段需要同步更新 ES Mapping，否则新字段走 dynamic mapping 可能产生不期望的类型
