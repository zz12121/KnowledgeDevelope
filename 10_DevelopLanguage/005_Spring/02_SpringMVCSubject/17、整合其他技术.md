###### 1. SpringMVC 如何整合 MyBatis？

SpringMVC 整合 MyBatis 的核心思路是：**用 Spring 的 IoC 容器统一管理 MyBatis 的核心组件**，关键角色是 `SqlSessionFactoryBean` 和 `MapperScannerConfigurer`。

**数据源与 SqlSessionFactory 配置：**

```xml
<!-- 数据源 -->
<bean id="dataSource" class="org.apache.commons.dbcp.BasicDataSource">
    <property name="driverClassName" value="${jdbc.driver}"/>
    <property name="url" value="${jdbc.url}"/>
    <property name="username" value="${jdbc.username}"/>
    <property name="password" value="${jdbc.password}"/>
</bean>

<!-- SqlSessionFactoryBean：桥接 MyBatis 与 Spring -->
<bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
    <property name="dataSource" ref="dataSource"/>
    <property name="typeAliasesPackage" value="com.example.entity"/>
    <property name="mapperLocations" value="classpath*:mapper/**/*.xml"/>
</bean>
```

**Mapper 接口自动扫描：**

```xml
<bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
    <property name="basePackage" value="com.example.mapper"/>
    <property name="sqlSessionFactoryBeanName" value="sqlSessionFactory"/>
</bean>
```

**底层原理简述：**
- `SqlSessionFactoryBean` 实现了 `FactoryBean` 和 `InitializingBean`，在 `afterPropertiesSet()` 里构建 `SqlSessionFactory`
- `MapperScannerConfigurer` 是个 `BeanDefinitionRegistryPostProcessor`，容器启动时扫描接口并注册为 `BeanDefinition`
- 每个 Mapper 接口对应一个 `MapperFactoryBean`，通过 `SqlSessionTemplate` 执行 SQL

**三层架构代码示意：**

```java
// Mapper 接口
public interface UserMapper {
    User selectById(Long id);
}

// Service 层
@Service
@Transactional
public class UserServiceImpl implements UserService {
    @Autowired
    private UserMapper userMapper;

    public User getUserById(Long id) {
        return userMapper.selectById(id);
    }
}

// Controller 层
@Controller
@RequestMapping("/user")
public class UserController {
    @Autowired
    private UserService userService;

    @GetMapping("/{id}")
    public String getUser(@PathVariable Long id, Model model) {
        model.addAttribute("user", userService.getUserById(id));
        return "user/detail";
    }
}
```

事务方面，配置一个 `DataSourceTransactionManager` 加上 `@Transactional` 注解就能搞定，不需要额外操作。

###### 2. SpringMVC 如何整合 Hibernate？

整合 Hibernate 的核心是用 `LocalSessionFactoryBean` 管理 `SessionFactory`，并配上对应的 `HibernateTransactionManager`。

**SessionFactory 配置：**

```xml
<bean id="sessionFactory"
      class="org.springframework.orm.hibernate5.LocalSessionFactoryBean">
    <property name="dataSource" ref="dataSource"/>
    <property name="packagesToScan" value="com.example.entity"/>
    <property name="hibernateProperties">
        <props>
            <prop key="hibernate.dialect">org.hibernate.dialect.MySQLDialect</prop>
            <prop key="hibernate.show_sql">true</prop>
            <prop key="hibernate.hbm2ddl.auto">update</prop>
        </props>
    </property>
</bean>
```

**事务管理器：**

```xml
<bean id="transactionManager"
      class="org.springframework.orm.hibernate5.HibernateTransactionManager">
    <property name="sessionFactory" ref="sessionFactory"/>
</bean>
<tx:annotation-driven transaction-manager="transactionManager"/>
```

**DAO 层实现：**

```java
@Repository
@Transactional
public class UserDaoImpl implements UserDao {
    @Autowired
    private SessionFactory sessionFactory;

    public User findById(Long id) {
        return sessionFactory.getCurrentSession().get(User.class, id);
    }
}
```

现代项目更推荐用 **Spring Data JPA**，直接继承 `JpaRepository` 接口，几行代码就能搞定常规的 CRUD 和复杂查询，不用手动管理 Session。

###### 3. SpringMVC 如何整合 Spring Security？

Spring Security 整合 SpringMVC 的核心机制是**过滤器链**，通过 `DelegatingFilterProxy` 把安全控制插入 Web 请求的最前端，在 DispatcherServlet 处理请求之前就完成认证鉴权。

**Java 配置方式（推荐）：**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests()
            .antMatchers("/admin/**").hasRole("ADMIN")
            .antMatchers("/user/**").hasAnyRole("USER", "ADMIN")
            .antMatchers("/public/**").permitAll()
            .anyRequest().authenticated()
            .and()
            .formLogin().loginPage("/login").permitAll()
            .and()
            .logout().permitAll();
    }

    @Override
    protected void configure(AuthenticationManagerBuilder auth) throws Exception {
        auth.userDetailsService(userDetailsService)
            .passwordEncoder(new BCryptPasswordEncoder());
    }
}
```

**方法级安全控制（细粒度授权）：**

```java
@Configuration
@EnableGlobalMethodSecurity(prePostEnabled = true)
public class MethodSecurityConfig extends GlobalMethodSecurityConfiguration {}

@Service
public class UserService {
    @PreAuthorize("hasRole('ADMIN') or #userId == authentication.principal.id")
    public User getUserProfile(Long userId) { ... }
}
```

Spring Security 的过滤器链在 DispatcherServlet 之前运行，这意味着即使一个请求通过了 URL 匹配，如果没有权限也会在最早阶段被拦下来，不会进入 MVC 处理流程。

###### 4. SpringMVC 如何整合 Thymeleaf？

Thymeleaf 整合通过配置 `TemplateEngine` 和 `ThymeleafViewResolver` 实现，它是目前 Spring 生态里最主流的服务端模板引擎。

**Java 配置方式：**

```java
@Configuration
@EnableWebMvc
public class ThymeleafConfig implements WebMvcConfigurer {

    @Bean
    public SpringResourceTemplateResolver templateResolver() {
        SpringResourceTemplateResolver resolver = new SpringResourceTemplateResolver();
        resolver.setPrefix("classpath:/templates/");
        resolver.setSuffix(".html");
        resolver.setTemplateMode(TemplateMode.HTML);
        resolver.setCharacterEncoding("UTF-8");
        resolver.setCacheable(false); // 开发阶段关闭缓存
        return resolver;
    }

    @Bean
    public SpringTemplateEngine templateEngine() {
        SpringTemplateEngine engine = new SpringTemplateEngine();
        engine.setTemplateResolver(templateResolver());
        return engine;
    }

    @Bean
    public ThymeleafViewResolver thymeleafViewResolver() {
        ThymeleafViewResolver resolver = new ThymeleafViewResolver();
        resolver.setTemplateEngine(templateEngine());
        resolver.setCharacterEncoding("UTF-8");
        resolver.setOrder(1);
        return resolver;
    }
}
```

**Thymeleaf 模板示例：**

```html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<body>
    <h1 th:text="'欢迎, ' + ${user.username}">欢迎用户</h1>

    <ul>
        <li th:each="item : ${user.items}" th:text="${item.name}">项目名</li>
    </ul>

    <!-- th:if 条件渲染 -->
    <div th:if="${user.active}">用户状态：活跃</div>
</body>
</html>
```

Thymeleaf 的模板是合法的 HTML 文件，可以直接在浏览器里打开预览，这是相比 JSP 最大的优势之一。

###### 5. SpringMVC 如何整合 FreeMarker？

FreeMarker 整合和 Thymeleaf 类似，通过 `FreeMarkerConfigurer` + `FreeMarkerViewResolver` 配置，适合对模板渲染性能要求较高的场景。

**Java 配置：**

```java
@Configuration
@EnableWebMvc
public class FreeMarkerConfig implements WebMvcConfigurer {

    @Bean
    public FreeMarkerConfigurer freeMarkerConfigurer() {
        FreeMarkerConfigurer configurer = new FreeMarkerConfigurer();
        configurer.setTemplateLoaderPath("classpath:/templates/");
        configurer.setDefaultEncoding("UTF-8");

        Properties settings = new Properties();
        settings.setProperty("datetime_format", "yyyy-MM-dd HH:mm:ss");
        settings.setProperty("number_format", "0.##");
        configurer.setFreemarkerSettings(settings);
        return configurer;
    }

    @Bean
    public FreeMarkerViewResolver freeMarkerViewResolver() {
        FreeMarkerViewResolver resolver = new FreeMarkerViewResolver();
        resolver.setCache(true);
        resolver.setSuffix(".ftl");
        resolver.setContentType("text/html;charset=UTF-8");
        return resolver;
    }
}
```

**FreeMarker 模板语法（.ftl 文件）：**

```ftl
<h1>欢迎, ${user.username!''}</h1>

<#if user.active>
    <p>用户状态：活跃</p>
</#if>

<#list user.items as item>
    <p>${item.name}</p>
</#list>
```

FreeMarker 支持宏（`<#macro>`）、函数、自定义指令等高级特性，在邮件模板、报表生成等场景里用得比较多。

###### 6. SpringMVC 如何整合 WebSocket？

WebSocket 整合提供全双工通信能力，Spring 通过 **STOMP 协议**和**消息代理**对其进行封装，让开发者不用直接操作底层 WebSocket API。

**核心配置：**

```java
@Configuration
@EnableWebSocketMessageBroker
public class WebSocketConfig implements WebSocketMessageBrokerConfigurer {

    @Override
    public void registerStompEndpoints(StompEndpointRegistry registry) {
        registry.addEndpoint("/ws")
                .setAllowedOriginPatterns("*")
                .withSockJS(); // 支持 SockJS 降级
    }

    @Override
    public void configureMessageBroker(MessageBrokerRegistry registry) {
        registry.enableSimpleBroker("/topic", "/queue"); // 消息订阅前缀
        registry.setApplicationDestinationPrefixes("/app"); // 客户端发送消息前缀
        registry.setUserDestinationPrefix("/user");         // 点对点消息前缀
    }
}
```

**WebSocket 控制器：**

```java
@Controller
public class WebSocketController {

    @MessageMapping("/chat.send")       // 接收客户端发到 /app/chat.send 的消息
    @SendTo("/topic/public")            // 广播到所有订阅 /topic/public 的客户端
    public ChatMessage sendMessage(@Payload ChatMessage chatMessage) {
        return chatMessage;
    }
}
```

**前端 JavaScript 连接示例：**

```javascript
const stompClient = Stomp.over(new SockJS('/ws'));
stompClient.connect({}, () => {
    stompClient.subscribe('/topic/public', message => {
        console.log(JSON.parse(message.body));
    });
});
```

需要注意的是，WebSocket 整合需要 Servlet 容器支持（Tomcat 7.0.47+ 或 Jetty 9.1+），并且过滤器和拦截器的配置也需要支持 WebSocket 升级请求。
