###### 1. SpringMVC 如何实现国际化？

SpringMVC 国际化（i18n）基于 **ResourceBundle 机制**和 **Locale 解析策略**实现，核心是 `MessageSource` 接口 + `LocaleResolver` 的组合。

整体流程是：请求进来 → `DispatcherServlet` 调用 `LocaleResolver` 解析出当前语言环境（`Locale`）→ 把 `Locale` 存到线程上下文 → 视图渲染时通过 `MessageSource` 查找对应语言的消息文本。

**典型配置示例：**

```java
@Configuration
@EnableWebMvc
public class I18nConfig implements WebMvcConfigurer {

    @Bean
    public LocaleResolver localeResolver() {
        SessionLocaleResolver resolver = new SessionLocaleResolver();
        resolver.setDefaultLocale(Locale.CHINA); // 默认简体中文
        return resolver;
    }

    @Bean
    public LocaleChangeInterceptor localeChangeInterceptor() {
        LocaleChangeInterceptor interceptor = new LocaleChangeInterceptor();
        interceptor.setParamName("lang"); // 通过 ?lang=en_US 切换语言
        return interceptor;
    }

    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(localeChangeInterceptor());
    }
}
```

**资源文件按语言命名，放在同一目录下：**

```
src/main/resources/i18n/
├── messages.properties          # 默认（通常是英文）
├── messages_zh_CN.properties   # 简体中文
├── messages_zh_TW.properties   # 繁体中文
└── messages_ja_JP.properties   # 日文
```

Spring 会按照 `messages_zh_CN.properties` → `messages_zh.properties` → `messages.properties` 的顺序依次 fallback 查找，找到就用。

###### 2. LocaleResolver 的作用是什么？

`LocaleResolver` 是国际化架构里的**语言环境决策器**，负责从 HTTP 请求中提取 `Locale` 信息，决定当前用户用什么语言。它采用策略模式，有几种开箱即用的实现：

- **`AcceptHeaderLocaleResolver`**：读取请求头里的 `Accept-Language`，尊重浏览器语言设置，但用户自己不能切换
- **`SessionLocaleResolver`**：把 Locale 存在 HttpSession 里，用户选一次语言，整个会话期间都生效，体验好
- **`CookieLocaleResolver`**：存在 Cookie 里，跨会话保持，用户下次来还是上次选的语言
- **`FixedLocaleResolver`**：固定 Locale，单语言应用或测试用，性能最好

实际项目里，`SessionLocaleResolver` 是最常用的，配合 `LocaleChangeInterceptor` 拦截器可以做到一个请求参数就能切语言：

```java
@Bean
public SessionLocaleResolver localeResolver() {
    SessionLocaleResolver resolver = new SessionLocaleResolver();
    resolver.setDefaultLocale(Locale.US);
    return resolver;
}
```

如果有更复杂的需求（比如先查 Session，没有再查 Cookie，最后才看浏览器设置），也可以自己实现 `LocaleResolver` 接口，做一个链式解析器。

###### 3. 如何配置国际化资源文件？

资源文件配置的核心是 `MessageSource` Bean，通常用 `ResourceBundleMessageSource`（生产环境，性能好）或 `ReloadableResourceBundleMessageSource`（开发环境，支持热重载）。

**标准配置：**

```java
@Bean
public MessageSource messageSource() {
    ResourceBundleMessageSource messageSource = new ResourceBundleMessageSource();
    messageSource.setBasenames("i18n/messages", "i18n/errors", "i18n/validation");
    messageSource.setDefaultEncoding("UTF-8");
    messageSource.setUseCodeAsDefaultMessage(true);      // 找不到 key 时直接返回 key 本身
    messageSource.setFallbackToSystemLocale(false);      // 禁用系统 Locale 回退
    return messageSource;
}
```

**支持热加载的配置（开发环境推荐）：**

```java
@Bean
public MessageSource messageSource() {
    ReloadableResourceBundleMessageSource messageSource = new ReloadableResourceBundleMessageSource();
    messageSource.setBasenames("classpath:i18n/messages");
    messageSource.setCacheSeconds(300); // 缓存 5 分钟，修改后最多等 5 分钟生效
    messageSource.setDefaultEncoding("UTF-8");
    return messageSource;
}
```

**资源文件格式：**

```properties
# messages_zh_CN.properties
welcome.message=欢迎使用我们的系统
user.login.title=用户登录
button.submit=提交

# 带参数的消息（{0} 是占位符）
greeting.message=你好，{0}！今天是 {1,date,long}
```

Spring 的 fallback 机制让资源文件可以分层维护：比如 `messages_zh_CN.properties` 只写中国大陆特有的措辞，通用中文放在 `messages_zh.properties`，找不到就一级级往上查。

###### 4. MessageSource 的作用是什么？

`MessageSource` 是 Spring 国际化架构的**核心接口**，负责根据 key + Locale 查找并返回对应的消息文本，支持参数替换和默认值回退。

接口定义很简洁：

```java
public interface MessageSource {
    // 查找消息，找不到返回 defaultMessage
    String getMessage(String code, Object[] args, String defaultMessage, Locale locale);

    // 查找消息，找不到抛 NoSuchMessageException
    String getMessage(String code, Object[] args, Locale locale) throws NoSuchMessageException;
}
```

在控制器或 Service 里注入后，使用非常直接：

```java
@Service
public class BusinessService {
    private final MessageSource messageSource;

    public String getWelcome(Locale locale) {
        // 简单查找
        return messageSource.getMessage("welcome.message", null, locale);
    }

    public String getGreeting(String name, Locale locale) {
        // 带参数
        return messageSource.getMessage("greeting.message", new Object[]{name, new Date()}, locale);
    }

    public String getOptional(Locale locale) {
        // 带默认值，找不到 key 不报错
        return messageSource.getMessage("custom.message", null, "默认消息", locale);
    }
}
```

**三个主要实现类：**

- `ResourceBundleMessageSource`：基于 JDK 的 ResourceBundle，性能好，适合生产
- `ReloadableResourceBundleMessageSource`：支持热加载，适合开发期频繁修改资源
- `StaticMessageSource`：编程式添加消息，不用资源文件，适合单元测试
