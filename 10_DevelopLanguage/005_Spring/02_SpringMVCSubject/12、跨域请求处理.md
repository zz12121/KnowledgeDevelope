###### 1. 什么是跨域问题？

跨域问题本质是浏览器的**同源策略（Same-Origin Policy）**与前后端分离架构之间的冲突。

同源策略规定：只有**协议 + 域名 + 端口**三者完全相同的两个 URL，才被浏览器认为是"同源"的。任何一项不匹配就构成跨域，浏览器会阻止页面发起的跨域 AJAX 请求。

几个典型的跨域场景：

- `http://a.com` 访问 `https://a.com`：协议不同（HTTP vs HTTPS）
- `http://a.com` 访问 `http://b.com`：域名不同
- `http://a.com:80` 访问 `http://a.com:8080`：端口不同
- `http://a.com` 访问 `http://api.a.com`：子域名不同，也算跨域

同源策略是 Web 安全的基石，能有效防止 CSRF（跨站请求伪造）和 XSS 等攻击。但前后端分离架构普及后，前端（如 `localhost:3000`）调用后端 API（如 `localhost:8080`）就成了合法的跨域需求，所以需要 CORS 机制来安全地开放跨域访问。

###### 2. SpringMVC 如何解决跨域问题？

Spring MVC 基于 **CORS（跨域资源共享）规范**提供了三种层次的跨域解决方案：

**方案一：`@CrossOrigin` 注解**（细粒度，控制单个接口）

**方案二：`WebMvcConfigurer.addCorsMappings()`**（全局配置，统一管理）

**方案三：`CorsFilter`**（Servlet 过滤器层面，优先级最高，适合与 Spring Security 集成）

底层工作原理：Spring MVC 的 `DispatcherServlet` 在处理请求时，会用 `DefaultCorsProcessor` 检查请求是否符合 CORS 配置。对于**预检请求**（OPTIONS 方法），直接返回 CORS 响应头；对于普通跨域请求，在响应中添加 `Access-Control-Allow-Origin` 等头信息，告诉浏览器这次跨域是被允许的。

关键 CORS 响应头：

- `Access-Control-Allow-Origin`：允许哪些来源（`*` 或具体域名）
- `Access-Control-Allow-Methods`：允许哪些 HTTP 方法
- `Access-Control-Allow-Headers`：允许哪些请求头
- `Access-Control-Allow-Credentials`：是否允许携带 Cookie 等凭证
- `Access-Control-Max-Age`：预检请求结果缓存时间（秒）

###### 3. @CrossOrigin 注解的作用是什么？

`@CrossOrigin` 是声明式跨域配置注解，可以加在类（对整个控制器生效）或方法（只对该方法生效）上，实现细粒度的跨域策略控制。

```java
@RestController
@RequestMapping("/api/users")
// 类级别：整个控制器允许来自两个域名的跨域请求
@CrossOrigin(
    origins = {"https://app.example.com", "https://admin.example.com"},
    methods = {RequestMethod.GET, RequestMethod.POST, RequestMethod.PUT},
    allowedHeaders = {"Content-Type", "Authorization"},
    allowCredentials = "true",
    maxAge = 3600
)
public class UserController {

    // 方法级别覆盖：这个接口对所有来源开放（比如公开数据接口）
    @CrossOrigin(origins = "*")
    @GetMapping("/public/{id}")
    public User getPublicUser(@PathVariable Long id) {
        return userService.findById(id);
    }

    // 继承类级别的 CORS 配置
    @PostMapping
    public User createUser(@RequestBody User user) {
        return userService.save(user);
    }
}
```

注意：`allowCredentials = "true"` 时，`origins` 不能设置为 `"*"`，必须指定具体的域名，否则浏览器会拒绝（这是 CORS 规范的安全限制）。

###### 4. 如何全局配置跨域？

全局跨域配置通过 `WebMvcConfigurer.addCorsMappings()` 实现，适合大型项目统一管理跨域策略：

```java
@Configuration
@EnableWebMvc
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addCorsMappings(CorsRegistry registry) {

        // API 路径：允许指定前端域名，支持凭证
        registry.addMapping("/api/**")
                .allowedOrigins("https://frontend.example.com", "https://admin.example.com")
                .allowedMethods("GET", "POST", "PUT", "DELETE", "OPTIONS")
                .allowedHeaders("Content-Type", "Authorization", "X-Requested-With")
                .allowCredentials(true)
                .maxAge(3600);

        // 公开接口：允许所有来源（只读接口）
        registry.addMapping("/public/**")
                .allowedOrigins("*")
                .allowedMethods("GET", "OPTIONS")
                .maxAge(1800);
    }
}
```

如果需要比 Spring MVC 层更早处理跨域（比如配合 Spring Security，在 Security 过滤器之前处理），用 `CorsFilter`：

```java
@Bean
public FilterRegistrationBean<CorsFilter> corsFilter() {
    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    CorsConfiguration config = new CorsConfiguration();

    config.setAllowCredentials(true);
    config.addAllowedOrigin("https://trusted-domain.com");
    config.addAllowedMethod("*");
    config.addAllowedHeader("*");
    config.setMaxAge(3600L);

    source.registerCorsConfiguration("/**", config);

    FilterRegistrationBean<CorsFilter> bean =
        new FilterRegistrationBean<>(new CorsFilter(source));
    bean.setOrder(Ordered.HIGHEST_PRECEDENCE);  // 最高优先级，在所有过滤器之前执行
    return bean;
}
```

**优先级**：`CorsFilter` > 全局配置（`addCorsMappings`）> `@CrossOrigin` 注解。通常项目里只选一种方式，混用容易出现配置冲突。

###### 5. CorsRegistry 的作用是什么？

`CorsRegistry` 是 Spring MVC 全局 CORS 配置的注册中心，通过 `WebMvcConfigurer.addCorsMappings()` 方法获取，提供流式 API 配置路径到 CORS 策略的映射关系。

它内部维护一组 `CorsRegistration` 对象，每个注册项对应一个路径模式及其 `CorsConfiguration`。当请求到达时，`HandlerMapping` 查找最匹配的 CORS 配置并应用。

```java
@Override
public void addCorsMappings(CorsRegistry registry) {

    // 根据部署环境动态配置允许的来源
    String[] allowedOrigins = resolveAllowedOrigins();

    registry.addMapping("/api/**")
            .allowedOrigins(allowedOrigins)
            .allowedMethods("GET", "POST", "PUT", "DELETE")
            .allowedHeaders("*")
            .allowCredentials(true)
            .maxAge(3600)
            .exposedHeaders("X-Total-Count", "X-Page-Number");  // 暴露自定义响应头给前端
}

private String[] resolveAllowedOrigins() {
    String env = System.getProperty("spring.profiles.active", "dev");
    return switch (env) {
        case "prod"    -> new String[]{"https://www.example.com"};
        case "staging" -> new String[]{"https://staging.example.com"};
        default        -> new String[]{"http://localhost:3000", "http://localhost:8080"};
    };
}
```

`exposedHeaders` 需要特别注意：浏览器默认只能读取少量响应头，如果后端返回了自定义头（如分页信息 `X-Total-Count`），需要在这里显式声明，前端才能读到。
