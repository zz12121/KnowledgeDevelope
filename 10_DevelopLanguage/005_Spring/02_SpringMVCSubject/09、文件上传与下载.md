###### 1. SpringMVC 如何实现文件上传？

SpringMVC 通过 `MultipartResolver` 策略接口实现文件上传。当请求的 `Content-Type` 是 `multipart/form-data` 时，`DispatcherServlet` 检测到文件上传请求，调用 `MultipartResolver.resolveMultipart()` 将原始 `HttpServletRequest` 包装成 `MultipartHttpServletRequest`，然后通过参数解析器把文件数据绑定到控制器方法的 `MultipartFile` 参数上。

```java
@Controller
public class FileUploadController {

    @PostMapping("/upload")
    public ResponseEntity<String> handleFileUpload(
        @RequestParam("file") MultipartFile file) {

        if (file.isEmpty()) {
            return ResponseEntity.badRequest().body("文件不能为空");
        }

        // 安全处理文件名，防止路径穿越攻击
        String originalFilename = StringUtils.cleanPath(file.getOriginalFilename());
        // 用 UUID 生成唯一文件名，避免同名文件覆盖
        String fileName = UUID.randomUUID() + "_" + originalFilename;

        Path uploadPath = Paths.get("uploads");
        try {
            if (!Files.exists(uploadPath)) {
                Files.createDirectories(uploadPath);
            }
            file.transferTo(uploadPath.resolve(fileName));
            return ResponseEntity.ok("上传成功：" + fileName);
        } catch (IOException ex) {
            throw new RuntimeException("文件保存失败", ex);
        }
    }
}
```

###### 2. MultipartFile 的作用是什么？

`MultipartFile` 是 Spring 文件上传的核心数据载体接口，封装了上传文件的元数据和内容，屏蔽了底层存储细节（文件可能暂存在内存，也可能写入临时磁盘，取决于文件大小阈值配置）。

接口主要方法：

- `getName()`：表单字段名
- `getOriginalFilename()`：原始文件名（含路径）
- `getContentType()`：MIME 类型
- `isEmpty()`：是否是空文件
- `getSize()`：文件大小（字节）
- `getBytes()`：读取为字节数组（适合小文件）
- `getInputStream()`：读取为输入流（适合大文件，避免 OOM）
- `transferTo(File dest)`：直接写入目标文件（性能最佳，内部做了优化）

安全验证示例——不能只看 `Content-Type`，还要验文件头（Magic Number）：

```java
@Service
public class FileValidationService {

    private static final Set<String> ALLOWED_TYPES =
        Set.of("image/jpeg", "image/png", "application/pdf");

    public void validate(MultipartFile file) {
        if (file.isEmpty()) throw new IllegalArgumentException("文件不能为空");

        if (!ALLOWED_TYPES.contains(file.getContentType())) {
            throw new IllegalArgumentException("不支持的文件类型");
        }

        // 验证文件头魔术数字，防止伪造 Content-Type
        try (InputStream is = file.getInputStream()) {
            byte[] header = new byte[4];
            is.read(header);
            if (!isValidMagicNumber(header, file.getContentType())) {
                throw new IllegalArgumentException("文件内容与声明类型不符");
            }
        } catch (IOException e) {
            throw new RuntimeException("文件读取失败", e);
        }
    }
}
```

###### 3. 如何配置文件上传解析器？

Spring Boot 项目通过 `application.yml` 配置即可，无需手动注册 Bean：

```yaml
spring:
  servlet:
    multipart:
      enabled: true
      max-file-size: 10MB       # 单个文件最大大小
      max-request-size: 100MB   # 整个请求最大大小
      file-size-threshold: 2MB  # 超过这个大小写入临时目录（而非内存）
      location: /tmp/uploads    # 临时文件目录
```

传统 Spring MVC 项目用 Java Config 手动配置：

```java
@Bean
public CommonsMultipartResolver multipartResolver() {
    CommonsMultipartResolver resolver = new CommonsMultipartResolver();
    resolver.setDefaultEncoding("UTF-8");
    resolver.setMaxUploadSize(10 * 1024 * 1024L);  // 10MB
    resolver.setMaxInMemorySize(40 * 1024);          // 40KB 以内放内存，超出写临时目录
    return resolver;
}
```

注意：传统 XML 配置中 Bean 的 id **必须**是 `multipartResolver`，这是 Spring 约定，名字错了就找不到。

###### 4. 如何限制上传文件的大小？

文件大小限制应该在多个层面配置，形成完整防护：

**第一层：配置文件（最简单）**

```yaml
spring.servlet.multipart.max-file-size=5MB
spring.servlet.multipart.max-request-size=20MB
```

**第二层：全局异常处理（给用户友好提示）**

```java
@RestControllerAdvice
public class FileUploadExceptionAdvice {

    @ExceptionHandler(MaxUploadSizeExceededException.class)
    public ResponseEntity<String> handleMaxSizeException() {
        return ResponseEntity.badRequest().body("文件大小超过限制，请上传 5MB 以内的文件");
    }
}
```

**第三层：业务层校验（精细控制）**

```java
@PostMapping("/upload")
public String uploadFile(@RequestParam MultipartFile file) {
    long maxSize = 2 * 1024 * 1024L;  // 2MB
    if (file.getSize() > maxSize) {
        throw new BusinessException("FILE_TOO_LARGE", "文件大小不能超过 2MB");
    }
    // 业务处理...
}
```

###### 5. 如何实现多文件上传？

多文件上传用数组或 List 接收同名参数：

```java
@RestController
public class MultiFileUploadController {

    // 方式一：数组接收
    @PostMapping("/upload-array")
    public ResponseEntity<List<String>> uploadFiles(
        @RequestParam("files") MultipartFile[] files) {

        List<String> names = new ArrayList<>();
        for (MultipartFile file : files) {
            if (!file.isEmpty()) {
                String savedName = fileService.save(file);
                names.add(savedName);
            }
        }
        return ResponseEntity.ok(names);
    }

    // 方式二：List 接收（更推荐）
    @PostMapping("/upload-list")
    public ResponseEntity<List<String>> uploadFileList(
        @RequestParam("files") List<MultipartFile> files) {

        List<String> names = files.stream()
            .filter(f -> !f.isEmpty())
            .map(f -> {
                // 生成唯一文件名避免冲突
                String fileName = UUID.randomUUID() + "_" +
                    StringUtils.cleanPath(f.getOriginalFilename());
                try {
                    f.transferTo(new File("/uploads/" + fileName));
                } catch (IOException e) {
                    throw new RuntimeException("文件保存失败", e);
                }
                return fileName;
            })
            .collect(Collectors.toList());

        return ResponseEntity.ok(names);
    }
}
```

大量文件上传时，可以考虑 `@Async` 异步处理，避免长时间占用请求线程。

###### 6. SpringMVC 如何实现文件下载？

文件下载的核心是正确设置 HTTP 响应头，告诉浏览器以附件形式处理响应体。推荐用 `ResponseEntity<Resource>` 的方式，更简洁、类型安全：

```java
@RestController
public class FileDownloadController {

    // 推荐方式：ResponseEntity<Resource>
    @GetMapping("/download/{fileName}")
    public ResponseEntity<Resource> downloadFile(
        @PathVariable String fileName) throws IOException {

        Path filePath = Paths.get("/secure/uploads")
            .resolve(fileName).normalize();

        // 安全检查：防止路径穿越
        if (!filePath.startsWith(Paths.get("/secure/uploads"))) {
            return ResponseEntity.badRequest().build();
        }
        if (!Files.exists(filePath)) {
            return ResponseEntity.notFound().build();
        }

        Resource resource = new UrlResource(filePath.toUri());

        // 处理中文文件名（RFC 5987 标准编码）
        String encodedFileName = URLEncoder.encode(fileName, StandardCharsets.UTF_8)
            .replaceAll("\\+", "%20");

        return ResponseEntity.ok()
            .contentType(MediaType.APPLICATION_OCTET_STREAM)
            .header(HttpHeaders.CONTENT_DISPOSITION,
                "attachment; filename*=UTF-8''" + encodedFileName)
            .body(resource);
    }
}
```

几个注意点：
- 下载前必须做路径安全检查，防止 `../../../etc/passwd` 这类路径穿越攻击
- 中文文件名需要正确编码，否则浏览器显示乱码
- 大文件建议用 `StreamingResponseBody` 流式响应，避免把整个文件加载进内存

###### 7. ResponseEntity 的作用是什么？

`ResponseEntity` 是对完整 HTTP 响应的封装，可以精确控制响应的状态码、响应头和响应体三个要素，在 RESTful API 开发中广泛使用。

它采用建造者模式，支持链式调用：

```java
@RestController
public class ApiController {

    // 查询成功：200 + 数据
    @GetMapping("/files/{id}")
    public ResponseEntity<FileInfo> getFileInfo(@PathVariable Long id) {
        FileInfo info = fileService.findById(id);
        if (info == null) {
            return ResponseEntity.notFound().build();  // 404
        }
        return ResponseEntity.ok(info);  // 200
    }

    // 创建成功：201 Created + Location 头
    @PostMapping("/files")
    public ResponseEntity<FileInfo> uploadFile(@RequestBody FileUploadRequest req) {
        FileInfo saved = fileService.save(req);
        URI location = ServletUriComponentsBuilder.fromCurrentRequest()
            .path("/{id}").buildAndExpand(saved.getId()).toUri();
        return ResponseEntity.created(location).body(saved);
    }

    // 带自定义响应头
    @GetMapping("/users")
    public ResponseEntity<List<User>> getUsers(Pageable pageable) {
        Page<User> page = userService.findAll(pageable);
        return ResponseEntity.ok()
            .header("X-Total-Count", String.valueOf(page.getTotalElements()))
            .header("X-Page-Number", String.valueOf(page.getNumber()))
            .body(page.getContent());
    }

    // 无内容响应：204 No Content
    @DeleteMapping("/files/{id}")
    public ResponseEntity<Void> deleteFile(@PathVariable Long id) {
        fileService.delete(id);
        return ResponseEntity.noContent().build();
    }
}
```

相比直接用 `@ResponseStatus`，`ResponseEntity` 的优势是可以在运行时动态决定状态码（比如找到了返回 200，没找到返回 404），更加灵活。
