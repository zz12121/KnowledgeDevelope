###### 1. SpringMVC 如何进行参数绑定？

参数绑定的本质是：把 HTTP 请求中的数据自动转换成控制器方法所需要的 Java 参数，这个过程由 `DataBinder` 机制驱动，背后是一套基于接口的参数解析器体系。

整个流程分三步：`DispatcherServlet` 收到请求后，`RequestMappingHandlerAdapter` 遍历所有注册的 `HandlerMethodArgumentResolver`，找到能处理当前参数类型的解析器，然后调用 `resolveArgument()` 方法完成实际的解析和类型转换；如果参数带有 `@Valid` 注解，还会触发数据验证并生成 `BindingResult`。

```java
public interface HandlerMethodArgumentResolver {
    // 判断是否支持当前参数类型
    boolean supportsParameter(MethodParameter parameter);

    // 实际解析参数值
    Object resolveArgument(MethodParameter parameter,
                           ModelAndViewContainer mavContainer,
                           NativeWebRequest webRequest,
                           WebDataBinderFactory binderFactory) throws Exception;
}
```

这套基于接口的设计让 SpringMVC 具有极强的扩展性——如果内置解析器不够用，直接实现这个接口就能加进来，完全不需要改框架本身的代码。

###### 2. SpringMVC 支持哪些参数类型？

SpringMVC 支持非常丰富的参数类型，基本覆盖了 Web 开发中的所有场景：

**基本类型与字符串**：`int`、`long`、`boolean`、`String` 等，会自动进行类型转换。

**POJO 对象绑定**：请求参数名与 POJO 的字段名匹配，Spring 自动填充，支持嵌套对象（比如 `User` 里有 `Address`）。

**集合和数组类型**：`List<String>`、`Map<String, String>` 等，支持多值参数和复杂嵌套结构。

**原生 Servlet API 对象**：`HttpServletRequest`、`HttpServletResponse`、`HttpSession`、`Principal` 等，直接注入即可使用。

**框架特有对象**：`Model`、`ModelAndView`、`BindingResult`、`RedirectAttributes` 等。

```java
// 原生 Servlet API 对象直接注入
@GetMapping("/native")
public String nativeParams(HttpServletRequest request,
                            HttpServletResponse response,
                            HttpSession session,
                            Principal principal) {
    return "result";
}

// 复杂嵌套绑定：前端提交 addressList[0].street, users[0].name
@PostMapping("/complex")
public String complexBinding(OrderForm form) {
    return "success";
}
```

核心要求：参数名必须与请求参数名匹配（区分大小写），POJO 需要提供 setter 方法。

###### 3. 如何在 SpringMVC 中接收 JSON 数据？

接收 JSON 数据靠 `@RequestBody` 注解加上 `HttpMessageConverter` 机制配合完成。

关键在于 `MappingJackson2HttpMessageConverter`——当 Spring 检测到请求的 `Content-Type` 是 `application/json`，就用它把请求体的 JSON 字符串反序列化成 Java 对象。

```java
@Configuration
@EnableWebMvc
public class WebConfig implements WebMvcConfigurer {

    @Override
    public void configureMessageConverters(List<HttpMessageConverter<?>> converters) {
        MappingJackson2HttpMessageConverter converter = new MappingJackson2HttpMessageConverter();
        // 可以在这里自定义 ObjectMapper 配置，比如日期格式、字段命名策略等
        ObjectMapper mapper = new ObjectMapper();
        mapper.configure(DeserializationFeature.FAIL_ON_UNKNOWN_PROPERTIES, false);
        converter.setObjectMapper(mapper);
        converters.add(converter);
    }
}

@RestController
public class UserController {

    // 接收 JSON 对象
    @PostMapping(value = "/users", consumes = MediaType.APPLICATION_JSON_VALUE)
    public ResponseEntity<User> createUser(@RequestBody @Valid User user) {
        return ResponseEntity.ok(userService.save(user));
    }

    // 接收 JSON 数组
    @PostMapping("/users/batch")
    public ResponseEntity<List<User>> createUsers(@RequestBody List<User> users) {
        return ResponseEntity.ok(userService.batchSave(users));
    }
}
```

使用 `@RequestBody` 时，前端请求头必须设置 `Content-Type: application/json`，否则 SpringMVC 找不到合适的转换器会报 415 Unsupported Media Type。

###### 4. 如何在 SpringMVC 中接收 XML 数据？

接收 XML 数据的思路和接收 JSON 一样，只是换了对应的 `HttpMessageConverter`——用 `MarshallingHttpMessageConverter` 配合 JAXB 来处理。

实体类需要加上 JAXB 注解，请求头的 `Content-Type` 要设置为 `application/xml`。

```java
@Configuration
public class WebConfig {

    @Bean
    public MarshallingHttpMessageConverter marshallingMessageConverter() {
        MarshallingHttpMessageConverter converter = new MarshallingHttpMessageConverter();
        converter.setMarshaller(jaxbMarshaller());
        converter.setUnmarshaller(jaxbMarshaller());
        return converter;
    }

    @Bean
    public Jaxb2Marshaller jaxbMarshaller() {
        Jaxb2Marshaller marshaller = new Jaxb2Marshaller();
        marshaller.setPackagesToScan("com.example.entity");
        return marshaller;
    }
}

// 实体类需要 JAXB 注解
@XmlRootElement(name = "user")
@XmlAccessorType(XmlAccessType.FIELD)
public class User {
    @XmlElement
    private String name;
    @XmlElement
    private Integer age;
}

// 控制器接收 XML
@PostMapping(value = "/users", consumes = MediaType.APPLICATION_XML_VALUE)
public User createUserFromXml(@RequestBody User user) {
    return userService.save(user);
}
```

实际项目中，如果既要支持 JSON 又要支持 XML，可以配合 `ContentNegotiatingViewResolver` 做内容协商，根据客户端的 `Accept` 头自动选择响应格式。

###### 5. 什么是数据绑定？

数据绑定是 SpringMVC 的核心特性之一，指的是**将 HTTP 请求参数自动绑定到 Java 对象的过程**。它不只是简单地把字符串赋值给字段，而是一个包含类型转换、格式化、数据验证的完整处理流水线。

核心流程：把 `HttpServletRequest` 传给 `DataBinder`，`DataBinder` 通过 `ConversionService` 完成类型转换，再通过 `Validator` 组件完成数据校验，最后把绑定结果和校验错误都封装在 `BindingResult` 中返回。

这套机制最大的价值就是把开发者从繁琐的手动参数提取和类型转换中解放出来，以前要写 `request.getParameter("age")` 然后 `Integer.parseInt(...)` 的那些样板代码，现在直接一个方法参数搞定。

###### 6. 如何自定义类型转换器？

当 Spring 内置的类型转换规则满足不了需求时，可以实现 `Converter<S, T>` 接口来自定义转换逻辑，或者实现 `Formatter<T>` 接口（更适合 Web 层，还支持本地化格式化）。

```java
// 方式一：实现 Converter 接口（通用类型转换）
public class StringToDateConverter implements Converter<String, Date> {
    private static final String DATE_PATTERN = "yyyy-MM-dd";

    @Override
    public Date convert(String source) {
        if (!StringUtils.hasText(source)) return null;
        try {
            return new SimpleDateFormat(DATE_PATTERN).parse(source);
        } catch (ParseException e) {
            throw new IllegalArgumentException("日期格式错误，应为：" + DATE_PATTERN);
        }
    }
}

// 方式二：实现 Formatter 接口（带本地化的格式化）
public class DateFormatter implements Formatter<Date> {
    @Override
    public Date parse(String text, Locale locale) throws ParseException {
        return new SimpleDateFormat("yyyy-MM-dd", locale).parse(text);
    }

    @Override
    public String print(Date object, Locale locale) {
        return new SimpleDateFormat("yyyy-MM-dd", locale).format(object);
    }
}

// 注册到 Spring MVC
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Override
    public void addFormatters(FormatterRegistry registry) {
        registry.addConverter(new StringToDateConverter());
        registry.addFormatter(new DateFormatter());
    }
}
```

###### 7. WebDataBinder 的作用是什么？

`WebDataBinder` 是 SpringMVC 数据绑定的核心组件，负责将请求参数绑定到目标对象，同时管理绑定过程中的类型转换和数据验证。

它的四个主要职责：**数据绑定**（把参数值设置到对象属性）、**类型转换**（通过 `ConversionService`）、**数据验证**（调用 Validator）、**结果管理**（维护绑定错误信息）。

通过 `@InitBinder` 可以在控制器方法执行前对 `WebDataBinder` 进行定制，常见用途有：

```java
@Controller
public class UserController {

    @InitBinder
    public void initBinder(WebDataBinder binder) {
        // 只允许绑定指定字段
        binder.setAllowedFields("name", "email", "age");

        // 禁止绑定敏感字段，防止用户篡改
        binder.setDisallowedFields("id", "createdAt", "role");

        // 注册自定义日期编辑器
        binder.registerCustomEditor(Date.class,
            new CustomDateEditor(new SimpleDateFormat("yyyy-MM-dd"), true));
    }
}
```

`setDisallowedFields` 是一个很重要的安全手段——它能防止所谓的"质量赋值漏洞"（Mass Assignment），避免用户通过构造请求参数来修改不应该被修改的字段（比如把自己的 role 改成 admin）。

###### 8. @InitBinder 注解的作用是什么？

`@InitBinder` 用于标记控制器中的数据绑定初始化方法，在每次数据绑定操作之前自动执行，允许对 `WebDataBinder` 进行精细化定制。

它的作用范围默认仅限当前控制器，如果需要全局生效，放到 `@ControllerAdvice` 类里即可。

常见应用场景：

```java
// 1. 安全控制：禁止绑定敏感字段
@InitBinder
public void initBinder(WebDataBinder binder) {
    binder.setDisallowedFields("id", "version", "createdBy");
}

// 2. 自定义属性编辑器：处理特殊格式
@InitBinder
public void initBinder(WebDataBinder binder) {
    binder.registerCustomEditor(Date.class, new PropertyEditorSupport() {
        @Override
        public void setAsText(String text) throws IllegalArgumentException {
            try {
                setValue(new SimpleDateFormat("yyyy/MM/dd").parse(text));
            } catch (ParseException e) {
                throw new IllegalArgumentException("日期格式错误");
            }
        }
    });
}

// 3. 针对特定控制器设置字段前缀
@InitBinder
public void initAdminBinder(WebDataBinder binder) {
    binder.setFieldDefaultPrefix("admin.");
}
```

###### 9. 如何处理日期类型的参数？

日期类型是 Web 开发中的老大难问题——前端传个字符串，后端要解析成 `Date` 或 `LocalDateTime`，格式还各种不统一。SpringMVC 提供了三种处理方式，按复杂度由低到高：

**方式一：@DateTimeFormat（最简单，最推荐）**

```java
public class User {
    @DateTimeFormat(pattern = "yyyy-MM-dd")
    private Date birthday;

    @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss")
    private LocalDateTime createTime;
}
```

**方式二：全局转换器（适合统一格式要求）**

```java
@Configuration
public class DateConfig {
    @Bean
    public ConversionService conversionService() {
        ConversionServiceFactoryBean factory = new ConversionServiceFactoryBean();
        factory.setConverters(Set.of(new StringToDateConverter()));
        return factory.getObject();
    }
}
```

**方式三：@InitBinder 配合 PropertyEditor（适合遗留代码）**

```java
@InitBinder
public void initBinder(WebDataBinder binder) {
    SimpleDateFormat dateFormat = new SimpleDateFormat("yyyy-MM-dd");
    dateFormat.setLenient(false);
    binder.registerCustomEditor(Date.class, new CustomDateEditor(dateFormat, true));
}
```

实际项目里，新代码优先选方式一；如果整个系统日期格式统一，考虑方式二全局配置；方式三主要是兼容旧项目用。

###### 10. PropertyEditor 和 Converter 的区别是什么？

两者都能做类型转换，但来自不同的体系，设计理念差异明显：

`PropertyEditor` 来自 JavaBeans 规范，最初是为 GUI 环境设计的，**非线程安全**，没有泛型支持，每次使用需要通过 `WebDataBinder` 注册，属于"老一代"方案。

`Converter<S, T>` 是 Spring 3.0 引入的新体系，**线程安全**，有完整的泛型支持，通过 `ConversionService` 统一管理，扩展性更好，是现代 Spring 应用的首选方案。

```java
// PropertyEditor：基于 JavaBean 规范，有状态，非线程安全
public interface PropertyEditor {
    void setAsText(String text) throws IllegalArgumentException;
    Object getValue();
}

// Converter：Spring 泛型设计，无状态，线程安全
public interface Converter<S, T> {
    T convert(S source);
}
```

**选择建议**：新项目统一用 `Converter`；遗留系统如果已经大量用了 `PropertyEditor`，维护期间可以并存，逐步迁移；简单的单例转换可以用 `Formatter`，它比 `Converter` 多了本地化格式化的能力，在 Web 层更合适。
