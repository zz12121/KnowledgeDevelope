###### 1. SpringMVC 如何处理静态资源？

SpringMVC 处理静态资源的核心挑战是：`DispatcherServlet` 通常映射到 `/`（根路径），会拦截所有请求，包括对 CSS、JS、图片等静态文件的请求。由于这些请求找不到对应的控制器，会直接返回 404。

Spring MVC 提供两种主流解决方案：

**方案一：`mvc:default-servlet-handler`（简单，委托给容器）**
让静态资源请求直接交还给 Servlet 容器的默认 Servlet 处理，相当于"我不管，你来处理"。

**方案二：`ResourceHandlerRegistry`（推荐，功能强大）**
用 Spring 自己的 `ResourceHttpRequestHandler` 处理，支持缓存控制、Gzip 压缩、版本化 URL 等高级特性。

```java
@Configuration
@EnableWebMvc
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        // classpath 下的静态资源，缓存 1 小时
        registry.addResourceHandler("/static/**")
                .addResourceLocations("classpath:/static/")
                .setCachePeriod(3600);

        // 服务器文件系统上的上传文件目录
        registry.addResourceHandler("/uploads/**")
                .addResourceLocations("file:/var/uploads/")
                .setCachePeriod(86400);
    }
}
```

###### 2. mvc:resources 标签的作用是什么？

`mvc:resources` 是 XML 配置时代配置静态资源处理器的标签，对应 Java Config 里的 `addResourceHandlers()`。核心实现是 `ResourceHttpRequestHandler`，提供比容器默认 Servlet 更强大的静态资源服务能力。

```xml
<!-- XML 配置 -->
<mvc:resources mapping="/static/**" location="classpath:/static/" cache-period="3600"/>
<mvc:resources mapping="/images/**" location="/var/www/images/" cache-period="86400"/>
```

`ResourceHttpRequestHandler` 在响应静态资源时会做几件重要的事：

1. **缓存控制**：设置 `Cache-Control` 头，告诉浏览器缓存策略
2. **条件请求**：支持 `If-Modified-Since` 头，文件未改变时返回 304，节省带宽
3. **内容类型检测**：根据文件扩展名自动设置 `Content-Type`

高级特性——版本化资源 URL（解决缓存更新问题）：

```xml
<mvc:resources mapping="/static/**" location="classpath:/static/">
    <mvc:resource-chain resource-cache="true">
        <mvc:resolvers>
            <mvc:version-resolver>
                <mvc:content-version-strategy patterns="/**"/>
            </mvc:version-resolver>
        </mvc:resolvers>
    </mvc:resource-chain>
</mvc:resources>
```

这个配置会自动给资源 URL 加上 MD5 指纹（如 `style-abc123.css`），彻底解决"改了代码但浏览器还用旧缓存"的问题。

###### 3. mvc:default-servlet-handler 的作用是什么？

`mvc:default-servlet-handler` 是一个回退方案，把 Spring MVC 找不到控制器的请求委托给 Servlet 容器的默认 Servlet 处理。

工作原理：注册一个 `DefaultServletHttpRequestHandler`，当 Spring MVC 的 `HandlerMapping` 找不到处理器时，这个 Handler 通过 `RequestDispatcher.forward()` 把请求转发给容器的默认 Servlet（Tomcat 叫 `default`，WebLogic 叫 `FileServlet`）。

```java
@Configuration
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void configureDefaultServletHandling(DefaultServletHandlerConfigurer configurer) {
        configurer.enable();  // 等同于 XML 里的 <mvc:default-servlet-handler/>
    }
}
```

**局限性**：只能提供基础文件服务，没有缓存控制、版本化等高级特性，依赖具体 Servlet 容器实现。现代项目更推荐用 `addResourceHandlers()` 方案，功能更完善，行为也更可控。

###### 4. ResourceHandlerRegistry 的作用是什么？

`ResourceHandlerRegistry` 是 Spring MVC Java Config 体系中静态资源的注册中心，通过 `WebMvcConfigurer.addResourceHandlers()` 使用，提供了类型安全的流式 API 配置静态资源处理器。

```java
@Configuration
@EnableWebMvc
public class WebMvcConfig implements WebMvcConfigurer {

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {

        // 基础配置：classpath 静态资源 + 本地文件系统
        registry.addResourceHandler("/static/**")
                .addResourceLocations(
                    "classpath:/static/",   // 先从 classpath 找
                    "file:/opt/static/"     // 找不到再从文件系统找
                )
                .setCachePeriod(3600);

        // 精细缓存控制：30 天强缓存，公共可缓存
        registry.addResourceHandler("/assets/**")
                .addResourceLocations("classpath:/assets/")
                .setCacheControl(CacheControl.maxAge(30, TimeUnit.DAYS)
                    .cachePublic());

        // 高级资源链：Gzip 压缩 + 版本化 URL
        registry.addResourceHandler("/js/**")
                .addResourceLocations("classpath:/static/js/")
                .resourceChain(true)
                .addResolver(new GzipResourceResolver())   // 优先返回 .gz 文件
                .addResolver(new VersionResourceResolver().addContentVersionStrategy("/**"))
                .addTransformer(new CssLinkResourceTransformer());
    }
}
```

多路径配置时，按顺序查找，前面的路径优先；`setCacheControl()` 比 `setCachePeriod()` 更灵活，可以设置 `no-cache`、`must-revalidate` 等指令。
