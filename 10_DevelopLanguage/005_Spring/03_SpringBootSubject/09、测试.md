###### 1. Spring Boot 如何进行单元测试?

Spring Boot 单元测试基于 JUnit 5 框架和 Spring TestContext 框架构建，通过 `spring-boot-starter-test` 传递引入完整测试工具链：JUnit 5、Mockito、AssertJ、JSONPath 等。

**引入依赖：**

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-test</artifactId>
    <scope>test</scope>
</dependency>
```

**测试分层策略（按轻到重选择）：**

- **纯单元测试**：`@ExtendWith(MockitoExtension.class)`，不加载 Spring 上下文，速度最快
- **Web 层测试**：`@WebMvcTest`，只加载 Controller 和 Web 相关组件
- **数据层测试**：`@DataJpaTest`，只加载 JPA 相关组件，自动配置内存数据库
- **集成测试**：`@SpringBootTest`，加载完整应用上下文，速度最慢

**实际建议：** 业务逻辑用纯 Mockito 测试，Controller 用 `@WebMvcTest`，Repository 用 `@DataJpaTest`，端到端流程用 `@SpringBootTest`。避免所有测试都加 `@SpringBootTest`，那样每次跑测试都要启动整个应用，非常慢。

###### 2. @SpringBootTest 注解的作用是什么?

`@SpringBootTest` 是 Spring Boot 测试体系的核心注解，内部包含 `@BootstrapWith(SpringBootTestContextBootstrapper)` 和 `@ExtendWith(SpringExtension)`，从主类（`@SpringBootApplication`）开始执行完整的组件扫描和自动配置。

**四种 Web 环境模式：**

```java
// MOCK（默认）：模拟 Servlet 环境，不启动真实服务器，配合 MockMvc 使用
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.MOCK)

// RANDOM_PORT：启动真实嵌入式服务器，随机端口，配合 TestRestTemplate 做真实 HTTP 测试
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)

// DEFINED_PORT：使用配置文件定义的端口

// NONE：不提供 Web 环境，纯 Spring 上下文
```

**上下文缓存（重要性能优化）：** Spring TestContext 框架会缓存相同配置的应用上下文，多个测试类共享同一个上下文，避免重复初始化。测试类加 `@DirtiesContext` 会强制刷新缓存（性能代价较大，谨慎使用）。

**属性覆盖（隔离测试环境）：**

```java
@SpringBootTest(properties = {
    "spring.datasource.url=jdbc:h2:mem:testdb",
    "spring.profiles.active=test"
})
class CustomConfigTest { ... }
```

###### 3. 如何进行 Mock 测试（Mockito）?

Mock 测试通过模拟依赖对象的行为实现测试隔离，是单元测试的核心手段，让你只测目标类的逻辑而不依赖外部真实资源。

**注解方式 Mock（推荐，最常用）：**

```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;  // 创建 Mock 对象

    @InjectMocks
    private UserService userService;         // 自动注入 Mock 依赖

    @Test
    void getUser_shouldReturnUser() {
        // 配置 Mock 行为
        User mockUser = new User("John");
        when(userRepository.findById(1L)).thenReturn(Optional.of(mockUser));

        // 执行
        User result = userService.getUser(1L);

        // 验证结果
        assertThat(result).isEqualTo(mockUser);
        // 验证方法被调用了一次
        verify(userRepository, times(1)).findById(1L);
    }
}
```

**`@MockBean`（Spring Boot 集成 Mock）：**

```java
@SpringBootTest
class ServiceIntegrationTest {

    @MockBean  // 注册到 Spring 上下文，自动替换同类型的真实 Bean
    private ExternalService externalService;

    @Autowired
    private MainService mainService;  // 注入的 externalService 是 Mock 的

    @Test
    void testWithMockBean() {
        when(externalService.callApi()).thenReturn("mocked response");
        String result = mainService.processWithExternal();
        assertThat(result).contains("mocked");
    }
}
```

**`@Mock` vs `@MockBean`：** `@Mock` 是纯 Mockito，不启动 Spring 上下文，速度快；`@MockBean` 会导致 Spring 上下文重新加载（因为 Bean 定义变了），适合需要 Spring 上下文的集成测试。

**高级用法：**

```java
// 模拟抛出异常
when(userRepository.findById(999L)).thenThrow(new RuntimeException("not found"));

// 灵活参数匹配
when(userRepository.findById(anyLong())).thenReturn(defaultUser);
when(userRepository.findByAge(argThat(age -> age > 18))).thenReturn(adults);

// 验证无多余调用
verifyNoMoreInteractions(userRepository);
```

###### 4. 如何测试 Controller 层?

Controller 层测试主要验证请求映射、参数绑定、响应生成等 Web 相关功能，使用 `@WebMvcTest` + `MockMvc`，不启动真实服务器，速度快。

**基础配置：**

```java
@WebMvcTest(UserController.class)  // 只加载 Web 相关组件，隔离其他 Bean
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private UserService userService;  // Service 层 Mock 掉

    @Test
    void getUser_shouldReturnUser() throws Exception {
        when(userService.getUser(1L)).thenReturn(new User(1L, "John"));

        mockMvc.perform(get("/users/1")
                .accept(MediaType.APPLICATION_JSON))
            .andDo(print())                                    // 打印请求/响应详情
            .andExpect(status().isOk())                        // 验证状态码
            .andExpect(content().contentType(MediaType.APPLICATION_JSON))
            .andExpect(jsonPath("$.id").value(1))              // JSON 路径断言
            .andExpect(jsonPath("$.name").value("John"));
    }

    @Test
    void createUser_shouldReturnCreated() throws Exception {
        mockMvc.perform(post("/users")
                .contentType(MediaType.APPLICATION_JSON)
                .content("""{"name": "John", "email": "john@example.com"}"""))
            .andExpect(status().isCreated())
            .andExpect(header().string("Location", containsString("/users/")));
    }

    @Test
    void getUser_notFound_shouldReturn404() throws Exception {
        when(userService.getUser(999L)).thenThrow(new UserNotFoundException("not found"));

        mockMvc.perform(get("/users/999"))
            .andExpect(status().isNotFound())
            .andExpect(jsonPath("$.error").value("USER_NOT_FOUND"));
    }
}
```

**带安全上下文的测试（配合 Spring Security）：**

```java
mockMvc.perform(get("/admin/users")
        .with(SecurityMockMvcRequestPostProcessors.user("admin").roles("ADMIN")))
    .andExpect(status().isOk());
```

Controller 测试要覆盖：正常场景、参数校验失败、权限不足、异常处理这四类。

###### 5. 如何测试 Service 层?

Service 层测试聚焦业务逻辑的正确性，策略分两种：

**纯单元测试（业务逻辑复杂时推荐）：**

```java
@ExtendWith(MockitoExtension.class)
class UserServiceUnitTest {

    @Mock private UserRepository userRepository;
    @Mock private EmailService emailService;
    @InjectMocks private UserServiceImpl userService;

    @Test
    void createUser_shouldSaveUserAndSendEmail() {
        UserCreateRequest request = new UserCreateRequest("John", "john@example.com");
        User savedUser = new User(1L, "John", "john@example.com");

        when(userRepository.save(any(User.class))).thenReturn(savedUser);
        doNothing().when(emailService).sendWelcomeEmail(anyString());

        User result = userService.createUser(request);

        assertThat(result.getId()).isNotNull();
        verify(userRepository).save(any(User.class));
        verify(emailService).sendWelcomeEmail("john@example.com");
        verifyNoMoreInteractions(userRepository, emailService);  // 验证无多余调用
    }

    @Test
    void createUser_duplicateEmail_shouldThrowException() {
        when(userRepository.existsByEmail("duplicate@example.com")).thenReturn(true);

        assertThatThrownBy(() -> userService.createUser(request))
            .isInstanceOf(DuplicateEmailException.class);

        verify(userRepository, never()).save(any());  // 验证 save 未被调用
    }
}
```

**事务性测试（验证数据访问逻辑，测试后自动回滚）：**

```java
@SpringBootTest
@Transactional  // 测试结束后自动回滚，不污染数据库
class UserServiceTxTest {

    @Autowired
    private UserService userService;

    @Test
    void testWithinTransaction() {
        User user = userService.createUser(new UserCreateRequest("Test", "test@example.com"));
        assertThat(userService.getUser(user.getId())).isPresent();
    }
}
```

###### 6. 如何测试 Repository 层?

Repository 层测试验证数据访问逻辑，`@DataJpaTest` 自动配置 H2 内存数据库和 JPA，不加载 Web 层，速度快。

```java
@DataJpaTest
class UserRepositoryTest {

    @Autowired
    private TestEntityManager entityManager;  // JPA 测试工具，用于准备数据

    @Autowired
    private UserRepository userRepository;

    @BeforeEach
    void setUp() {
        entityManager.persistFlushFind(new User("John", "john@example.com"));
    }

    @Test
    void findByEmail_shouldReturnUser() {
        Optional<User> found = userRepository.findByEmail("john@example.com");
        assertThat(found).isPresent();
        assertThat(found.get().getName()).isEqualTo("John");
    }

    @Test
    void existsByEmail_shouldReturnTrue() {
        assertThat(userRepository.existsByEmail("john@example.com")).isTrue();
    }

    @Test
    void save_shouldPersistUser() {
        User newUser = new User("Alice", "alice@example.com");
        User saved = userRepository.save(newUser);
        assertThat(saved.getId()).isNotNull();
    }
}
```

**用 SQL 脚本初始化测试数据：**

```java
@Sql("/test-data/users.sql")
@Test
void testWithSqlData() {
    assertThat(userRepository.findAll()).hasSize(5);
}
```

**需要真实数据库（Testcontainers）：**

```java
@SpringBootTest
@Testcontainers
class RealDatabaseTest {
    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:13");

    @DynamicPropertySource
    static void configureProperties(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
    }
}
```

###### 7. 如何进行集成测试?

集成测试验证多个组件协同工作的正确性，使用真实或接近真实的环境进行端到端的流程验证。

**完整 HTTP 集成测试（启动真实服务器）：**

```java
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
class ApplicationIntegrationTest {

    @Autowired
    private TestRestTemplate restTemplate;  // 支持真实 HTTP 调用

    @LocalServerPort
    private int serverPort;

    @Autowired
    private UserRepository userRepository;

    @Test
    void fullWorkflow_shouldWork() {
        User user = userRepository.save(new User("Integration", "integration@example.com"));

        ResponseEntity<User> response = restTemplate.getForEntity(
            "http://localhost:" + serverPort + "/users/{id}", User.class, user.getId());

        assertThat(response.getStatusCode()).isEqualTo(HttpStatus.OK);
        assertThat(response.getBody().getName()).isEqualTo("Integration");
    }
}
```

**消息队列集成测试：**

```java
@Test
void messageFlow_shouldWork() throws InterruptedException {
    rabbitTemplate.convertAndSend("test.exchange", "test.routing", "Test message");

    // 使用 Awaitility 等待异步处理完成
    await().atMost(5, TimeUnit.SECONDS)
        .untilAsserted(() -> {
            assertThat(messageListener.getLastMessage()).isEqualTo("Test message");
        });
}
```

集成测试建议独立运行（不在每次提交时执行），在 CI/CD 的特定阶段触发，避免影响开发速度。

###### 8. 如何测试异步方法?

异步方法测试需要特殊处理，因为主线程直接断言时异步任务可能还没执行完。

**CompletableFuture 测试（使用 AssertJ 的 future 断言）：**

```java
@SpringBootTest
class AsyncServiceTest {

    @Autowired
    private AsyncUserService asyncUserService;

    @Test
    void processAsync_shouldComplete() {
        CompletableFuture<String> future = asyncUserService.processAsync("data");

        assertThat(future)
            .succeedsWithin(5, TimeUnit.SECONDS)  // 等待最多 5s
            .isEqualTo("processed_data");
    }

    @Test
    void processAsync_exception_shouldFail() {
        CompletableFuture<String> future = asyncUserService.processAsync("error_input");

        assertThat(future)
            .failsWithin(5, TimeUnit.SECONDS)
            .withThrowableOfType(ExecutionException.class)
            .withCauseInstanceOf(BusinessException.class);
    }
}
```

**响应式（Reactor）测试（使用 StepVerifier）：**

```java
@Test
void fluxOperation_shouldEmitItems() {
    Flux<String> result = reactiveUserService.getUsers();

    StepVerifier.create(result)
        .expectNext("user1", "user2", "user3")
        .verifyComplete();  // 验证流正常结束
}

@Test
void monoOperation_shouldEmitSingle() {
    StepVerifier.create(reactiveUserService.findUser(1L))
        .assertNext(user -> assertThat(user.getName()).isEqualTo("John"))
        .verifyComplete();
}
```

**事件监听异步测试（Mockito verify + timeout）：**

```java
// 验证在 5000ms 内异步事件监听器被调用
verify(eventListener, timeout(5000)).handleUserCreated(any(UserCreatedEvent.class));
```
