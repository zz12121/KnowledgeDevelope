# 05、故障排查与监控

## Q1. Tomcat 内存泄漏和频繁 Full GC 如何排查？

**症状**：堆内存持续增长不释放，Full GC 越来越频繁，最终 OOM。

```bash
# 步骤1：确认堆内存增长趋势
jstat -gcutil $(pgrep java) 1000  # 每秒观察一次
# 老年代（O列）持续增长不下降 → 内存泄漏
# O列满了就触发 Full GC（FGC 列）

# 步骤2：Heap Dump 分析（生产上先用轻量命令快速定位）
jmap -histo $(pgrep java) | head -30   # 快速看哪些类实例最多
# 发现异常多的类 → 是内存泄漏的嫌疑

# 步骤3：完整 Heap Dump 分析
jmap -dump:format=b,file=/tmp/heap.hprof $(pgrep java)
# 用 Eclipse MAT 或 VisualVM 分析 heap.hprof
# 重点看：
# - Leak Suspects Report（MAT 自动识别泄漏嫌疑）
# - Dominator Tree（占内存最多的对象树）
```

**Tomcat 常见内存泄漏原因**：
```
1. WebappClassLoader 泄漏（热部署/卸载后 ClassLoader 无法被GC）
   → 原因：ThreadLocal 未清理、JDBC Driver 未注销、定时任务线程未停止
   → Tomcat 日志：SEVERE: The web application [/myapp] appears to have started a thread named [xxx]

2. Session 泄漏
   → 大量 Session 未过期，内存被占满
   → 检查：jmap -histo | grep Session

3. 静态集合持有对象引用
   → static Map/List 持续向里塞对象，没有清理
```

## Q2. Tomcat 请求响应慢如何排查？

```bash
# ==================== 第一步：确认慢在哪里 ====================

# 1. 查看访问日志中的响应时间
# AccessLogValve 中加 %D（毫秒）或 %T（秒）
# pattern="%h %l %u %t &quot;%r&quot; %s %b %D"
awk '{print $NF}' access.log | sort -n | tail -20   # 最慢的请求

# 2. 实时监控线程状态（线程阻塞是慢的常见原因）
jstack $(pgrep java) | grep -A5 "BLOCKED\|WAITING"

# 3. 查看线程池是否满
# Tomcat Manager App: http://localhost:8080/manager/status
# 或 JMX：Catalina:type=ThreadPool,name=...

# ==================== 常见慢请求原因 ====================

# 慢 SQL：开启 Spring Boot 的 SQL 日志
# logging.level.org.hibernate.SQL=DEBUG
# 或 MyBatis 慢 SQL 日志（>2s 记录）

# 外部服务超时：HTTP Client 超时设置太长
# RestTemplate/Feign 设置合理的 connectTimeout/readTimeout

# GC 停顿：查看 GC 日志，看 STW 时间
grep "Pause" /app/logs/gc.log | awk '{print $NF}' | sort -rn | head

# 线程池满导致请求排队
# 看 Tomcat 的 currentThreadsBusy 指标
```

## Q3. Tomcat 启动失败的常见原因？

```bash
# ==================== 端口占用 ====================
# Address already in use: bind
ss -tlnp | grep 8080
kill -9 $(lsof -ti :8080)

# ==================== web.xml 配置错误 ====================
# 查看 catalina.out 中的错误信息
# 常见：ContextLoaderListener 加载失败（Spring 配置错误）
tail -100 logs/catalina.out | grep -i "error\|exception\|failed"

# ==================== 类文件版本不兼容 ====================
# UnsupportedClassVersionError: xxx has been compiled by a more recent version
# 应用编译的 Java 版本 > Tomcat 运行的 JDK 版本
java -version  # 检查 JDK 版本

# ==================== 内存不足 ====================
# Cannot create new native thread / GC overhead limit exceeded
# 增大 -Xmx，或检查是否 JVM 参数配置有误

# ==================== 权限问题 ====================
# Permission denied
ls -la logs/  # 检查日志目录权限
id tomcat     # 确认 Tomcat 运行用户
```

## Q4. Tomcat 的日志体系是什么？

```bash
logs/
├── catalina.out        # 所有标准输出/错误（System.out.println、未捕获异常）
├── catalina.YYYY-MM-DD.log  # Tomcat 组件日志（按天滚动）
├── localhost.YYYY-MM-DD.log # 应用相关日志（Context 启动失败等）
├── manager.YYYY-MM-DD.log   # Manager App 操作日志
├── host-manager.YYYY-MM-DD.log
└── localhost_access_log.YYYY-MM-DD.txt  # HTTP 访问日志
```

**访问日志格式配置（AccessLogValve）**：
```xml
<Valve className="org.apache.catalina.valves.AccessLogValve"
       directory="logs"
       prefix="localhost_access_log"
       suffix=".txt"
       pattern="%{X-Real-IP}i %l %u %t &quot;%r&quot; %s %b %D %{User-Agent}i"
       rotatable="true" />
```

**常用 pattern 变量**：
- `%h`：远端主机 IP
- `%t`：时间
- `%r`：请求行（方法+URL+协议）
- `%s`：响应状态码
- `%b`：响应字节数
- `%D`：请求处理时间（毫秒）
- `%{Header-Name}i`：请求头值

## Q5. Tomcat 安全加固措施有哪些？

```xml
<!-- 1. 修改默认端口（8080 → 非常见端口）-->
<Connector port="18080" .../>
<!-- 2. 修改 shutdown 端口和命令（8005 → 随机）-->
<Server port="18005" shutdown="RANDOM_STRING_XYZ">

<!-- 3. 移除不需要的默认应用 -->
<!-- 删除 webapps 下的 ROOT、docs、examples、manager、host-manager -->
<!-- 或至少修改 manager 应用的访问密码和 IP 限制 -->

<!-- 4. 隐藏 Tomcat 版本信息 -->
```

```java
// conf/server.xml 中 Connector 加
server=" "   // 清空 Server 响应头（空格）
```

```xml
<!-- conf/web.xml 中 DefaultServlet 禁止目录列表 -->
<init-param>
    <param-name>listings</param-name>
    <param-value>false</param-value>   <!-- 禁止目录列表 -->
</init-param>

<!-- 5. 禁用 PUT/DELETE 等危险方法（conf/web.xml）-->
<security-constraint>
    <web-resource-collection>
        <web-resource-name>Restricted Methods</web-resource-name>
        <url-pattern>/*</url-pattern>
        <http-method>PUT</http-method>
        <http-method>DELETE</http-method>
        <http-method>TRACE</http-method>
    </web-resource-collection>
    <auth-constraint/>   <!-- 空 auth-constraint 表示拒绝所有 -->
</security-constraint>
```

## Q6. Tomcat 与 Spring Boot 的关系？

Spring Boot 将 Tomcat 作为**嵌入式容器**内嵌在应用 jar 中：

```
传统 Web 部署：               Spring Boot 嵌入式部署：
外部 Tomcat 进程              应用自包含 jar（含内嵌 Tomcat）
    ↑                             ↑
  war 包（放入 webapps/）        java -jar myapp.jar
```

**Spring Boot 集成嵌入式 Tomcat 的关键步骤**：
1. `spring-boot-starter-web` 引入了 `spring-boot-starter-tomcat`
2. `TomcatServletWebServerFactory` 负责创建和配置 Tomcat
3. `SpringApplication.run()` → 创建 `TomcatWebServer` → 启动 Tomcat → 注册 `DispatcherServlet`

**可以换掉嵌入式 Tomcat**：
```xml
<!-- 换成 Undertow -->
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <exclusions>
        <exclusion>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-tomcat</artifactId>
        </exclusion>
    </exclusions>
</dependency>
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-undertow</artifactId>
</dependency>
```

## Q7. Tomcat 与 Docker/Kubernetes 集成注意事项

```dockerfile
# 官方镜像
FROM tomcat:10.1-jdk17-temurin

# 删除默认应用（安全）
RUN rm -rf /usr/local/tomcat/webapps/*

# 复制 war 到 ROOT（根路径）
COPY myapp.war /usr/local/tomcat/webapps/ROOT.war

# 设置 JVM 参数
ENV JAVA_OPTS="-Xms512m -Xmx1g -XX:+UseG1GC"

EXPOSE 8080
```

**Kubernetes 关键配置**：
```yaml
# 优雅关闭
spec:
  terminationGracePeriodSeconds: 60  # 给 Tomcat 足够时间处理完请求
  containers:
  - name: app
    # 健康检查
    livenessProbe:
      httpGet:
        path: /actuator/health/liveness
        port: 8080
      initialDelaySeconds: 30
    readinessProbe:
      httpGet:
        path: /actuator/health/readiness
        port: 8080
    # 资源限制（影响 JVM MaxRAMPercentage 计算）
    resources:
      limits:
        memory: "1Gi"
      requests:
        memory: "512Mi"
```

> **追问：Spring Boot 嵌入式 Tomcat 如何调优线程池？**
> ```yaml
> server:
>   tomcat:
>     threads:
>       max: 400          # maxThreads
>       min-spare: 50     # minSpareThreads
>     max-connections: 10000
>     accept-count: 200
>     connection-timeout: 5s
>     keep-alive-timeout: 15s
>     max-keep-alive-requests: 100
> ```
