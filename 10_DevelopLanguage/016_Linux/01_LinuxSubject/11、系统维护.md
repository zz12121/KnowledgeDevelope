# 11、系统维护

## Q1. 如何安全地重启和关闭 Linux 系统？

```bash
# 立即关机
shutdown -h now         # 等价 poweroff / halt
# 定时关机
shutdown -h +10         # 10分钟后关机
shutdown -h 22:00       # 今晚22点关机
# 取消定时关机
shutdown -c

# 重启
reboot                  # 等价 shutdown -r now
shutdown -r +5 "5分钟后重启维护" # 带提示消息的定时重启

# 系统维护模式（单用户/救援模式）
systemctl rescue        # 救援模式（类似runlevel 1）
systemctl emergency     # 紧急模式（最小服务，挂载只读）

# 确认重启前的准备清单
who                     # 确认是否有其他用户在线
ss -tnp | wc -l        # 确认活跃连接数
systemctl list-jobs     # 确认没有长时间运行的systemd任务
```

## Q2. 如何进行磁盘和文件系统维护？

```bash
# ==================== 磁盘使用分析 ====================
df -h                   # 文件系统空间使用
df -i                   # inode 使用情况
du -sh /var/*           # 各目录大小（-s 汇总，-h 人性化）
du -h --max-depth=2 /   # 递归2层
# 找出最大的目录
du -h /var | sort -rh | head -20

# 查找大文件
find / -type f -size +1G 2>/dev/null    # 大于1GB的文件
find /var/log -name "*.log" -size +100M  # 日志目录中大于100M的
# 最近修改的大文件
find / -type f -size +100M -mtime -7 2>/dev/null

# ==================== 文件系统检查和修复 ====================
# 检查前必须卸载（或以只读挂载）
umount /dev/sdb1
# 文件系统检查
fsck /dev/sdb1              # 自动选择工具
e2fsck -f /dev/sdb1         # ext4 检查（-f 强制检查）
xfs_repair /dev/sdb1        # XFS 修复

# 强制下次启动时检查根文件系统
touch /forcefsck            # CentOS 6
# 或在 /etc/fstab 中将 pass 字段改为非0

# ==================== 磁盘分区和格式化 ====================
# 查看磁盘和分区
lsblk                       # 块设备树状视图
fdisk -l                    # 所有磁盘分区信息
parted -l                   # 支持GPT的查看方式

# 创建分区（交互式）
fdisk /dev/sdb              # MBR分区（<2TB）
gdisk /dev/sdb              # GPT分区（推荐）
parted /dev/sdb             # 统一工具

# 格式化
mkfs.ext4 /dev/sdb1         # 格式化为 ext4
mkfs.xfs /dev/sdb1          # 格式化为 XFS（大文件/高性能）

# 挂载
mount /dev/sdb1 /data
# 永久挂载（/etc/fstab）
echo "/dev/sdb1  /data  ext4  defaults  0  2" >> /etc/fstab
mount -a    # 测试 fstab 是否正确
```

## Q3. 如何进行系统备份和恢复？

```bash
# ==================== tar 备份 ====================
# 全量备份
tar -czpf /backup/full-$(date +%Y%m%d).tar.gz \
    --exclude=/proc --exclude=/sys --exclude=/dev \
    --exclude=/backup --exclude=/tmp \
    /

# 增量备份（与上次比较）
tar -czpf /backup/inc-$(date +%Y%m%d).tar.gz \
    --newer-mtime="2026-03-01" /app

# 恢复
tar -xzpf backup.tar.gz -C /restore/dir

# ==================== rsync 同步备份 ====================
# 本地同步
rsync -av --delete /source/ /destination/
# 远程同步（SSH）
rsync -avz --delete -e "ssh -p 22222" /app/ user@backup-server:/backup/app/

# rsync 常用参数
# -a  归档模式（保留权限/时间/链接等，等价 -rlptgoD）
# -v  显示进度
# -z  传输时压缩
# --delete  删除目标中源没有的文件
# --exclude='*.log'  排除文件
# --dry-run  演习，不实际执行

# ==================== 定期自动备份脚本 ====================
# /etc/cron.d/app-backup
0 2 * * * root /opt/scripts/backup.sh >> /var/log/backup.log 2>&1
```

**数据库备份**：
```bash
# MySQL/MariaDB
mysqldump -u root -p dbname > /backup/db-$(date +%Y%m%d).sql
mysqldump --all-databases > /backup/all-db-$(date +%Y%m%d).sql

# 恢复
mysql -u root -p dbname < /backup/db-20260315.sql
```

## Q4. 如何分析系统日志排查问题？

```bash
# ==================== systemd journal 日志 ====================
journalctl                          # 所有日志
journalctl -b                       # 本次启动日志
journalctl -b -1                    # 上次启动日志
journalctl -u nginx -f              # 实时查看服务日志
journalctl -u nginx --since "1 hour ago"
journalctl -p err                   # 只看错误级别
journalctl --disk-usage             # 日志占用空间
journalctl --vacuum-time=7d         # 清理7天前的日志

# ==================== 系统日志文件 ====================
/var/log/messages      # CentOS 通用系统消息
/var/log/syslog        # Ubuntu 通用系统消息
/var/log/secure        # CentOS 安全/认证日志
/var/log/auth.log      # Ubuntu 认证日志
/var/log/kern.log      # 内核日志
/var/log/dmesg         # 硬件/驱动启动日志（也可用 dmesg 命令）
/var/log/cron          # cron 执行记录
/var/log/nginx/        # Nginx 日志
/var/log/mysql/        # MySQL 日志

# ==================== 常用日志分析命令 ====================
# 统计 Nginx 访问IP排名
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

# 统计 HTTP 状态码分布
awk '{print $9}' /var/log/nginx/access.log | sort | uniq -c | sort -rn

# 找出 502 错误的时间段
grep " 502 " /var/log/nginx/access.log | awk '{print $4}' | cut -d: -f1-2 | uniq -c

# 分析慢请求（>1秒的请求）
awk '($NF > 1) {print $0}' /var/log/nginx/access.log | head

# 实时监控错误日志
tail -f /var/log/nginx/error.log | grep -i "error\|crit\|alert"
```

## Q5. crontab 定时任务如何配置和排查？

```bash
# ==================== crontab 基础 ====================
crontab -l          # 查看当前用户的定时任务
crontab -e          # 编辑
crontab -r          # 删除所有（危险！）
crontab -u root -l  # 查看root的定时任务

# crontab 格式（5个时间字段 + 命令）
# 分  时  日  月  周  命令
# 0   2   *   *   *   /bin/backup.sh        每天凌晨2点
# 0   */4 *   *   *   /bin/check.sh         每4小时
# 30  8   *   *   1-5 /bin/report.sh        工作日早8:30
# 0   0   1   *   *   /bin/monthly.sh       每月1号零点
# */5 *   *   *   *   /bin/monitor.sh       每5分钟
# @reboot             /bin/startup.sh       开机时执行
# @daily              /bin/daily.sh         每天一次（=0 0 * * *）

# ==================== /etc/cron.d 系统级定时任务 ====================
# 文件格式与 crontab 类似，但多一个用户字段
# 0 2 * * * root /opt/scripts/backup.sh

# ==================== 排查定时任务不执行的问题 ====================
# 1. 确认 crond 在运行
systemctl status crond   # CentOS
systemctl status cron    # Ubuntu

# 2. 查看 cron 执行日志
grep cron /var/log/syslog
grep CRON /var/log/syslog   # Ubuntu
grep crond /var/log/cron    # CentOS

# 3. 常见问题
# - 脚本没有执行权限：chmod +x script.sh
# - 脚本使用了相对路径（cron 环境变量不同，建议用绝对路径）
# - cron 的 PATH 不含命令所在目录（在脚本开头加 source /etc/profile）
# - 输出重定向：0 2 * * * /bin/script.sh >> /var/log/script.log 2>&1
```

## Q6. 如何进行内核升级和系统升级？

```bash
# ==================== 升级前准备 ====================
# 1. 全量备份（最重要！）
# 2. 在非生产环境验证
# 3. 准备回滚方案
# 4. 选择业务低峰期

# ==================== 只安装安全更新（推荐生产策略）====================
yum update --security          # CentOS/RHEL 只安全更新
yum update-minimal --security  # 最小化安全更新（更保守）
apt-get upgrade --dry-run      # Ubuntu 预演

# 查看可用的安全更新
yum updateinfo list security

# ==================== 内核升级 ====================
# 查看当前内核
uname -r

# 安装新内核（不会删除旧内核，方便回滚）
yum install kernel kernel-headers

# 查看已安装的内核
rpm -qa | grep kernel
grub2-editenv list              # 查看默认启动项

# 设置默认内核（重启后生效）
grub2-set-default 0             # 0 表示第一个（最新内核）
grub2-mkconfig -o /boot/grub2/grub.cfg  # 重新生成grub配置

# 如果新内核有问题，重启时在GRUB菜单选择旧内核

# ==================== 清理旧内核 ====================
# 只保留最新2个内核
package-cleanup --oldkernels --count=2   # yum-utils
# 或
dnf remove $(dnf repoquery --installonly --latest-limit=-2 -q)
```

## Q7. 如何使用 `strace` 和 `lsof` 排查进程问题？

```bash
# ==================== strace — 追踪系统调用 ====================
# 追踪进程的系统调用
strace ls /tmp
strace -p 1234              # 附加到已运行的进程
strace -f -p 1234           # 包括子进程

# 统计各系统调用的耗时（非常有用）
strace -cp 1234 sleep 10    # 采样10秒，输出统计

# 只看特定系统调用
strace -e trace=open,read,write ls /tmp
strace -e trace=network -p 1234   # 只看网络相关调用

# ==================== lsof — 查看打开的文件和连接 ====================
# 查看进程打开的所有文件
lsof -p 1234

# 查看哪个进程打开了某个文件
lsof /var/log/nginx/access.log

# 查看被删除但仍有进程持有fd的文件（占用磁盘空间但ls看不到）
lsof | grep deleted
# 解决方案：找到进程 kill 或等待它释放，或者 > /proc/pid/fd/3 清空文件内容

# 查看网络连接
lsof -i                    # 所有网络连接
lsof -i :8080              # 特定端口
lsof -i TCP:1-1024         # TCP 1-1024 端口

# 查看特定用户的文件
lsof -u appuser
```

## Q8. 虚拟内存和 swap 如何管理？

```bash
# 查看 swap 使用情况
free -h
swapon --show              # 详细信息
cat /proc/swaps

# 创建 swap 文件（动态扩展 swap）
dd if=/dev/zero of=/swapfile bs=1G count=4  # 创建4GB swap文件
chmod 600 /swapfile
mkswap /swapfile
swapon /swapfile

# 永久配置（/etc/fstab）
echo "/swapfile swap swap defaults 0 0" >> /etc/fstab

# 关闭 swap
swapoff /swapfile
# 关闭所有 swap（Java 应用推荐，避免GC时换出）
swapoff -a
# 永久禁用：注释掉 /etc/fstab 中的 swap 行

# swap 倾向配置（vm.swappiness）
# 0  尽量不用swap，内存快满才用
# 60 默认值（Ubuntu/CentOS默认）
# 100 积极使用swap
# Java 服务器推荐设置为 1 或 10
echo "vm.swappiness=10" >> /etc/sysctl.conf
sysctl -p
```

## Q9. 如何处理磁盘空间满的紧急情况？

```bash
# ==================== 快速找到罪魁祸首 ====================
df -h            # 先确认哪个分区满了

# 找大文件
du -sh /* 2>/dev/null | sort -rh | head   # 根目录下各目录大小
du -sh /var/* | sort -rh | head
find /var/log -type f -size +1G            # 找1G以上的日志

# ==================== 常见清理操作 ====================
# 清理日志
truncate -s 0 /var/log/nginx/access.log   # 清空但保留文件（比rm安全）
find /var/log -name "*.gz" -mtime +30 -delete  # 删除30天前压缩日志

# 清理 yum/apt 缓存
yum clean all
apt-get clean

# 清理 /tmp
find /tmp -mtime +7 -delete   # 删除7天前的临时文件
find /var/tmp -mtime +30 -delete

# 清理 Docker 资源（如果有 Docker）
docker system prune -a -f     # 清理所有未使用的镜像、容器、网络
docker volume prune -f

# 清理 Java 应用日志
find /app/logs -name "*.log.*" -mtime +7 -delete  # 滚动日志

# ==================== 处理"文件已删除但空间未释放" ====================
# 文件被删除但进程还在持有fd，需要重启进程或清空文件内容
lsof | grep deleted           # 找到这些文件
# 方案1：重启持有文件的进程
# 方案2：清空文件内容（不用关进程）
cat /dev/null > /proc/PID/fd/FD_NUM
```

## Q10. 如何制定 Linux 系统的日常巡检规范？

```bash
#!/bin/bash
# 每日巡检脚本

echo "===== 系统巡检报告 $(date) ====="

# 1. 系统基本信息
echo "--- 系统信息 ---"
uname -a
uptime

# 2. CPU 使用率
echo "--- CPU 负载 ---"
uptime
mpstat 1 3 | tail -2

# 3. 内存使用
echo "--- 内存使用 ---"
free -h
grep -E "MemTotal|MemFree|MemAvailable|SwapTotal|SwapFree" /proc/meminfo

# 4. 磁盘使用（超过80%告警）
echo "--- 磁盘使用 ---"
df -h | awk 'NR==1 || int($5) > 80 {print}'

# 5. 系统负载
echo "--- 当前负载 ---"
cat /proc/loadavg

# 6. 服务状态
echo "--- 关键服务状态 ---"
for svc in nginx mysql redis java; do
    if systemctl is-active --quiet $svc; then
        echo "$svc: 运行中"
    else
        echo "$svc: [警告] 未运行！"
    fi
done

# 7. 最近异常日志
echo "--- 最近错误日志 ---"
journalctl -p err -b --no-pager | tail -20

# 8. 安全事件
echo "--- 最近失败登录 ---"
lastb | head -5

echo "===== 巡检完成 ====="
```

> **追问：生产环境如何实现自动化运维（不用人工巡检）？**
> ①**监控告警**：Prometheus 采集指标 + Alertmanager 发告警（邮件/钉钉/企微）+ Grafana 看板，当 CPU/内存/磁盘超阈值自动通知；②**日志聚合**：ELK/EFK Stack 集中日志，配合告警规则发现错误激增；③**自动恢复**：systemd Restart 策略（`Restart=on-failure; RestartSec=5`）+ 自定义 Watchdog 脚本（健康检查失败自动重启）；④**基础设施即代码**：Ansible Playbook 标准化部署和配置，避免"雪花服务器"；⑤**变更管理**：Jenkins/GitLab CI 自动化发布，减少人工操作失误。
