###### 1. 查看文件有哪些命令

```bash
cat file.txt          # 一次性输出全部内容（小文件用）
less file.txt         # 分页查看，支持上下翻页、搜索（推荐）
more file.txt         # 分页查看，只能向下翻页（较旧）
head -n 20 file.txt   # 查看前 20 行
tail -n 50 file.txt   # 查看后 50 行
tail -f app.log       # 实时追踪日志（生产必备）
vi / vim file.txt     # 用编辑器查看（可编辑）
grep "关键字" file.txt # 带条件查看
wc -l file.txt        # 查看行数
```

###### 2. 列举几个常用的 Linux 命令

**文件操作**：`ls -la`、`cp -r`、`mv`、`rm -rf`、`mkdir -p`、`touch`、`chmod`、`chown`

**查看内容**：`cat`、`tail -f`、`grep`、`awk`、`sed`

**进程管理**：`ps aux`、`top`、`kill -9 PID`、`nohup ... &`

**网络**：`ping`、`curl`、`wget`、`netstat -lntp`、`ss -lntp`、`lsof -i :8080`

**系统状态**：`df -h`（磁盘）、`free -m`（内存）、`uptime`（负载）、`iostat`（I/O）

**文本处理**：`grep -r "关键字" .`、`awk '{print $1}'`、`sed 's/old/new/g'`

**压缩解压**：`tar -czf out.tar.gz dir/`、`tar -xzf file.tar.gz`

###### 3. 绝对路径用什么符号表示？当前目录、上层目录用什么表示？主目录用什么表示？切换目录用什么命令？

- **绝对路径**：从根目录 `/` 开始，如 `/usr/local/bin`
- **当前目录**：`.`（一个点），或 `./`
- **上层目录**：`..`（两个点），或 `../`
- **主目录（home）**：`~`（波浪号）— 等价于 `/home/username`（root 用户是 `/root`）
- **切换目录**：`cd` 命令
  ```bash
  cd /usr/local    # 切到绝对路径
  cd ../config     # 切到上级目录的 config 子目录
  cd ~             # 切到当前用户的主目录
  cd -             # 切回上一个工作目录（很实用）
  ```

###### 4. 如何创建、删除、复制、移动文件和目录？

```bash
# 创建
touch file.txt           # 创建空文件（或更新时间戳）
mkdir mydir              # 创建目录
mkdir -p a/b/c           # 递归创建多层目录（-p 不报错）

# 删除
rm file.txt              # 删除文件
rm -f file.txt           # 强制删除（不提示）
rm -r dir/               # 递归删除目录（谨慎！）
rm -rf dir/              # 强制递归删除（生产环境慎用！）

# 复制
cp file.txt dest/        # 复制文件到目录
cp -r dir/ dest/         # 递归复制目录
cp -a src/ dest/         # 保留所有属性复制（包含权限/时间戳）

# 移动/重命名
mv file.txt newname.txt  # 重命名
mv file.txt /tmp/        # 移动到其他目录
```

###### 5. 如何查找文件和目录？find 和 locate 命令的区别是什么？

```bash
# find：实时查找，功能强大
find /var/log -name "*.log"                   # 按文件名查找
find /home -type f -mtime -7                  # 7天内修改的文件
find /tmp -size +100M                         # 大于100MB的文件
find /app -name "*.java" | xargs grep "TODO"  # 查找包含TODO的Java文件

# locate：数据库查找，极快（但不是实时的）
locate nginx.conf                             # 查找包含 nginx.conf 的路径
updatedb                                      # 手动更新数据库（root权限）
```

| 维度 | find | locate |
|------|------|--------|
| 速度 | 慢（实时扫描磁盘） | **极快**（查数据库） |
| 实时性 | ✅ 实时 | ❌ 有延迟（数据库每天更新一次） |
| 功能 | **强大**（可按权限/大小/时间/类型过滤，可执行操作） | 简单（只能按文件名查找） |
| 适用场景 | 脚本/自动化/复杂条件查找 | 快速定位已知文件名 |

###### 6. 如何查看和修改文件权限？

```bash
# 查看权限
ls -la file.txt
# -rw-r--r-- 1 user group 1024 Jan 1 10:00 file.txt
#  │││ │││ │││
#  所有者 组 其他人（r=读 w=写 x=执行）

# chmod 数字方式（每组：r=4, w=2, x=1，加权求和）
chmod 755 script.sh    # rwxr-xr-x（所有者全权，组和其他只读执行）
chmod 644 config.conf  # rw-r--r--（所有者读写，其他只读）
chmod 600 ~/.ssh/id_rsa  # 私钥必须 600（否则 SSH 报错）

# chmod 符号方式
chmod +x script.sh     # 给所有用户加执行权限
chmod u+x,g-w file     # 所有者加执行，组去掉写
chmod o-r secret.conf  # 其他用户去掉读权限
chmod -R 755 /var/www/ # 递归修改目录及内容

# 修改所有者
chown user:group file.txt
chown -R nginx:nginx /var/www/html
```

---

###### 7. chmod、chown、chgrp 命令的作用分别是什么？

- **chmod**（change mode）：修改文件/目录的**读写执行权限**
- **chown**（change owner）：修改文件/目录的**所有者（用户和组）**
- **chgrp**（change group）：只修改文件/目录的**所属组**

###### 8. 软链接和硬链接的区别是什么？如何创建？

```bash
# 硬链接：指向同一 inode（磁盘数据块）
ln file.txt hardlink.txt

# 软链接（符号链接）：类似 Windows 快捷方式，存储路径
ln -s /usr/local/nginx/sbin/nginx /usr/bin/nginx
ln -s /absolute/path target  # 推荐使用绝对路径
```

| 维度 | 硬链接 | 软链接 |
|------|--------|--------|
| 本质 | 同一个 inode 的多个文件名 | 存储目标路径的特殊文件 |
| 跨文件系统 | ❌ 不支持 | ✅ 支持 |
| 链接目录 | ❌ 不支持（Linux） | ✅ 支持 |
| 原文件删除后 | ✅ 仍可访问（inode 还在） | ❌ 链接失效（悬挂链接） |
| 查看 | `ls -l` 看链接数 | `ls -l` 显示 `->` 指向 |
| 适用场景 | 多处引用同一文件，防止误删 | 版本切换（如 `/usr/bin/python -> python3.11`） |

###### 9. 如何压缩和解压文件？tar、gzip、zip 命令的使用？

```bash
# tar（打包归档，常配合 gzip/bzip2/xz）
tar -czf backup.tar.gz  dir/    # 打包+gzip压缩（c=创建 z=gzip f=文件名）
tar -cjf backup.tar.bz2 dir/   # 打包+bzip2压缩（压缩率更高，但慢）
tar -xzf backup.tar.gz          # 解压（x=提取 z=gzip）
tar -xzf backup.tar.gz -C /tmp/ # 解压到指定目录
tar -tzf backup.tar.gz           # 查看压缩包内容（不解压）

# gzip（只压缩单个文件）
gzip large.log          # 压缩（原文件被替换为 large.log.gz）
gunzip large.log.gz     # 解压

# zip/unzip
zip -r archive.zip dir/ # 压缩目录
unzip archive.zip       # 解压（保留原文件）
unzip archive.zip -d /tmp/  # 解压到指定目录
```

###### 10. 如何查看磁盘空间使用情况？df 和 du 命令的区别？

```bash
# df：查看文件系统磁盘使用情况（Disk Filesystem）
df -h          # 人性化单位（GB/MB）
df -h /        # 只看根目录
df -ih         # 查看 inode 使用情况（inode 耗尽也会导致"磁盘满"）

# du：查看目录/文件占用空间（Disk Usage）
du -sh /var/log/      # 查看目录总大小
du -sh /var/log/*     # 查看各子目录大小
du -sh * | sort -rh | head -10  # 找出当前目录最大的10个文件/目录

# 组合技：找出磁盘占用最多的目录
df -h                          # 先找占满的分区
du -sh /var/* | sort -rh | head -20  # 再找该分区下的大目录
```

| 维度 | df | du |
|------|----|----|
| 统计对象 | 文件系统整体 | 目录/文件 |
| 速度 | 极快 | 较慢（需递归扫描） |
| 适用场景 | 查看分区剩余空间 | 找出占用大空间的目录 |

###### 追问：为什么 df 显示磁盘已满，但 du 统计总和比较小？

**原因**：被删除但仍被进程打开的文件占用空间。
- 在 Linux 中，文件被删除（`rm`）后，如果还有进程持有该文件的文件描述符，磁盘空间不会立即释放
- 常见于日志文件被 `rm` 但日志进程仍在写入

**解决**：`lsof | grep deleted` 找出这些进程，重启相关进程或清空文件内容（`> /path/to/log`）。
