# 04、网络管理

## Q1. Linux 中如何查看网络接口信息？

`ifconfig` 是老牌命令（需安装 net-tools），`ip` 命令是现代替代品，功能更强：

```bash
# 查看所有网络接口
ip addr show          # 简写 ip a
ifconfig -a           # 老版本

# 查看特定接口
ip addr show eth0
ifconfig eth0

# 查看接口统计（收发包数、错误数）
ip -s link show eth0

# 启用/禁用接口
ip link set eth0 up
ip link set eth0 down
```

`ip addr` 输出关键字段：
- `inet`：IPv4 地址和子网掩码
- `inet6`：IPv6 地址
- `ether`：MAC 地址
- `UP LOWER_UP`：接口已启用且物理链路正常

> **追问：`ifconfig` 和 `ip` 命令的区别？**
> `ifconfig` 来自 net-tools 包，很多最小化安装的 Linux 发行版已不预装；`ip` 来自 iproute2 包，功能更完整（支持路由策略、网络命名空间、隧道等），且是 RHEL 7+ 的推荐工具。生产环境建议统一用 `ip` 系列命令。

## Q2. `ping` 和 `traceroute` 的使用场景分别是什么？

**`ping`** — 测试网络可达性和延迟：
```bash
ping -c 4 8.8.8.8          # 发送4个包
ping -i 0.2 -c 100 host    # 每0.2s一个包，共100个（测丢包率）
ping -s 1400 host          # 指定包大小（测MTU问题）
ping6 ::1                  # IPv6 测试
```

**`traceroute`** — 追踪数据包经过的每一跳路由：
```bash
traceroute 8.8.8.8          # 默认UDP探测
traceroute -I 8.8.8.8       # 用ICMP（穿防火墙更好）
traceroute -T -p 80 host    # 用TCP，测试特定端口可达性
mtr 8.8.8.8                 # 动态版traceroute，实时显示丢包率（推荐）
```

**典型排查思路**：
1. `ping` 不通 → 网络层或以上问题
2. `ping` 通但延迟高 → 用 `mtr` 找到具体哪一跳延迟高
3. `traceroute` 某跳 `***` → 该节点不响应ICMP，但不代表包真的丢了

## Q3. `netstat` 和 `ss` 命令怎么用？

`ss` 是 `netstat` 的现代替代，速度更快（直接读 `/proc/net` 而非 socket）：

```bash
# 查看所有监听端口
ss -tlnp           # t=TCP l=监听 n=不解析域名 p=显示进程
netstat -tlnp      # 等价老命令

# 查看所有建立的TCP连接
ss -tnp state established

# 查看某个端口占用
ss -tlnp | grep :8080
lsof -i :8080      # 更直观，显示进程名和PID

# 查看连接状态统计
ss -s
netstat -s | grep -i tcp

# 查看某个进程的连接（按PID）
ss -tnp | grep pid=1234
```

**常用状态说明**：
- `LISTEN`：监听中，等待连接
- `ESTABLISHED`：已建立连接
- `TIME_WAIT`：主动关闭方等待2MSL，高并发服务常见
- `CLOSE_WAIT`：被动关闭方未调用 close()，代码层面的连接泄漏

> **追问：TIME_WAIT 大量出现怎么处理？**
> 短连接高并发时 TIME_WAIT 是正常现象（每个连接关闭后等待2MSL=60s）。优化方向：①开启 `tcp_tw_reuse`（允许新连接复用 TIME_WAIT 端口，需开启 `tcp_timestamps`）；②增大 `ip_local_port_range`（扩大可用端口范围）；③应用层改用长连接（HTTP Keep-Alive、连接池）从根本上减少连接创建销毁。

## Q4. Linux 如何查看和配置路由表？

```bash
# 查看路由表
ip route show         # 简写 ip r
route -n              # 老命令，-n 不解析域名

# 添加路由
ip route add 192.168.2.0/24 via 192.168.1.1 dev eth0
# 添加默认网关
ip route add default via 192.168.1.1

# 删除路由
ip route del 192.168.2.0/24

# 查看到某个IP的具体路由
ip route get 8.8.8.8
```

路由表关键字段：
- `default via x.x.x.x`：默认网关
- `proto kernel`：内核自动生成（直连路由）
- `scope link`：本地直连网段
- `src x.x.x.x`：出包时使用的源IP

## Q5. iptables 防火墙如何配置？

iptables 是 Linux 内核 netfilter 框架的用户空间工具，基于"表+链+规则"模型：

```bash
# 查看规则（-v 显示计数，-n 不解析）
iptables -L -vn
iptables -L INPUT -vn --line-numbers   # 显示行号，方便删除

# 允许特定端口
iptables -A INPUT -p tcp --dport 8080 -j ACCEPT
# 拒绝某个IP
iptables -A INPUT -s 192.168.1.100 -j DROP
# 允许已建立连接的响应包
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 删除规则（按行号）
iptables -D INPUT 3

# 保存规则（重启后仍生效）
iptables-save > /etc/iptables/rules.v4     # Debian/Ubuntu
service iptables save                       # CentOS 6
# CentOS 7+ 推荐用 firewalld
firewall-cmd --permanent --add-port=8080/tcp
firewall-cmd --reload
```

**实际工作中**：直接操作 iptables 容易误配，生产上常用 `firewalld`（CentOS 7+）或云厂商安全组，iptables 多用于排查网络问题或写自定义脚本。

## Q6. 如何用 `tcpdump` 抓包分析网络问题？

```bash
# 抓取 eth0 上的所有包（ctrl+c 停止）
tcpdump -i eth0

# 过滤特定端口
tcpdump -i eth0 port 8080

# 过滤特定IP
tcpdump -i eth0 host 192.168.1.100

# 组合过滤
tcpdump -i eth0 'tcp port 80 and host 192.168.1.100'

# 保存到文件，用Wireshark分析
tcpdump -i eth0 -w /tmp/capture.pcap

# 常用参数
# -n     不解析域名（更快）
# -v/-vv  详细模式
# -s 0   抓完整包（默认只抓68字节）
# -c 100 只抓100个包
# -X     十六进制+ASCII显示包内容

# 排查HTTP问题：看完整请求响应
tcpdump -i eth0 -A -s 0 'tcp port 80'
```

**排查 Java 应用网络问题**：
```bash
# 确认应用端口是否在监听
ss -tlnp | grep 8080
# 抓包看是否收到请求
tcpdump -i any port 8080 -n
# 如果有包但应用无响应，说明应用层问题（线程阻塞/OOM等）
```

## Q7. 如何测试网络带宽？

```bash
# iperf3 是标准带宽测试工具
# 服务端
iperf3 -s

# 客户端（测TCP带宽）
iperf3 -c server_ip

# 测UDP带宽（附丢包率）
iperf3 -c server_ip -u -b 100M

# 双向测试
iperf3 -c server_ip --bidir

# 测试10秒，并行4条流
iperf3 -c server_ip -t 10 -P 4
```

其他工具：
```bash
# wget/curl 粗略测试下载速度
wget -O /dev/null http://server/bigfile

# speedtest-cli（测公网）
pip install speedtest-cli
speedtest-cli
```

## Q8. Linux 如何排查 DNS 解析问题？

```bash
# 基本DNS查询
nslookup example.com
dig example.com           # 更详细，推荐

# dig 常用选项
dig example.com A          # 查A记录
dig example.com MX         # 查邮件服务器
dig @8.8.8.8 example.com  # 指定DNS服务器查询
dig +trace example.com     # 追踪完整解析链

# 查看本机DNS配置
cat /etc/resolv.conf

# 清除DNS缓存
systemctl restart nscd      # 如果有nscd
systemd-resolve --flush-caches  # systemd-resolved
```

**排查步骤**：
1. `ping` 不通但 `ping IP` 通 → DNS 问题
2. `dig @8.8.8.8 domain` 能解析但本地不行 → 本地 DNS 服务器问题
3. 检查 `/etc/resolv.conf` 中 nameserver 配置
4. 检查 `/etc/nsswitch.conf` 中 `hosts` 行的解析顺序（files → dns）
5. 检查 `/etc/hosts` 是否有错误的静态映射

## Q9. 如何在 Linux 上配置静态 IP？

**CentOS 7/RHEL（NetworkManager）**：
```bash
# 编辑网卡配置文件
vi /etc/sysconfig/network-scripts/ifcfg-eth0
```
```ini
TYPE=Ethernet
BOOTPROTO=none          # 改为none（静态）
NAME=eth0
DEVICE=eth0
ONBOOT=yes
IPADDR=192.168.1.100
PREFIX=24               # 子网掩码
GATEWAY=192.168.1.1
DNS1=8.8.8.8
DNS2=8.8.4.4
```
```bash
systemctl restart network   # 或 nmcli con reload
```

**Ubuntu 20.04+（Netplan）**：
```yaml
# /etc/netplan/00-installer-config.yaml
network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
```
```bash
netplan apply
```

## Q10. 如何配置和使用 SSH？

```bash
# 远程连接
ssh user@host
ssh -p 2222 user@host          # 指定端口
ssh -i ~/.ssh/id_rsa user@host # 指定私钥

# 生成密钥对（推荐Ed25519）
ssh-keygen -t ed25519 -C "comment"
# 复制公钥到服务器
ssh-copy-id user@host
# 等价手动操作：
cat ~/.ssh/id_ed25519.pub >> ~/.ssh/authorized_keys

# SSH 隧道（本地端口转发）
ssh -L 8080:localhost:80 user@server   # 本地8080 → 服务器80
# 反向隧道（内网穿透）
ssh -R 9090:localhost:8080 user@server # 服务器9090 → 本地8080

# SSH 跳板机
ssh -J jumphost user@target

# 保活配置（~/.ssh/config）
Host *
  ServerAliveInterval 60
  ServerAliveCountMax 3
  ConnectTimeout 10
```

**sshd 安全加固**（`/etc/ssh/sshd_config`）：
```ini
Port 22222                    # 改非标准端口
PermitRootLogin no            # 禁止root直接登录
PasswordAuthentication no     # 禁用密码登录，只允许密钥
MaxAuthTries 3                # 最大尝试次数
AllowUsers deploy appuser     # 白名单用户
```

## Q11. 网络不通时的系统排查步骤是什么？

按 OSI 模型从底层到上层逐步排查：

```
1. 物理层/数据链路层
   ip link show eth0        # 确认接口 UP 且有流量
   ethtool eth0             # 查看链路速率、双工模式

2. 网络层
   ip addr show             # 确认IP地址正确
   ip route show            # 确认路由/网关正确
   ping 网关IP              # 测试到网关是否通

3. 传输层
   ss -tlnp | grep 端口    # 确认服务在监听
   telnet host port        # 测试TCP端口是否可达
   nc -zv host port        # 更好的telnet替代

4. 应用层
   curl -v http://host      # 测试HTTP
   tcpdump -i any port 端口 # 抓包确认是否有流量到达

5. 防火墙
   iptables -L -vn          # 检查iptables规则
   firewall-cmd --list-all  # firewalld规则

6. DNS
   dig @8.8.8.8 域名        # 绕过本地DNS测试
```

> **追问：Java 微服务之间调用失败，如何排查网络问题？**
> ①先看应用日志，确认是 ConnectTimeout（连不上）还是 ReadTimeout（连上但无响应）；②ConnectTimeout → 检查目标服务是否启动（`ss -tlnp`）、防火墙是否放行；③ReadTimeout → 目标服务可能线程满/GC停顿，看 GC 日志和线程池；④抓包确认请求是否真的发出去了（`tcpdump port 端口`）；⑤检查服务发现/DNS，看 IP 是否正确；⑥对于 HTTP 调用用 `curl -v` 直接测试，排除客户端代码问题。
