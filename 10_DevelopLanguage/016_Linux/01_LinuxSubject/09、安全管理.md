# 09、安全管理

## Q1. Linux 系统安全加固的基本步骤是什么？

生产服务器上线前的安全加固清单：

```bash
# ==================== 1. SSH 加固 ====================
# /etc/ssh/sshd_config
Port 22222                      # 改非默认端口
PermitRootLogin no              # 禁止root直接登录
PasswordAuthentication no       # 禁用密码，只用密钥
MaxAuthTries 3                  # 最大认证尝试次数
LoginGraceTime 30               # 30秒内未完成认证断开
X11Forwarding no                # 禁用X11转发
AllowTcpForwarding no           # 禁用TCP转发（如不需要）
UseDNS no                       # 禁用DNS反查（加快连接速度）
ClientAliveInterval 300         # 空闲5分钟踢出
AllowUsers deploy appuser       # 白名单（比AllowGroups更严格）

systemctl restart sshd

# ==================== 2. 防火墙 ====================
# 默认拒绝所有，只开放必要端口
firewall-cmd --set-default-zone=drop
firewall-cmd --add-port=22222/tcp --permanent   # SSH
firewall-cmd --add-port=80/tcp --permanent      # HTTP
firewall-cmd --add-port=443/tcp --permanent     # HTTPS
firewall-cmd --reload

# ==================== 3. 更新系统 ====================
yum update -y --security     # 只安装安全更新
apt-get upgrade -y           # Ubuntu

# ==================== 4. 最小化安装原则 ====================
# 关闭不需要的服务
systemctl disable --now bluetooth cups avahi-daemon postfix

# 删除不需要的包
yum remove telnet ftp rsh
```

## Q2. 如何防止暴力破解 SSH？

```bash
# ==================== Fail2ban ====================
yum install fail2ban

# 配置（/etc/fail2ban/jail.local）
[DEFAULT]
bantime  = 3600      # 封禁1小时
findtime = 600       # 10分钟内
maxretry = 3         # 失败3次触发封禁
banaction = firewallcmd-ipset

[sshd]
enabled  = true
port     = 22222
logpath  = /var/log/secure
maxretry = 3

systemctl enable --now fail2ban

# 查看封禁列表
fail2ban-client status sshd
# 手动解封
fail2ban-client set sshd unbanip 192.168.1.100

# ==================== 限制单IP连接数（iptables）====================
iptables -A INPUT -p tcp --dport 22222 -m state --state NEW \
  -m recent --set --name SSH
iptables -A INPUT -p tcp --dport 22222 -m state --state NEW \
  -m recent --update --seconds 60 --hitcount 4 --name SSH -j DROP
```

## Q3. Linux 文件完整性检测如何实现？

```bash
# ==================== AIDE（文件完整性检测）====================
yum install aide

# 初始化基准数据库（系统初始状态）
aide --init
mv /var/lib/aide/aide.db.new.gz /var/lib/aide/aide.db.gz

# 检测文件变化（与基准比较）
aide --check

# 定期检查（加入cron）
0 3 * * * /usr/sbin/aide --check 2>&1 | mail -s "AIDE Report" admin@company.com

# ==================== 手动文件完整性检查 ====================
# 对关键文件做哈希，定期对比
sha256sum /etc/passwd /etc/shadow /etc/sudoers > /root/baseline.sha256
# 验证
sha256sum -c /root/baseline.sha256

# 查找 SUID/SGID 文件（异常的SUID文件可能是后门）
find / -perm /4000 -type f 2>/dev/null   # 找SUID文件
find / -perm /2000 -type f 2>/dev/null   # 找SGID文件
# 对比正常系统的SUID文件列表，发现异常
```

## Q4. SELinux 是什么？如何管理？

SELinux（Security-Enhanced Linux）是内核级强制访问控制（MAC）框架，在 DAC（chmod/chown）之上增加额外安全层：

```bash
# 查看 SELinux 状态
getenforce              # Enforcing / Permissive / Disabled
sestatus               # 详细状态

# 三种模式
# Enforcing   - 强制模式，违规操作被阻止
# Permissive  - 宽容模式，只记录日志不阻止（排查问题用）
# Disabled    - 完全禁用

# 临时切换模式（重启失效）
setenforce 0    # Enforcing → Permissive
setenforce 1    # → Enforcing

# 永久修改（/etc/selinux/config）
SELINUX=enforcing   # 或 permissive 或 disabled
# 注意：从 disabled 改为 enabled 需要重启并重新标记文件系统

# 排查 SELinux 拒绝记录
ausearch -m avc -ts recent | audit2why    # 解读拒绝原因
ausearch -m avc -ts recent | audit2allow  # 生成允许规则

# 查看文件的 SELinux 上下文
ls -Z /etc/hosts
stat /etc/hosts | grep Context

# 修改文件上下文
chcon -t httpd_sys_content_t /var/www/html/index.html
# 恢复默认上下文
restorecon -v /var/www/html/index.html
```

**生产实践**：很多运维人员遇到问题直接 `setenforce 0` 是不好的习惯，正确做法是用 `audit2allow` 生成针对性规则，保持 SELinux 启用。

## Q5. 如何审计系统中的安全事件？

```bash
# ==================== auditd（内核审计框架）====================
systemctl enable --now auditd

# 配置审计规则（/etc/audit/rules.d/audit.rules）
# 监控关键文件的读写
-w /etc/passwd -p wa -k passwd_changes
-w /etc/shadow -p wa -k shadow_changes
-w /etc/sudoers -p wa -k sudoers_changes
-w /etc/ssh/sshd_config -p wa -k sshd_config

# 监控命令执行（sudo）
-a always,exit -F arch=b64 -S execve -F euid=0 -k root_commands

# 重新加载规则
augenrules --load
auditctl -l   # 查看当前规则

# 查询审计日志
ausearch -k passwd_changes    # 按关键字查询
ausearch -ua appuser          # 查询特定用户的操作
ausearch -m LOGIN -ts today   # 今天的登录事件
aureport -au                  # 认证报告汇总

# ==================== 系统日志中的安全事件 ====================
# 查看失败登录
grep "Failed password" /var/log/secure
grep "authentication failure" /var/log/auth.log   # Ubuntu

# 查看 sudo 操作
grep "sudo" /var/log/secure

# 查看新建用户
grep "useradd\|adduser" /var/log/secure
```

## Q6. 如何检测系统是否被入侵？

```bash
# ==================== 基本入侵迹象检查 ====================

# 1. 检查异常登录
last | head -20                    # 最近登录记录
lastb | head -20                   # 失败的登录尝试
w                                  # 当前登录用户

# 2. 检查异常进程
ps auxf                            # 进程树
ps aux | awk '{print $1}' | sort -u  # 有哪些用户在跑进程
lsof -i -n | grep LISTEN           # 监听的网络服务

# 3. 检查异常网络连接
ss -tnp | grep ESTABLISHED         # 当前连接
netstat -plant | grep -v 127.0.0.1 # 对外连接

# 4. 检查可疑文件
find /tmp /var/tmp -type f -newer /etc/passwd   # 新建文件
find / -name "*.php" -newer /etc/passwd 2>/dev/null  # 新建PHP文件（webshell）
find / -perm -4000 -type f 2>/dev/null   # 异常SUID文件

# 5. 检查计划任务
crontab -l
ls -la /etc/cron*
cat /etc/crontab
for user in $(cut -f1 -d: /etc/passwd); do
    crontab -u $user -l 2>/dev/null && echo " <- $user的crontab"
done

# 6. 检查启动项
systemctl list-units --type=service --state=enabled
ls /etc/rc.d/rc.local

# 7. 检查 /etc/passwd 中的异常账户
awk -F: '($3==0) {print $1}' /etc/passwd  # 除root外的uid=0账户（极危险）
awk -F: '($7 != "/sbin/nologin" && $7 != "/bin/false") {print $1}' /etc/passwd
```

## Q7. 常见的 Web 服务器安全加固措施有哪些？

```bash
# ==================== Nginx 安全配置 ====================
# 隐藏版本信息
server_tokens off;

# 防止点击劫持
add_header X-Frame-Options "SAMEORIGIN";

# 防止 MIME 类型嗅探
add_header X-Content-Type-Options "nosniff";

# XSS 保护
add_header X-XSS-Protection "1; mode=block";

# HSTS（HTTPS 才开启）
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";

# 限制请求方法
if ($request_method !~ ^(GET|HEAD|POST)$) {
    return 444;
}

# 限制请求体大小（防止 DOS）
client_max_body_size 10m;

# 防止路径遍历
location ~* \.\. { return 403; }

# ==================== 应用安全 ====================
# 应用账号不要用 root 运行
User nginx;

# 目录权限
chown -R root:nginx /etc/nginx        # nginx 配置只读
chmod 750 /etc/nginx
chown -R appuser:nginx /var/www/html  # 应用文件 appuser 可写
chmod 755 /var/www/html
```

> **追问：如何快速排查 Java 应用是否存在安全漏洞？**
> ①检查依赖包版本：用 Maven 的 `mvn dependency:tree` 梳理依赖，结合 [OWASP Dependency-Check](https://owasp.org/www-project-dependency-check/) 扫描已知 CVE；②检查 Spring Boot Actuator 是否对外暴露（`/actuator/env`、`/actuator/heapdump` 等必须加认证或关闭）；③Log4j 漏洞（CVE-2021-44228）：确认 Log4j 版本 ≥ 2.17.1 或已配置 `log4j2.formatMsgNoLookups=true`；④敏感配置（数据库密码/AK-SK）不要硬编码，用环境变量或 Vault/KMS 管理。
