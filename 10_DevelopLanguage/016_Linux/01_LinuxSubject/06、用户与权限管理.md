# 06、用户与权限管理

## Q1. Linux 用户和用户组的基本概念是什么？

Linux 是多用户操作系统，每个文件/进程都有所有者。

**核心文件**：
```bash
/etc/passwd    # 用户账户信息
/etc/shadow    # 密码（加密存储）
/etc/group     # 用户组信息
/etc/gshadow   # 用户组密码

# /etc/passwd 格式
# 用户名:密码占位:UID:GID:注释:主目录:Shell
# root:x:0:0:root:/root:/bin/bash
# appuser:x:1001:1001::/home/appuser:/bin/bash
```

**关键 UID 范围**：
- `0`：root（超级用户）
- `1~999`：系统账号（守护进程用，不能登录）
- `1000+`：普通用户

```bash
# 查看当前用户
whoami
id                     # 显示uid/gid/groups

# 切换用户
su - username          # 完整切换（加载环境变量）
sudo command           # 以root权限执行单条命令
sudo -u appuser cmd    # 以指定用户身份执行
```

## Q2. 如何创建和管理用户？

```bash
# 创建用户
useradd appuser                        # 基本创建
useradd -m -s /bin/bash appuser        # 创建主目录，指定Shell
useradd -m -u 1500 -g 1500 appuser     # 指定UID和GID
useradd -G docker,sudo appuser         # 加入附加组

# 设置密码
passwd appuser          # 交互式设置
echo "password" | passwd --stdin appuser  # 非交互（脚本用）

# 修改用户
usermod -aG docker appuser    # 追加到docker组（-a 必须，否则覆盖原有组）
usermod -s /bin/bash appuser  # 修改默认Shell
usermod -L appuser            # 锁定账户（禁止登录）
usermod -U appuser            # 解锁账户

# 删除用户
userdel appuser          # 保留主目录
userdel -r appuser       # 同时删除主目录

# 创建/管理组
groupadd mygroup
groupmod -n newname mygroup
groupdel mygroup

# 查看用户所在组
groups appuser
id appuser
```

## Q3. Linux 文件权限体系是怎样的？

```bash
ls -l /etc/hosts
# -rw-r--r-- 1 root root 223 Jan 1 00:00 /etc/hosts
#  ↑           ↑    ↑
#  权限位       所有者  所属组

# 权限位解读（10位）
# 第1位：文件类型（- 普通文件，d 目录，l 符号链接，b 块设备，c 字符设备）
# 第2-4位：所有者(user)权限
# 第5-7位：所属组(group)权限
# 第8-10位：其他人(other)权限

# r=4, w=2, x=1
# rw-r--r-- = 644（所有者读写，组和其他只读）
# rwxr-xr-x = 755（所有者全权，组和其他读和执行）
# rwx------ = 700（只有所有者可以访问）
```

**目录权限的特殊含义**：
- `r`：可以 `ls` 列出目录内容
- `w`：可以在目录中创建/删除文件
- `x`：可以进入目录（`cd`），访问目录内文件

## Q4. `chmod` 和 `chown` 怎么使用？

```bash
# chmod — 修改权限
chmod 755 file          # 数字方式
chmod u+x file          # 字母方式：给所有者加执行权限
chmod g-w file          # 去掉组的写权限
chmod o=r file          # 设置其他人只有读权限
chmod a+x file          # 所有人加执行权限
chmod -R 755 /app       # 递归修改目录

# chown — 修改所有者
chown appuser file
chown appuser:appgroup file   # 同时修改所有者和组
chown :appgroup file          # 只修改组
chown -R appuser:appgroup /app  # 递归修改

# chgrp — 只修改组
chgrp docker /var/run/docker.sock

# 特殊权限位
chmod u+s file    # SetUID：执行时以文件所有者身份运行（如 passwd 命令）
chmod g+s dir     # SetGID：目录下新文件继承目录的组
chmod +t dir      # Sticky Bit：目录中只有文件所有者才能删除自己的文件（如 /tmp）

# 数字表示特殊权限：在三位数字前加一位
# 4=SetUID, 2=SetGID, 1=Sticky
chmod 4755 /usr/bin/passwd   # 带SetUID的755
chmod 1777 /tmp              # 带Sticky的777
```

## Q5. `sudo` 如何配置和使用？

```bash
# 编辑 sudo 配置（必须用 visudo，语法检查避免配置错误）
visudo
# 或
visudo -f /etc/sudoers.d/appuser

# /etc/sudoers 格式
# 用户  主机=(以谁的身份) 命令
root    ALL=(ALL:ALL) ALL           # root 无限制
%sudo   ALL=(ALL:ALL) ALL           # sudo 组的成员
appuser ALL=(root) NOPASSWD: /bin/systemctl restart myapp   # 特定命令免密

# 常用 sudo 操作
sudo command              # 以root执行
sudo -u username cmd      # 以指定用户执行
sudo -l                   # 查看当前用户可执行的sudo命令
sudo -i                   # 切换到root的交互式Shell

# 查看 sudo 日志
tail -f /var/log/auth.log    # Ubuntu
tail -f /var/log/secure      # CentOS/RHEL
```

**最小权限原则**：生产上不要给应用账号 `ALL ALL=(ALL) ALL`，只授予必要的命令，避免权限提升攻击。

## Q6. umask 是什么？

`umask` 是文件创建时的权限掩码，决定新文件/目录的默认权限：

```bash
# 查看当前umask
umask              # 输出如 0022

# 权限计算
# 新文件的默认权限 = 0666 - umask
# 新目录的默认权限 = 0777 - umask
# umask 0022 → 文件 644，目录 755

# 修改umask（临时）
umask 027    # 文件 640，目录 750（同组只读，其他人无权限）

# 永久修改（加到 ~/.bashrc 或 /etc/profile）
echo "umask 027" >> ~/.bashrc
```

## Q7. 如何查看和审计用户操作记录？

```bash
# 查看登录历史
last                       # 所有用户登录记录（读取 /var/log/wtmp）
last appuser               # 指定用户
lastb                      # 失败的登录尝试（读取 /var/log/btmp）
w                          # 当前在线用户
who                        # 当前在线用户（简洁版）
last reboot                # 系统重启记录

# 查看命令历史
history                    # 当前用户的历史
cat ~/.bash_history         # 历史文件
# 查看指定用户的历史（需root）
cat /home/appuser/.bash_history

# 系统认证日志（登录、sudo等）
tail -f /var/log/auth.log    # Ubuntu
tail -f /var/log/secure      # CentOS

# 更完整的审计：auditd
ausearch -m USER_CMD -sv no   # 查看失败的sudo
ausearch -f /etc/passwd        # 查看谁访问了passwd文件
```

## Q8. PAM（可插拔认证模块）是什么？

PAM（Pluggable Authentication Modules）是 Linux 认证框架，允许灵活配置认证策略：

```bash
# PAM 配置文件位置
/etc/pam.d/            # 每个服务对应一个配置文件
/etc/pam.d/sshd        # SSH 登录认证
/etc/pam.d/sudo        # sudo 认证
/etc/pam.d/login       # 本地登录

# 常见 PAM 模块
pam_unix.so       # 传统Unix认证（/etc/passwd/shadow）
pam_ldap.so       # LDAP认证
pam_google_authenticator.so  # 双因素认证
pam_limits.so     # 系统资源限制（读取 /etc/security/limits.conf）
pam_tally2.so     # 账户锁定（登录失败次数限制）

# 配置密码强度（/etc/security/pwquality.conf）
minlen = 12         # 最小长度
dcredit = -1        # 至少1个数字
ucredit = -1        # 至少1个大写字母
lcredit = -1        # 至少1个小写字母
ocredit = -1        # 至少1个特殊字符
```

> **追问：如何限制用户只能在特定时间段内登录？**
> 在 `/etc/pam.d/sshd` 中添加 `pam_time.so`，然后在 `/etc/security/time.conf` 中配置允许登录的时间段，如 `sshd;*;appuser;Wk0800-1800`（工作日8点到18点）。也可以直接在 `/etc/ssh/sshd_config` 中用 `AllowGroups` 配合 `Match` 块来限制访问时段。
