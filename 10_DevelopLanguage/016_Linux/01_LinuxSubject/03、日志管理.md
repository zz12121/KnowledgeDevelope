### 三、日志管理类

---

###### 1. 你平时是怎么查看日志的？

```bash
# 1. 实时追踪（最常用）
tail -f /var/log/nginx/error.log
tail -f /app/logs/app.log

# 2. 多文件同时追踪
tail -f /app/logs/app.log -f /var/log/nginx/error.log

# 3. 关键字过滤（实时）
tail -f app.log | grep "ERROR\|WARN"

# 4. 查看最近 N 行
tail -n 200 app.log

# 5. 按时间范围查看（大日志文件）
grep "2024-01-15 14:" app.log | head -100

# 6. 多条件查询（Java 异常堆栈上下文）
grep -A 30 "NullPointerException" app.log | head -60  # 匹配行后30行

# 7. 大文件搜索（比 grep 快）
ag "OutOfMemory" /var/log/     # ag（silver searcher）
rg "OutOfMemory" /var/log/     # ripgrep（更快）
```

###### 2. tail、head、cat、more、less 命令的区别和使用场景？

| 命令 | 作用 | 适用场景 |
|------|------|---------|
| `tail -f` | 实时追踪文件末尾 | **生产环境查看实时日志（最常用）** |
| `tail -n N` | 查看最后N行 | 快速看最近的日志 |
| `head -n N` | 查看前N行 | 查看文件头/配置文件 |
| `cat` | 一次性输出全部内容 | 小文件、脚本输出重定向 |
| `less` | 分页查看，支持前后翻页和搜索 | **查看大文件（推荐，不会撑爆内存）** |
| `more` | 分页查看，只能向下翻页 | 基本被 less 取代 |

**less 的实用快捷键**：
- `G` = 跳到文件末尾（类似 tail），`gg` = 回到开头
- `/pattern` = 向下搜索，`?pattern` = 向上搜索，`n` = 下一个，`N` = 上一个
- `q` = 退出

###### 3. 如何实时监控日志文件的变化？

```bash
# 基础：tail -f（最常用）
tail -f app.log

# 升级版：grep 过滤 + 高亮
tail -f app.log | grep --color=auto "ERROR\|WARN"

# 高级：multitail（同时监控多个文件，有窗口划分）
multitail app.log nginx_access.log

# 带时间戳监控
tail -f app.log | ts '%Y-%m-%d %H:%M:%S'  # 需要 moreutils 包

# 当日志以日期命名时，需要脚本
while true; do tail -f $(ls -t /app/logs/*.log | head -1); sleep 1; done
```

###### 4. 如何在日志文件中搜索关键字？grep 命令的使用？

```bash
# 基础用法
grep "ERROR" app.log
grep -i "error" app.log          # -i 忽略大小写
grep -n "ERROR" app.log          # -n 显示行号
grep -c "ERROR" app.log          # -c 只显示匹配行数

# 上下文（排查问题最实用）
grep -A 5 "OutOfMemory" app.log  # 匹配行 + 后5行（A=After）
grep -B 5 "OutOfMemory" app.log  # 匹配行 + 前5行（B=Before）
grep -C 10 "OutOfMemory" app.log # 匹配行 + 前后各10行（C=Context）

# 多条件
grep -E "ERROR|WARN" app.log     # -E 扩展正则（OR）
grep "Exception" app.log | grep -v "ignore"  # 排除含 ignore 的行

# 递归搜索目录
grep -r "NullPointerException" /app/logs/

# 只显示文件名
grep -rl "OutOfMemory" /app/logs/

# 统计每种错误的出现次数
grep "ERROR" app.log | awk '{print $5}' | sort | uniq -c | sort -rn
```

###### 5. awk 和 sed 命令在日志分析中的应用？

**awk** — 按列处理，适合结构化日志分析：
```bash
# Nginx 访问日志：统计各状态码数量
awk '{print $9}' access.log | sort | uniq -c | sort -rn

# 统计接口响应时间 > 1s 的请求
awk '$NF > 1 {print $7, $NF}' access.log | sort -k2 -rn | head -20

# 计算平均响应时间
awk '{sum += $NF; count++} END {print "avg:", sum/count}' access.log

# 按小时统计 QPS
awk '{print substr($4,14,2)}' access.log | sort | uniq -c
```

**sed** — 流编辑器，适合文本替换和过滤：
```bash
# 删除日志中的颜色转义码
sed 's/\x1B\[[0-9;]*m//g' colored.log

# 替换敏感信息（脱敏）
sed 's/password=[^ ]*/password=****/g' app.log

# 提取某个时间段的日志（日期匹配）
sed -n '/2024-01-15 14:00/,/2024-01-15 15:00/p' app.log

# 删除空行和注释行
sed -e '/^$/d' -e '/^#/d' config.conf

# 在线修改文件（-i）
sed -i 's/old_host/new_host/g' config.conf
sed -i.bak 's/old/new/g' config.conf  # 备份原文件为 .bak
```

###### 6. Linux 系统日志一般存放在哪里？

```bash
/var/log/syslog         # 系统日志（Ubuntu/Debian）
/var/log/messages       # 系统日志（CentOS/RHEL）
/var/log/auth.log       # 认证/SSH 登录日志（Ubuntu）
/var/log/secure         # 认证/SSH 登录日志（CentOS）
/var/log/kern.log       # 内核日志
/var/log/dmesg          # 内核启动消息（也可用 dmesg 命令查看）
/var/log/cron           # cron 定时任务日志
/var/log/boot.log       # 系统启动日志
/var/log/nginx/         # Nginx 日志
/var/log/mysql/         # MySQL 日志
/var/log/journal/       # systemd 日志（journalctl 命令查看）

# 查看 systemd 日志
journalctl -u nginx -n 100         # 查看 nginx 服务最近100条日志
journalctl --since "2024-01-15" --until "2024-01-16"  # 按时间范围
journalctl -k                      # 只看内核日志
```

###### 7. 如何清空或删除日志文件？

```bash
# 方式1：截断文件（保留文件，清空内容）—— 进程仍可写入
> /var/log/app.log              # 截断为空
cat /dev/null > /var/log/app.log # 等效写法
truncate -s 0 /var/log/app.log  # 同效

# 方式2：删除文件（注意：如果进程还在写，磁盘空间不会立即释放！）
rm /var/log/app.log

# 正确删除并释放空间的方式：
# 先截断（让进程感知到文件变化，继续向新文件写入）
> /var/log/app.log
# 如果进程不感知截断，需要重启进程或发送 USR1 信号（如 nginx -s reopen）
```

###### 8. 日志轮转是什么？logrotate 的作用？

日志轮转（Log Rotation）是定期对日志文件进行**重命名/压缩/删除**的管理机制，防止日志文件无限增长耗尽磁盘。

```bash
# logrotate 配置示例（/etc/logrotate.d/myapp）
/app/logs/*.log {
    daily                     # 每天轮转
    missingok                 # 文件不存在不报错
    rotate 30                 # 保留最近 30 份
    compress                  # gzip 压缩旧日志
    delaycompress             # 最新一份不压缩（可能还在被写入）
    notifempty                # 空文件不轮转
    dateext                   # 用日期命名（app.log.2024-01-15）
    create 640 appuser appgroup  # 轮转后新建文件的权限和所有者
    postrotate
        # 通知应用重新打开日志文件
        /bin/kill -USR1 $(cat /var/run/app.pid 2>/dev/null) 2>/dev/null || true
    endscript
}

# 手动测试 logrotate 配置
logrotate -d /etc/logrotate.d/myapp    # 调试模式（不实际执行）
logrotate -f /etc/logrotate.d/myapp    # 强制立即执行
```

###### 追问：生产环境 Java 应用日志最佳实践是什么？

1. **使用 logback / log4j2**，配置滚动策略（按大小 + 按时间双条件）
2. **异步日志**（`AsyncAppender`）：写日志异步处理，不阻塞业务线程
3. **结构化日志（JSON 格式）**：方便 ELK/Loki 收集和搜索
4. **日志级别管理**：生产用 INFO，排查问题时动态调整为 DEBUG（通过 Actuator/配置中心）
5. **敏感信息脱敏**：手机号/密码/身份证不能出现在日志中
6. **合理保留周期**：热日志 7 天，压缩归档 30 天，超过删除
