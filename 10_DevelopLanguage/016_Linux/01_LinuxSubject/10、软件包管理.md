# 10、软件包管理

## Q1. RPM 和 YUM/DNF 的区别是什么？

**RPM（Red Hat Package Manager）** 是底层包格式和工具，处理单个 `.rpm` 包但不自动解决依赖：

```bash
# rpm 常用命令
rpm -ivh package.rpm          # 安装（i=install，v=verbose，h=进度条）
rpm -Uvh package.rpm          # 升级（upgrade）
rpm -e package_name           # 卸载
rpm -qa                       # 查询所有已安装包
rpm -qa | grep nginx          # 搜索已安装包
rpm -qi nginx                 # 查询包的详细信息
rpm -ql nginx                 # 查询包安装了哪些文件
rpm -qf /usr/sbin/nginx       # 查询某文件属于哪个包（逆向查询）
rpm -qc nginx                 # 查询包的配置文件
rpm -V nginx                  # 验证包文件完整性
```

**YUM/DNF** 是高层包管理器，自动解决依赖关系（调用 RPM 完成实际安装）：

```bash
# YUM（CentOS 7）/ DNF（CentOS 8+/Fedora，dnf 是 yum 的下一代）
yum install nginx             # 安装（自动解决依赖）
yum remove nginx              # 卸载
yum update                    # 更新所有包
yum update nginx              # 更新指定包
yum search nginx              # 搜索包
yum info nginx                # 查看包信息
yum list installed            # 查看已安装的包
yum provides /usr/sbin/nginx  # 查询哪个包提供该文件
yum history                   # 安装/删除历史记录
yum history undo 5            # 撤销第5次操作
```

## Q2. APT 包管理器（Ubuntu/Debian）如何使用？

```bash
# 更新软件包索引（安装前必做）
apt update
apt-get update    # 等价旧命令

# 安装包
apt install nginx
apt install -y nginx     # 非交互模式

# 删除包
apt remove nginx         # 删除程序文件（保留配置）
apt purge nginx          # 完全删除（包括配置文件）
apt autoremove           # 删除不再需要的依赖包

# 更新
apt upgrade              # 更新已安装的包
apt full-upgrade         # 更激进的升级（可能删除包解决冲突）
apt dist-upgrade         # 发行版升级

# 查询
apt search nginx         # 搜索
apt show nginx           # 显示包信息
apt list --installed     # 已安装包列表
dpkg -L nginx            # 列出包安装的文件
dpkg -S /usr/sbin/nginx  # 查询文件属于哪个包
dpkg -l | grep nginx     # 检查包是否安装

# 查看可更新的包
apt list --upgradable
```

## Q3. 如何配置 YUM/APT 源？

**YUM 源配置（CentOS/RHEL）**：
```bash
# 配置文件位置
/etc/yum.repos.d/         # 所有 .repo 文件

# repo 文件格式
[repo-id]
name=描述
baseurl=http://mirrors.xxx.com/centos/7/os/x86_64/
enabled=1
gpgcheck=1
gpgkey=http://mirrors.xxx.com/RPM-GPG-KEY-CentOS-7

# 常用国内镜像（阿里云）
# 替换CentOS 7的默认源
curl -o /etc/yum.repos.d/CentOS-Base.repo \
  http://mirrors.aliyun.com/repo/Centos-7.repo
yum makecache fast    # 重建缓存

# 添加 EPEL 源（Extra Packages for Enterprise Linux）
yum install epel-release
# 或
rpm -ivh https://dl.fedoraproject.org/pub/epel/epel-release-latest-7.noarch.rpm
```

**APT 源配置（Ubuntu）**：
```bash
# 主配置文件
/etc/apt/sources.list
/etc/apt/sources.list.d/   # 额外源文件

# sources.list 格式
deb https://mirrors.aliyun.com/ubuntu/ jammy main restricted universe multiverse
deb https://mirrors.aliyun.com/ubuntu/ jammy-updates main restricted universe multiverse

# 添加第三方源（以 Docker 为例）
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | apt-key add -
add-apt-repository "deb https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
apt update
```

## Q4. 如何从源码编译安装软件？

```bash
# 以编译安装 Nginx 为例
# 1. 安装编译依赖
yum groupinstall "Development Tools"
yum install gcc make pcre-devel zlib-devel openssl-devel

# 2. 下载源码
wget http://nginx.org/download/nginx-1.24.0.tar.gz
tar -xzf nginx-1.24.0.tar.gz
cd nginx-1.24.0

# 3. 配置编译选项
./configure \
    --prefix=/usr/local/nginx \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_gzip_static_module

# 4. 编译和安装
make -j$(nproc)   # 并行编译（-j 线程数）
make install

# 5. 配置环境变量
echo 'export PATH=/usr/local/nginx/sbin:$PATH' >> /etc/profile
source /etc/profile

# 常见编译问题
# configure: error: xxx not found → 缺少依赖，yum install xxx-devel
# make: *** Error → 查看 make 输出的具体错误信息
```

## Q5. pip 和 Python 包管理的常见问题

```bash
# pip 基本使用
pip install requests              # 安装
pip install requests==2.28.0      # 安装指定版本
pip install -r requirements.txt   # 从文件安装
pip install --upgrade requests    # 升级
pip uninstall requests            # 卸载
pip list                          # 已安装包
pip show requests                 # 包信息
pip freeze > requirements.txt     # 导出依赖列表

# 虚拟环境（强烈推荐，隔离项目依赖）
python3 -m venv venv              # 创建虚拟环境
source venv/bin/activate          # 激活（Linux/Mac）
pip install -r requirements.txt   # 在虚拟环境中安装
deactivate                        # 退出虚拟环境

# 使用国内镜像（阿里云）
pip install package -i https://mirrors.aliyun.com/pypi/simple/
# 永久配置
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/
```

## Q6. 如何管理 Java 应用的版本和依赖？

**多版本 JDK 管理**：
```bash
# 安装多个 JDK
yum install java-11-openjdk java-17-openjdk

# 切换默认 Java 版本（RHEL/CentOS）
alternatives --config java
# 或
update-alternatives --config java   # Ubuntu

# 查看当前 Java 版本
java -version
which java
ls -la $(which java)   # 查看软链接指向

# 手动设置环境变量（切换项目）
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$JAVA_HOME/bin:$PATH

# SDKMAN（推荐，开发环境多版本管理神器）
curl -s "https://get.sdkman.io" | bash
sdk install java 17.0.8-tem    # 安装指定版本
sdk use java 11.0.21-tem       # 切换当前shell使用的版本
sdk default java 17.0.8-tem    # 设置默认版本
sdk list java                   # 查看可用版本
```

> **追问：CentOS 7 如何安装高版本 GCC？**
> CentOS 7 默认的 GCC 4.8 太旧，有些软件需要更高版本：
> ```bash
> # 方案1：开发者工具集（devtoolset）
> yum install centos-release-scl
> yum install devtoolset-11-gcc devtoolset-11-gcc-c++
> scl enable devtoolset-11 bash  # 激活（当前shell）
> echo "source /opt/rh/devtoolset-11/enable" >> ~/.bashrc  # 永久激活
> 
> # 方案2：自行编译（耗时，但版本随意）
> wget https://ftp.gnu.org/gnu/gcc/gcc-12.2.0/gcc-12.2.0.tar.gz
> # ... 编译步骤
> ```
