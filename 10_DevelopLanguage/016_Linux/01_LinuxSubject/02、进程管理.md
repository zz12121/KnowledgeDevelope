###### 1. 怎么查看当前进程？怎么执行退出？怎么查看当前路径？

```bash
# 查看当前进程（Shell 会话中的进程）
ps               # 只显示当前 Shell 的进程
jobs             # 查看后台任务

# 查看当前路径
pwd              # Print Working Directory

# 退出 Shell / 登出
exit
logout
Ctrl + D         # EOF 信号，效果同 exit
```

###### 2. 如何查看系统中所有进程？ps 命令的常用参数有哪些？

```bash
ps aux           # 显示所有用户的所有进程（最常用）
# a = 所有终端的进程  u = 用户格式（含CPU/内存）  x = 包含无终端进程

ps -ef           # 另一种格式（含 PPID 父进程ID）
ps aux | grep nginx            # 查找 nginx 进程
ps aux | sort -k3 -rn | head   # 按 CPU 使用率排序，取前10
ps aux --sort=-%mem | head     # 按内存使用率排序

# 输出字段说明
# USER  PID  %CPU  %MEM  VSZ  RSS  TTY  STAT  START  TIME  COMMAND
# STAT：R=运行 S=睡眠 Z=僵尸 T=停止 D=不可中断睡眠（等I/O）
```

###### 3. top 和 htop 命令的作用是什么？如何使用？

**top**（Linux 内置）：
```bash
top             # 实时进程监控
# 交互命令：
# P = 按 CPU 排序（默认）
# M = 按内存排序
# k = 终止进程（输入PID）
# 1 = 展开所有CPU核
# q = 退出
```

**关键指标解读**：
```
top - 14:30:00 up 10 days,  3:45,  2 users,  load average: 1.23, 1.45, 1.67
#                                              1分钟负载  5分钟  15分钟（≤CPU核数为正常）
Tasks: 210 total,   2 running, 208 sleeping
%Cpu(s): 15.0 us,  5.0 sy,  0.0 ni, 78.0 id,  2.0 wa,  0.0 hi,  0.0 si
#         用户空间  系统空间                    空闲    等待IO（高=磁盘瓶颈）
MiB Mem:  16000 total,  2000 free,  10000 used,  4000 buff/cache
```

**htop**（需安装）：彩色界面、可用方向键操作、显示更直观，功能基本同 top。

###### 4. 如何终止一个进程？kill、killall、pkill 命令的区别？

```bash
# kill：向指定 PID 发送信号
kill 1234            # 默认发送 SIGTERM（15），请求优雅退出
kill -9 1234         # 发送 SIGKILL（9），强制杀死（不可被捕获/忽略）
kill -HUP 1234       # 发送 SIGHUP（1），让进程重新加载配置

# killall：按进程名杀死
killall nginx        # 杀死所有名为 nginx 的进程
killall -9 java      # 强制杀死所有 java 进程

# pkill：按名称/属性杀死（支持正则）
pkill -f "python app.py"  # 杀死命令行包含 "python app.py" 的进程
pkill -u apache          # 杀死属于 apache 用户的所有进程
```

**常用信号**：
| 信号 | 编号 | 作用 |
|------|------|------|
| SIGTERM | 15 | 优雅退出（进程可以做清理工作），`kill` 默认 |
| SIGKILL | 9 | 强制立即杀死（不可屏蔽），最后手段 |
| SIGHUP | 1 | 挂起/重载配置（Nginx/sshd 用于热重载） |
| SIGINT | 2 | 中断（Ctrl+C） |

###### 5. 什么是守护进程？如何创建守护进程？

守护进程（Daemon）是在后台运行、**不与任何终端关联**的系统服务进程，如 `nginx`、`sshd`、`mysqld`。

**特点**：
- 脱离控制终端（tty），终端关闭不受影响
- 通常以 `d` 结尾命名（nginx**d**、ssh**d**）
- 父进程是 init（PID=1）或 systemd

**创建守护进程的方式**：
```bash
# 方式1：nohup（最简单）
nohup ./app.sh &

# 方式2：使用 systemd（推荐，生产环境标准）
# 创建 /etc/systemd/system/myapp.service
[Unit]
Description=My App Service

[Service]
ExecStart=/usr/bin/java -jar /app/app.jar
Restart=always
User=appuser

[Install]
WantedBy=multi-user.target

# 启动
systemctl daemon-reload
systemctl start myapp
systemctl enable myapp  # 开机自启
```

###### 6. 什么是僵尸进程和孤儿进程？如何处理？

**僵尸进程（Zombie）**：
- 子进程已经退出，但父进程**还没有调用 wait() 回收**子进程的资源（PCB 还在）
- `ps aux` 中 STAT 列显示 `Z`
- 危害：占用 PID，PID 耗尽后无法创建新进程
- **处理**：杀死父进程（孤儿进程会被 init 接管并自动回收）；修复父进程代码让其正确调用 wait()

**孤儿进程（Orphan）**：
- 父进程比子进程**先退出**，子进程被 init（PID=1）收养
- **基本无害**：init 会负责调用 wait() 回收孤儿进程的资源

###### 7. 前台进程和后台进程的区别？如何将进程放到后台运行？

```bash
# 前台进程：占用终端，Ctrl+C 可中断
python server.py

# 后台运行（启动时加 &）
python server.py &
# 打印 [1] 12345（任务号和PID）

# 将正在运行的前台进程切到后台
Ctrl+Z    # 暂停进程并切到后台
bg %1     # 让任务1在后台继续运行（bg = background）

# 将后台任务切回前台
fg %1     # fg = foreground

# 查看所有后台任务
jobs -l   # 显示任务号、PID、状态
```

###### 8. nohup 命令的作用是什么？

`nohup`（No Hang Up）让命令**在终端关闭后继续运行**（忽略 SIGHUP 信号）：

```bash
# 基本用法
nohup python server.py &
# 输出默认重定向到 nohup.out

# 指定输出文件
nohup python server.py > /var/log/app.log 2>&1 &
# 2>&1：将 stderr 重定向到 stdout（都写到 app.log）

# 查看 PID
echo $!   # 最后一个后台命令的 PID

# 注意：nohup 不等于守护进程，进程仍关联当前会话
# 生产环境建议使用 systemd 或 supervisor 管理服务
```

###### 9. 如何查看进程占用的资源（CPU、内存）？

```bash
# 方式1：top/htop（实时动态）
top -p 1234     # 只监控指定 PID

# 方式2：ps（快照）
ps aux | grep java
# %CPU = CPU使用率  %MEM = 物理内存使用率  RSS = 实际内存(KB)  VSZ = 虚拟内存(KB)

# 方式3：/proc 文件系统（更详细）
cat /proc/1234/status  # 查看指定进程状态
ls -la /proc/1234/fd/  # 查看进程打开的文件描述符

# 方式4：pidstat（需 sysstat 包）
pidstat -p 1234 1      # 每秒输出该进程的 CPU/内存/I/O 详情
```

###### 10. cron 和 at 命令的作用是什么？如何定时执行任务？

```bash
# cron：周期性定时任务
crontab -e    # 编辑当前用户的定时任务
# 格式：分 时 日 月 星期  命令
# 分钟(0-59) 小时(0-23) 天(1-31) 月(1-12) 周(0-7,0和7都是周日)

# 示例
0 2 * * *   /opt/backup.sh          # 每天凌晨2点执行备份
*/5 * * * * /usr/bin/health_check.sh # 每5分钟检查一次
0 */6 * * * /opt/sync.sh             # 每6小时同步一次
0 8 * * 1   /opt/weekly_report.sh   # 每周一上午8点

crontab -l    # 查看当前任务
crontab -r    # 删除所有任务

# at：一次性定时任务
at 14:30                  # 今天14:30执行
at 2 days + 8:00          # 2天后8点执行
# 输入命令后 Ctrl+D 确认
atq                       # 查看待执行的 at 任务
atrm 1                    # 删除任务1
```

###### 追问：如何让 Java 进程在服务器重启后自动启动？

**推荐方式：systemd（Linux 标准方式）**：
```bash
# /etc/systemd/system/myapp.service
[Unit]
Description=My Java Application
After=network.target

[Service]
Type=simple
User=appuser
ExecStart=/usr/bin/java -Xmx2g -jar /app/myapp.jar
ExecStop=/bin/kill -SIGTERM $MAINPID
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target

# 操作
systemctl daemon-reload          # 重载 systemd 配置
systemctl enable myapp           # 设置开机自启
systemctl start myapp            # 立即启动
systemctl status myapp           # 查看状态
journalctl -u myapp -f           # 查看日志
```
