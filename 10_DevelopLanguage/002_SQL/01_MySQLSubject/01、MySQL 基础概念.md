###### 1. 什么是 MySQL？

[[../../../21_MySqlKnowledge/01_MySQL基础/01、MySQL架构与存储引擎|📖]]
MySQL 是一款开源的关系型数据库管理系统，遵循 GPL 协议，由瑞典 MySQL AB 公司开发，现在隶属于 Oracle 旗下。它以高性能、高可靠性和低成本闻名，是全球最流行的开源数据库之一。

架构上，MySQL 采用经典的客户端-服务器模式，整体分三层。**连接层**负责管理客户端连接、身份认证和权限校验，Java 里常用的 HikariCP 连接池就工作在这一层。**SQL 层**负责解析 SQL、优化执行计划和结果缓存，其中优化器会用基于代价的模型（CBO）为每条 SQL 找到它认为最优的执行路径。**存储引擎层**是 MySQL 最有特色的设计——可插拔架构，不同的表可以选不同的存储引擎，InnoDB 和 MyISAM 是最常用的两种。

在 Java 应用中，我们通过 JDBC 驱动与 MySQL 通信，驱动把 Java 的 SQL 调用翻译成 MySQL 网络协议格式，通过网络传给服务器执行。

###### 2. MySQL 与 PostgreSQL、Oracle 的区别？

[[../../../21_MySqlKnowledge/01_MySQL基础/01、MySQL架构与存储引擎|📖]]
三者定位不同，侧重点各有差异。

**MySQL** 轻量、快速、易上手，互联网公司首选，可插拔存储引擎是核心亮点，默认隔离级别是可重复读（RR），SQL 标准支持不算最完整，但社区活跃、运维成本低。

**PostgreSQL** 功能更强大，SQL 标准兼容性极高，支持窗口函数、复杂子查询、自定义类型等高级特性，JSONB 处理也比 MySQL 的 JSON 类型更成熟，默认隔离级别是读已提交（RC），适合 GIS、金融分析等复杂业务。

**Oracle** 是商业数据库，功能最全面，有 PL/SQL、RAC 高可用集群、强大的性能诊断工具，稳定性和安全性一流，但授权费用昂贵，主要出现在大型银行、电信等核心系统里。

从 MVCC 实现来看，InnoDB 通过 ReadView + Undo Log 版本链来判断行可见性；PostgreSQL 把多版本数据直接存在表页面中，用 xmin/xmax 字段标记；Oracle 则依赖回滚段存储旧版本。三者思路不同，但目的都是让读写不互相阻塞。

从索引来看，InnoDB 的主键索引是聚簇索引，数据直接存在叶节点，二级索引叶节点存的是主键值，查询非索引列时需要回表。PostgreSQL 是堆表结构，所有索引都指向数据行的物理位置，没有回表的概念。

###### 3. 什么是关系型数据库？

[[../../../21_MySqlKnowledge/01_MySQL基础/01、MySQL架构与存储引擎|📖]]
关系型数据库建立在关系模型基础上，用**表**来组织数据，表由行和列构成。每一行代表一条记录（实体），每一列代表一个字段（属性）。表之间通过主键和外键建立关联，外键保证参照完整性，防止出现"引用了不存在记录"的情况。

关系型数据库的核心是 **ACID** 特性：**原子性**（事务要么全成功要么全回滚）、**一致性**（事务前后数据必须合法）、**隔离性**（并发事务互不干扰）、**持久性**（提交后数据永久保存，即使宕机也不丢）。这四个特性是银行转账、订单扣库存这类需要强一致性场景的根基。

###### 4. MySQL 的主要特点有哪些？

[[../../../21_MySqlKnowledge/01_MySQL基础/01、MySQL架构与存储引擎|📖]]
**开源免费**是 MySQL 最大的优势，遵循 GPL 协议，显著降低了项目成本。

**可插拔存储引擎**让你可以按业务需求选最合适的引擎，比如需要事务就用 InnoDB，需要全文检索就用 MyISAM（MySQL 5.5 起 InnoDB 是默认引擎）。

**高性能与可扩展性**方面，通过多种索引策略、连接池、主从复制等手段，MySQL 能扛住海量数据和高并发，读能力可以通过加从库水平扩展。

**高可用**方面提供了主从复制、组复制（Group Replication）等方案，可以做到故障自动切换。

此外还有**跨平台**（Linux/Windows/macOS 都支持）、**庞大社区**（文档和第三方工具丰富）、**完善的权限与安全体系**（SSL 连接、列级权限、数据加密等）。

###### 5. MySQL 支持哪些数据类型？

[[../../../21_MySqlKnowledge/01_MySQL基础/02、数据类型与字符集|📖]]
MySQL 的数据类型主要分以下几类：

**数值类型**：`INT`、`BIGINT` 用于整数；`DECIMAL(M, D)` 用于精确小数，M 是总位数，D 是小数点后位数，金额字段首选；`FLOAT`/`DOUBLE` 是浮点数，存在精度问题，别用在钱上。

**日期时间类型**：`DATE` 只存日期，`TIME` 只存时间，`DATETIME` 存完整日期时间且不受时区影响，范围到 9999 年；`TIMESTAMP` 存的是 Unix 时间戳，受时区影响，上限是 2038 年。业务里推荐用 `DATETIME`，避免 2038 年问题。

**字符串类型**：`CHAR(N)` 定长，不足补空格，适合 MD5、手机号这类长度固定的字段；`VARCHAR(N)` 变长，更省空间，适合大多数场景；`TEXT` 用于长文本；`BLOB` 用于二进制数据。

**JSON 类型**：MySQL 5.7+ 原生支持 JSON 存储和索引，可以在 JSON 字段上建虚拟列索引，方便查询。

**空间类型**：`GEOMETRY`、`POINT`、`POLYGON` 等，用于 GIS 地理信息系统。

在 Java 里，`DATETIME` 对应 `LocalDateTime`，`TIMESTAMP` 对应 `Instant`（要处理时区），`BLOB` 对应 `byte[]`。

###### 6. 什么是 SQL？SQL 有哪些分类？

[[../../../21_MySqlKnowledge/01_MySQL基础/03、SQL语言与执行流程|📖]]
SQL（Structured Query Language）是关系型数据库的标准操作语言，按功能分为四大类。

**DDL（数据定义语言）**：用 `CREATE`、`ALTER`、`DROP`、`TRUNCATE` 来定义和修改数据库结构，比如建表、加字段、删索引。注意 DDL 操作在 MySQL 里会自动隐式提交事务，无法回滚。

**DML（数据操作语言）**：用 `INSERT`、`UPDATE`、`DELETE` 对数据做增删改。在 Java 里执行 DML 后需要显式 `commit()`，或者开启自动提交模式。

**DQL（数据查询语言）**：核心就是 `SELECT`，虽然有时被归入 DML，但因为地位特殊一般单独列出。复杂查询会涉及 JOIN、子查询、聚合、窗口函数等，MySQL 8.0+ 完整支持窗口函数。

**DCL（数据控制语言）**：用 `GRANT`、`REVOKE` 控制数据库访问权限，比如给某个用户赋予某张表的读写权限。
