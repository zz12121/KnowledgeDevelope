##题库要点（原始材料）
为了减少获得锁和释放锁带来的性能开销，JVM 会对 `synchronized`锁进行优化，其升级过程是单向的（不可降级）。下面的流程图清晰地展示了这一过程：

```mermaid
flowchart TD
    A[线程访问同步资源] --> B{锁状态}
    B -->|无锁| C[尝试获取锁]
    C --> D{是否启用偏向锁<br>且当前无竞争?}
    D -->|是| E[偏向锁]
    D -->|否| F[轻量级锁]
    E --> G{存在线程竞争?}
    G -->|是| F
    F --> H{自旋获取锁成功?<br>或竞争加剧?}
    H -->|竞争加剧| I[重量级锁]
    H -->|成功获取| J[线程执行同步代码]
    I --> J
    J --> K[释放锁]
    K --> A
```
