# HashMap 为什么线程不安全？

## 一句话说明（白话）

hashCode 是对象散列标识，用于加速查找。

## 它解决什么问题 / 为什么重要

HashMap/HashSet先用 hashCode 定位桶，再用 equals 判断。

## 核心原理（一步步讲清楚）

equals 相等必须 hashCode 相等。

##典型使用场景

Map key、Set 去重。

## 简单例子 /伪代码

equals 基于 id，hashCode也应基于 id。

## 常见坑与误区

hashCode 相同不代表 equals 相同。

##题库要点（原始材料）
HashMap 在设计上未考虑同步，在多线程并发环境下使用主要存在以下问题：
- **数据覆盖**：多个线程同时执行 `put`操作，可能导致后一个线程覆盖前一个线程放入的值，造成数据丢失。
- **环形链表（JDK 1.7）**：在扩容过程中，线程不安全的转移操作可能导致链表形成环，进而引起后续的 `get`操作出现无限循环。
- **数据不一致**：一个线程在遍历，另一个线程在修改，可能会抛出 `ConcurrentModificationException`异常。
