# MyCAT2 分库分表概述

## 1. 分库分表类型

### 1.1 分片表（Sharding Table）

分片表是将一个逻辑表的数据分散存储到多个数据库（分片）中。

```json
{
    "schemaName": "db1",
    "shardingTables": {
        "travelrecord": {
            "function": {
                "properties": {
                    "dbNum": "2",           // 分库数量
                    "tableNum": "2",       // 分表数量
                    "tableMethod": "hash(id)",   // 分表函数
                    "dbMethod": "hash(id)",      // 分库函数
                    "storeNum": 2           // 存储节点数量
                }
            }
        }
    }
}
```

### 1.2 广播表（Broadcast Table）

广播表表示在所有数据分片上都相同的表，通过数据冗余使分片表的分区与该表的数据在同一个数据库实例里，达到 join 运算能够直接在该数据库实例里执行。

```sql
CREATE TABLE dict_type (
  id INT PRIMARY KEY,
  type_id INT NOT NULL,
  name VARCHAR(255)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 BROADCAST;
```

### 1.3 ER表（ER Table）

ER 表是指子表（通常是明细表）的分片键指向父表（主表）的分片键，而且两表的分片算法相同。

## 2. 分片表配置

### 2.1 mappingFormat 格式

```
targetIndex, dbIndex, tableIndex 总是从0开始计算，支持 groovy 运算生成目标名、库名、表名
```

```json
{
    "shardingTables": {
        "travelrecord": {
            "function": {
                "properties": {
                    "dbNum": "2",
                    "tableNum": "2",
                    "tableMethod": "hash(id)",
                    "dbMethod": "hash(id)",
                    "mappingFormat": "c${targetIndex}/db1_${dbIndex}/travelrecord_${tableIndex}"
                }
            }
        }
    }
}
```

### 2.2 自定义分片算法

```json
{
    "shardingTables": {
        "travelrecord": {
            "function": {
                "clazz": "io.mycat.router.mycat1xfunction.PartitionByHotDate",
                "properties": {
                    "lastTime": 90,
                    "partionDay": 180,
                    "dateFormat": "yyyy-MM-dd",
                    "columnName": "create_time"
                },
                "ranges": {}
            },
            "partition": {
                "targetNames": "c$0-1",      // 集群或数据源
                "schemaNames": "db1_$0-1",   // 物理库
                "tableNames": "t1_$0-1"      // 物理表
            }
        }
    }
}
```

## 3. 分库分表示例

### 3.1 创建分片表 SQL

```sql
-- 使用 MyCAT2 语法创建分片表
CREATE TABLE db1.travelrecord (
    id BIGINT NOT NULL AUTO_INCREMENT,
    user_id VARCHAR(100),
    create_time DATE,
    fee DECIMAL(10,0),
    days INT,
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY MOD_HASH(id) DBPARTITIONS 6
  TBPARTITION BY MOD_HASH(id) TBPARTITIONS 6;
```

### 3.2 分库分表原理

**分库键和分表键是同键**：
- 分库下标 = 分片值 % 分库数量
- 分表下标 = (分片值 % 分库数量) * 分表数量 + (分片值 / 分库数量) % 分表数量

**分库键和分表键是不同键**：
- 分表下标 = 分片值 % 分表数量
- 分片下标 = 分片值 % 分库数量

## 4. 分库分表查询

### 4.1 跨分片查询

MyCAT2 支持跨分片的 SQL 查询，包括：
- JOIN（需要使用 ER 表或广播表）
- ORDER BY
- GROUP BY
- LIMIT

### 4.2 排序分页

```sql
-- 分页查询（会归并结果）
SELECT * FROM travelrecord ORDER BY id LIMIT 10, 20;
```

---

## 5. 分库分表注意事项

1. **选择合适的分片键** - 避免数据倾斜和热点问题
2. **跨分片事务** - 需要使用 XA 事务
3. **分页查询** - 深分页可能导致性能问题
4. **唯一键** - 全局唯一性需要使用分布式 ID 生成器
