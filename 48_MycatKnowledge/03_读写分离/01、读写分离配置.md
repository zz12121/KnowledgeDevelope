## 2. 读写分离配置流程

### 2.1 通过 SQL 语句创建配置

MyCAT2 支持通过 SQL 注释方式动态创建数据源、集群等配置。

### 2.2 创建写数据源（主库）

```sql
/*+ mycat:createDataSource{
    "dbType":"mysql",
    "idleTimeout":60000,
    "initSqls":[],
    "initSqlsGetConnection":true,
    "instanceType":"READ_WRITE",
    "maxCon":1000,
    "maxConnectTimeout":3000,
    "maxRetryCount":5,
    "minCon":1,
    "name":"dw0",
    "password":"root",
    "type":"JDBC",
    "url":"jdbc:mysql://127.0.0.1:3306?useUnicode=true&serverTimezone=UTC&characterEncoding=UTF-8",
    "user":"root",
    "weight":0
} */;
```

### 2.3 创建读数据源（从库）

```sql
/*+ mycat:createDataSource{
    "dbType":"mysql",
    "idleTimeout":60000,
    "initSqls":[],
    "initSqlsGetConnection":true,
    "instanceType":"READ",
    "maxCon":1000,
    "maxConnectTimeout":3000,
    "maxRetryCount":5,
    "minCon":1,
    "name":"dr0",
    "password":"root",
    "type":"JDBC",
    "url":"jdbc:mysql://127.0.0.1:3308?useUnicode=true&serverTimezone=UTC&characterEncoding=UTF-8",
    "user":"root",
    "weight":0
} */;
```

### 2.4 查看数据源配置

```sql
/*+ mycat:showDataSources*/;
```

### 2.5 创建集群

```sql
/*! mycat:createCluster{
    "clusterType":"MASTER_SLAVE",
    "heartbeat":{
        "heartbeatTimeout":1000,
        "maxRetry":3,
        "minSwitchTimeInterval":300,
        "slaveThreshold":0
    },
    "masters":["dw0"],           -- 主节点
    "maxCon":2000,
    "name":"c0",
    "readBalanceType":"BALANCE_ALL",
    "replicas":["dr0"],          -- 从节点
    "switchType":"SWITCH"
} */;
```

### 2.6 查看集群配置

```sql
/*+ mycat:showClusters*/;
```

## 3. 逻辑库与逻辑表配置

```sql
CREATE DATABASE db1;
```

## 4. 测试读写分离

### 4.1 连接 MyCAT

```bash
# MyCAT 默认端口 8066
mysql -h 127.0.0.1 -P 8066 -u root -p123456
```

### 4.2 执行测试

```sql
-- 切换到逻辑库
USE db1;

-- 插入数据（写操作，走主库）
INSERT INTO user (name, age) VALUES ('张三', 20);

-- 查询数据（读操作，走从库）
SELECT * FROM user;
```

### 4.3 验证

1. 在主库插入数据后，立刻在从库查询，应该能查到（如果延迟为0）
2. 在从库插入数据，不会同步到主库（单向复制）

## 5. 负载均衡策略配置

### 5.1 readBalanceType（集群级别）

| 值 | 说明 |
|----|------|
| BALANCE_ALL | 获取集群中所有数据源 |
| BALANCE_ALL_READ | 获取集群中允许读的数据源 |
| BALANCE_READ_WRITE | 获取集群中允许读写的数据源，但允许读的数据源优先 |
| BALANCE_NONE | 获取集群中允许写数据源，即主节点中选择 |

### 5.2 负载均衡实现类（读写分离级别）

| 实现类 | 说明 |
|--------|------|
| BalanceLeastActive | 最少正在使用的连接数的 MySQL 数据源被选中 |
| BalanceRandom | 随机算法产生随机数 |
| BalanceRoundRobin | 加权轮询算法 |
| BalanceRunOnReplica | 把请求尽量发往从节点 |

## 6. 高可用配置

### 6.1 主从切换

```json
{
    "switchType": "SWITCH",  // 开启主从自动切换
    "heartbeat": {
        "heartbeatTimeout": 1000,
        "maxRetry": 3,
        "minSwitchTimeInterval": 300,
        "slaveThreshold": 0
    }
}
```

### 6.2 切换类型

| 值 | 说明 |
|----|------|
| NOT_SWITCH | 不进行主从切换 |
| SWITCH | 进行主从切换 |

### 6.3 集群心跳

```json
"timer": {
    "initialDelay": 30,
    "period": 5,
    "timeUnit": "SECONDS"
}
```
