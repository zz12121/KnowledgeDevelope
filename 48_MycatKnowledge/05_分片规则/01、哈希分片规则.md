# MyCAT2 分片规则详解

## 1. 哈希分片规则概述

MyCAT2 支持多种分片规则，用于将数据分散到不同的分片中。

---

## 2. 哈希分片规则详解

### 2.1 MOD_HASH

**取模分片**：根据分片键的哈希值取模，映射到具体的分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY MOD_HASH(id) DBPARTITIONS 6
  TBPARTITION BY MOD_HASH(id) TBPARTITIONS 6;
```

**计算公式**：
- 分库下标 = 分片值 % 分库数量
- 分表下标 = 分片值 % 分表数量

### 2.2 RANGE_HASH

**范围哈希分片**：对多个字段进行哈希后取模。

```sql
-- 使用两个字段
DBPARTITION BY RANGE_HASH(id, user_id) DBPARTITIONS 3
```

**计算公式**：
- `RANGE_HASH(字段1, 字段2, 截取开始下标)`
- 仅支持数值类型和字符串类型
- 当时字符串类型时，第三个参数生效
- 计算时优先选择第一个字段，找不到选择第二个字段

### 2.3 RIGHT_SHIFT

**右移分片**：数值类型的分片键右移指定位数后取模。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    user_id BIGINT,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY RIGHT_SHIFT(id, 4) DBPARTITIONS 3
  TBPARTITION BY RIGHT_SHIFT(user_id, 4) TBPARTITIONS 3;
```

**计算公式**：
- 分片值右移二进制位数，然后按分片数量取余
- 例如：`id=16, 右移4位=1, 1%3=1`

### 2.4 UNI_HASH

**统一哈希分片**：与 MOD_HASH 类似，但计算方式略有不同。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY UNI_HASH(id) DBPARTITIONS 6
  TBPARTITION BY UNI_HASH(id) TBPARTITIONS 6;
```

**特点**：
- 仅用于分库
- 如果分片值是字符串则先对字符串进行 hash 转换为数值类型

## 3. 日期分片规则

### 3.1 WEEK

**按周分片**：根据一周之中的星期（1-7）进行取余运算。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATETIME,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY WEEK(create_time) DBPARTITIONS 7;
```

**限制**：
- 仅用于分表
- 仅支持 DATE/DATETIME
- 分表数不超过 7

### 3.2 YYYYDD

**按一年中的天数分片**：根据一年中的第几天进行分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATE,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY YYYYDD(create_time) DBPARTITIONS 8;
```

**计算公式**：
- `(YYYY * 366 + DD) % 分库数`
- 仅用于分库

### 3.3 YYYYMM

**按年月分片**：根据年份和月份进行分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATE,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  DBPARTITION BY YYYYMM(create_time) DBPARTITIONS 8;
```

**计算公式**：
- `(YYYY * 12 + MM) % 分库数`
- 仅用于分库

### 3.4 DD

**按日期分片**：根据一月中的第几天进行分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATE,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  TBPARTITION BY DD(create_time) TBPARTITIONS 31;
```

**计算公式**：
- `(DD) % 分表数`
- 仅用于分表
- 分表数不能超过 31

### 3.5 YYYYWEEK

**按年份周分片**：根据一年中的第几周进行分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATE,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  TBPARTITION BY YYYYWEEK(create_time) TBPARTITIONS 12;
```

**计算公式**：
- `(YYYY * 54 + WEEK) % 分片数`
- WEEK 范围是 1-53
- 仅用于分表
- 支持分库分表

### 3.6 MM

**按月份分片**：根据月份进行分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATE,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  TBPARTITION BY MM(create_time) TBPARTITIONS 12;
```

**计算公式**：
- `MM % 分表数`
- 仅支持 DATE/DATETIME
- 分表数不超过 12

### 3.7 MMDD

**按月日分片**：根据月日进行分片。

```sql
CREATE TABLE travelrecord (
    id BIGINT NOT NULL,
    create_time DATE,
    ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8 
  TBPARTITION BY MMDD(create_time) TBPARTITIONS 366;
```

**计算公式**：
- `一年之中第几天 % 分表数`
- 仅用于分表
- 分表数不超过 366

---

## 4. 分片规则选择建议

| 场景 | 推荐规则 |
|------|---------|
| 订单数据（按用户ID） | MOD_HASH |
| 订单数据（按时间） | YYYYMM / YYYYDD / WEEK |
| 流水数据（按时间） | MMDD / DD |
| 多字段组合 | RANGE_HASH |
