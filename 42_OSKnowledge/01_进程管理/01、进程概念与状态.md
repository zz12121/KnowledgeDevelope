# 进程概念与状态

## 一、进程基本概念
### 1.1 进程定义
进程是计算机中正在运行的程序的实例，包括：
- 程序代码（文本段）
- 程序计数器（PC）
- 寄存器集合
- 堆栈段（局部变量、函数参数）
- 数据段（全局变量）
- 堆段（动态分配的内存）

### 1.2 进程与程序的区别
| 特性 | 程序 | 进程 |
|------|------|------|
| 存在形式 | 静态的代码文件 | 动态的执行实体 |
| 生命周期 | 永久存储 | 创建→运行→终止 |
| 资源占用 | 不占用系统资源 | 占用CPU、内存等资源 |
| 并发性 | 无 | 可以并发执行 |

## 二、进程状态
### 2.1 五状态模型
```
新建(New) → 就绪(Ready) ↔ 运行(Running) → 终止(Terminated)
                    ↑         ↓
                    └── 阻塞(Blocked)
```

### 2.2 状态说明
1. **新建(New)**：进程刚被创建，还未被操作系统接受
2. **就绪(Ready)**：进程已准备好运行，等待CPU分配
3. **运行(Running)**：进程正在CPU上执行
4. **阻塞(Blocked)**：进程等待某些事件发生（如I/O完成）
5. **终止(Terminated)**：进程执行完毕或被强制终止

### 2.3 状态转换条件
- **就绪 → 运行**：进程调度器选择该进程
- **运行 → 就绪**：时间片用完或更高优先级进程就绪
- **运行 → 阻塞**：等待I/O操作、信号量、消息等
- **阻塞 → 就绪**：等待的事件发生

## 三、进程控制块（PCB）
### 3.1 PCB结构
每个进程都有一个PCB，包含：
- **进程标识信息**：PID、PPID（父进程ID）
- **处理器状态信息**：寄存器值、程序计数器、堆栈指针
- **进程控制信息**：
  - 进程状态（运行、就绪、阻塞等）
  - 进程优先级
  - 程序计数器
  - 内存管理信息（基址寄存器、界限寄存器）
  - 记账信息（CPU时间、时间限制）
  - I/O状态信息（打开的I/O设备列表）

### 3.2 PCB的作用
- 进程调度依据
- 上下文切换时保存和恢复现场
- 进程间通信的基础
- 资源管理的核心数据结构

## 四、进程创建与终止
### 4.1 进程创建
**创建方式**：
1. **系统初始化**：启动时创建系统进程
2. **系统调用**：运行中的进程调用fork()创建子进程
3. **用户请求**：用户通过shell命令创建新进程
4. **批处理作业**：批处理系统初始化时创建

**创建过程**：
1. 分配唯一的进程标识符（PID）
2. 分配进程空间
3. 初始化进程控制块（PCB）
4. 设置正确的链接（如就绪队列）
5. 创建或扩充其他数据结构

### 4.2 进程终止
**正常终止**：
- 自愿退出（main函数return或调用exit()）
- 错误退出（程序检测到错误条件）

**异常终止**：
- 致命错误（如除以零、非法指令）
- 被其他进程杀死（kill信号）
- 被操作系统终止（资源超限等）

**终止过程**：
1. 释放进程占用的所有资源
2. 通知父进程子进程的终止状态
3. 将PCB从所有队列中移除
4. 销毁PCB

## 五、进程层次结构
### 5.1 进程树
在Unix/Linux系统中，进程形成树状结构：
- init进程（PID=1）是所有用户进程的祖先
- 每个进程都有父进程（PPID）
- 进程可以创建子进程

### 5.2 进程组和会话
- **进程组**：一组相关进程的集合
- **会话**：一个用户登录后创建的所有进程的集合

## 六、实战应用
### 6.1 Linux进程查看
```bash
# 查看所有进程
ps aux
ps -ef

# 查看进程树
pstree

# 实时查看进程
top
htop
```

### 6.2 进程管理命令
```bash
# 杀死进程
kill <PID>
kill -9 <PID>  # 强制杀死

# 根据名称杀死进程
pkill <process_name>

# 优雅停止
kill -TERM <PID>
```

## 八、性能考虑
- 进程创建和销毁的开销较大
- 进程间切换需要保存和恢复大量上下文
- 进程间通信（IPC）比线程间通信更复杂
