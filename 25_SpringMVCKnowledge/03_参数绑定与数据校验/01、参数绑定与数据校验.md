# 参数绑定与数据校验

## 参数绑定机制

SpringMVC 参数绑定的本质是：**将 HTTP 请求的各类数据自动转换为控制器方法的 Java 参数**。整个过程由 `HandlerMethodArgumentResolver` 接口的各种实现负责，核心处理流程在 `InvocableHandlerMethod.invokeForRequest()` 中：遍历方法参数 → 找到匹配的解析器 → 解析参数值 → 类型转换 → 最终通过反射调用方法。

**支持的参数类型：**
- 基本类型和字符串（自动解析请求参数）
- POJO 对象（自动将同名请求参数映射到字段）
- 集合/数组类型（支持同名多值）
- 原生 Servlet API（`HttpServletRequest`/`HttpServletResponse`/`HttpSession`）
- `@RequestBody` 接收 JSON/XML 请求体
- `BindingResult`/`Errors` 接收校验结果

## 数据绑定（DataBinder）

`WebDataBinder` 是 SpringMVC 数据绑定的核心组件，职责包括：
- **数据绑定**：将请求参数绑定到目标对象
- **类型转换**：通过 `ConversionService` 进行类型转换
- **数据校验**：调用 `Validator` 执行校验逻辑
- **结果管理**：收集错误信息到 `BindingResult`

`@InitBinder` 注解的方法在每次请求处理前执行，可以定制 `WebDataBinder` 的行为：设置禁止绑定的字段（防止恶意参数覆盖）、注册自定义属性编辑器、设置必要字段。

```java
@Controller
public class UserController {
    
    @InitBinder
    public void initBinder(WebDataBinder binder) {
        // 禁止绑定 id 字段（防止外部传参覆盖）
        binder.setDisallowedFields("id", "createTime");
        // 注册日期格式
        binder.registerCustomEditor(Date.class, 
            new CustomDateEditor(new SimpleDateFormat("yyyy-MM-dd"), true));
    }
}
```

`@InitBinder` 只对当前控制器有效，全局的需要放在 `@ControllerAdvice` 类中。

## 类型转换

**`Converter<S, T>`（推荐）**：Spring 类型转换体系的核心接口，线程安全，支持泛型，通过 `ConversionService` 统一管理：

```java
@Component
public class StringToDateConverter implements Converter<String, Date> {
    @Override
    public Date convert(String source) {
        try {
            return new SimpleDateFormat("yyyy-MM-dd").parse(source);
        } catch (ParseException e) {
            throw new IllegalArgumentException("日期格式不正确: " + source);
        }
    }
}
```

通过 `WebMvcConfigurer.addFormatters()` 注册自定义转换器。

**`@DateTimeFormat`（最简便）**：直接在字段或方法参数上注解，处理日期格式转换：
```java
@GetMapping("/users")
public List<User> findByDate(@RequestParam @DateTimeFormat(pattern = "yyyy-MM-dd") Date startDate) { ... }
```

**`PropertyEditor`（老式）**：Java Bean 规范，非线程安全，不支持泛型，现代项目已被 `Converter` 取代。

## 数据校验

**JSR-303/349 Bean Validation** 是标准校验规范，Hibernate Validator 是参考实现，Spring Validation 做了整合。常用注解：

| 注解 | 说明 |
|------|------|
| `@NotNull` | 不能为 null |
| `@NotBlank` | 不能为空串（Hibernate 扩展） |
| `@NotEmpty` | 集合/数组不能为空 |
| `@Size(min, max)` | 字符串/集合大小限制 |
| `@Min/@Max` | 数值范围 |
| `@Email` | 邮箱格式（Hibernate 扩展） |
| `@Pattern(regexp)` | 正则匹配 |
| `@Past/@Future` | 日期过去/未来 |

**基本用法**：在 DTO 字段上加注解，在 Controller 方法参数上加 `@Valid` 或 `@Validated` 触发校验，用 `BindingResult` 接收校验结果：

```java
@PostMapping("/users")
public Result createUser(@Valid @RequestBody UserDTO user, BindingResult result) {
    if (result.hasErrors()) {
        return Result.fail(result.getAllErrors().stream()
            .map(DefaultMessageSourceResolvable::getDefaultMessage)
            .collect(Collectors.joining(", ")));
    }
    return Result.ok(userService.create(user));
}
```

## @Valid vs @Validated

两者都触发校验，区别是：

`@Valid` 来自 JSR-303 标准，不支持分组校验，支持嵌套对象的级联校验（在字段上加 `@Valid` 触发级联）。

`@Validated` 是 Spring 的扩展，支持**分组校验**（根据不同场景应用不同的校验规则）：

```java
public class UserDTO {
    @NotNull(groups = Update.class)          // 更新时必须有 id
    private Long id;
    
    @NotBlank(groups = {Create.class, Update.class})  // 创建和更新都需要
    private String name;
    
    public interface Create {}
    public interface Update {}
}

@PostMapping("/create")
public Result create(@Validated(UserDTO.Create.class) @RequestBody UserDTO dto) { ... }

@PutMapping("/update")
public Result update(@Validated(UserDTO.Update.class) @RequestBody UserDTO dto) { ... }
```

## 自定义校验注解

当内置注解不满足需求时，可以自定义：

```java
// 1. 定义注解
@Target({ElementType.FIELD, ElementType.PARAMETER})
@Retention(RetentionPolicy.RUNTIME)
@Constraint(validatedBy = PhoneValidator.class)
public @interface ValidPhone {
    String message() default "手机号格式不正确";
    Class<?>[] groups() default {};
    Class<? extends Payload>[] payload() default {};
}

// 2. 实现校验器
public class PhoneValidator implements ConstraintValidator<ValidPhone, String> {
    private static final Pattern PHONE_PATTERN = Pattern.compile("^1[3-9]\\d{9}$");
    
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        return value == null || PHONE_PATTERN.matcher(value).matches();
    }
}
```

跨字段校验（如密码确认）用类级别注解，在 `isValid()` 中同时获取两个字段的值进行比较。

## 全局异常处理校验错误

不传 `BindingResult` 时，校验失败会抛出 `MethodArgumentNotValidException`，用全局异常处理器统一处理更干净：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result handleValidationException(MethodArgumentNotValidException ex) {
        List<String> errors = ex.getBindingResult().getFieldErrors().stream()
            .map(error -> error.getField() + ": " + error.getDefaultMessage())
            .collect(Collectors.toList());
        return Result.fail(400, "参数校验失败", errors);
    }
}
```
