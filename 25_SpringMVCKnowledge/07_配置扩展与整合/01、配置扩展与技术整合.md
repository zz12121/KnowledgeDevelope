# 配置扩展与技术整合

## Java 配置方式（推荐）

SpringMVC 推荐使用 Java 配置替代 XML，通过实现 `WebMvcConfigurer` 接口（不要加 `@EnableWebMvc`，否则会覆盖 Spring Boot 的自动配置）：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    
    // 拦截器
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new AuthInterceptor()).addPathPatterns("/api/**");
    }
    
    // 静态资源
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/static/**")
                .addResourceLocations("classpath:/static/");
    }
    
    // 跨域
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/api/**").allowedOrigins("*");
    }
    
    // 视图控制器（简单重定向/转发，不需要写 Controller）
    @Override
    public void addViewControllers(ViewControllerRegistry registry) {
        registry.addRedirectViewController("/", "/index");
    }
    
    // 消息转换器（扩展而不是替换）
    @Override
    public void extendMessageConverters(List<HttpMessageConverter<?>> converters) {
        // 调整 Jackson 配置
        converters.stream()
            .filter(c -> c instanceof MappingJackson2HttpMessageConverter)
            .map(c -> (MappingJackson2HttpMessageConverter) c)
            .forEach(c -> {
                c.getObjectMapper().configure(SerializationFeature.WRITE_DATES_AS_TIMESTAMPS, false);
                c.getObjectMapper().registerModule(new JavaTimeModule());
            });
    }
    
    // 内容协商
    @Override
    public void configureContentNegotiation(ContentNegotiationConfigurer configurer) {
        configurer.defaultContentType(MediaType.APPLICATION_JSON);
    }
}
```

注意：`configureMessageConverters()` 会替换所有默认转换器，`extendMessageConverters()` 是在已有转换器基础上修改，通常用后者。

## HttpMessageConverter

`HttpMessageConverter` 负责 HTTP 请求/响应体与 Java 对象之间的转换，是 `@RequestBody`/`@ResponseBody` 工作的核心。

常用实现：
- `MappingJackson2HttpMessageConverter`：JSON 转换（classpath 有 Jackson 时自动注册）
- `StringHttpMessageConverter`：字符串处理
- `FormHttpMessageConverter`：处理表单数据（`application/x-www-form-urlencoded`）
- `ByteArrayHttpMessageConverter`：字节数组
- `ResourceHttpMessageConverter`：文件资源下载

转换器选择过程：根据请求的 `Content-Type` 和方法参数类型（用于读），或根据 `Accept` 头和返回值类型（用于写），遍历所有注册的转换器，找到第一个满足条件的。

## Session 与 Cookie

**操作 Session**：
```java
// 方式一：注入 HttpSession
@GetMapping("/session-test")
public void testSession(HttpSession session) {
    session.setAttribute("user", currentUser);
    Object user = session.getAttribute("user");
    session.invalidate();  // 让整个 session 失效
}

// 方式二：@SessionAttribute（从 session 中取已有属性）
@GetMapping("/profile")
public String profile(@SessionAttribute("loginUser") User user, Model model) {
    model.addAttribute("user", user);
    return "profile";
}
```

**操作 Cookie**：
```java
// 设置 Cookie
@GetMapping("/set-cookie")
public ResponseEntity<String> setCookie(HttpServletResponse response) {
    Cookie cookie = new Cookie("token", generateToken());
    cookie.setMaxAge(7 * 24 * 3600);  // 7天
    cookie.setPath("/");
    cookie.setHttpOnly(true);   // 防 XSS
    cookie.setSecure(true);     // 仅 HTTPS
    response.addCookie(cookie);
    return ResponseEntity.ok("Cookie 已设置");
}

// 读取 Cookie
@GetMapping("/user-info")
public String getUserInfo(@CookieValue(value = "token", required = false) String token) {
    // ...
}
```

## 异步请求处理

SpringMVC 基于 Servlet 3.0 规范支持异步请求，核心是释放 Tomcat 请求线程，避免线程长时间阻塞：

**`Callable`**：Spring 管理线程，适合内部耗时任务：
```java
@GetMapping("/async-callable")
public Callable<Result> asyncCallable() {
    return () -> {
        // 在 Spring 的 TaskExecutor 线程池中执行
        Thread.sleep(2000);
        return Result.ok("completed");
    };
}
```

**`DeferredResult`**：应用代码控制何时设置结果，适合长轮询、消息推送等场景：
```java
@GetMapping("/async-deferred")
public DeferredResult<Result> asyncDeferred() {
    DeferredResult<Result> result = new DeferredResult<>(5000L, Result.fail("timeout"));
    
    // 可以在任意线程中（甚至外部系统回调中）设置结果
    asyncService.processAsync(result);
    
    return result;  // Tomcat 线程立即释放
}
```

两者的核心区别：`Callable` 把任务提交给 Spring 线程池，线程资源还是 Spring 管；`DeferredResult` 完全由应用程序控制结果的设置时机，可以等待外部系统（MQ 回调、WebHook）的通知。

## 国际化（i18n）

SpringMVC 国际化通过 `MessageSource` + `LocaleResolver` 实现：

```java
@Configuration
public class I18nConfig {
    
    @Bean
    public MessageSource messageSource() {
        ResourceBundleMessageSource ms = new ResourceBundleMessageSource();
        ms.setBasenames("i18n/messages");
        ms.setDefaultEncoding("UTF-8");
        return ms;
    }
    
    // 从请求头 Accept-Language 解析 Locale（最常用）
    @Bean
    public LocaleResolver localeResolver() {
        AcceptHeaderLocaleResolver resolver = new AcceptHeaderLocaleResolver();
        resolver.setDefaultLocale(Locale.CHINA);
        return resolver;
    }
    
    // 支持通过请求参数切换语言，需要配合 LocaleChangeInterceptor
    // /api/users?lang=en_US
}
```

`LocaleResolver` 的几种实现：`AcceptHeaderLocaleResolver`（从 HTTP 头解析，推荐）、`SessionLocaleResolver`（存 Session，用户自选语言）、`CookieLocaleResolver`（存 Cookie，长期记忆）。

## 整合 Spring Security

Spring Security 通过 `DelegatingFilterProxy` 过滤器链机制集成到 SpringMVC：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {
    
    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.csrf().disable()
            .sessionManagement().sessionCreationPolicy(SessionCreationPolicy.STATELESS)
            .and()
            .authorizeRequests()
                .antMatchers("/api/auth/**").permitAll()
                .antMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            .and()
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class);
    }
}
```

方法级安全注解（需要 `@EnableGlobalMethodSecurity(prePostEnabled = true)`）：

```java
@PreAuthorize("hasRole('ADMIN') or #userId == principal.id")
public User getUser(Long userId) { ... }
```

## 整合 Thymeleaf

Spring Boot 引入 `spring-boot-starter-thymeleaf` 即可自动配置，无需额外配置：

```html
<!-- Thymeleaf 模板示例 -->
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org"
      xmlns:sec="http://www.thymeleaf.org/extras/spring-security">
<head>
    <title th:text="${pageTitle}">Title</title>
</head>
<body>
    <div th:if="${user != null}">
        <p th:text="|欢迎，${user.name}！|">欢迎</p>
        <ul>
            <li th:each="item : ${items}" th:text="${item.name}">Item</li>
        </ul>
    </div>
    <div sec:authorize="hasRole('ADMIN')">管理员专区</div>
</body>
</html>
```

Thymeleaf 相比 JSP 的优势：模板是合法的 HTML（不依赖服务器可直接用浏览器预览）、支持 Spring Security 标签、支持国际化（`#{message.key}`）、IDE 友好。
