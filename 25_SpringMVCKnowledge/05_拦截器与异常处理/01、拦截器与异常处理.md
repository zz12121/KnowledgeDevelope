# 拦截器与异常处理

## 拦截器（HandlerInterceptor）

拦截器是基于 AOP 思想的横切关注点实现机制，通过 `HandlerExecutionChain` 管理拦截器链。与过滤器相比，拦截器在 Spring MVC 层面工作，能访问 Spring 上下文和 Handler 信息。

`HandlerInterceptor` 接口有三个回调方法：

- **`preHandle()`**：Handler 方法执行前调用，返回 `false` 则中断请求（后续拦截器和 Handler 不再执行），适合做权限验证、请求预处理
- **`postHandle()`**：Handler 方法执行后、视图渲染前调用，可以修改 `ModelAndView`，适合添加通用模型数据
- **`afterCompletion()`**：整个请求处理完成后调用（类似 `finally`，即使发生异常也会执行），适合资源清理（清理 ThreadLocal、记录请求日志）

---

## 多拦截器执行顺序

```
preHandle1 → preHandle2 → preHandle3
→ 控制器方法执行 →
postHandle3 → postHandle2 → postHandle1
→ 视图渲染 →
afterCompletion3 → afterCompletion2 → afterCompletion1
```

`preHandle` 按注册顺序正序执行，`postHandle` 和 `afterCompletion` 逆序执行（栈式结构）。如果某个 `preHandle` 返回 false，后续拦截器的 `preHandle` 不执行，之前已执行的拦截器会执行 `afterCompletion`。

---

## 拦截器 vs 过滤器

| 特性 | 拦截器 | 过滤器 |
|------|--------|--------|
| 规范 | Spring MVC 特有 | Servlet 规范标准 |
| 作用范围 | 只拦截 Spring MVC 控制器 | 所有 Web 请求（含静态资源） |
| Spring 依赖注入 | 支持 | 不支持（需要特殊方式） |
| 访问 Handler 信息 | 可以获取 HandlerMethod | 不能 |
| 实现方式 | 实现 `HandlerInterceptor` | 实现 `Filter` 接口 |

拦截器能做的，过滤器大多也能做；但拦截器能访问 HandlerMethod（知道是哪个 Controller 的哪个方法），这是过滤器做不到的，特别适合基于注解的权限控制。

## 自定义拦截器

```java
@Component
public class AuthInterceptor implements HandlerInterceptor {
    
    @Autowired
    private TokenService tokenService;  // 支持 Spring 依赖注入
    
    @Override
    public boolean preHandle(HttpServletRequest request, 
                             HttpServletResponse response, Object handler) throws Exception {
        // 判断是否需要校验（静态资源等不需要）
        if (!(handler instanceof HandlerMethod)) return true;
        
        HandlerMethod handlerMethod = (HandlerMethod) handler;
        // 检查是否有免认证注解
        if (handlerMethod.hasMethodAnnotation(NoAuth.class)) return true;
        
        String token = request.getHeader("Authorization");
        if (!tokenService.validateToken(token)) {
            response.setStatus(HttpStatus.UNAUTHORIZED.value());
            return false;
        }
        return true;
    }
    
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response,
                                Object handler, Exception ex) {
        // 清理 ThreadLocal，防止内存泄漏
        UserContext.clear();
    }
}
```

注册拦截器：

```java
@Configuration
public class WebConfig implements WebMvcConfigurer {
    @Autowired
    private AuthInterceptor authInterceptor;
    
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(authInterceptor)
                .addPathPatterns("/api/**")
                .excludePathPatterns("/api/auth/login", "/api/auth/register")
                .order(1);
    }
}
```

## 异常处理体系

SpringMVC 异常处理基于**责任链模式**，通过 `HandlerExceptionResolver` 接口实现。`DispatcherServlet` 在 `doDispatch()` 中捕获异常，调用 `processHandlerException()` 遍历解析器链，返回第一个非 null 的 `ModelAndView`。

默认注册的解析器链（按优先级）：
1. `ExceptionHandlerExceptionResolver`：处理 `@ExceptionHandler` 注解的方法
2. `ResponseStatusExceptionResolver`：处理 `@ResponseStatus` 注解
3. `DefaultHandlerExceptionResolver`：处理 Spring MVC 内部异常（如 `MethodArgumentNotValidException`）

---

## @ExceptionHandler 与全局异常处理

`@ExceptionHandler` 在控制器类内定义异常处理方法，只对当前控制器有效。要实现全局处理，需结合 `@ControllerAdvice`（传统 MVC）或 `@RestControllerAdvice`（前后端分离）：

```java
@RestControllerAdvice
public class GlobalExceptionHandler {
    
    // 业务异常
    @ExceptionHandler(BusinessException.class)
    public Result handleBusinessException(BusinessException ex) {
        return Result.fail(ex.getCode(), ex.getMessage());
    }
    
    // 参数校验异常
    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result handleValidationException(MethodArgumentNotValidException ex) {
        List<String> errors = ex.getBindingResult().getFieldErrors().stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.toList());
        return Result.fail(400, "参数校验失败", errors);
    }
    
    // 系统异常（兜底）
    @ExceptionHandler(Exception.class)
    public Result handleException(Exception ex) {
        log.error("系统异常", ex);
        return Result.fail(500, "系统繁忙，请稍后重试");
    }
}
```

Spring 会优先匹配**最具体**的异常类型（子类优先于父类）。

## 自定义异常类

```java
@Getter
public class BusinessException extends RuntimeException {
    private final int code;
    
    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }
    
    // 预定义常用异常
    public static BusinessException notFound(String resource) {
        return new BusinessException(404, resource + " 不存在");
    }
    
    public static BusinessException forbidden() {
        return new BusinessException(403, "没有操作权限");
    }
}
```

不推荐在自定义异常上用 `@ResponseStatus` 注解，因为一旦异常被全局处理器捕获，`@ResponseStatus` 就不会生效，容易造成混乱。
