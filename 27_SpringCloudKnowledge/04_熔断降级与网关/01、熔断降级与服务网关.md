# 熔断降级与服务网关

## 一、服务雪崩问题

**服务雪崩**是分布式系统中的级联故障现象：一个服务不可用 → 调用方线程阻塞 → 线程池耗尽 → 调用方也不可用 → 故障扩散到整个系统。

根本原因是**同步阻塞调用模型**：线程在等待下游响应时被阻塞，当下游响应慢，线程池很快就被耗尽。

```
服务A → 调用服务B（响应慢）→ A的线程全部阻塞 → A线程池耗尽 → A也挂了 → C依赖A也挂了 → 雪崩
```

## 二、熔断机制

### 熔断器状态机

熔断器（Circuit Breaker）借鉴电路熔断器思想，有三个状态：

- **CLOSED（关闭）**：正常状态，请求正常通过，监控失败率
- **OPEN（打开）**：故障达到阈值，熔断器打开，后续请求**直接快速失败**，不访问实际服务
- **HALF_OPEN（半开）**：等待一段时间后进入半开状态，放行少量**试探请求**，若成功则恢复 CLOSED，若失败则回到 OPEN

```java
// 熔断器核心逻辑（简化）
if (state == OPEN) {
    return fallback(); // 直接降级，不访问服务
}
if (state == HALF_OPEN) {
    return trialRequest(); // 试探请求
}
return normalRequest(); // 正常调用
```

**核心参数**：
- 失败率阈值（如 50%）
- 熔断持续时间（OPEN → HALF_OPEN 的等待时间）
- 最小请求数（触发统计的最小请求量）

### 熔断 vs 降级

| 维度 | 服务熔断 | 服务降级 |
|------|----------|----------|
| 触发原因 | 依赖服务故障（高错误率） | 系统资源整体不足 |
| 实现层次 | 框架层自动处理 | 需业务层设计兜底逻辑 |
| 状态变化 | 有明确状态机 | 通常是配置开关 |
| 时效 | 临时性，服务恢复后自动关闭 | 可持续到系统压力缓解 |

## 三、Hystrix

Hystrix 是 Netflix 开源的延迟和容错库，是微服务架构中最早成熟的容错方案（目前已停止更新，新项目推荐 Sentinel）。

### 工作原理

Hystrix 采用**命令模式**封装依赖调用：

```java
// 使用 HystrixCommand 封装调用
@HystrixCommand(fallbackMethod = "fallback")
public User getUser(Long id) {
    return restTemplate.getForObject("http://user-service/users/" + id, User.class);
}

public User fallback(Long id) {
    return new User(-1L, "降级用户");
}
```

### 隔离策略

**线程池隔离（默认）**：每个依赖使用独立线程池，互不影响，开销较大但隔离彻底。

**信号量隔离**：通过计数器限制并发，同一线程执行，适合访问延迟低的本地缓存或高频调用。

## 四、Sentinel

Sentinel 是阿里巴巴开源的**流量控制组件**，是国内微服务的主流容错方案，功能比 Hystrix 更全面。

### 核心功能

- **流量控制**：根据 QPS 或并发线程数限流
- **熔断降级**：基于慢调用比例、异常比例、异常数触发熔断
- **系统保护**：根据系统整体负载（CPU、内存）保护
- **热点限流**：对特定参数值进行细粒度限流

### Hystrix vs Sentinel

| 维度 | Hystrix | Sentinel |
|------|---------|---------|
| 隔离策略 | 线程池/信号量 | 信号量（轻量） |
| 熔断策略 | 错误率 | 慢调用比例、异常比例、异常数 |
| 实时监控 | 简单 Dashboard | 实时流量监控 |
| 控制台 | Hystrix Dashboard（需部署） | Sentinel Dashboard（开箱即用） |
| 状态 | 停止更新 | 持续活跃 |

### 基本使用

```java
// 注解方式（配合 OpenFeign）
@SentinelResource(value = "getUser", blockHandler = "blockHandler", fallback = "fallback")
public User getUser(Long id) { ... }

public User blockHandler(Long id, BlockException e) {
    return new User(-1L, "流量控制降级");
}

public User fallback(Long id, Throwable t) {
    return new User(-1L, "异常降级");
}
```

## 五、Spring Cloud Gateway

### 什么是 Spring Cloud Gateway

Spring Cloud Gateway 是 Spring 官方的 API 网关，基于 **Spring WebFlux（响应式/非阻塞 IO）**，性能优于基于 Servlet 的 Zuul 1.x，是目前 Spring Cloud 生态的推荐网关。

### 三个核心概念

- **路由（Route）**：基本转发单元，包含 ID、目标 URI、断言集合、过滤器集合
- **断言（Predicate）**：判断请求是否匹配路由的条件（路径、Header、参数、时间等）
- **过滤器（Filter）**：在请求转发前后执行的逻辑（鉴权、限流、日志等）

### 配置示例

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service-route
          uri: lb://user-service        # lb:// 表示负载均衡
          predicates:
            - Path=/api/users/**        # 路径匹配断言
            - Method=GET,POST          # HTTP 方法断言
          filters:
            - StripPrefix=1            # 移除路径前缀
            - AddRequestHeader=X-Source, Gateway  # 添加请求头
```

### 常用断言

- `Path=/api/**`：路径匹配
- `Method=GET`：HTTP 方法匹配
- `Header=X-Token, \d+`：请求头匹配
- `Query=token`：查询参数匹配
- `After=2024-01-01T00:00:00+08:00[Asia/Shanghai]`：时间匹配

### 自定义全局过滤器

```java
@Component
@Order(0)  // 优先级，值越小越先执行
public class AuthFilter implements GlobalFilter {
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        String token = exchange.getRequest().getHeaders().getFirst("Authorization");
        if (StringUtils.isBlank(token)) {
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);
            return exchange.getResponse().setComplete();
        }
        return chain.filter(exchange);  // 放行
    }
}
```

### Gateway vs Zuul

| 特性 | Spring Cloud Gateway | Zuul 1.x |
|------|---------------------|---------|
| 编程模型 | 响应式（WebFlux，非阻塞） | 阻塞 Servlet |
| 性能 | 高，适合高并发 | 线程模型，高并发有瓶颈 |
| 维护状态 | 持续活跃 | Zuul 1.x 已停更 |
| Spring 生态集成 | 深度集成 | 第三方 |

## 六、Zuul

Zuul 是 Netflix 开源的微服务网关，通过**过滤器链**处理请求：

- `pre` 过滤器：路由前执行（认证、日志）
- `route` 过滤器：路由转发（HTTP/Ribbon）
- `post` 过滤器：响应后执行（响应头处理、指标收集）
- `error` 过滤器：发生错误时执行

Zuul 1.x 基于阻塞 IO，Zuul 2.x 升级为非阻塞（基于 Netty），但 Spring Cloud 官方未集成 Zuul 2.x，新项目应直接使用 Spring Cloud Gateway。
