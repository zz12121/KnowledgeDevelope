# 服务调用与负载均衡

## 一、服务间通信方式

### 同步调用

- **RESTful HTTP**：最常用，基于 HTTP 协议，JSON/XML 格式。开发简单，但存在调用链耦合风险。
- **RPC 框架**（如 gRPC、Dubbo）：基于二进制协议，性能更高，像调用本地方法一样调用远程服务。

### 异步消息

- **消息队列**（Kafka、RabbitMQ）：服务解耦，吞吐量高，但需处理消息可靠性和最终一致性。
- **事件驱动**：发布/订阅模式，服务自主协作，去中心化。

**选型原则**：同步调用简单直接但有耦合风险；异步消息解耦性好但引入额外复杂度。

## 二、RestTemplate

RestTemplate 是 Spring 提供的**同步 HTTP 客户端工具类**，用于消费 RESTful 服务。

### 常用方法

```java
// GET 请求
User user = restTemplate.getForObject("http://api/users/1", User.class);
ResponseEntity<User> resp = restTemplate.getForEntity("http://api/users/1", User.class);

// POST 请求
User created = restTemplate.postForObject("http://api/users", user, User.class);

// PUT / DELETE
restTemplate.put("http://api/users/1", user);
restTemplate.delete("http://api/users/1");

// 通用方法（支持自定义请求头）
ResponseEntity<User> resp = restTemplate.exchange(
    "http://api/users/1", HttpMethod.GET, httpEntity, User.class);
```

### 配合 Ribbon 使用

加上 `@LoadBalanced` 注解后，RestTemplate 的 URL 中可以直接用服务名，Ribbon 会拦截并做负载均衡：

```java
@Bean
@LoadBalanced
public RestTemplate restTemplate() {
    return new RestTemplate();
}

// 调用时用服务名替代 IP:Port
String result = restTemplate.getForObject("http://user-service/users/1", String.class);
```

## 三、Ribbon

### 什么是 Ribbon

Ribbon 是 Netflix 开源的**客户端负载均衡器**，嵌入在服务消费者中，解决如何从多个服务实例中选一个的问题。

**客户端负载均衡 vs 服务端负载均衡**：

| 对比维度 | 服务端（Nginx） | 客户端（Ribbon） |
|----------|-----------------|------------------|
| 位置 | 独立中间件 | 嵌入应用 |
| 请求路径 | 客户端 → 负载器 → 实例 | 客户端 → 实例（直连） |
| 单点风险 | 存在 | 无 |
| 适用场景 | 南北流量（外部入口） | 东西流量（服务间调用） |

### 工作原理

```java
// LoadBalancerInterceptor 拦截 RestTemplate 请求
public ClientHttpResponse intercept(HttpRequest request, byte[] body, ...) {
    String serviceName = request.getURI().getHost(); // 提取服务名 "user-service"
    return loadBalancer.execute(serviceName, ...);   // 交给 Ribbon 做负载均衡
}
```

**执行步骤**：
1. 从 URL 中解析服务名（如 `user-service`）
2. 从注册中心（Eureka/Nacos）获取该服务的健康实例列表
3. 根据负载均衡策略选一个实例
4. 将服务名替换为实际 IP:Port，发送请求
5. 失败时自动重试其他实例

### 负载均衡策略

| 策略 | 类名 | 说明 |
|------|------|------|
| 轮询（默认） | `RoundRobinRule` | 按顺序依次选择 |
| 随机 | `RandomRule` | 随机选择 |
| 重试 | `RetryRule` | 轮询失败后在指定时间内重试 |
| 最低并发 | `BestAvailableRule` | 选并发请求数最小的实例 |
| 响应时间加权 | `WeightedResponseTimeRule` | 响应越快被选中概率越高 |
| 可用过滤 | `AvailabilityFilteringRule` | 过滤故障实例后选并发较小的 |

```yaml
# 为特定服务配置策略
user-service:
  ribbon:
    NFLoadBalancerRuleClassName: com.netflix.loadbalancer.RandomRule
```

### 饥饿加载

Ribbon 默认**懒加载**，首次请求时才初始化，可能导致第一次请求较慢。开启饥饿加载在应用启动时提前初始化：

```yaml
ribbon:
  eager-load:
    enabled: true
    clients: user-service,order-service
```

## 四、OpenFeign

### 什么是 OpenFeign

OpenFeign 是 Spring Cloud 对 Netflix Feign 的增强封装，提供**声明式 HTTP 客户端**，让调用远程服务像调用本地方法一样简单。

**核心优势**：
- 用接口+注解定义 HTTP 调用，代码极简
- 原生集成 Ribbon（或 Spring Cloud LoadBalancer），天然支持负载均衡
- 支持 Spring MVC 注解（`@RequestMapping`、`@PathVariable` 等）
- 可与 Sentinel/Hystrix 集成实现熔断降级

### 基本使用

```java
// 1. 定义 Feign 客户端接口
@FeignClient(name = "user-service")
public interface UserServiceClient {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}

// 2. 在 Service 中直接注入使用
@Service
public class OrderService {
    @Autowired
    private UserServiceClient userServiceClient;

    public User getUser(Long userId) {
        return userServiceClient.getUser(userId); // 像调本地方法
    }
}

// 3. 启动类开启 Feign 扫描
@SpringBootApplication
@EnableFeignClients
public class Application { ... }
```

### 实现原理

**核心是动态代理 + 注解处理**：

1. **启动阶段**：`FeignClientFactoryBean` 扫描所有 `@FeignClient` 接口，通过 JDK 动态代理生成代理对象，注入 Spring 容器。

2. **调用阶段**：
   - `Contract` 组件解析方法注解（`@GetMapping` 等），构建 `RequestTemplate`（请求模板）
   - `Encoder` 将方法参数应用到模板，生成完整 HTTP 请求
   - `LoadBalancerFeignClient` 委托 Ribbon 做负载均衡，选出目标实例
   - 真正的 HTTP 客户端（默认 `HttpURLConnection`）向目标地址发送请求
   - `Decoder` 将响应体解码成方法的返回类型

### 性能优化

```yaml
# 替换底层 HTTP 客户端（默认 HttpURLConnection → OKHttp）
feign:
  okhttp:
    enabled: true
  httpclient:
    enabled: false

# 配置超时
  client:
    config:
      default:
        connectTimeout: 5000
        readTimeout: 5000

# 开启 GZIP 压缩
  compression:
    request:
      enabled: true
    response:
      enabled: true
```

### 开启日志

Feign 日志级别：`NONE`（默认）→ `BASIC`（请求/响应简报）→ `HEADERS`（含请求头）→ `FULL`（全部）

```java
@Configuration
public class FeignConfig {
    @Bean
    Logger.Level feignLoggerLevel() {
        return Logger.Level.BASIC; // 生产环境用 BASIC，排查问题用 FULL
    }
}
```

```yaml
logging:
  level:
    com.yourcompany.client.UserServiceClient: DEBUG
```

### 熔断降级

```java
// 降级实现类
@Component
public class UserServiceFallback implements UserServiceClient {
    @Override
    public User getUser(Long userId) {
        return new User(-1L, "降级用户", "服务暂时不可用");
    }
}

// Feign 客户端指定降级类
@FeignClient(name = "user-service", fallback = UserServiceFallback.class)
public interface UserServiceClient {
    @GetMapping("/users/{id}")
    User getUser(@PathVariable("id") Long id);
}
```

```yaml
feign:
  sentinel:
    enabled: true
```

### Ribbon vs Feign vs RestTemplate

- **Ribbon**：负责"选哪个实例"，是负载均衡的底层基础
- **RestTemplate**：模板化 HTTP 客户端，需手动拼 URL，配合 `@LoadBalanced` + Ribbon 使用
- **OpenFeign**：声明式 HTTP 客户端，内部集成 Ribbon，是现代微服务的推荐方式

Feign 是上层建筑，Ribbon 是底层基础，两者协作而非竞争。

## 源码深度：OpenFeign 工作原理全流程

### 1. 自动装配入口

`spring-cloud-openfeign-core.jar` 的 `spring.factories` 中注册了四个自动配置类：

```properties
org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
  org.springframework.cloud.openfeign.FeignAutoConfiguration,\
  org.springframework.cloud.openfeign.encoding.FeignAcceptGzipEncodingAutoConfiguration,\
  org.springframework.cloud.openfeign.encoding.FeignContentGzipEncodingAutoConfiguration,\
  org.springframework.cloud.openfeign.loadbalancer.FeignLoadBalancerAutoConfiguration
```

**FeignAutoConfiguration** 的核心职责：
- 注册 `FeignContext`（继承 `NamedContextFactory`）：为每个 FeignClient 创建独立的 IOC 子容器，实现配置隔离
- 注册 `Targeter`：默认是 `DefaultTargeter`，若容器有 `CircuitBreakerFactory` 则注册 `FeignCircuitBreakerTargeter`
- 注册 HTTP 客户端（`ApacheHttpClient` 或 `OkHttpClient`）

**FeignContext 的隔离机制**：
```
主容器
 ├── FeignContext（NamedContextFactory）
 │    ├── "user-service" 子容器：FeignClientsConfiguration + @FeignClient(configuration=A.class)
 │    └── "order-service" 子容器：FeignClientsConfiguration + @FeignClient(configuration=B.class)
```
不同 FeignClient 的 Encoder、Decoder、Contract 等组件完全隔离，互不影响。

---

### 2. @EnableFeignClients 扫描注册流程

`@EnableFeignClients` 通过 `@Import(FeignClientsRegistrar.class)` 触发，`FeignClientsRegistrar` 实现 `ImportBeanDefinitionRegistrar`：

```java
// 核心注册逻辑（简化）
public void registerBeanDefinitions(AnnotationMetadata metadata, BeanDefinitionRegistry registry) {
    // 1. 注册 @EnableFeignClients(defaultConfiguration=...) 指定的全局配置
    registerDefaultConfiguration(metadata, registry);
    // 2. 扫描包下所有 @FeignClient 接口，注册为 BeanDefinition
    registerFeignClients(metadata, registry);
}
```

每个 `@FeignClient` 接口被映射为 `FeignClientFactoryBean` 的 `BeanDefinition`，真正的代理对象由 `FeignClientFactoryBean.getObject()` 创建。

---

### 3. FeignClientFactoryBean#getObject() 代理创建

```java
public Object getObject() throws Exception {
    // 1. 从容器拿 FeignContext
    FeignContext context = beanFactory.getBean(FeignContext.class);

    // 2. 获取 Feign.Builder 并配置（从对应 name 的子容器中获取）
    Feign.Builder builder = feign(context);
    //    - 设置 Logger.Level、Retryer、ErrorDecoder、Options
    //    - 设置 Encoder、Decoder、Contract（SpringMvcContract，支持 Spring MVC 注解）
    //    - 设置 RequestInterceptor、QueryMapEncoder

    // 3. 获取负载均衡 Client（FeignBlockingLoadBalancerClient）
    Client client = getOptional(context, Client.class);
    builder.client(client);

    // 4. 委托 Targeter 生成代理对象
    Targeter targeter = get(context, Targeter.class);
    return targeter.target(this, builder, context,
        new Target.HardCodedTarget<>(type, name, url));
}
```

**Feign.Builder 最终通过 JDK 动态代理生成接口代理对象**：
```java
// Feign 内部 ReflectiveFeign.newInstance()
InvocationHandler handler = factory.create(target, methodToHandler);
T proxy = (T) Proxy.newProxyInstance(target.type().getClassLoader(),
    new Class<?>[]{ target.type() }, handler);
```

### 4. 方法调用全流程（请求发送）

当调用 `@FeignClient` 接口的某个方法时，JDK 动态代理的 `InvocationHandler.invoke()` 被触发：

```
接口方法调用
  └── FeignInvocationHandler.invoke()
       └── SynchronousMethodHandler.invoke(args)
            ├── 1. 通过 RequestTemplate.Factory 构建 RequestTemplate
            │       Contract（SpringMvcContract）解析方法注解，提取 path、method、headers、params
            │       Encoder 将参数对象序列化填入模板
            ├── 2. 执行 RequestInterceptor 链（在 template 上添加请求头等）
            ├── 3. 调用 Client.execute(request, options)
            │       FeignBlockingLoadBalancerClient.execute()
            │           → LoadBalancerClientFactory 做负载均衡，选择服务实例
            │           → 将 serviceId 替换为实际 IP:Port
            │           → 委托底层 Client（OkHttp/Apache HTTP）发送请求
            └── 4. 调用 Decoder 将响应反序列化为返回类型
```

### 5. 断路器集成：FeignCircuitBreakerTargeter

当配置 `feign.circuitbreaker.enabled=true` 且容器有 `CircuitBreakerFactory`，使用 `FeignCircuitBreakerTargeter`：

```java
// FeignCircuitBreakerInvocationHandler.invoke()
public Object invoke(Object proxy, Method method, Object[] args) {
    String circuitName = feignClientName + "_" + method.getName();
    CircuitBreaker circuitBreaker = factory.create(circuitName);

    // 将方法执行包装成 Supplier
    Supplier<Object> supplier = () -> dispatch.get(method).invoke(args);

    if (nullableFallbackFactory != null) {
        // 有 fallback 时，方法执行委托给 CircuitBreaker，失败调 fallback
        return circuitBreaker.run(supplier,
            throwable -> {
                Object fallback = nullableFallbackFactory.create(throwable);
                return fallbackMethodMap.get(method).invoke(fallback, args);
            });
    }
    return circuitBreaker.run(supplier);
}
```

### 6. FeignClientsConfiguration 核心组件

每个 FeignClient 子容器默认加载的配置，定义了 Feign 的核心组件：

| 组件 | 默认实现 | 职责 |
|------|---------|------|
| `Contract` | `SpringMvcContract` | 解析 `@RequestMapping`、`@GetMapping` 等 Spring MVC 注解 |
| `Encoder` | `SpringEncoder` | 将 Java 对象序列化为请求体（使用 `HttpMessageConverter`） |
| `Decoder` | `SpringDecoder` → `OptionalDecoder` | 将响应体反序列化为 Java 对象 |
| `Retryer` | `Retryer.NEVER_RETRY` | 默认不重试（重试由 Ribbon 负责） |
| `Feign.Builder` | `Feign.Builder` 或 `FeignCircuitBreaker.Builder` | Feign 构建器 |

所有组件都有 `@ConditionalOnMissingBean`，可通过自定义配置类覆盖。
