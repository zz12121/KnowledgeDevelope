# Kafka 安装与实战

## 一、环境准备

### 1.1 JDK 安装

Kafka 是基于 Java 开发，需要 JDK 环境支持。建议使用 JDK 8 或 JDK 11+。

```bash
# 验证 Java 环境
java -version
```

### 1.2 Zookeeper 安装

Kafka 依赖 Zookeeper 进行集群管理（Kafka 3.0+ 支持 KRaft 模式，可不依赖 Zookeeper）。

**下载 Zookeeper：**
```
官网：https://zookeeper.apache.org/
下载地址：https://zookeeper.apache.org/releases.html
```

**配置 Zookeeper：**
1. 解压到本地路径
2. 进入 `conf` 目录，复制 `zoo_sample.cfg` 为 `zoo.cfg`
3. 修改 `zoo.cfg` 配置文件：

```properties
# 基础配置
dataDir=/usr/local/zookeeper/data
dataLogDir=/usr/local/zookeeper/log
clientPort=2181

# 心跳时间（单位：毫秒）
tickTime=2000
# 初始化同步时间
initLimit=10
# 同步超时时间
syncLimit=5

# 最大客户端连接数
maxClientCnxns=60
```

**启动 Zookeeper：**
```bash
# Linux/Mac
./zkServer.sh start

# Windows
.\bin\zkServer.cmd

# 验证运行状态
./zkServer.sh status
```

### 1.3 Kafka 安装

**下载 Kafka：**
```
官网：https://kafka.apache.org/
下载地址：https://kafka.apache.org/downloads
```

**配置 Kafka：**
进入 `config` 目录，编辑 `server.properties`：

```properties
# Broker 唯一标识（集群中每个 Broker 必须唯一）
broker.id=0

# 监听地址
listeners=PLAINTEXT://localhost:9092

# 日志存储目录
log.dirs=/usr/local/kafka/kafka-logs

# Topic 默认分区数
num.partitions=1

# 日志留存时间（默认 7 天）
log.retention.hours=168

# 日志段大小
log.segment.bytes=1073741824

# Zookeeper 连接地址
zookeeper.connect=localhost:2181
```

**启动 Kafka：**
```bash
# Linux/Mac（后台启动）
./kafka-server-start.sh -daemon ../config/server.properties

# Windows
.\bin\windows\kafka-server-start.bat .\config\server.properties
```

## 二、Kafka 基本操作

### 2.1 Topic 操作

```bash
# 创建 Topic
./kafka-topics.sh --create \
    --topic my-topic \
    --bootstrap-server localhost:9092 \
    --partitions 3 \
    --replication-factor 1

# 查看 Topic 列表
./kafka-topics.sh --list \
    --bootstrap-server localhost:9092

# 查看 Topic 详情
./kafka-topics.sh --describe \
    --topic my-topic \
    --bootstrap-server localhost:9092

# 修改分区数（只能增加）
./kafka-topics.sh --alter \
    --topic my-topic \
    --partitions 5 \
    --bootstrap-server localhost:9092

# 删除 Topic
./kafka-topics.sh --delete \
    --topic my-topic \
    --bootstrap-server localhost:9092
```

### 2.2 生产者与消费者

```bash
# 启动生产者（发送消息）
./kafka-console-producer.sh \
    --topic my-topic \
    --bootstrap-server localhost:9092

# 启动消费者（从最新消息开始消费）
./kafka-console-consumer.sh \
    --topic my-topic \
    --bootstrap-server localhost:9092

# 启动消费者（从最早消息开始消费）
./kafka-console-consumer.sh \
    --topic my-topic \
    --bootstrap-server localhost:9092 \
    --from-beginning
```

### 2.3 消费组操作

```bash
# 查看消费组列表
./kafka-consumer-groups.sh \
    --bootstrap-server localhost:9092 \
    --list

# 查看消费组详情
./kafka-consumer-groups.sh \
    --bootstrap-server localhost:9092 \
    --group my-group \
    --describe

# 重置消费位点
./kafka-consumer-groups.sh \
    --bootstrap-server localhost:9092 \
    --group my-group \
    --reset-offsets \
    --to-earliest \
    --topic my-topic \
    --execute
```

## 三、Kafka 集群搭建

### 3.1 多 Broker 配置

**准备多台 Kafka 实例：**

```properties
# broker.id 必须唯一
broker.id=0
listeners=PLAINTEXT://localhost:9092
log.dirs=/path/to/kafka1/logs

---

broker.id=1
listeners=PLAINTEXT://localhost:9093
log.dirs=/path/to/kafka2/logs

---

broker.id=2
listeners=PLAINTEXT://localhost:9094
log.dirs=/path/to/kafka3/logs
```

**创建多副本 Topic：**
```bash
./kafka-topics.sh --create \
    --topic my-replicated-topic \
    --bootstrap-server localhost:9092,localhost:9093,localhost:9094 \
    --partitions 3 \
    --replication-factor 3
```

### 3.2 集群验证

```bash
# 查看 Broker 列表
./kafka-broker-api-versions.sh \
    --bootstrap-server localhost:9092

# 查看 Topic 副本分布
./kafka-topics.sh --describe \
    --topic my-replicated-topic \
    --bootstrap-server localhost:9092
```

## 四、EFAK 监控工具

EFAK（Eagle For Apache Kafka）是一个 Kafka 集群监控管理系统。

### 4.1 安装配置

**下载 EFAK：**
```
官网：http://download.kafka-eagle.org/
```

**配置 EFAK：**
编辑 `conf/system-config.properties`：

```properties
# Zookeeper 地址
kafka.eagle.zk.list=localhost:2181

# Kafka 集群别名
kafka.eagle.cluster=localhost

# Kafka 客户端连接地址（SASL 认证需要额外配置）
kafka.eagle.kafka.endpoints=localhost:9092

# 数据库配置（可选，使用 MySQL 存储元数据）
kafka.eagle.driver=com.mysql.cj.jdbc.Driver
kafka.eagle.url=jdbc:mysql://localhost:3306/ke?useUnicode=true&characterEncoding=UTF-8
kafka.eagle.username=root
kafka.eagle.password=123456
```

### 4.2 启动 EFAK

```bash
# 配置环境变量
export KE_HOME=/usr/local/ke
export PATH=$PATH:$KE_HOME/bin

# 启动
./ke.sh start

# 访问界面
# 地址：http://localhost:8048
# 默认用户名：admin
# 默认密码：123456
```

## 五、Spring Boot 集成

### 5.1 添加依赖

```xml
<dependency>
    <groupId>org.springframework.kafka</groupId>
    <artifactId>spring-kafka</artifactId>
</dependency>
```

### 5.2 配置 application.yml

```yaml
spring:
  kafka:
    bootstrap-servers: localhost:9092
    # 生产者配置
    producer:
      # 序列化
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.apache.kafka.common.serialization.StringSerializer
      # 可靠性配置
      acks: all
      retries: 3
      # 批处理配置
      batch-size: 16384
      linger-ms: 5
      buffer-memory: 33554432
      compression-type: lz4
    # 消费者配置
    consumer:
      # 反序列化
      key-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      value-deserializer: org.apache.kafka.common.serialization.StringDeserializer
      # 消费组
      group-id: my-group
      # 自动提交偏移量
      enable-auto-commit: true
      auto-offset-reset: earliest
      # 拉取配置
      max-poll-records: 500
      fetch-min-size: 1
      fetch-max-wait-ms: 500
    # 监听器配置
    listener:
      ack-mode: manual
      concurrency: 3
```

### 5.3 生产者示例

```java
@RestController
public class KafkaProducerController {

    @Autowired
    private KafkaTemplate<String, String> kafkaTemplate;

    @GetMapping("/send/{message}")
    public String sendMessage(@PathVariable String message) {
        // 异步发送
        ListenableFuture<SendResult<String, String>> future = 
            kafkaTemplate.send("my-topic", message);
        
        // 同步等待
        try {
            SendResult<String, String> result = future.get();
            return "发送成功，partition=" + result.getRecordMetadata().partition() 
                   + ", offset=" + result.getRecordMetadata().offset();
        } catch (Exception e) {
            return "发送失败：" + e.getMessage();
        }
    }

    // 带回调的异步发送
    @GetMapping("/send/async/{message}")
    public String sendMessageAsync(@PathVariable String message) {
        kafkaTemplate.send("my-topic", message)
            .addCallback(
                success -> System.out.println("发送成功"),
                failure -> System.out.println("发送失败：" + failure.getMessage())
            );
        return "异步发送中...";
    }
}
```

### 5.4 消费者示例

```java
@Component
public class KafkaConsumer {

    // 简单消费
    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void consume(String message) {
        System.out.println("收到消息：" + message);
    }

    // 指定分区消费
    @KafkaListener(
        topicPartitions = @TopicPartition(
            topic = "my-topic",
            partitions = {"0", "1"}
        )
    )
    public void consumePartition(ConsumerRecord<String, String> record) {
        System.out.println("分区：" + record.partition() 
            + "，偏移量：" + record.offset() 
            + "，消息：" + record.value());
    }

    // 手动提交偏移量
    @KafkaListener(topics = "my-topic", groupId = "my-group")
    public void consumeWithAck(@Payload String message, 
                                Acknowledgment ack) {
        try {
            System.out.println("处理消息：" + message);
            ack.acknowledge();  // 手动提交
        } catch (Exception e) {
            System.err.println("处理失败：" + e.getMessage());
        }
    }
}
```

## 六、高频面试题

### 6.1 如何防止消息丢失？

**发送端：**
- 设置 `acks=all`（或 `-1`），确保消息同步到所有 ISR 副本后才返回成功
- 配置 `min.insync.replicas=2`（确保有足够的同步副本）
- 设置 `retries=Integer.MAX_VALUE`，开启重试机制

```java
props.put(ProducerConfig.ACKS_CONFIG, "all");
props.put(ProducerConfig.RETRIES_CONFIG, Integer.MAX_VALUE);
props.put(ProducerConfig.MIN_IN_SYNC_REPLICAS_CONFIG, 2);
```

**消费端：**
- 关闭自动提交，改为手动提交 offset
- 在业务处理完成后才提交偏移量

```java
props.put(ConsumerConfig.ENABLE_AUTO_COMMIT_CONFIG, false);
```

### 6.2 如何防止消息重复消费？

**消息重复消费原因：**
- 消费者消费消息后未提交 offset，触发 Rebalance 导致重复消费
- 生产者重试导致消息重复发送

**解决方案：**
1. **业务幂等性处理**
   - 数据库主键唯一性
   - Redis 分布式锁
   - 消息去重表

```java
// Redis 去重示例
if (redisTemplate.opsForValue().get("msg:" + messageId) != null) {
    // 消息已处理，直接返回
    return;
}
// 处理消息
redisTemplate.opsForValue().set("msg:" + messageId, "1", 10, TimeUnit.MINUTES);
```

2. **消费者幂等**
```java
@KafkaListener(topics = "my-topic")
public void consume(ConsumerRecord<String, String> record) {
    String uniqueKey = record.key() + "_" + record.offset();
    // 基于唯一键实现幂等
    processMessage(uniqueKey, record.value());
}
```

### 6.3 如何保证消息顺序？

**问题分析：**
- 分区内消息有序，分区间无序
- 生产者重试可能导致乱序

**解决方案：**
1. **发送端**：将相关消息发送到同一分区
```java
// 使用相同的 Key 保证进入同一分区
new ProducerRecord<>("order-topic", orderId, orderMessage);
```

2. **消费端**：单分区单消费者
```java
// 配置一个消费组只有一个消费者
@KafkaListener(topics = "order-topic", groupId = "order-group")
// 或者在配置中指定
spring:
  kafka:
    consumer:
      properties:
        partition.assignment.strategy: org.apache.kafka.clients.consumer.RoundRobinAssignor
```

### 6.4 如何解决消息积压？

**消息积压原因：**
- 消费者消费能力不足
- 消费者故障
- 生产者速度远大于消费者速度

**解决方案：**

1. **提升单消费者处理能力**
```java
// 增加并发线程数
@KafkaListener(topics = "my-topic", concurrency = "10")
public void consume(String message) {
    // 处理逻辑
}
```

2. **增加消费者数量**
```java
// 部署多个消费者实例（注意：消费者数 ≤ 分区数）
```

3. **消费者多线程处理**
```java
@KafkaListener(topics = "my-topic")
public void consume(String message) {
    // 提交到线程池异步处理
    executor.submit(() -> processMessage(message));
}
```

4. **紧急处理**：跳过积压消息或清理旧消息
```bash
# 修改消费位点到最新
./kafka-consumer-groups.sh --reset-offsets \
    --to-latest \
    --topic my-topic \
    --group my-group \
    --execute
```

### 6.5 如何实现延迟队列？

**应用场景：** 订单超时未支付自动取消

**实现方案：**

```java
// 生产者：发送延迟消息
public void sendDelayMessage(String orderId, long delayMinutes) {
    long delayTime = System.currentTimeMillis() + delayMinutes * 60 * 1000;
    kafkaTemplate.send("order-delay-topic", delayTime + "_" + orderId, orderId);
}

// 消费者：检查延迟时间
@KafkaListener(topics = "order-delay-topic")
public void consumeDelayMessage(ConsumerRecord<String, String> record) {
    String key = record.key();
    String[] parts = key.split("_");
    long delayTime = Long.parseLong(parts[0]);
    String orderId = parts[1];
    
    if (System.currentTimeMillis() >= delayTime) {
        // 检查订单状态，如果未支付则取消
        orderService.cancelOrder(orderId);
    } else {
        // 重新发送（延迟检查）
        kafkaTemplate.send("order-delay-topic", key, orderId);
    }
}
```

**更优方案：**
- 使用 RocketMQ 的定时消息功能
- 使用 Redis 延迟队列
- 使用定时任务轮询数据库

## 七、Kafka 高吞吐原理

### 7.1 写入数据原理

| 技术 | 说明 |
|------|------|
| **顺序写磁盘** | 追加写入，避免随机磁头寻址 |
| **Page Cache** | 写入先到 OS Page Cache，异步刷盘 |
| **批量发送** | 消息在内存积累后批量发送 |

### 7.2 读取数据原理

| 技术 | 说明 |
|------|------|
| **零拷贝（Zero-Copy）** | 使用 sendfile 系统调用，直接从 Page Cache 到网卡 |
| **顺序读取** | 预读（Readahead）优化 |
| **压缩** | 支持 LZ4、Snappy、Zstd 压缩 |

### 7.3 零拷贝对比

```
传统方式（4次拷贝）：
磁盘 → 内核缓冲区 → 用户缓冲区 → Socket 缓冲区 → 网卡

零拷贝（2次拷贝）：
磁盘/Page Cache → 网卡（直接通过 sendfile）
```
